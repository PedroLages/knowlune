# Test Plan: Ollama Dev Proxy
Generated by /plan-eng-review on 2026-03-25

## Affected Pages/Routes
- `/api/ai/ollama/tags` — Ollama model listing proxy endpoint
- `/api/ai/ollama/health` — Ollama health check proxy endpoint
- `/api/ai/ollama/chat` — Ollama chat completions proxy endpoint (non-streaming)
- Settings page → AI Configuration → Ollama direct mode warning text

## Key Interactions to Verify
- User configures Ollama on localhost:11434 → proxy returns model list
- User configures Ollama on LAN address (192.168.x) → proxy connects successfully
- User starts dev server without Ollama running → clear "Cannot reach Ollama" error
- User selects "direct mode" in AI settings → sees updated CORS guidance text

## Edge Cases
- Ollama server URL with trailing slashes → normalized correctly
- Missing `serverUrl` query param on GET endpoints → 400 response
- Missing `ollamaServerUrl` in POST body → 400 response
- Blocked URL (169.254.x metadata) → 403 response
- Ollama server timeout → 504 response (not silent hang)
- Ollama returns non-200 status → error message forwarded to client
- Non-GET request to /tags or /health → passed to next middleware
- Non-POST request to /chat → passed to next middleware

## Critical Paths
- Full Ollama setup: configure URL → health check → list models → send chat → receive response
- SSRF protection: ensure cloud metadata addresses are always blocked
- isAllowedOllamaUrl: shared module used by both Vite plugin and Express server
