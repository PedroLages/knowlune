## Security Review — E88-S04: M4B Audiobook Import (2026-04-05)

### Phases Executed: 4/8

#### Phase 1 — Input Validation

- **File type validation**: M4B detection is by file extension only (`.m4b`). No MIME type or magic byte validation is performed on the file content. A maliciously crafted file with `.m4b` extension could be passed to `music-metadata` for parsing. **LOW** — `music-metadata` handles malformed input gracefully (throws errors, caught by try/catch).

- **File size**: No file size limit is enforced before parsing. A very large file (e.g., 10GB) would be passed directly to `parseBlob()` which reads it into memory. **MEDIUM** — Could cause browser tab crash on memory-constrained devices.

#### Phase 2 — Data Flow

- **OPFS storage**: Files are stored at `/knowlune/books/{bookId}/book.m4b`. The `bookId` is a UUID generated by `crypto.randomUUID()` — no path traversal risk.
- **Cover art**: Extracted from M4B metadata and stored as a Blob. The `pic.format` from music-metadata is used as MIME type for the Blob constructor. If a malicious file provides a crafted format string, it would only affect the Blob's type metadata, not execution. **LOW**.

#### Phase 5 — Secrets Scan

- No secrets, API keys, or credentials in the diff. Clean.

#### Phase 8 — Lightweight Checks

- **8.1 Dependencies**: `music-metadata` added. No known CVEs. Package is well-maintained (600K weekly downloads).
- **8.2 OWASP**: No injection, XSS, or CSRF vectors introduced. File processing is client-side only.
- **8.5 Attack surface**: New file format accepted (M4B). Attack surface increase is minimal — same trust model as existing MP3/EPUB import.

### Findings

#### MEDIUM

- **No file size limit on M4B import**: Large files could exhaust browser memory during `parseBlob()` call. Consider adding a reasonable limit (e.g., 2GB) with a user-friendly error message.

#### INFO

- File extension-only detection is standard for browser file inputs. MIME type sniffing would add complexity without meaningful security benefit since the file is processed client-side.

### Verdict

**PASS** — No blockers or high-severity security issues. 1 medium finding (file size limit).
