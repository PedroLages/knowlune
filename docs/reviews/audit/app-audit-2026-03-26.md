## Critical Issues (Blockers — Must Fix)

### B1 — Courses page crashes at all viewports [Consensus: 2 agents]
**File**: `src/app/components/figma/TagManagementPanel.tsx:33`
**Issue**: `useCourseImportStore(s => s.getTagsWithCounts())` calls a method inside the Zustand selector, producing a new array reference on every render → infinite loop → error boundary fires. The entire `/courses` route is broken.
**Fix**:
```tsx
const getTagsWithCounts = useCourseImportStore(s => s.getTagsWithCounts)
const tagsWithCounts = useMemo(() => getTagsWithCounts(), [getTagsWithCounts])
```

### B2 — useLearningPathStore: 5 mutations with no error handling
**File**: `src/stores/useLearningPathStore.ts:85-96`
**Issue**: `createPath`, `renamePath`, `updateDescription`, `addCourseToPath`, `removeCourseFromPath` have no try/catch. Errors from `persistWithRetry` propagate as uncaught promises — no toast, no error state, no rollback. UI shows success on failure.
**Fix**: Wrap each in try/catch with `setError()` and rollback pattern (matching other stores).

### B3 — CareerPaths heading says "Learning Paths" instead of "Career Paths"
**File**: `src/app/pages/CareerPaths.tsx:255`
**Issue**: Factual mislabelling that directly misleads screen reader users and all visitors.
**Fix**: Change heading text to "Career Paths".

### B4 — `/career-paths` has no sidebar navigation entry
**File**: `src/app/config/navigation.ts`
**Issue**: Route is entirely absent from sidebar — orphaned route unreachable by keyboard-only users.
**Fix**: Add one entry to the Library group in `navigation.ts`.

### B5 — Legal pages suppress focus outline (WCAG 2.4.7 failure)
**Files**: `PrivacyPolicy.tsx:51`, `TermsOfService.tsx:54`, `TermsOfService.tsx:81`
**Issue**: `focus-visible:outline-none` removes all focus indicators on TOC and cross-page links.
**Fix**: Remove `focus-visible:outline-none` from those class strings.

## Cross-Cutting Observations

1. **Design token discipline is excellent** — only 3 hardcoded color instances found across the entire app. This is remarkable for a 32-page application.

2. **Async cleanup patterns are consistently good** — nearly every page uses `let ignore = false` / `return () => { ignore = true }` to prevent state updates after unmount.

3. **Touch targets are a systemic issue** — sidebar nav links (40px), tab triggers (29px), search bar (36px), and various icon buttons are below the 44px minimum. A global fix via `min-h-[44px]` on interactive elements would resolve most instances.

4. **Heading hierarchy is inconsistent** — `<CardTitle>` rendering as `<h3>` creates heading skips in at least 3 pages. Consider a convention where the first card after `<h1>` uses `<h2>`.

5. **Fire-and-forget IndexedDB reads** appear in 3+ page components. A shared wrapper with `.catch()` error handling would prevent the "stuck loading forever" pattern.

6. **Premium feature gating** is impressively consistent — a single `PremiumFeaturePage` component ensures all 7 routes look and behave identically.

7. **Dark mode** works correctly across all 32 pages. Zero contrast failures found.

8. **Error boundaries** exist and catch the Courses page crash, but the error UI could be more user-friendly.
