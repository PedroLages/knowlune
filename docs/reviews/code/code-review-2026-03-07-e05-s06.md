## Summary

Clean separation of concerns. 1 blocker (test data issue), 4 high-priority findings, 4 medium, 3 nits.

## Findings

### Blocker

1. **AC7 E2E test missing `streakStartDate`** (confidence: 95) — `tests/e2e/story-e05-s06.spec.ts:165-171`: Seeded milestone object omits `streakStartDate`. The `getUncelebratedMilestones()` function at `streakMilestones.ts:62-64` compares `m.streakStartDate === streakStart` — undefined will never match, so the test passes for the wrong reason. Fix: Add `streakStartDate: '2026-02-01'` to the seeded milestone.

### High Priority

2. **`getStreakStartDate()` off-by-one ambiguity** (confidence: 90) — `streakMilestones.ts:43-47`: Computes streak start as `today - (N-1)` days. Whether the user studied today affects correctness. Could cause double-celebrations or missed milestones. Fix: Derive from actual study-log entries or store streak start date authoritatively.

3. **Mount-effect captures stale snapshot** (confidence: 88) — `StudyStreakCalendar.tsx:130-132`: `celebrateMilestones` missing from deps. Users may see duplicate toasts on every navigation to Overview. Fix: Add to deps; consider session-level dedup.

4. **String interpolation instead of `cn()`** (confidence: 85) — `StreakMilestoneToast.tsx:91`, `MilestoneGallery.tsx:52,66`: Use `cn()` from `@/app/components/ui/utils` for className construction.

5. **No unit tests for `streakMilestones.ts`** (confidence: 92) — Core milestone detection logic has zero unit tests. Fix: Add `src/lib/__tests__/streakMilestones.test.ts`.

### Medium

6. **Synchronous localStorage read on every render** (confidence: 78) — `MilestoneGallery.tsx:14-15`: `getMilestones()` called directly in render. Fix: Use `useState(() => getMilestones())`.

7. **Hardcoded hex colors in confetti** (confidence: 72) — `StreakMilestoneToast.tsx:26-56`: Acceptable for canvas-confetti but document the deviation.

8. **`text-[10px]` below readability minimum** (confidence: 75) — `MilestoneGallery.tsx:61`: 10px is below 12px minimum. Fix: Use `text-xs`.

9. **No error handling around localStorage writes** (confidence: 80) — `StudyStreakCalendar.tsx:120-127`: `detectAndRecordMilestones` calls `saveMilestones()` which can throw if localStorage is full. Fix: Add try/catch.

### Nits

10. **useEffect dependency `[tier]` vs `[milestone.milestoneValue]`** (confidence: 65) — `StreakMilestoneToast.tsx:86`.
11. **Trophy icon `size-3` is small** (confidence: 60) — `StudyStreakCalendar.tsx:253`.
12. **`getTierConfig` exported from toast, imported by gallery** (confidence: 70) — Consider extracting to `streakMilestones.ts`.

## Verdict

**1 blocker** must be fixed. Otherwise clean architecture with good separation of concerns.
