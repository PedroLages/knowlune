## New Findings

### HIGH

1. **TypeScript error in TopicTreemap** — `TopicTreemap.tsx:120`: `Type 'TreemapDataItem[]' is not assignable to type 'readonly TreemapDataType[]'`. The index signature was removed (fixing R1 Finding 3) but the Recharts `<Treemap>` component requires its data to conform to `TreemapDataType`. Fix: cast at call site `data={data as unknown as TreemapDataType[]}` or add `& Record<string, unknown>` to the interface.

### MEDIUM

2. **E2E tests fail: AC2 and AC8** — `knowledge-map-widget.spec.ts:14,55`: Tests expect `getByRole('heading', { name: 'Knowledge Map' })` to be visible, but the heading only renders in the non-empty state. Without IndexedDB seeding, the widget shows the empty state (no heading). Either seed data before asserting the heading, or assert the empty state testid instead.

3. **Inline style objects in TopicTreemap SVG** — `TopicTreemap.tsx:77,85-89,100-104`: Uses `style={{ fill, stroke, ... }}` on SVG elements. While SVG `<rect>` and `<text>` elements require inline styles (Tailwind cannot target SVG paint properties), the ESLint `no-inline-styles` rule may flag these. Add `// eslint-disable-next-line react-best-practices/no-inline-styles` comments or verify the rule excludes SVG elements.

### LOW

4. **`computeRelevanceScore` uses `Date.now()` directly** — `dashboardOrder.ts:172`: Carried forward from R1. Not blocking since tests use relative comparisons.
