# Code Review: E03-S08 — Global Notes Dashboard

**Date**: 2026-03-01
**Reviewer**: Code review agent (Opus 4.6)

## What Works Well

1. **Well-structured data pipeline.** `buildLookups()` at module level, `enrichNotes()` via `useMemo`, and the filter/sort chain are cleanly separated. Architecture guidance to "compute lookups ONCE in parent" was followed.
2. **Proper Zustand selector usage.** Lines 123-125 use individual selectors correctly, avoiding unnecessary re-renders.
3. **Solid E2E test coverage.** 18 tests across all 5 ACs with thoughtful locator refinements.

## Findings

### Blockers

1. **All implementation files uncommitted (confidence: 98)** — `src/app/pages/Notes.tsx` (untracked), `NoteCard.tsx`, `navigation.ts`, `routes.tsx` (modified) exist only as unstaged working tree changes. If PR is created from committed state, it ships a broken `/notes` route.

2. **Tag filter bar badges not keyboard accessible (confidence: 92)** — `Notes.tsx:379-388`: `<Badge>` renders as `<span>` with no `tabIndex`, `role`, or keyboard handlers. Same issue on in-card tag badges (:257-269). WCAG 2.1 SC 2.1.1 violation.

### High Priority

3. **Silent error swallowing on `getAllNoteTags()` (confidence: 90)** — `Notes.tsx:142`: `.catch(() => {})` silently discards IndexedDB errors. Tag filter bar just never appears with no user feedback.

4. **`courseName` prop added but never used (confidence: 95)** — `NoteCard.tsx:29,67`: Prop added to interface per Task 6.2 but never destructured or rendered. Notes.tsx doesn't use NoteCard at all. Dead code.

5. **Nested interactive Badge elements inside role="button" (confidence: 85)** — `Notes.tsx:230-243`: Card container is `role="button"` with `tabIndex={0}`, containing clickable Badge spans inside. Creates confusing keyboard/screen reader experience. Flagged in E03-S06 review.

### Medium

6. **String interpolation instead of `cn()` (confidence: 82)** — `Notes.tsx:261,383`: Uses template literals for conditional classes instead of project convention `cn()` utility.

7. **`stripHtml` and `ReadOnlyContent` duplicated from NoteCard (confidence: 78)** — Both exist in NoteCard.tsx and Notes.tsx. Should extract to shared modules.

8. **`searchNotesWithContext` caps results at 20 (confidence: 80)** — `noteSearch.ts:150` slices to 20 results. No "showing X of Y" indicator for users with many matching notes.

9. **TipTap `useEditor` created on each expansion (confidence: 72)** — New editor instance on every expand/collapse cycle. Acceptable for single-card-at-a-time but won't update if content changes between expansions.

### Nits

10. `buildLookups()` at module level is fine for static data but would become stale if `allCourses` is ever lazily loaded.
11. Timestamp button and "Open in Lesson" button are functionally identical (both navigate to same URL).
12. Timestamp test only verifies visibility, not click behavior.
