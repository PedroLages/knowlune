## Findings

### HIGH

1. **Missing `catch` block in `handleSave`** — `LinkFormatsDialog.tsx:242-256`
   `handleSave` uses `try/finally` but no `catch`. If `linkBooks()` rejects, the error propagates as an unhandled promise rejection with no user feedback. The `unlinkBooks` store action shows `toast.error`, but `linkBooks` failures from network/Dexie errors would be silent at the dialog level.
   **Fix**: Add `catch (err) { toast.error('Failed to link book formats') }`.

2. **Missing `catch` block in `handleUnlink`** — `LinkFormatsDialog.tsx:259-268`
   Same pattern — `try/finally` without `catch`. Defense-in-depth requires surfacing errors at the call site even if the store action has its own error handling.
   **Fix**: Add `catch` block with `toast.error`.

### MEDIUM

3. **`setTimeout` race in `handleOpenChange`** — `LinkFormatsDialog.tsx:183`
   `setTimeout(() => setView('select'), 300)` can fire after a rapid re-open, resetting view state unexpectedly. Should use a ref to track and clear the timeout.

4. **Stale closure risk in `handlePairPressed`** — `LinkFormatsDialog.tsx:221-239`
   `handlePairPressed` captures `selectedBook` from closure. If the books array updates between render and callback execution, `selectedBook` could be stale. Low probability in practice but worth noting.

### LOW

5. **Duplicated epub/audio book identification logic** — `LinkFormatsDialog.tsx:224-226` and `530-531`
   The `epubBook`/`audioBook` computation is duplicated in `handlePairPressed` and the IIFE in the editor view JSX. Extract to a shared helper.

6. **`void finalMappings` placeholder** — `LinkFormatsDialog.tsx:249`
   The TODO comment references E103-S01 for persisting mappings. This is acceptable for now but should be tracked.

### POSITIVE

- Design tokens used correctly throughout (no hardcoded colors)
- All interactive elements have min-h-[44px] touch targets
- Proper ARIA attributes: `aria-pressed`, `aria-label`, `role="progressbar"`, `role="list"`
- `unlinkBooks` follows the established optimistic update + atomic tx + rollback pattern
- Focus trap handled by shadcn/ui Dialog
- Empty state provides clear guidance on what to do next
- Confidence bar has proper color coding with design tokens (success/warning/destructive)

## Verdict

**HIGH: 2** | **MEDIUM: 2** | **LOW: 2**

The HIGH issues (missing catch blocks) should be fixed before shipping — they violate the project's "Catch Blocks Must Surface Errors" pattern. The MEDIUM and LOW issues are acceptable to defer.
