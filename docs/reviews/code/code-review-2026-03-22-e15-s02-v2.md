The `ACCOMMODATION_MULTIPLIERS` record with typed keys ensures exhaustive coverage -- adding a new tier forces updates at compile time. The `getAccommodationMultiplier` function is the single call site for multiplier logic across the entire codebase.

2. **Robust Zod validation pipeline**. Every entry point for accommodation values (localStorage read in Quiz.tsx:67, localStorage read in QuizResults.tsx:84, radio value change in TimerAccommodationsModal.tsx:55) uses `safeParse` with graceful fallback. No raw string casts remain.

### Findings

#### Blockers

#### High Priority

- **[src/app/pages/QuizResults.tsx:87] (confidence: 85)**: New empty catch block introduced by the v1 fix. The accommodation loading in `handleRetake` has `catch { // Best-effort -- fall back to standard }` with no logging. This is the exact anti-pattern the v1 review flagged in Quiz.tsx (which was fixed with `console.warn`). The fix for one finding introduced the same pattern in another file. Why: If localStorage throws (SecurityError in private browsing, quota exceeded), the learner silently falls back to standard time with no diagnostic trail. Fix: Add `console.warn('[QuizResults] Failed to load accommodation:', e)` matching the convention in Quiz.tsx:70.

- **[src/app/pages/QuizResults.tsx:80, src/app/pages/Quiz.tsx:62-72] (confidence: 80)**: Accommodation loading logic is duplicated instead of extracted. The v1 review recommended extracting `loadSavedAccommodation` to a shared utility. Quiz.tsx has it as a clean module-level function (lines 62-72), but QuizResults.tsx (lines 80-88) reimplements the same localStorage read + safeParse inline within `handleRetake`. If the localStorage key format changes (e.g., namespacing), both files must be updated in sync. Why: Divergent key formats or validation logic would cause one path to load the preference and another to miss it, silently breaking AC5 for retakes. Fix: Import `loadSavedAccommodation` from Quiz.tsx (or extract to `src/lib/quiz-utils.ts`) and call it in QuizResults:
  ```typescript
  const accommodation = loadSavedAccommodation(lessonId)
  await retakeQuiz(lessonId, accommodation)
  ```

#### Medium

- **[src/app/components/quiz/TimerAccommodationsModal.tsx:50-52] (confidence: 72)**: The `useEffect(() => setSelected(value), [value])` is dead code. Because Radix `Dialog` unmounts its Portal content when `open` is false (no `forceMount` on DialogContent), the `TimerAccommodationsModal` component fully unmounts on close and remounts on open. The `useState(value)` initializer at line 47 already provides the correct value on each mount. The effect can never fire in a meaningful scenario (the component is unmounted before `value` could change). Why: Dead code obscures intent -- a future reader may assume the effect is necessary and add `forceMount` thinking it's required, introducing the opposite bug. Fix: Remove the `useEffect` and add a comment explaining the mount/unmount lifecycle:
  ```typescript
  // Dialog unmounts content on close — useState initializer handles resync on reopen
  const [selected, setSelected] = useState<TimerAccommodation>(value)
  ```

- **[src/app/components/quiz/QuizStartScreen.tsx:35-43, src/app/components/quiz/TimerAccommodationsModal.tsx:24-29] (confidence: 70)**: Duration formatting logic is duplicated. `formatTimeBadge` and `formatDuration` both compute `Math.floor(minutes)`, `fractional * 60`, `Math.min(Math.round(...), 59)` -- identical math with slightly different output strings ("min" vs "minutes"). While the multiplier duplication was correctly extracted, the formatting duplication remains. Fix: Extract a shared `formatAdjustedDuration(minutes: number, abbreviated?: boolean)` utility alongside `getAccommodationMultiplier` in `types/quiz.ts` or a new `lib/quiz-utils.ts`.

#### Nits

- **Nit** [src/app/components/quiz/QuizStartScreen.tsx:1] (confidence: 55): `useState` is imported but only used for the `modalOpen` state. The `accommodation` state was lifted to the parent (`Quiz.tsx`), but the `useState` import and `modalOpen` local state remain. This is fine architecturally, just noting the import is used -- no action needed.

- **Nit** [src/app/components/quiz/QuizTimer.tsx:53] (confidence: 60): The 10% threshold announcement at line 53 does not include the `annotation` suffix, while the 25% threshold at line 47 does. This inconsistency means a learner with extended time hears "(Extended Time)" at the 25% warning but not at the 10% urgent warning. Consider adding the suffix to both or neither for consistency.

- **Nit** [src/app/components/quiz/QuizHeader.tsx:29] (confidence: 55): The annotation logic checks `progress.timerAccommodation === '150%' || progress.timerAccommodation === '200%'` explicitly rather than checking `!== 'standard'`. If a new accommodation tier is added, this condition must be updated. A more resilient check: `accommodation !== 'standard' && accommodation !== 'untimed'` (untimed never reaches here because the timer is hidden).

3. **Remove dead `useEffect`** (Medium) -- Improves clarity, no behavioral change.
4. **Extract shared duration formatter** (Medium) -- Lower priority since the math is correct in both places.
