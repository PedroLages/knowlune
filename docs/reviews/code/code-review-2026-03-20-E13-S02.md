## Findings

### [High] isAnswered array handling incorrect — QuestionGrid.tsx:25-26
**Confidence:** 85%

`isAnswered` check does not handle array answers correctly. For multiple-select questions, an answer of `[]` (empty array) passes `!== undefined && !== ''` and appears as "answered". Learners who click then deselect all options see a misleadingly "answered" bubble.

**Fix:**
```typescript
const isAnswered = questionId
  ? (() => {
      const a = answers[questionId]
      if (a === undefined || a === '') return false
      if (Array.isArray(a)) return a.length > 0
      return true
    })()
  : false
```

### [High] Count mismatch in ReviewSummary — ReviewSummary.tsx:22
**Confidence:** 80%

Count text uses `markedForReview.length` but buttons render from `markedIndices` (which filters IDs not in `questionOrder`). If a marked question ID is orphaned, the text says "2 questions marked" but only 1 button renders.

**Fix:** Use `markedIndices.length` for the count text.

### [High] onCheckedChange type mismatch — MarkForReview.tsx:18
**Confidence:** 78%

`onCheckedChange` passes `boolean | 'indeterminate'` but `onToggle` expects `() => void`. Works now but the type mismatch could mask future bugs if indeterminate state is added.

**Fix:** Change to `onCheckedChange={() => onToggle()}` to be explicit.

### [High] w-8 h-8 instead of size-8 — QuestionGrid.tsx:38
**Confidence:** 90%

Tailwind v4 provides `size-*` shorthand and this is a recurring pattern flagged since E02-S05.

**Fix:** Replace `w-8 h-8` with `size-8` (or `size-11` if adopting the min-size nit below).

### [Medium] aria-label on non-landmark div — ReviewSummary.tsx:20
**Confidence:** 75%

`aria-label` on a plain `<div>` won't create an announced region for screen readers.

**Fix:** Add `role="region"` alongside `aria-label`, or use a heading element `<h3>` for the section title.

### [Medium] O(n*m) includes() in QuestionGrid render — QuestionGrid.tsx:29
**Confidence:** 72%

`markedForReview.includes(questionId)` is O(n) per question. Negligible now, but scales poorly at 50+ questions.

**Fix:** Convert to a Set at component top: `const markedSet = useMemo(() => new Set(markedForReview), [markedForReview])`

### [Medium] Inline function in AlertDialog — Quiz.tsx:296-299
**Confidence:** 72%

`onJumpToQuestion` handler creates a new arrow function on every render inside the dialog.

**Fix:**
```typescript
const handleJumpFromDialog = useCallback((idx: number) => {
```
