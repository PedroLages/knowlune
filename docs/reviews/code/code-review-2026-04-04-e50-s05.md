### Findings

#### Blockers

- **[AC3 Integration — CourseHeader.tsx:303 — src/app/routes.tsx:251]**: The `StudyScheduleEditor` and "Schedule study time" button were integrated into `CourseHeader.tsx`, but `CourseHeader` is exclusively used by `UnifiedCourseDetail.tsx`, which is **no longer routed**. The active course detail route maps to `CourseOverview`. AC3 cannot be triggered by any user navigation. Fix: Integrate the schedule button + `StudyScheduleEditor` into `CourseOverview.tsx`.

#### High Priority

None.

#### Medium

- **[StudyScheduleEditor.tsx:73 — `existing` identity]**: `existing` is computed via `schedules.find(...)` inline on every render. While `open` being in the reset `useEffect` deps means form resets work correctly, the `existing` variable changes identity on every render (new object reference each time `schedules` is updated), which will cause the reset effect to re-fire even when the schedule content hasn't changed. Use `useMemo` to stabilize `existing`:
  ```ts
  const existing = useMemo(
    () => (isEdit ? schedules.find(s => s.id === scheduleId) : undefined),
    [isEdit, schedules, scheduleId]
  )
  ```

- **[TimePicker.tsx:39 — unpadded hour value]**: If a stored schedule has a non-padded time (e.g., `"6:00"` instead of `"06:00"`), the hour `<Select>` value won't match any item and will appear blank. Normalize on entry:
  ```ts
  const [rawHour, minute] = value.split(':')
  const hour = rawHour.padStart(2, '0')
  ```

#### Nits

- **[StudyScheduleEditor.tsx:247 — aria-live]**: Error message for days uses `aria-live="assertive"`. Since this fires on user-initiated save attempt, `aria-live="polite"` is more appropriate — assertive should be reserved for time-sensitive interruptions.

- **[DayPicker.tsx:44-45 — token comment]**: The story spec says selected state should use `bg-brand text-brand-foreground`. The implementation uses `data-[state=on]:bg-brand data-[state=on]:text-brand-foreground` which is correct, but `data-[state=off]` styling would be more precisely `hover:bg-muted/80` to indicate interactivity.
