# Code Review — E71-S01 Action Suggestion Data Layer

**Date:** 2026-04-13
**Reviewer:** Claude (Opus)
**Files:** `src/lib/actionSuggestions.ts`, `src/lib/__tests__/actionSuggestions.test.ts`

## Summary

Clean, well-structured pure-function module with comprehensive test coverage. No React/Zustand/Dexie coupling — good separation of concerns. Minor input validation gaps.

## Findings

### MEDIUM

1. **No input clamping on `recencyDecayFactor`** — `recencyDecayFactor(120)` returns `-20`, which would reduce urgency below what a perfect score should produce. Same applies to `fsrsDecayFactor` with negative stability values.
   - File: `src/lib/actionSuggestions.ts:90-91`
   - Recommendation: Clamp input or output to `[0, 100]` range.

2. **No input validation on `calculateUrgencyScore`** — accepts any number, including negatives and values > 100. While callers currently pass reasonable values, the function is exported and could be misused.
   - File: `src/lib/actionSuggestions.ts:73-75`
   - Recommendation: Add `Math.max(0, Math.min(100, ...))` clamping on result.

### LOW

3. **`canonicalName` injected into URL without encoding** — `actionRoute` uses template literals with `topic.canonicalName` directly. If canonicalName ever contains `&`, `=`, or spaces, the route will break.
   - File: `src/lib/actionSuggestions.ts:104,118`
   - Recommendation: Use `encodeURIComponent(topic.canonicalName)` for query params.

4. **Non-null assertion on Map.get()** — Line 198 uses `fsrsStability.get(topic.canonicalName)!` after a `.has()` check. Safe but fragile if refactored.
   - File: `src/lib/actionSuggestions.ts:198`
   - Recommendation: Consider `const stability = fsrsStability.get(...); if (stability !== undefined)`.

### NIT

5. **FIXED_DATE declared and immediately voided** — Test file declares `FIXED_DATE` and `void`s it for ESLint compliance, but the module under test doesn't use dates. Unnecessary noise.
   - File: `src/lib/__tests__/actionSuggestions.test.ts:11-12`

## Verdict

**PASS** — No blockers. Two MEDIUM issues are defensive improvements, not bugs in current usage.
