### MEDIUM

**M1. `selectedCoverImage` excluded from useEffect dependency array â€” stale closure risk**

In `ImportWizardDialog.tsx:62-95`, the `useEffect` for loading image previews references `selectedCoverImage` on line 82 but excludes it from the dependency array (line 95). The comment says "intentionally excluded" because re-running when the selection changes would revoke URLs. However, the stale closure means if the user navigates back and re-scans with different images, the auto-select logic on line 82 (`!selectedCoverImage`) will see the old value. This is mitigated by `handleRescan` resetting state, but it's fragile.

**File:** `src/app/components/figma/ImportWizardDialog.tsx:82-95`

**M2. Object URL leak potential on rapid re-scans**

If the user rapidly clicks Back then re-scans, the cleanup function (line 92-94) revokes URLs from the first scan's `objectUrls` array. But `objectUrls` is local to the async `loadPreviews` function, which captures it in a closure. If the component unmounts while `loadPreviews` is still running (e.g., dialog closed mid-scan), some URLs may never be revoked.

**File:** `src/app/components/figma/ImportWizardDialog.tsx:66-94`
