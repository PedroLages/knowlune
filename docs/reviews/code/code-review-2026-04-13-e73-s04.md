### MEDIUM

1. **`ASSESSMENT:` marker text not stripped from displayed message content**
   - `src/app/components/chat/MessageBubble.tsx:110-115`
   - The `DebugTrafficLight` badge renders above the message, but the raw `ASSESSMENT: green` text from the LLM response remains in `message.content` and is displayed via `renderContentWithCitations()`. Users will see both the badge AND the raw marker text.
   - **Fix**: Strip the `ASSESSMENT: ...` line from displayed content, e.g., filter it in `renderContentWithCitations()` or clean the content before storing.

2. **`debugAssessments` array never reset on mode switch or session end**
   - `src/stores/useTutorStore.ts:484`
   - `switchMode()` (line 232) resets `quizState` when leaving quiz mode but does not reset `debugAssessments` when leaving debug mode. The array grows unbounded across mode switches within a session.
   - **Fix**: Add `debugAssessments: previousMode === 'debug' ? [] : state.debugAssessments` in `switchMode()`, mirroring the `quizState` reset pattern.

### LOW

3. **Redundant screen reader output in DebugTrafficLight**
   - `src/app/components/tutor/DebugTrafficLight.tsx:43-44`
   - The visible text ("Solid") and the `sr-only` span ("Assessment: Solid") will both be announced by screen readers, resulting in "Solid Assessment: Solid". Either make the visible text `aria-hidden` or use `role="img"` with `aria-label` on the outer span.

### NITS

4. **`_context` parameter unused in `buildDebugPrompt`**
   - `src/ai/prompts/modes/debug.ts:22`
   - Intentionally unused per tests, but a `// TODO: adapt prompt when hasTranscript=false` comment would clarify intent.
