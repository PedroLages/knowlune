### HIGH

**H1: `CourseDetail.tsx` uses `useAuthorStore.getState()` inside render — no reactivity, no `loadAuthors()` call**
- File: `src/app/pages/CourseDetail.tsx:88`
- `useAuthorStore.getState().getAuthorById(course.authorId)` is called inside an IIFE within JSX. This has two problems:
  1. It bypasses Zustand's subscription mechanism, so if an author is edited/deleted, CourseDetail won't re-render.
  2. `loadAuthors()` is never called on this page, so if a user navigates directly to CourseDetail (before visiting Authors page), the store will be empty and author data will never show.
- **Fix:** Use `useAuthorStore(state => state.getAuthorById)` as a hook at the component level, and add a `useEffect` calling `loadAuthors()`.
- **Classification:** STORY-RELATED (file changed by this story)

**H2: `getAuthorForCourse()` in `lib/authors.ts` uses `getState()` — same reactivity issue**
- File: `src/lib/authors.ts:18`
- `useAuthorStore.getState().getAuthorById(course.authorId)` won't trigger re-renders when the store updates.
- Used by `CourseCard.tsx` which was also modified by this story.
- This is acceptable for a utility function called from render (since CourseCard already re-renders from other state changes), but the `loadAuthors()` issue persists: if the author store hasn't been loaded yet, this will always return `undefined`.
- **Classification:** STORY-RELATED
