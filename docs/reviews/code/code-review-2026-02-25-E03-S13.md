## Findings

### Blocker

**B1 — `replaceAll` stale decorations** — `SearchReplaceExtension.ts:107-120`
`replaceAll` dispatches the replacement transaction (line 116) then updates `storage.results` afterward (line 118). The ProseMirror decoration callback runs during dispatch and reads `storage.results` with OLD positions from the pre-replacement document. If replacement text differs in length, those positions are invalid, causing visual glitches or RangeError crash.

Fix: Add `editor.view.dispatch(editor.state.tr)` after updating storage (matching `replaceCurrent` pattern).

### High Priority

**H1 — Cmd+F handler scoped to `document`** — `NoteEditor.tsx:409-418`
Intercepts ALL Cmd+F keypresses page-wide while Notes tab is mounted. Hijacks browser native Find.
Fix: Scope handler to editor container element.

**H2 — 7 `eslint-disable` for `any` casts** — `FindReplacePanel.tsx:26-62`
Custom commands not typed via module augmentation. Renaming a command silently breaks at runtime.
Fix: Add `declare module '@tiptap/core'` extension in `SearchReplaceExtension.ts`.

**H3 — `scrollIntoView` ignores `prefers-reduced-motion`** — `SearchReplaceExtension.ts:175`, `TableOfContentsPanel.tsx:26`
Programmatic `scrollIntoView({ behavior: 'smooth' })` is not affected by CSS `scroll-behavior` rule.
Fix: Create shared `getScrollBehavior()` utility checking `window.matchMedia`.
