## Critical Gap

**Streaming Ollama fails silently without Express server.** The Vite plugin was created to eliminate the need for `npm run server`, but streaming chat (the primary AI interaction mode) still requires it. Fix approved: add SSE streaming support to the Vite plugin (~20 lines).

### 2. Delete SSRF validation from Vite plugin

**File:** `vite.config.ts`
**What to remove:**
- The entire `isAllowedOllamaUrl()` function (lines 20-31)
- All calls to `isAllowedOllamaUrl()` inside `getValidatedServerUrl()` and the `/chat` endpoint
- The 403 "disallowed address" error responses

**What to keep:**
- Basic URL validation (is it a valid URL? does it have http/https protocol?) — but not IP-range blocking
- The `isAllowedOllamaUrl()` function in `server/index.ts` — the Express server has a legitimate SSRF concern since it could run on a shared network

**Rationale:** The Vite dev server only runs on the developer's own machine during `npm run dev`. SSRF protection on a localhost-only dev tool is security theater that actively blocks the most common Ollama setup (`http://localhost:11434`).

### 3. Add streaming SSE support to Vite plugin

**File:** `vite.config.ts`
**What:** Add a new middleware handler for `POST /api/ai/ollama` (the streaming endpoint) that pipes the response as SSE.

**Implementation approach:**
```
POST /api/ai/ollama → read body → extract ollamaServerUrl → fetch Ollama with stream:true
→ set response headers (Content-Type: text/event-stream, Cache-Control: no-cache)
→ pipe response.body (ReadableStream) directly to res
```

**Key details:**
- The existing `/api/ai/ollama/chat` endpoint handles non-streaming (returns full JSON response)
- The streaming endpoint (`/api/ai/ollama`) returns Server-Sent Events (SSE)
- Use `response.body.getReader()` to read chunks and write to `res` as they arrive
- Set `Transfer-Encoding: chunked` and `Connection: keep-alive`
- Handle client disconnect: listen for `req.on('close')` to abort the upstream fetch
- Timeout: use `AbortSignal.timeout(120_000)` (2 min for streaming — longer than non-streaming)

**Why this matters:** The Vite plugin's value proposition is "no separate `npm run server` needed for Ollama." Without streaming support, the most important AI interaction (real-time chat) still requires Express. This makes the plugin feel broken.

### 6. Production Ollama proxy — tracked in TODOS.md

Already tracked. The architectural solution for production:
- In production (Vercel), there is no server-side proxy
- Users must configure Ollama with `OLLAMA_ORIGINS=*` to allow browser CORS
- The app detects production environment and makes direct browser→Ollama requests
- This is the same pattern as running Ollama with Open WebUI or other browser-based tools
