## Story-Related Findings

### HIGH

1. **`checkPagesGoalMet` is dead code — never called** — `src/stores/useReadingGoalStore.ts:151`
   The function is defined but never invoked from any component or hook. AC-5 states streak tracking should work with pages mode, but the streak is never updated when pages are read. The `DailyGoalRing` displays pages progress but doesn't trigger streak advancement.

### MEDIUM

2. **`usePagesReadToday` only runs once on mount** — `src/app/hooks/usePagesReadToday.ts:111`
   The `useEffect` has an empty dependency array `[]`. The pages count won't update during a reading session — the user must remount the component (navigate away and back) to see updated pages. Consider adding a polling interval or a dependency on a reading-session-end event.

3. **Pages estimation is unreliable for EPUB** — `src/app/hooks/usePagesReadToday.ts:59,82`
   Uses a fixed "2 min per page" heuristic to convert reading time to pages. This varies wildly by book density and reader speed. The comment says "approximate" but users setting a pages goal may expect accurate tracking.

4. **No E2E tests** — Tasks 8.1-8.3 from the story spec were not implemented. Genre filter, genre override, and pages goal ring are untested at the integration level.

5. **No unit tests for `checkPagesGoalMet`** — Task 7.2 "Test pages-per-day streak calculation" was not implemented. Coverage report confirms lines 152-172 are uncovered.

### LOW

6. **`removeChip` type cast excludes genre** — `src/app/components/library/LibraryFilters.tsx:61`
   Cast `as 'sort' | 'format' | 'authors'` should include `'genre'` for type safety. Works at runtime but misleading.

7. **`BookMetadataEditor` component size warning** — 432 lines exceeds 300-line threshold. Pre-existing but slightly worsened by this story (+6 lines).

### NITS

8. **Duplicate session query logic in `usePagesReadToday`** — Lines 45-55 and 68-78 are near-identical for PDF and EPUB paths. Could be extracted to a helper function.

9. **`ALL_GENRES` order differs from `BookGenre` type union order** — Not a bug but could cause confusion if someone assumes they match.

## Summary

Genre detection service is well-designed with good keyword matching, proper tie-breaking, and solid unit tests. The genre filter UI integration is clean and consistent. However, the pages-per-day goal tracking has gaps: the streak function is dead code, the pages hook doesn't live-update, and there are no E2E or unit tests for the pages goal path.
