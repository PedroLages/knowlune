## Story-Related Findings

### LOW: `aria-controls` targets non-existent elements when collapsed

**Files:** `CourseOverview.tsx:356`, `YouTubeCourseDetail.tsx:338`

Both `aria-controls` attributes reference IDs of elements that are conditionally rendered and not present in the DOM when collapsed:
- `CourseOverview.tsx`: `aria-controls={module-content-${mod.id}}` — the target `<div>` only renders when `isExpanded` is true
- `YouTubeCourseDetail.tsx`: `aria-controls="ai-summary-content"` — Radix `CollapsibleContent` unmounts when closed

**Impact:** Screen readers may not find the controlled element when the section is collapsed. This is a known limitation of conditional rendering with `aria-controls`. WCAG does not strictly require `aria-controls`, and most screen readers handle missing targets gracefully.

**Recommendation:** Accept as-is. The `aria-expanded` attribute (the primary requirement) works correctly in all cases.

### NIT: ImportProgressOverlay missing `aria-controls` (inconsistency)

**File:** `ImportProgressOverlay.tsx:219`

`aria-expanded` was added but no matching `aria-controls` attribute, unlike the other two components. This is not a bug (the story notes say `aria-controls` is optional), but it's inconsistent with the pattern applied to CourseOverview and YouTubeCourseDetail.
