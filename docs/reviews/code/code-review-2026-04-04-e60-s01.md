#### Nits

- **Nit** `src/services/NotificationService.ts:25` [Maintainability] (confidence: 75): `DECAY_THRESHOLD = 50` duplicates the value of `FADING_THRESHOLD = 50` in `src/lib/retentionMetrics.ts`. Both represent the same conceptual boundary (retention below 50% = fading/decaying). If the threshold ever changes, both must be updated in lockstep. The story file explicitly acknowledges this design choice ("FADING_THRESHOLD = 50 constant is already defined there but not exported -- define DECAY_THRESHOLD = 50 in NotificationService"), so this is intentional. Consider exporting a shared constant in a future refactor.

Pre-existing pattern (17+ instances since E01-S03).
