## Story-Related Issues

### HIGH: `handleManageBilling` finally-block defeats redirect loading state

**File:** `src/app/components/settings/SubscriptionCard.tsx:215-216`
**Severity:** HIGH

The `finally { setIsPortalLoading(false) }` block runs immediately after `window.location.href = result.url`, resetting the loading spinner before the browser actually navigates. The 5-second fallback timer on lines 209-211 never gets a chance to hold the loading state.

Compare with `handleCancelSubscription` (line 220-242) which correctly handles each path individually without a `finally` block.

**Fix:** Remove the `finally` block and add `setIsPortalLoading(false)` only to the error path (after `setPortalError`), matching the pattern in `handleCancelSubscription`.

```typescript
// Current (buggy):
} catch {
  setPortalError('Unable to open billing portal. Please try again.')
} finally {
  setIsPortalLoading(false)  // <-- defeats redirect loading
}

// Fixed:
} catch {
  setPortalError('Unable to open billing portal. Please try again.')
  setIsPortalLoading(false)
}
```

### MEDIUM: Missing URL validation on portal redirect

**File:** `src/app/components/settings/SubscriptionCard.tsx:206`
**Severity:** MEDIUM

`handleUpgrade` validates checkout URLs against `https://checkout.stripe.com/` before redirect (line 173), but `handleManageBilling` and `handleCancelSubscription` redirect to portal URLs without any validation. For defense-in-depth consistency, portal URLs should be validated against `https://billing.stripe.com/`.
