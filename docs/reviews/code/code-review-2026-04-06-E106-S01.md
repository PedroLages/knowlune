## Findings

### MEDIUM — TypeScript errors in useAudiobookshelfStore.test.ts (AbsSeries mock shape)

**File:** `src/stores/__tests__/useAudiobookshelfStore.test.ts:242-243, 297-311`

The `AbsSeries` mock objects include a non-existent `description` field and are missing three required fields: `nameIgnorePrefix`, `type`, and `totalDuration`. This causes TS2353 and TS2322 errors.

**Fix:** Update mock series objects to match the `AbsSeries` interface:
```typescript
{ id: 'series-1', name: 'Test Series', nameIgnorePrefix: 'Test Series', type: 'series', books: [], totalDuration: 0, addedAt: 0, updatedAt: 0 }
```

### MEDIUM — TypeScript errors in useAudiobookshelfStore.test.ts (vi.mocked type mismatch)

**File:** `src/stores/__tests__/useAudiobookshelfStore.test.ts:37, 57-60, 87, 99, 108-109, 131, 184, 206`

The `vi.mocked(db)` call doesn't properly type the mock DB methods because the module is mocked with `vi.mock` but the imported `db` object's type doesn't reflect the mock. The mock methods (`mockResolvedValue`, `mockRejectedValueOnce`, etc.) are not recognized by TypeScript on the original Dexie table types.

**Fix:** Cast the mocked table methods explicitly, e.g.:
```typescript
const mockToArray = db.audiobookshelfServers.toArray as ReturnType<typeof vi.fn>
```
Or use `vi.mocked()` with `{ deep: true }` if supported.

### LOW — Unused import `afterEach` in useAudiobookshelfStore.test.ts

**File:** `src/stores/__tests__/useAudiobookshelfStore.test.ts:10`

`afterEach` is imported but never used (TS6133).

**Fix:** Remove `afterEach` from the import.

### LOW — Unused variable `FIXED_DATE` in useNotificationPrefsStore.test.ts

**File:** `src/stores/__tests__/useNotificationPrefsStore.test.ts:13`

`FIXED_DATE` is declared but never read (TS6133). The test file doesn't use fake timers.

**Fix:** Remove the `FIXED_DATE` declaration.

### NIT — Inconsistent mock strategy across test files

Some test files (useBookStore, useStudyScheduleStore, useNotificationPrefsStore) use `vi.resetModules()` + dynamic `import()` for clean isolation, while useAudiobookshelfStore uses top-level `vi.mock()` with manual mock resetting in `beforeEach`. Both approaches work, but the inconsistency may confuse future contributors.

**Recommendation:** Not a blocker; document the rationale in a code comment if intentional.
