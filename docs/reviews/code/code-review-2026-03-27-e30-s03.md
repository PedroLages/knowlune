#### Medium

- **[src/app/components/ui/card.tsx:35] (confidence: 68)**: The `CardTitle` component types `...props` as `React.ComponentProps<'div'>` but renders a heading element (`h1`-`h6`). This is a minor type mismatch — `ComponentProps<'div'>` allows `div`-specific attributes that are not valid on heading elements. In practice this is harmless since heading elements accept the same global HTML attributes used in this codebase (className, children, data-*, etc.). No fix needed now, but if stricter typing is desired later, consider using `React.ComponentProps<'h3'>` or `React.HTMLAttributes<HTMLHeadingElement>`. This is a pre-existing pattern from shadcn/ui's original code; the story merely extended it.
