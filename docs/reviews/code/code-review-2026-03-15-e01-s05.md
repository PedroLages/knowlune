### Findings

#### Blockers

None.

#### High Priority

- **[Recurring] [src/hooks/useFileStatusVerification.ts:86] (confidence: 82)**: `verifyAll()` is fire-and-forget inside useEffect -- rejected promise is unhandled. Although `verifyAll` has internal error handling via `Promise.allSettled`, the function itself is `async` and any unexpected throw before or after the `allSettled` call (e.g., if `items.map()` throws due to a corrupt item) would produce an unhandled promise rejection. Why: Silent failure means the learner sees all items stuck in "checking" state with no feedback. Fix: Add `.catch(err => console.error('File verification failed:', err))` to the `verifyAll()` call: `verifyAll().catch(err => console.error('File verification failed:', err))`. Pattern: fire-and-forget async without `.catch()` is recurring since E03-S03.

- **[src/app/pages/ImportedCourseDetail.tsx:55-67] (confidence: 78)**: Dexie `.catch()` logs to console but provides no user feedback. When IndexedDB fails (corrupt DB, missing store, quota exceeded), the component renders an empty content list with the course header visible -- the learner sees a course page with no content and no indication of what went wrong. Why: Distinguishing "empty course" from "failed to load" is critical for learner trust. Fix: Add an `error` state variable, set it in the catch block, and render an error message when set (e.g., "Failed to load course content. Try reloading the page.").

#### Medium

- **[src/app/pages/ImportedCourseDetail.tsx:148,172] (confidence: 75)**: Content item rows lack `flex-wrap` for narrow mobile viewports. The design spec (story file line 121) states "Badge wraps below filename on very narrow viewports (flex-wrap)" but the `flex items-center gap-3` container has no `flex-wrap` class. On narrow screens, the badge and duration/page-count metadata will overflow or be clipped. Fix: Add `flex-wrap` to the content row container classes.

- **[src/app/pages/ImportedCourseDetail.tsx:170-199] (confidence: 72)**: All PDF items render as non-clickable `<div>` with `cursor-not-allowed` and `aria-disabled="true"` regardless of file status. When a PDF is `'available'`, it still appears disabled (opacity-75, not clickable). While PDFs may not have a viewer route yet, the visual treatment contradicts AC3 ("available files are fully functional") for PDF content items. Why: A learner who imported PDFs and sees them all grayed out may think their import failed. Fix: Either add a Link wrapper for available PDFs (even to a future route) or clearly indicate the "PDF viewer coming soon" state instead of making them look broken.

- **[tests/e2e/story-e01-s05.spec.ts:66-69] (confidence: 70)**: `beforeEach` seeds localStorage but does not clean up IndexedDB between tests. Each test seeds new data (courses, videos, PDFs) into IndexedDB, but there is no `afterEach` cleanup. If Playwright reuses the page context between tests within a describe block, data accumulates. The risk is low because test IDs are unique, but stale data could cause false positives if future tests use broader selectors. Fix: Add an `afterEach` block that clears the relevant IndexedDB stores, or verify that Playwright creates a fresh context per test in the fixture configuration.

#### Nits

- **Nit** [src/hooks/useFileStatusVerification.ts:43] (confidence: 60): `setStatusMap(new Map(initial))` creates a redundant copy. `initial` is already a freshly constructed Map that will never be mutated. `setStatusMap(initial)` would suffice.

- **Nit** [src/app/pages/ImportedCourseDetail.tsx:101] (confidence: 55): `new Date(course.importedAt).toLocaleDateString()` uses the user's locale, which is fine, but inconsistent with the project's `toLocaleDateString('sv-SE')` pattern noted in the pre-review checklist (line 161 of the story file). This may produce different formats across locales.

- **Nit** [tests/e2e/story-e01-s05.spec.ts:42-43] (confidence: 55): `format: 'mp4' as const` type assertion is unnecessary if the test factory or type already constrains the field. Minor noise in test data definitions.

### Recommendations

1. **First**: Add `.catch()` to the `verifyAll()` call to prevent unhandled rejections.
2. **Second**: Add user-facing error state when Dexie query fails.
3. **Third**: Add `flex-wrap` to content item rows per design spec.
4. **Fourth**: Clarify available PDF visual treatment (either make them interactive or visually distinguish "no viewer yet" from "file missing").
