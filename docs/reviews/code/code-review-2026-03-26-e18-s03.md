# Code Review: E18-S03 — Semantic HTML and ARIA Attributes for Quiz Components

**Date:** 2026-03-26
**Reviewer:** Claude Opus 4.6 (automated)
**Branch:** `feature/e18-s03-semantic-html-aria`
**Files Changed:** 20 (quiz components, question types, pages, tests)

## Summary

This story converts quiz components from generic `<div>` wrappers to proper semantic HTML (`<main>`, `<section>`, `<fieldset>`, `<legend>`, `<nav>`) and adds ARIA attributes (`role`, `aria-label`, `aria-expanded`, `aria-live`, `aria-hidden`, `role="progressbar"`). The changes improve screen reader navigation, keyboard accessibility, and WCAG compliance across the quiz flow.

## Changes Reviewed

### Semantic HTML Upgrades
- **QuizStartScreen, QuizReviewContent, QuizResults, Quiz (active state)**: `<div>` -> `<main>` for primary content landmark
- **QuizResults**: Content sections wrapped in `<section aria-label="...">` for score summary, analytics, question details, and actions
- **QuizReviewContent**: Added `<section>` landmarks with aria-labels, sr-only `<h2>` for question context, sr-only `role="progressbar"`
- **All question types (MC, TF, MS, FIB)**: Replaced empty `<legend className="sr-only" />` + `aria-labelledby` pattern with proper `<legend>` containing question text via `<MarkdownRenderer>`

### ARIA Improvements
- **QuestionBreakdown**: Added `aria-label` to expandable question rows (correct/incorrect/not answered status + points)
- **ReviewSummary**: Added `aria-label="Jump to question N"` to jump buttons
- **QuizReviewContent**: `aria-label` on Previous/Next buttons
- **QuestionDisplay**: Changed unsupported type fallback from `role="status"` to `role="alert"` (correct semantics)
- **FillInBlankQuestion**: Added `role="status"` and `aria-atomic="true"` to review feedback
- **QuizReview**: Added `role="alert"` to error state

## Findings

### STORY-RELATED

#### MEDIUM

1. **Multiple `<main>` landmarks possible on quiz pages** — `QuizStartScreen` renders `<main>`, `Quiz` active state renders `<main>`, and these are nested inside `Layout.tsx` which likely already has a `<main>`. Multiple `<main>` elements on a page violate HTML spec and confuse screen readers. The `<main>` in these components should be `<div>` or `<article>` if `Layout.tsx` already provides the `<main>` landmark.
   - Files: `src/app/components/quiz/QuizStartScreen.tsx:75`, `src/app/components/quiz/QuizReviewContent.tsx:50`, `src/app/pages/QuizResults.tsx:146`, `src/app/pages/Quiz.tsx:445`

2. **sr-only `role="progressbar"` missing `aria-valuetext`** — The sr-only progressbar in `QuizReviewContent` has `aria-valuenow`, `aria-valuemin`, `aria-valuemax` but no `aria-valuetext`. Screen readers will announce raw numbers without context. Adding `aria-valuetext={`Question ${currentIndex + 1} of ${questions.length}`}` would provide better narration.
   - File: `src/app/components/quiz/QuizReviewContent.tsx:61-68`

#### LOW

3. **Duplicate "Review progress" aria-labels** — The `<section aria-label="Review progress">` and the nested `<div role="progressbar" aria-label="Review progress">` share the same label, which may confuse screen reader landmark navigation.
   - File: `src/app/components/quiz/QuizReviewContent.tsx:52,63`

4. **Redundant sr-only heading in QuizReviewContent** — The `<h2 className="sr-only">Question {currentIndex + 1} of {questions.length}</h2>` duplicates the visible text "Question X of Y" in the paragraph above. This creates double narration for screen readers.
   - File: `src/app/components/quiz/QuizReviewContent.tsx:73-75`

## Verdict

The semantic HTML and ARIA changes are well-implemented and substantially improve accessibility. The `<legend>` pattern replacing empty `<legend className="sr-only" />` is a clear improvement. The `<section>` landmark structure in QuizResults is well-organized.

The main concern is the potential for multiple `<main>` landmarks (MEDIUM), which should be verified against the Layout component.
