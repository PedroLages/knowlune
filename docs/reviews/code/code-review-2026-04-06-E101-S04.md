## Findings

### BLOCKER

None.

### HIGH

**H1: Unit test not updated for `isSingleFileAudiobook` behavior change**
- File: `src/services/__tests__/M4bParserService.test.ts:274`
- The test "returns false for a remote source book" now fails because E101-S04 intentionally changed `isSingleFileAudiobook()` to return `true` for remote books. The test must be updated to expect `true` (or renamed to "returns true for a remote ABS book").

**H2: 4/5 E2E tests failing — stream URL assertion fails in test environment**
- File: `tests/e2e/audiobookshelf/streaming.spec.ts`
- Tests assert `audio.src` contains `abs.test:13378`, but the audio element fires an error (server unreachable) and `audio.src` appears empty. The test needs either:
  - A route intercept (`page.route()`) to mock the streaming endpoint, or
  - Assert `audio.src` was *set* (before error clears it), or
  - Use `page.evaluate` to check the src was assigned by hooking into the `loadChapterInternal` flow

### MEDIUM

**M1: `setTimeout(() => seekTo(savedSeconds), 500)` is a hard wait for session resume**
- File: `src/app/components/audiobook/AudiobookRenderer.tsx:133`
- Uses a 500ms timeout to wait for chapter load before seeking. This is fragile — slow devices may not have loaded in 500ms. Better: listen for `canplay` event or use a state-based approach (e.g., after `isLoading` transitions to `false`).

**M2: Promise rejection in `loadChapterInternal` remote branch not caught by caller**
- File: `src/app/hooks/useAudioPlayer.ts:302-320`
- If the `canplay` promise rejects (via `onError`), the rejection propagates to `loadChapter`. The caller in `AudiobookRenderer` line 72 (`loadChapter(0, false)`) does not `await` or `.catch()` this — resulting in an unhandled promise rejection. The error handler at line 187 shows a toast, but the rejected promise still propagates.

### LOW

**L1: `new Date().toISOString()` in production code (AudiobookRenderer:94)**
- Not a test issue (test patterns rule doesn't apply to production), but worth noting this is called on every pause. Acceptable.

### NITS

**N1: Redundant `revokeSharedObjectUrl()` call for remote books**
- File: `src/app/hooks/useAudioPlayer.ts:293`
- Remote books never create object URLs, so this call is always a no-op for remote→remote transitions. Harmless but adds noise.

## Architecture Assessment

The implementation correctly mirrors the single-file M4B pattern for remote ABS streaming. Key design decisions are sound:
- `isSingleFileAudiobook()` returning `true` for remote books reuses the seek-based chapter nav
- Bearer token via `?token=` query param is the standard ABS approach
- `_loadedBookId` caching prevents unnecessary stream reloads on chapter seeks
- Error handler correctly shows toast without auto-retry
- Position persistence via Dexie with optimistic store update is robust

The `BookContentService.ts` change (auth type guard) is minimal and correct — switches from `?.username` to `'username' in` for discriminated union compatibility.

## Verdict

**ISSUES FOUND** — 2 HIGH (failing tests), 2 MEDIUM, 1 LOW, 1 NIT
