## Findings

### MEDIUM

1. **Memory leak risk in `processFile` ‚Äî object URLs not revoked on re-process** ‚Äî `BookImportDialog.tsx:135,152`
   When a user selects a second file without clicking "Remove", `processFile` creates new object URLs via `URL.createObjectURL()` without revoking the previous one from the first file processing. The `reset()` function does revoke, but `processFile` itself does not check for an existing `coverPreviewUrl` before creating a new one.

2. **No file size validation before processing** ‚Äî `BookImportDialog.tsx:119-173`
   Large EPUB files (hundreds of MB) will be fully read into memory via `file.arrayBuffer()` in `EpubMetadataService.ts:25`. There is no size guard. Consider rejecting files above a reasonable threshold (e.g., 500 MB) with a user-friendly message.

### LOW

3. **ISBN regex may produce false positives** ‚Äî `EpubMetadataService.ts:70`
   The ISBN-10 pattern `/\d{9}[\dXx]/` matches any 10-digit sequence, which could match non-ISBN identifiers. This is acceptable for best-effort metadata but worth noting.

4. **`setTimeout` in `handleImport` for dialog close** ‚Äî `BookImportDialog.tsx:245-248`
   Uses `setTimeout(fn, 600)` for delayed dialog close. This is a minor UX pattern (success feedback delay) and acceptable, but could be replaced with a transition callback for cleaner lifecycle management.

### NITS

5. **Book grid shows placeholder for both coverUrl and no-coverUrl cases** ‚Äî `Library.tsx:68-78`
   Both branches render the same Library icon placeholder. This is intentional (comment says "implemented in E83-S03") but the conditional is unnecessary until cover rendering is implemented.
