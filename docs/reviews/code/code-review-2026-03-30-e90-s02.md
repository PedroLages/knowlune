## Story-Related Issues

### MEDIUM — Ollama feature-aware path ignores `resolved.model`

**File:** `src/ai/llm/factory.ts:66-67`

In the feature-aware Ollama branch (lines 56-68), the resolved model from the cascade is ignored. Instead, `getOllamaSelectedModel()` is always used. If a user sets a per-feature override like `{ provider: 'ollama', model: 'codellama' }`, the `model` field is discarded.

```typescript
// Line 66-67: resolved.model is never passed to OllamaLLMClient
const selectedModel = getOllamaSelectedModel() || undefined
return new OllamaLLMClient(serverUrl, isOllamaDirectConnection(), selectedModel)
```

**Suggested fix:** Use `resolved.model` as the primary model, falling back to `getOllamaSelectedModel()`:
```typescript
const selectedModel = resolved.model || getOllamaSelectedModel() || undefined
```

### LOW — `console.warn` in catch blocks may trigger `no-silent-catch` lint rule

**File:** `src/lib/aiConfiguration.ts:416,427`

The catch blocks use `console.warn` but not `toast.error()`. Currently passing lint because the ESLint rule checks for `toast.error()` or `// silent-catch-ok`. These are internal utility functions where warning is appropriate, but adding `// silent-catch-ok` comments would be more explicit.

### NIT — Redundant `AI_FEATURE_IDS` re-export

**File:** `src/lib/aiConfiguration.ts:19`

`AI_FEATURE_IDS` is re-exported but not used by any consumer in this story. Not harmful, but added proactively.
