## Story-Related Issues

### MEDIUM

**M1: Unsafe `as unknown as` type cast in `getAuthorForCourse()`**
- File: `src/lib/authors.ts:141`
- `return state.getAuthorById(course.authorId) as unknown as Author | undefined`
- This double-cast from `ImportedAuthor` to `Author` silently hides type mismatches. `ImportedAuthor` lacks `avatar`, `title`, `shortBio`, `yearsExperience`, `education`, `featuredQuote` fields that `Author` has. Callers expecting `Author` shape will get runtime `undefined` for these fields.
- **Recommendation:** Return `ImportedAuthor | undefined` or create a proper adapter function. The caller (`CourseDetail.tsx`) already adapted to use `photoUrl` instead of `avatar`, suggesting the cast is a workaround.

**M2: `getAuthorForCourse()` fires async `loadAuthors()` without awaiting**
- File: `src/lib/authors.ts:139`
- `state.loadAuthors()` is called fire-and-forget inside a synchronous function. On first render, this will return `undefined` because authors haven't loaded yet. The function returns stale data until the next re-render.
- **Recommendation:** This is a known pattern in the codebase (lazy initialization) but could cause flicker. Consider documenting this behavior or returning a loading sentinel.

**M3: Prettier formatting not applied before commit**
- Files: `src/app/pages/AuthorProfile.tsx`, `src/app/pages/Authors.tsx`, `src/app/pages/__tests__/Authors.test.tsx`
- Three story files had formatting issues that were auto-fixed during review.
- **Action Required:** These need to be committed.

**L2: `deleteAuthor` shows double toast**
- File: `src/stores/useAuthorStore.ts:152-164` and `src/app/components/authors/DeleteAuthorDialog.tsx:38`
- The store calls `toastWithUndo()` on success, and the dialog also calls `toast.success('Author deleted')`. The user may see two success toasts.
- **Recommendation:** Remove `toast.success('Author deleted')` from DeleteAuthorDialog since the store already handles the success notification via undo toast.

**L3: Delete confirmation dialog says "cannot be undone" but undo IS available**

- **Recommendation:** Change to "You can undo this within a few seconds." for accuracy.

### NITS

**N1: `AuthorFormDialog` builds `Omit<Author, 'id'>` but passes it to `addAuthor(NewAuthorData)`**
- File: `src/app/components/authors/AuthorFormDialog.tsx:167`
- The constructed `authorData` object has `Author` shape (with `title`, `shortBio`, `yearsExperience`, etc.) but `addAuthor` expects `NewAuthorData` (which has `name`, `bio`, `photoUrl`, `courseIds`, `specialties`, `socialLinks`). Extra fields like `title`, `shortBio`, `yearsExperience`, `education`, `featuredQuote` are silently ignored by the store.
- This means the form collects data that never gets persisted. The form fields for title, shortBio, yearsExperience, education, and featuredQuote are cosmetic — they don't save.

**N2: `AuthorFormDialog` line 71 uses `as unknown as Record<string, unknown>` for field access**
- Somewhat fragile pattern. Consider using optional chaining with type narrowing instead.
