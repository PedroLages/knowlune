#### High Priority
- **[src/lib/actionSuggestions.ts:219](https://github.com/knowlune/knowlune/blob/HEAD/src/lib/actionSuggestions.ts#L219) (confidence: 92)**: **Deduplication loses the most-urgent action.** When a topic has multiple action types, the code keeps whichever action type has the highest *static* priority (flashcard > quiz > lesson), discarding actions with higher *urgency scores*. This directly conflicts with the design goal of urgency-based ranking.  
  Example: A "weak" topic with score 10 that has flashcards gets a `flashcard-review` suggestion. Another "fading" topic with score 60 that only has lessons gets a `lesson-rewatch`. The lesson suggestion has lower urgency, so it sorts lower — that's correct. **But** if two topics both have multiple action types, the wrong one can be kept for a topic, placing it at a misleading rank in the final sorted list. When AC 6 (FSRS stability) causes different urgency scores for the same topic's different action types, discarding the less-prioritized but higher-urgency action produces wrong sort order.  
  **Fix:** Keep the suggestion with the highest `urgencyScore` per topic (breaking ties by `ACTION_PRIORITY`), or compute urgency once per topic (not per action) since urgency is a topic-level metric:
  ```typescript
  if (
    !existing ||
    suggestion.urgencyScore > existing.urgencyScore ||
    (suggestion.urgencyScore === existing.urgencyScore &&
      ACTION_PRIORITY[suggestion.actionType] < ACTION_PRIORITY[existing.actionType])
  ) {
    bestPerTopic.set(suggestion.canonicalName, suggestion)
  }
  ```

#### Medium
- **[src/lib/actionSuggestions.ts:153](https://github.com/knowlune/knowlune/blob/HEAD/src/lib/actionSuggestions.ts#L153) (confidence: 75)**: **XSS via unsanitized `topic.topicName` in `actionLabel`.** The label is constructed via string interpolation (`Review 5 flashcards on ${topic.topicName}`). If `topicName` contains user-controlled content (e.g., from a user-named topic), it could contain malicious strings. If this label is ever rendered via `dangerouslySetInnerHTML` or passed to a non-React context (toast notifications, PDF generation, etc.), it's an XSS vector. Even in React JSX (`<span>{label}</span>`), some icon/tooltip libraries interpret HTML.  
  **Fix:** Sanitize `topicName` before interpolation, or document that the caller must escape the label, or use a structured approach (separate `topicName` from display text).

#### Nits
- **[src/lib/actionSuggestions.ts:162](https://github.com/knowlune/knowlune/blob/HEAD/src/lib/actionSuggestions.ts#L162) (confidence: 95)**: **URL-unsafe characters in `canonicalName` not encoded.** `actionRoute` uses template literals (`/flashcards?topic=${topic.canonicalName}`) without `encodeURIComponent`. While `canonicalName` is likely a slug (e.g., `linear-algebra`), non-ASCII or special characters (spaces, `&`, `#`, `%`) would produce malformed URLs or query parameter injection.  
  **Fix:** Use `encodeURIComponent(topic.canonicalName)` when constructing query parameters.
