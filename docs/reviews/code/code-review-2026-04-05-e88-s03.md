# Code Review — E88-S03: Remote EPUB Streaming (2026-04-05)

## Summary

4 files changed, 587 insertions, 30 deletions. New `BookContentService` with remote EPUB fetch, Cache API caching with LRU eviction, auth header injection, structured error handling, and BookReader integration with cached fallback UI.

## Architecture Assessment

**GOOD**: Clean content-source abstraction pattern. `BookContentService` correctly routes by `source.type` and returns a uniform `ArrayBuffer`. The `RemoteEpubError` subclass with structured `code` property enables type-safe error handling in the UI layer.

**GOOD**: Cache API choice over IndexedDB is appropriate for request/response-shaped binary data. LRU eviction with `X-Cached-At` header timestamp is a pragmatic approach.

**GOOD**: Abort controller with 30s timeout prevents hanging requests. Cleanup pattern in useEffect prevents stale state updates.

## Findings

### HIGH

**H1 — `btoa()` fails on non-ASCII credentials** — `BookContentService.ts:89`

`btoa()` throws on non-ASCII characters (e.g., `password: "pässwörd"`). The Basic auth spec (RFC 7617) recommends UTF-8 encoding. Fix: use `btoa(unescape(encodeURIComponent(...)))` or `TextEncoder` + base64 encoding.

```typescript
// Current (breaks on non-ASCII):
headers['Authorization'] = `Basic ${btoa(`${source.auth.username}:${source.auth.password}`)}`

// Fix:
const encoded = new TextEncoder().encode(`${source.auth.username}:${source.auth.password}`)
const base64 = btoa(String.fromCharCode(...encoded))
headers['Authorization'] = `Basic ${base64}`
```

Severity: HIGH — users with international characters in Calibre-Web passwords will get a runtime error.

### MEDIUM

**M1 — `clearTimeout` not called on successful non-error response paths for error status codes** — `BookContentService.ts:97-124`

When the response is received but has a 401/403/404/500 status, `clearTimeout(timeoutId)` is called on line 97 before checking status. This is correct. No issue here upon closer inspection.

**M2 — Cache "Read cached version" button uses raw `bg-brand` on a `<button>` instead of `variant="brand"`** — `BookReader.tsx:591`

The styling rules state to use `variant="brand"` on `<Button>` components, not `className="bg-brand"`. However, this is a native `<button>`, not a shadcn `<Button>`. Using a shadcn `<Button variant="brand">` would be more consistent with the component library.

Severity: MEDIUM — functional but inconsistent with UI library patterns.

### LOW

**L1 — LRU eviction reads all cached responses to sort by timestamp** — `BookContentService.ts:204-222`

The `evictOldestIfNeeded` method opens every cached response to read the `X-Cached-At` header. With MAX_CACHED_BOOKS=10, this is only 10-11 reads, so performance is fine. But the pattern is O(n) reads per cache write. Not a concern at current scale.

**L2 — Missing CORS mention in error messages** — `BookContentService.ts:154-158`

The story notes mention CORS as a likely failure mode. The network error message says "Could not reach your library server" but doesn't hint at CORS. Since CORS errors manifest as `TypeError: Failed to fetch`, users won't get a helpful diagnostic. Consider appending "If using a browser, check CORS settings on your server." to the network error.

### NITS

**N1 — `hasCachedFallback` state could be derived** — `BookReader.tsx:113`

The `hasCachedFallback` state is only set when a `RemoteEpubError` is caught. It could potentially be derived from the error object itself rather than stored as separate state. Current approach works fine though.

## Verdict

1 HIGH, 1 MEDIUM, 2 LOW, 1 NIT. The HIGH (btoa non-ASCII) should be fixed before shipping. Overall architecture is clean and well-tested.
