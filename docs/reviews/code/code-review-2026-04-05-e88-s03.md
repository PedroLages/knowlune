### LOW

**L1 — `String.fromCharCode(...encoded)` spread has theoretical argument limit** — `BookContentService.ts:107`

The spread operator passes every byte as a separate argument. V8 has a ~65K argument limit. For normal credentials this is fine, but `Array.from(encoded, b => String.fromCharCode(b)).join('')` would be safer. Not a practical concern for auth credentials.

Severity: LOW — theoretical only. Credentials would need to be >65KB.

**L2 — `password: undefined` produces `"user:undefined"` string** — `BookContentService.ts:97`

If `source.auth` has `username` but `password` is `undefined`, the credential string becomes `"admin:undefined"`. Guard only checks `source.auth?.username`. Consider also checking `source.auth?.password`.

Severity: LOW — OPDS catalog stores both fields as required. Defensive check would be nice but not blocking.

### NITS

**N1 — Double `clearTimeout` on error response paths** — `BookContentService.ts:116,155`

When response has error status (401/403/404/500), `clearTimeout` runs on line 116, then a RemoteEpubError is thrown, caught in outer catch, and `clearTimeout` runs again on line 155. Harmless (clearing already-cleared timer is a no-op) but slightly asymmetric control flow.
