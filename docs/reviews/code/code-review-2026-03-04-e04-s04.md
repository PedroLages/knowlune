#### Blockers

- **[`src/app/pages/SessionHistory.tsx:53`] (confidence: 95)**: Type mismatch between `StudySession.startTime` (typed as `string` in `src/data/types.ts:177`) and actual usage as `number`. The `StudySession` interface declares `startTime: string` and `endTime?: string` (ISO 8601), but the component casts `allSessions` to `DisplaySession[]` and the E2E tests seed numeric timestamps (`new Date(...).getTime()`). The `toDate()` helper handles both, but this means `DisplaySession extends StudySession` is semantically incorrect. Additionally, line 292 `session.videosWatched?.length > 0` â€” TypeScript with strict null checks would flag `> 0` on `number | undefined` (the `?.` makes it possibly `undefined`). This expression evaluates `undefined > 0` as `false`, which happens to work at runtime, but is a type-unsafe comparison. Fix: Add `videosWatched?: string[]` to `DisplaySession` interface (making it explicitly optional), or use `(session.videosWatched?.length ?? 0) > 0`.
