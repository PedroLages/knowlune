## Code Review: E27-S01 — Add Analytics Tabs To Reports Page

### What Works Well

1. **URL-aware tab pattern is clean and correct.** The `VALID_TABS` array with `.includes()` guard and fallback to `'study'` is a robust validation pattern. Moving it outside the component as a const is the right call.

2. **QuizAnalyticsTab follows established async-loading patterns well.** The `ignore` flag for cleanup, `Promise.all` for parallel fetches, proper error handling with both `console.error` and `toast.error`, loading skeletons with `aria-busy`, and the `sr-only` heading all demonstrate attention to conventions.

3. **E2E test coverage is thorough.** All tab URL permutations, invalid param fallback, empty state, tab click URL update, and three-tab visibility are covered. The `beforeEach` seeding pattern with welcome wizard and sidebar keys is solid.

### Findings

#### Blockers

- **[src/app/pages/Reports.tsx:202] (confidence: 92)**: `replace: true` in `setSearchParams` breaks AC 5 ("the browser back button returns to the previous tab"). With `replace: true`, each tab click replaces the current history entry instead of pushing a new one, so the browser back button skips over tab changes entirely. The AC explicitly requires back-button navigation between tabs. Why: Learners expect to press Back to return to the previous tab they were viewing; with `replace`, they get sent out of the Reports page entirely. Fix: Remove `{ replace: true }` from the `onValueChange` callback, or at minimum change to `{ replace: false }`.

  ```typescript
  // Current (breaks back button):
  onValueChange={value => setSearchParams({ tab: value }, { replace: true })}
  // Fix:
  onValueChange={value => setSearchParams({ tab: value })}
  ```

- **[src/app/components/reports/QuizAnalyticsTab.tsx] (confidence: 88)**: AC 7 specifies "Best/worst performing quizzes" as part of the aggregate quiz statistics, but the QuizAnalyticsTab component does not display any best/worst performing quiz information. The data is loaded (`attempts` array has `quizId`) but never aggregated by quiz to show top/bottom performers. Why: Learners navigating to Quiz Analytics expect to see which quizzes they excelled at or struggled with -- this is a key analytics insight missing from the implementation. Fix: Group attempts by `quizId`, compute per-quiz average scores, and display the highest and lowest performing quizzes (at minimum quiz title + average score).

#### High Priority

- **[src/app/components/reports/QuizAnalyticsTab.tsx:31] (confidence: 82)**: `db.quizAttempts.toArray()` loads ALL quiz attempt records into memory just to compute an average score. This is a full table scan that grows linearly with the learner's quiz history. For learners who retake quizzes frequently, this could mean hundreds or thousands of records loaded at once. Why: Unnecessary memory pressure and slower load times as data grows. Fix: Use a Dexie cursor or `each()` to compute the running average without loading all records, or at minimum use a targeted query with `.toArray()` on only the needed fields. For a simple count + sum, consider:

  ```typescript
  const allAttempts = await db.quizAttempts.toArray()
  // Better: compute in a single pass without holding all records
  let sum = 0, count = 0
  await db.quizAttempts.each(a => { sum += a.percentage; count++ })
  const averageScore = count > 0 ? sum / count : 0
  ```

- **[tests/e2e/regression/story-e27-s01.spec.ts] (confidence: 85)**: No E2E test for the back-button AC (AC 5: "the browser back button returns to the previous tab"). Even if the `replace: true` blocker is fixed, there is no test verifying `page.goBack()` returns to the previous tab. Why: This is an explicitly stated acceptance criterion with no test coverage, meaning regressions would go undetected. Fix: Add a test that navigates between tabs, calls `page.goBack()`, and verifies the previous tab is restored.

- **[tests/e2e/regression/story-e27-s01.spec.ts] (confidence: 80)**: No E2E test for Quiz Analytics with seeded data (AC 7: "I see aggregate quiz statistics: Total quizzes taken, Average score, Average retakes"). Why: The entire data-populated view of the Quiz Analytics tab is untested in E2E, meaning stat card rendering, retake frequency detail, and score display could break without detection. Fix: Add a test that seeds quiz data via IndexedDB helpers and asserts the stat cards show correct values.

#### Medium

- **[src/app/pages/Reports.tsx:270-271] (confidence: 72)**: Inline `style={{ height: barChartHeight }}` is used for the chart container. The project's ESLint rule `react-best-practices/no-inline-styles` flags inline style objects. While this is a dynamic value that legitimately needs runtime computation, it could use a CSS custom property approach or the existing Tailwind arbitrary value pattern. Fix: Consider `style={{ '--chart-h': `${barChartHeight}px` } as React.CSSProperties}` with `h-[var(--chart-h)]`, or accept the inline style with a suppress comment explaining why.

- **[src/app/components/reports/QuizAnalyticsTab.tsx:79] (confidence: 75)**: The empty state condition `!data || data.totalAttempts === 0` conflates a load failure with having no data. If the `catch` block fires, `data` stays `null` and `isLoading` becomes `false`, causing the empty state to render instead of an error state. The learner sees "No quizzes taken yet" when the real issue is a database read failure. Why: Silent mismapping of errors to empty states makes debugging harder for learners and support. Fix: Track an `error` state separately and show an error UI (or at least a different message) when the load failed.

- **[src/app/pages/Reports.tsx:177-181] (confidence: 70)**: The `hasActivity` guard wraps the entire tab structure including the Quiz and AI tabs. If a learner has quiz data but no study activity (e.g., they only took quizzes through a direct link), they would see the empty state and never reach the Quiz Analytics tab. This may be intentional (study activity is the primary flow), but it prevents quiz-only users from accessing their analytics. Fix: Consider whether `hasActivity` should also check quiz attempt count, or move the guard to the Study tab content only.
