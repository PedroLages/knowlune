## Findings

### BLOCKER

1. **All 4 E2E tests fail — onboarding dialog blocks test interaction** — `tests/e2e/story-e83-s06.spec.ts`
   - The `seedBooksAndHighlights()` helper seeds `knowlune-onboarding-v1` in localStorage then navigates to `/library` and reloads, but the onboarding dialog still appears and blocks all interaction with the book cards.
   - Root cause: likely the `page.reload()` on line 79 or a race between localStorage write and app initialization. Other passing tests (e.g., `story-e83-s04.spec.ts`) use a similar pattern but may not reload.
   - **Fix:** Remove the `page.reload()` or ensure the onboarding dismiss is seeded before initial navigation. Compare with the working pattern in `story-e83-s04.spec.ts`.

### MEDIUM

2. **No `await` return value check on `deleteBook`** — `src/stores/useBookStore.ts:102`
   - `handleConfirmDelete` in BookContextMenu calls `deleteBook(book.id)` without awaiting. If the delete fails and rolls back, the dialog has already closed and the user sees the book disappear then reappear without explanation.
   - **Suggestion:** `await deleteBook(book.id)` or keep dialog open until promise resolves.

### LOW

3. **Duplicate AC label in test descriptions** — `tests/e2e/story-e83-s06.spec.ts:83,126`
   - Two tests reference "AC1" (one for dialog content, one for cancel). The cancel test should be labeled differently (e.g., "AC1-cancel" or separate AC).

### NIT

4. **`min-h-[44px]` on AlertDialogAction** — `src/app/components/library/BookContextMenu.tsx:189`
   - Good touch target addition, but this is an arbitrary value. Consider using a design token or shared utility class if 44px touch targets are a recurring pattern.

## Architecture Assessment

The cascade delete pattern (highlights → book → OPFS) with best-effort OPFS cleanup is well-designed. The double-delete guard and optimistic update with rollback follow existing store patterns correctly.

## Security Assessment

No security concerns. Delete operations are local-only (IndexedDB + OPFS), no network calls involved.
