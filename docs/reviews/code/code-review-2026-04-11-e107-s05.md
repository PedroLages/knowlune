### What Works Well

1. **Clean single-source-of-truth architecture.** Extracting `readerThemeConfig.ts` as the shared module eliminates 5 independent hardcoded color maps across reader components. The `THEME_COLORS` matrix cleanly maps every reader theme x color scheme combination, and the separation between raw hex values (for epub.js iframe injection) and pre-computed Tailwind classes (for JIT scanning) shows good understanding of the two different rendering contexts.

2. **Tailwind JIT scanning handled correctly.** Using literal string lookups in `BG_CLASSES` and `TEXT_CLASSES` rather than template literals ensures Tailwind v4 can detect all arbitrary-value classes at build time. This is a non-obvious constraint that was handled proactively.

The E2E tests verify runtime scheme switching (AC-1), component color derivation (AC-2), and per-scheme correctness (AC-4). The `waitForTimeout` calls in E2E are properly justified with comments.

#### Medium

- **[Correctness] `src/app/components/reader/readerThemeConfig.ts:53-54` (confidence: 72)**: `getReaderThemeColors` does not guard against an invalid `colorScheme` value. If `getSettings().colorScheme` returns a corrupted string (e.g., `'foobar'`), the chain `THEME_COLORS[theme][colorScheme]` returns `undefined`, and the caller at line 94 (`colors.background`) throws a TypeError. The `??` fallback on `BG_CLASSES` (line 95) only handles unknown hex values, not an `undefined` `colors` object. The root cause is that `settings.ts` validates most enum fields but not `colorScheme` -- however, this module should be defensive regardless. Fix: Add a guard at the top of `getReaderThemeColors`:
  ```ts
  const VALID_SCHEMES: ColorScheme[] = ['professional', 'vibrant', 'clean']
  const safeScheme = VALID_SCHEMES.includes(colorScheme) ? colorScheme : 'professional'
  return THEME_COLORS[theme][safeScheme]
  ```
  Effort: ~2 minutes.

#### Nits

- **[Recurring] [Maintainability] `src/app/components/reader/EpubRenderer.tsx:214` (confidence: 75)**: String interpolation for className (`` `relative h-full w-full ${containerBg} ${animationClass}` ``) instead of `cn()`. This is a recurring pattern across the codebase (tracked since E01-S03). Using `cn()` would handle edge cases like empty strings and merge conflicts better. Fix: `className={cn('relative h-full w-full', containerBg, animationClass)}`. Effort: ~1 minute.

### Recommendations

1. Add the defensive guard in `getReaderThemeColors` to prevent a TypeError on corrupted settings (Medium finding). Quick fix, low risk.
2. Optionally switch to `cn()` in EpubRenderer line 214 if touching the file anyway.
