## Code Review: E107-S04 -- Wire About Book Dialog

### What Works Well

1. **Clean component architecture**: `AboutBookDialog` follows the established `LinkFormatsDialog` pattern exactly -- controlled state via `open`/`onOpenChange`, shadcn/ui Dialog primitives, `useBookCoverUrl` hook for cover resolution. No surprises.

2. **Thorough fallback handling**: Every optional field (`author`, `description`, `isbn`, `fileSize`) has explicit fallback text. The cover fallback differentiates audiobook vs EPUB with appropriate icons. This is good defensive coding.

3. **Consistent menu integration**: The "About Book" item was added to both `ContextMenu` and `DropdownMenu` with proper `data-testid` attributes, maintaining the dual-input pattern for desktop right-click and mobile touch.

### Findings

#### Blockers

- **`tests/e2e/story-e107-s04.spec.ts:16` (confidence: 95)** `[Testing]`: Test file references `libraryPage` fixture which does not exist. The project's merged fixtures (`tests/support/fixtures/index.ts`) only provide `localStorage` and `indexedDB` helpers -- there is no `libraryPage` fixture anywhere in the codebase. Every test in this file will fail at registration time with an "Unknown fixture" error. The fixture would need to provide `goto()`, `openAboutBookDialog()`, `openBookCardContextMenu()`, `openBookListItemContextMenu()`, and `switchToListView()`. Fix: Either create the `libraryPage` page object fixture, or rewrite tests to use raw `page` and `indexedDB` fixtures from the existing merged test.

- **`src/app/components/library/AboutBookDialog.tsx:90-97` (confidence: 95)** `[Testing]`: `data-testid="about-book-author"` is only on the truthy branch (line 91). When `book.author` is falsy, the "Unknown author" fallback (line 95-97) renders WITHOUT that testid. The AC-3 test at `story-e107-s04.spec.ts:75-77` selects `[data-testid="about-book-author"]` and expects "Unknown author" -- but that element will not exist in the DOM for books with empty/missing authors. Fix: Add `data-testid="about-book-author"` to the fallback `<p>` element as well, or move the testid to a wrapper element that exists in both branches.

- **`src/app/components/library/AboutBookDialog.tsx:110-126` (confidence: 95)** `[Testing]`: `data-testid="about-book-description"` is only on the truthy branch (line 115). When `book.description` is falsy, no element has that testid. The AC-3 test at `story-e107-s04.spec.ts:81-83` selects `[data-testid="about-book-description"]` and expects "No description" -- it will fail because the element does not exist. Fix: Add `data-testid="about-book-description"` to the fallback `<p>` at line 124 as well, or use a wrapper element.

#### High Priority

- **`src/app/components/library/AboutBookDialog.tsx:90` (confidence: 75)** `[Correctness]`: `Book.author` is typed as `string` (required, not optional) in `src/data/types.ts:682`. The falsy check `book.author ?` only triggers on empty string `""`, not on `undefined`. If test data never produces an empty-string author, the "Unknown author" fallback is dead code. This is not a runtime bug (empty string still triggers the fallback), but it means the fallback is unlikely to fire with real data since the field is required. The same applies to `book.description` (optional `string?`) which is correct, but `author` should arguably be `author?: string` in the type or the component should check differently. Fix: Either update the Book type to `author?: string` if authors can genuinely be absent, or remove the falsy check since the type guarantees a value. Alternatively, keep it as defensive coding and add a `// Intentional: defensive check for empty string` comment.

#### Medium

- **`src/app/components/library/AboutBookDialog.tsx:35-39` (confidence: 70)** `[Performance]`: `formatFileSize` is defined inside the component body, creating a new function reference on every render. Since this is a pure function that doesn't capture any component state, it should be hoisted outside the component or memoized. Fix: Move `formatFileSize` to module scope (before the component definition). Effort: ~1 minute.

### Recommendations

1. Fix the `libraryPage` fixture issue first -- without it, no tests can run. Either create the page object fixture or rewrite tests to use raw Playwright APIs with the existing `indexedDB`/`localStorage` fixtures.
2. Add `data-testid` attributes to the fallback branches in `AboutBookDialog` so AC-3 tests can locate the fallback elements.
3. Hoist `formatFileSize` outside the component body.
