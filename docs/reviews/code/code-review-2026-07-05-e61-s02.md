### Findings

#### Medium

- **`src/sw.ts:209` (confidence: 75) `[Silent Failure]`**: `fetch('/api/push/subscriptions', ...)` in the `pushsubscriptionchange` handler doesn't check `response.ok`. The `catch` block only handles network errors (offline, DNS failure, connection refused) — HTTP 4xx/5xx responses silently succeed. If the backend returns 503 during a subscription renewal, the subscription is lost without any log, violating the handler's documented intent to "log but don't throw."

  **Why it matters**: During a backend outage, push subscriptions silently disappear. The user won't receive reminders until they re-open the app (when `usePushSubscription` detects the gap). Adding a `response.ok` check would surface the failure in the browser console, helping debug production subscription losses.

  **Fix** (~3 lines):
  ```ts
  const response = await fetch('/api/push/subscriptions', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(newSubscription.toJSON()),
  })
  if (\!response.ok) {
    console.error('[SW] Push subscription POST failed:', response.status, response.statusText)
    return
  }
  ```

### Recommendations

1. Add `response.ok` check to the `pushsubscriptionchange` handler's `fetch` call (Medium, ~3 lines). This is the only actionable finding — the rest of the code is clean.
