## Summary

13 files changed, 1184 insertions, 67 deletions. The integration is architecturally sound — reusing `createOpenAI` with a custom `baseURL` for Ollama's OpenAI-compatible API is the right call (zero new dependencies). The dual connection modes (proxy vs. direct) are cleanly separated. However, two security issues require fixing before shipping.

---

## Findings

### BLOCKER

**B1 — CSP injection via unsanitized user URL** (`src/lib/aiConfiguration.ts:347`, confidence: 92)

`applyOllamaCSP()` inserts the user-provided `baseUrl` directly into the CSP `connect-src` directive via string replacement: `content.replace('connect-src', \`connect-src ${baseUrl}\`)`. A crafted URL like `http://evil.com; script-src 'unsafe-inline'` injects arbitrary CSP directives, effectively disabling CSP protections.

**Fix**: Parse URL with `new URL(baseUrl)` and inject only `url.origin`. Reject URLs that fail parsing.

---

**B2 — SSRF via Ollama proxy** (`server/providers.ts:49-53`, confidence: 90)

The proxy accepts an arbitrary URL from the browser (`apiKey` field) and the server makes HTTP requests to it. An attacker could send requests to `http://169.254.169.254/latest/meta-data/` (cloud metadata), internal network services, or loopback.

**Fix**: Validate URL server-side before proxying — reject non-HTTP(S) schemes, reject private/reserved IPs (127.x, 169.254.x, 10.x, 192.168.x unless explicitly opted in).

---

### HIGH

**H1 — Hardcoded model `'llama3.2'` with no user override** (`src/ai/llm/factory.ts:68`, confidence: 88)

The model is hardcoded in factory (line 68) and `server/providers.ts:53`. Users with `mistral`, `codellama`, `phi3`, etc. cannot use the integration without code changes.

**Fix**: Add `ollamaModel` field to `AIConfigurationSettings` (default: `'llama3.2'`), expose in UI, pass through factory.

---

**H2 — Duplicated CSP logic between `main.tsx` IIFE and `applyOllamaCSP()`** (`src/main.tsx:7-28`, confidence: 85)

The startup IIFE manually reimplements the CSP update logic that already exists in `applyOllamaCSP()`. If the CSP fix for B1 is applied to `applyOllamaCSP()`, the IIFE will not get it.

**Fix**: Import and call `applyOllamaCSP()` from the IIFE instead of duplicating.

---

**H3 — Silent catch in `main.tsx` startup IIFE** (`src/main.tsx:25-27`, confidence: 82)

Empty `catch {}` swallows all errors including JSON parse failures on corrupted localStorage data.

**Fix**: Add `console.warn('[CSP] Failed to apply Ollama CSP:', error)`.

---

**H4 — Contradictory default URL handling** (`src/ai/llm/factory.ts:37-38`, confidence: 80)

`getDecryptedApiKey()` returns `null` when no `ollamaBaseUrl` is set, causing the error "No Ollama URL configured." But the factory falls back to `'http://localhost:11434'` at line 67. Two code paths disagree on whether a missing URL is an error or a valid default.

**Fix**: Either remove the fallback in factory (making URL mandatory) or return the default URL from `getDecryptedApiKey()` when none is configured.

---

### MEDIUM

**M1 — Weak URL validation regex** (`src/lib/aiConfiguration.ts:161`, confidence: 75)

`^https?:\/\/.+` accepts URLs with spaces, control characters, and fragments like `http://foo bar`.

**Fix**: Use `new URL(key.trim())` in a try/catch for proper validation.

---

**M2 — Ollama save path skips connection testing** (`src/app/components/figma/AIConfigurationSettings.tsx:126`, confidence: 72)

For API-key providers, `handleSave` calls `testAIConnection()` before marking `connected`. For Ollama, it immediately sets `connectionStatus: 'connected'` without reachability check.

**Fix**: Add a basic fetch to `ollamaUrl/v1/models` to validate the server is reachable before marking connected.

---

**M3 — HTTP 429 test has no assertion if error isn't thrown** (`src/ai/llm/__tests__/ollama-client.test.ts:153-169`, confidence: 78)

Test uses bare `try/catch` — silently passes if no error thrown.

**Fix**: Use `caughtError` pattern from 401 test, or `expect(async () => ...).rejects.toThrow()`.

---

**M4 — Same issue on NETWORK_ERROR test** (`src/ai/llm/__tests__/ollama-client.test.ts:171-186`, confidence: 78)

Same pattern — assert that an error was actually caught.
