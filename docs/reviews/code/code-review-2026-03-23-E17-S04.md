## Findings

### High Priority

**[HIGH] `src/lib/analytics.ts:335-336` — unanswered questions counted as wrong (confidence: 90)**
`Array.find()` inside `Array.map()` for each attempt creates O(n*m) complexity. More critically, when an attempt has no answer for a question (skipped, quiz structure changed), `answer?.isCorrect` evaluates to `undefined`, mapping to `questionCorrect = 0`. This silently treats missing answers as incorrect for rpb purposes — statistically different from "learner skipped."
**Fix:** Filter out unanswered questions (matching `calculateItemDifficulty`'s behavior), and pre-build an answer lookup map:
```typescript
const answerMap = new Map(attempt.answers.map(a => [a.questionId, a]))
const answer = answerMap.get(question.id)
if (!answer || answer.userAnswer === undefined) continue
```

**[HIGH] `src/app/components/quiz/DiscriminationAnalysis.tsx` — missing `useMemo` (confidence: 80)**
`calculateDiscriminationIndices` is called on every render without memoization. The parent `QuizResults.tsx` already uses `useMemo` for `calculateImprovement` and `calculateNormalizedGain`. The full rpb computation (variance + square root for every question) fires on every Zustand store update.
**Fix:**
```tsx
const discriminationResults = useMemo(
  () => calculateDiscriminationIndices(quiz, attempts),
  [quiz, attempts]
)
```

**[HIGH] Missing negative rpb behavior (confidence: 75)**
The ACs don't define a "Negative discriminator" label, but negative rpb (high scorers getting a question wrong) is the most diagnostic signal the feature provides. Currently all negative values fall into "Low discriminator" silently. No unit test exercises negative rpb. Consider adding a branch and test, or at minimum documenting the behavior.

### Medium

**[MEDIUM] `src/app/components/quiz/DiscriminationAnalysis.tsx:23` — card not using `rounded-[24px]` (confidence: 75)**
Project convention (CLAUDE.md) specifies `rounded-[24px]` for cards. The sibling `ItemDifficultyAnalysis` has the same gap — not a regression, but worth addressing.

**[MEDIUM] `src/lib/__tests__/analytics.test.ts:574` — `makeTestQuiz` duplicates shared factory (confidence: 72)**
`makeTestQuiz` builds a quiz inline, bypassing the `makeQuiz` factory already imported and used in `calculateItemDifficulty` tests. If the `Quiz` type evolves, this helper won't be updated.
**Fix:** Replace with `makeQuiz({ questions: Array.from(...) })`.

**[MEDIUM] `src/lib/__tests__/analytics.test.ts:787` — low discriminator test has thin margin (confidence: 65)**
The computed rpb is ~0.193 vs threshold 0.2. The margin is thin. A clearly negative or near-zero example would be more reliable.

### Nits

**[NIT] `src/app/components/quiz/DiscriminationAnalysis.tsx:29-30`** — `quiz.questions.find()` lookup per rendered item is redundant. `calculateDiscriminationIndices` already iterates `quiz.questions`. Add `questionText` to `DiscriminationResult` type (matching `ItemDifficulty`).

**[NIT] `src/lib/analytics.ts:334`** — Results are in question order but `aria-label` says "ranked by discrimination index." Either sort descending or fix the label.

**[NIT] `analytics.test.ts:625` and `:844`** — Near-duplicate tests using identical 5-attempt data both asserting rpb ≈ 0.894. Consolidate.
