## What Works Well

1. **Clean separation of concerns**: `VectorStorePersistence` neatly encapsulates the dual IndexedDB + in-memory pattern, and the `EmbeddingPipeline` class provides a clear API surface for note indexing. The singleton pattern with `vectorStorePersistence` keeps the architecture simple.

2. **Defensive embedding pipeline**: `embeddingPipeline.indexNote()` wraps all work in try/catch with informative logging, ensuring that embedding failures never block note creation — exactly the right tradeoff for a non-critical enhancement layer.

3. **Proper async cleanup in Notes.tsx**: The semantic search effect at line 152 uses a `cancelled` flag for race condition protection, preventing stale results from overwriting newer ones. The debounced query pattern is also correctly implemented.

## Findings

### Blockers

**[BLOCKER] src/app/pages/Notes.tsx:305,451 — Hardcoded Tailwind colors (confidence: 95)**
`bg-blue-600 text-white hover:bg-blue-700` instead of theme tokens. This is a recurring violation (E03-S05+). Bypasses the design token system, breaking dark mode and preventing global theme changes.
Fix: Replace with `bg-brand text-brand-foreground hover:bg-brand-hover`.

**[BLOCKER] src/app/pages/Notes.tsx:116 — AC7 violated: toggle not hidden when workers unavailable (confidence: 90)**
AC7 states: "the semantic search toggle is hidden" when `typeof Worker === 'undefined'`. The current code computes `semanticSearchAvailable` and disables the toggle, but the toggle, label, and tooltip are always rendered regardless of worker support.
Fix: Wrap the semantic toggle section (lines 408–433) in `{supportsWorkers() && (...)}` to conditionally render it.

**[BLOCKER] src/ai/workers/coordinator.ts:209 — `activeRequests` never decremented (confidence: 95)**
`activeRequests` is incremented in `updateWorkerActivity` but never decremented in `resolvePendingRequest` or `rejectPendingRequest`. The idle termination check (`entry.activeRequests === 0`) will never be true after the first request — workers are never terminated despite the 60-second idle timeout. Memory leak that compounds over a session.
Fix: Decrement `activeRequests` in both `resolvePendingRequest` and `rejectPendingRequest`.

### High Priority

**[HIGH] src/ai/workers/coordinator.ts:291-294 — `handleWorkerError` rejects ALL pending requests (confidence: 92)**
When one worker crashes, every in-flight request to every other worker is rejected with "Worker crashed."
Fix: Filter `pendingRequests` by the crashed worker's type before rejecting.

**[HIGH] src/ai/vector-store.ts:28-29 — Non-atomic dual-write (confidence: 82)**
If `db.embeddings.put(record)` succeeds but `this.store.insert(noteId, embedding)` throws (e.g., dimension mismatch), the embedding persists in IndexedDB but is absent from memory. On next load, `loadAll()` will silently fail to re-insert it.
Fix: Validate dimensions before the DB write, or rollback via `db.embeddings.delete(noteId)` on `insert` failure.

**[HIGH] src/ai/workers/embedding.worker.ts:37 — Mock embeddings return all-zero vectors (confidence: 85)**
`new Float32Array(384)` all-zeros means all mock embeddings are identical — every note gets similarity 0 to every query, making semantic search non-functional at the mock stage. The E2E test works only because it seeds pre-computed non-zero vectors, masking this.
Fix: Generate deterministic non-zero mock vectors (e.g., hash-based) so the pipeline is testable end-to-end without pre-seeded data.

**[HIGH] src/app/pages/Notes.tsx:116 — `semanticSearchAvailable` is not reactive (confidence: 80)**
Computed as a plain `const` outside any hook, reading `vectorStorePersistence.size` synchronously. If Notes renders before `loadAll()` completes (async in App.tsx useEffect), the toggle will be permanently disabled for that render cycle.
Fix: Track vector store readiness in a Zustand store or state variable that updates after `loadAll()` resolves.
