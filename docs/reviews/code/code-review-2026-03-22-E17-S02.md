## Findings

### [High] H1 — Wrong localStorage key for sidebar dismissal in E2E tests

**File**: `tests/e2e/regression/story-e17-s02.spec.ts:24`
**Confidence**: 92

The test uses `'eduvi-sidebar-v1'` but the canonical key is `'knowlune-sidebar-v1'`. While `Layout.tsx` has a one-time migration from the legacy key, this relies on that migration remaining in place. If the migration code is ever removed, all E17-S02 E2E tests will break because the sidebar will remain open, potentially obscuring the Reports page content.

13 out of 15 other test files use `knowlune-sidebar-v1`. This is an outlier.

**Fix**: Change to `localStorage.setItem('knowlune-sidebar-v1', 'false')`.

---

### [High] H2 — `interpretRetakeFrequency` has no guard for 0 or negative input

**File**: `src/lib/analytics.ts:232`
**Confidence**: 78

When `averageRetakes` is 0, `avg <= 1.0` returns `true`, producing "No retakes yet — each quiz taken once." — semantically wrong for zero attempts. The UI guards this in Reports.tsx with `totalAttempts === 0`, but the function is a public export callable independently.

**Fix**: Add `if (avg <= 0) return 'No quizzes attempted yet.'` at the top, or add JSDoc documenting that callers must not pass 0.

---

### [High] H3 — `vi.mock('@/db')` blast radius covers all describe blocks in the file

**File**: `src/lib/__tests__/analytics.test.ts:18-24`
**Confidence**: 82

The module-level `vi.mock('@/db')` is hoisted by Vitest and applies to every `describe` block including unrelated suites. The `beforeEach(() => vi.clearAllMocks())` is scoped only inside `calculateRetakeFrequency`, so mock state can leak to other tests in theory.

**Fix**: Move retake frequency tests to `analytics-retake.test.ts`, or use `vi.hoisted()` with a factory scoped to `quizAttempts`.

---

### [Medium] M1 — Missing boundary tests at 2.0 and 3.0 for `interpretRetakeFrequency`

**File**: `src/lib/__tests__/analytics.test.ts:497-517`
**Confidence**: 72

Tests cover 1.0, 1.5, 2.5, 4.0 but not the boundary values 2.0 and 3.0, where off-by-one errors in `<=` vs `<` comparisons hide.

**Fix**: Add `interpretRetakeFrequency(2.0)` (expect "Light review") and `interpretRetakeFrequency(3.0)` (expect "Active practice") tests.

---

### [Medium] M2 — `db.quizAttempts.toArray()` loads entire table into memory

**File**: `src/lib/analytics.ts:213`
**Confidence**: 70

Loads all records to count unique quiz IDs and total count. Acceptable for a personal app today, but O(n) memory cost.

**Fix**: Consider `db.quizAttempts.count()` + Dexie `uniqueKeys()` for a more efficient approach, or document acceptable scope.

### [Medium] M3 — Empty-state E2E test assumes no other data sources trigger `hasActivity`

**File**: `tests/e2e/regression/story-e17-s02.spec.ts:89-98`
**Confidence**: 68

`hasActivity` checks `completedLessons`, `studyNotes`, and activity data beyond quizAttempts. If any of those leak between tests, the empty state won't render and this test will fail. The `afterEach` only clears `quizAttempts`.

**Fix**: Clear all relevant IndexedDB stores in `afterEach`, or make the assertion specific to the retake card state rather than the global empty state.
