## Blocker

### [Blocker] Duplicate React keys in sidebar render — `Layout.tsx:122` (confidence: 95)

Three navigation items now share `path: '/reports'`, but `key={item.path}` means React receives three siblings with the same key (`"/reports"`). Only one of the three Report links will render reliably, React's reconciliation produces unpredictable results on re-renders, and React emits a console warning.

**Fix:** Use a composite key: `key={item.tab ? \`${item.path}?tab=${item.tab}\` : item.path}` or use `item.name`.

---

## High Priority

### [High] BottomNav duplicate keys + missing tab awareness — `navigation/BottomNav.tsx:82` (confidence: 90)

`BottomNav` uses `key={item.path}` (line 82) and its local `isActive` function (lines 16–18) uses only `location.pathname.startsWith(path)` without tab awareness. All three Reports items share `path: '/reports'`, so:
- Duplicate React keys in the mobile drawer
- All three links simultaneously marked active when on any `/reports` URL
- `<Link to={item.path}>` on line 84 strips the `?tab=` parameter entirely

**Fix:** Update key to `item.tab ? \`${item.path}?tab=${item.tab}\` : item.path`, import and use `getIsActive` from `navigation.ts`, and pass `?tab=` in the link href.

### [High] SearchCommandPalette uses same icon for all three entries — `figma/SearchCommandPalette.tsx:83–101` (confidence: 80)

All three Report entries use `BarChart3`. The sidebar uses distinct icons per tab (`BarChart3`, `ClipboardList`, `BrainCircuit`). No visual differentiation when searching "analytics".

**Fix:** Use the same icons as the sidebar — `ClipboardList` for Quiz Analytics and `BrainCircuit` for AI Analytics.

### [High] Hardcoded default tab `'study'` creates hidden coupling — `navigation.ts:35` (confidence: 85)

`getIsActive` hardcodes `item.tab === 'study'` as the default tab. If the Reports page default tab ever changes, the sidebar active state silently desyncs.

**Fix:** Add `defaultTab?: boolean` to `NavigationItem` and set `defaultTab: true` on Study Analytics. Use `item.defaultTab === true && search === '' && pathname === item.path` in `getIsActive`.

## Medium

`NavLink` uses template literals for conditional classes instead of the project's `cn()` utility. Recurring pattern flagged since E01-S03.

**Fix:** Use `cn()` for class merging.

Recurring since E02-S05. Tailwind v4 prefers `size-5`.

### [Medium] Missing edge case test for unknown tab values — `NavLink.test.tsx` (confidence: 72)

No test for `getIsActive({ path: '/reports', tab: 'study' }, '/reports', '?tab=invalid')`.

**Fix:** Add test: `it('does not activate study when tab param is unrecognized', () => { ... })`.

### [Medium] `BottomNav.isActive` ignores `tab` field — `BottomNav.tsx:16–18` (confidence: 75)

Even after the duplicate-key fix, the `isMoreActive` check will incorrectly trigger for all three Report items at once.

**Fix:** Import and use `getIsActive` from `navigation.ts`, passing `location.search`.
