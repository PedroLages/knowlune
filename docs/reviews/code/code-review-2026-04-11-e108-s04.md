## Findings

### MEDIUM

#### M1: `skipSilence` and `defaultSleepTimer` not wired to behavior (AC-3, AC-4 partial)
**File:** `src/app/components/audiobook/AudiobookRenderer.tsx`

The settings panel allows toggling "Skip Silence" and selecting a "Default Sleep Timer", and these are persisted to localStorage. However, neither preference has any functional effect:
- `skipSilence` is never read by any audio processing code
- `defaultSleepTimer` is never used to auto-start a sleep timer on session start (Task 5 in story)

The story notes acknowledge skip silence is non-trivial, but `defaultSleepTimer` integration (Task 5.1) appears to be missing. The existing `useSleepTimer` hook is already available for wiring.

**Recommendation:** Wire `defaultSleepTimer` to auto-start the sleep timer in the session-start `useEffect`. For `skipSilence`, add a comment noting it's UI-ready but the audio processing backend is deferred.

#### M2: Auto-bookmark `.catch()` triggers ESLint warning
**File:** `src/app/components/audiobook/AudiobookRenderer.tsx:145`

The catch block uses `// Intentional:` comment but the ESLint rule `error-handling/no-silent-catch` expects `// silent-catch-ok`. This causes a lint warning on story-changed code.

**Recommendation:** Change comment to include `// silent-catch-ok` to suppress the warning.

### LOW

#### L1: `setDefaultSpeed` clamping vs validation inconsistency
**File:** `src/stores/useAudiobookPrefsStore.ts:93-94`

`setDefaultSpeed` clamps to [0.5, 3.0] range but doesn't snap to 0.25 increments (per AC-2). A programmatic call like `setDefaultSpeed(1.3)` would persist 1.3, which isn't in `VALID_SPEEDS` and would reset to 1.0 on next load. Since the UI only offers preset buttons this is unlikely to be hit in practice, but the asymmetry between write (accepts any float) and read (validates against VALID_SPEEDS) is surprising.

#### L2: `SPEED_OPTIONS` in UI differs from `VALID_SPEEDS` in store
**File:** `src/app/components/audiobook/AudiobookSettingsPanel.tsx:31` vs `src/stores/useAudiobookPrefsStore.ts:41-43`

UI offers `[0.5, 0.75, 1.0, 1.25, 1.5, 1.75, 2.0, 2.5, 3.0]` (9 values) but the store validates against a set that also includes `2.25` and `2.75` (11 values). Not a bug, but could be confusing if someone loads persisted data with 2.25 — the UI wouldn't show it as selected.

#### L3: Missing E2E test
**Story AC-7 (Task 7):** The story specifies E2E tests for opening the settings panel and verifying speed persistence. No E2E spec was created.

### NIT

#### N1: `formatSpeed` could use simpler logic
**File:** `src/app/components/audiobook/AudiobookSettingsPanel.tsx:42-43`

`Number.isInteger(rate)` is clearer than `rate % 1 === 0`.
