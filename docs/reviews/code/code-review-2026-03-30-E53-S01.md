## Findings

### MEDIUM — `new Date().toISOString()` in bookmarkExport.ts is non-deterministic

**File:** `src/lib/bookmarkExport.ts:99`
**Issue:** `const exportedAt = new Date().toISOString()` makes output non-deterministic. While the test handles this with a regex match (`/exported: "\d{4}-\d{2}-\d{2}T/`), this means the exact output can never be snapshot-tested. The existing `exportService.ts` has the same pattern, so this is consistent — but worth noting for future refactoring.
**Recommendation:** Accept as-is for consistency with existing codebase patterns. Consider injecting a `now?: Date` parameter in a future refactor.

### MEDIUM — YAML frontmatter tags not escaped in flashcardExport.ts

**File:** `src/lib/flashcardExport.ts:83`
**Issue:** Tag values are interpolated as `"${t}"` without escaping internal quotes. If a tag contains a double quote (e.g., from a note tag), the YAML frontmatter would be malformed:
```yaml
tags: ["react-mastery", "say "hello""]  # broken YAML
```
**Recommendation:** Apply the same `yamlString()` escaping to tag values:
```typescript
`tags: [${tags.map(t => yamlString(t)).join(', ')}]`
```
**Severity rationale:** MEDIUM because tags are derived from course names (kebab-cased, safe) and note tags (normalized lowercase/trimmed per `Note.tags` definition). The probability of quotes in tags is very low but non-zero.

### LOW — Duplicated `yieldToUI()` helper

**File:** `src/lib/flashcardExport.ts:16-18`, `src/lib/bookmarkExport.ts:14-16`
**Issue:** Both files define an identical `yieldToUI()` function. This same function also exists (as an inline concept) in `exportService.ts`.
**Recommendation:** Extract to a shared utility (e.g., `src/lib/exportUtils.ts`) in a future cleanup story. Not blocking for this story.

### LOW — `formatTimestamp` does not handle negative values explicitly

**File:** `src/lib/bookmarkExport.ts:27`
**Issue:** `Math.abs(seconds)` silently converts negative timestamps to positive. While bookmarks should never have negative timestamps, silently normalizing invalid data hides bugs upstream. No test covers the negative case.
**Recommendation:** Accept as defensive coding. The `Math.abs` is reasonable for a display function.
