## Findings

### STORY-RELATED

#### MEDIUM

1. **Bloom's drop logic only checks single previous answer, not consecutive count** — `useTutorStore.ts:445-447`. The AC says "drop one level after 2 consecutive incorrect." The current logic drops when `!correct && lastAnswerCorrect === false`, which is correct for detecting 2 consecutive. However, `currentStreak` resets to 0 on incorrect, so there's no explicit "incorrect streak" counter. If 3+ consecutive incorrect happen, it only drops once (on the 2nd), then `lastAnswerCorrect` is already `false` so the 3rd incorrect drops again — which is actually correct behavior. On closer inspection this works as intended. **Downgraded to NON-ISSUE.**

2. **SCORE regex may match in middle of LLM prose** — `useTutor.ts:350`. The regex `^SCORE:\s*(correct|incorrect)/im` uses multiline mode (`m` flag), so `^` matches start of any line. This is intentional per the prompt ("on its own line"). However, if the LLM outputs something like "SCORE: correct" as part of quoting the format instructions, it would trigger a false positive. LOW risk since the prompt says "after evaluating" and the regex only runs in quiz mode.

#### LOW

1. **Pulse animation duration mismatch with visual perception** — `QuizScoreTracker.tsx:39`. The `setTimeout` is 100ms which matches the `duration-100` CSS transition, but 100ms is very fast — users may not perceive the scale-105 pulse. The AC specifies 100ms, so this follows the spec. Consider bumping to 200-300ms in a future polish pass.

2. **`buildTutorSystemPrompt` signature is getting long (8 params)** — `tutorPromptBuilder.ts:120-127`. Adding `bloomLevel` as the 8th positional parameter is functional but increasingly unwieldy. Consider an options object in a future refactor. Not blocking.
