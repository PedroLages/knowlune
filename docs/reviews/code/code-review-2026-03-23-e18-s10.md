## Code Review: E18-S10 — Export Quiz Results

### What Works Well

1. **Clean architecture separation**: The export logic in `src/lib/quizExport.ts` is entirely decoupled from the UI component. Pure functions for CSV generation (`buildAttemptsCsv`, `buildQuestionsCsv`) are independently testable with no side effects. The `QuizExportCard` only orchestrates — it doesn't own export logic. This is textbook separation of concerns.

2. **Solid lazy-loading strategy for jsPDF**: Dynamic `import()` of `jspdf` and `jspdf-autotable` (lines 194-195) keeps the ~391KB chunk out of the initial bundle. The lessons-learned section documents this trade-off explicitly. The functional API (`autoTable(doc, opts)`) instead of prototype augmentation is the right call to avoid import ordering issues.

3. **Thorough unit test coverage (24 cases)**: The unit tests cover formatter edge cases (0ms, boundary padding), empty-state handling, per-quiz filtering, RFC 4180 CSV quoting for commas, multi-select answer formatting, and both export functions. The mock structure (`setupAttemptsWhere`) is clean and reusable.

### Findings

#### Blockers

- **`src/lib/quizExport.ts:93-100` (confidence: 85)**: **CSV formula injection vulnerability.** The `escapeCsv` function handles RFC 4180 delimiters (commas, quotes, newlines) but does not sanitize formula-injection payloads. If a quiz title or question text starts with `=`, `+`, `-`, `@`, `\t`, or `\r`, spreadsheet applications (Excel, Google Sheets, LibreOffice) will interpret the cell as a formula. Since quiz titles and question text are user-generated content stored in IndexedDB, a self-crafted quiz titled `=HYPERLINK("http://evil.com","Click Here")` would pass through `escapeCsv` unmodified and execute when opened. **Why**: A learner opens their own export in a spreadsheet and gets unexpected formula execution — or a shared export could be weaponized in educational contexts where instructors review student quiz data. **Fix**: Prefix values starting with `=`, `+`, `-`, `@`, `\t`, or `\r` with a single quote or tab character. Standard mitigation:
  ```typescript
  const FORMULA_CHARS = /^[=+\-@\t\r]/
  function escapeCsv(value: unknown): string {
    if (value === null || value === undefined) return ''
    let str = String(value)
    if (FORMULA_CHARS.test(str)) str = `'${str}` // or prefix with tab
    if (str.includes(',') || str.includes('"') || str.includes('\n') || str.includes('\r')) {
      return `"${str.replace(/"/g, '""')}"`
    }
    return str
  }
  ```

#### High Priority

- **[Recurring] `src/app/components/reports/QuizExportCard.tsx:51` (confidence: 90)**: **Silent failure on IndexedDB count query.** The `.catch(err => console.error(...))` swallows the error — the user sees 0 attempts and a permanently disabled export button with no indication that something went wrong. If IndexedDB is temporarily unavailable (private browsing quota exceeded, storage pressure), learners with actual quiz data would be unable to export with no feedback. This matches the recurring silent-failure anti-pattern tracked in agent memory since E03-S03. **Why**: A learner who has completed quizzes sees "no attempts" and a disabled button with a misleading tooltip ("Complete a quiz to enable export") — they think they have no data instead of knowing the system had an error. **Fix**: Add error state to the component and surface it:
  ```typescript
  const [loadError, setLoadError] = useState(false)
  // In the catch:
  loadCounts().catch(err => {
    console.error('Failed to load quiz counts:', err)
    if (!ignore) setLoadError(true)
  })
  // In the JSX, show an error state instead of the disabled button
  ```

- **`src/lib/quizExport.ts:170,189` (confidence: 78)**: **Non-deterministic `new Date()` in library code makes unit testing fragile.** The `exportQuizResultsCsv` (line 170) and `exportQuizResultsPdf` (line 189) both call `new Date().toLocaleDateString('sv-SE')` for the filename. The unit tests work around this with regex matching (`/\d{4}-\d{2}-\d{2}/`), which means the tests cannot verify the exact date in the filename — they only verify the shape. If a bug caused the date to always be `1970-01-01`, the test would still pass. More importantly, if a test runs across a date boundary (midnight), the regex still passes but with an unexpected date. **Why**: Reduces test precision. If the date formatting logic breaks, tests won't catch it. **Fix**: Accept an optional `dateStr` parameter (or use a `getNowFn` injector) so tests can pin the date:
  ```typescript
  export async function exportQuizResultsCsv(dateStr?: string): Promise<void> {
    const date = dateStr ?? new Date().toLocaleDateString('sv-SE')
    // ...
  }
  ```

- **`src/lib/quizExport.ts:84` (confidence: 72)**: **`Math.max(...scores)` will throw `RangeError` on very large arrays.** The spread operator pushes all array elements onto the call stack. V8's limit is ~65,536 arguments. While unlikely in a personal learning app, a learner who imports quiz data or uses the app over years could accumulate enough attempts to hit this limit. The early return at line 78 handles the empty case, but very large arrays are unguarded. **Why**: An enthusiastic learner who has taken thousands of quizzes over time gets a cryptic `RangeError: Maximum call stack size exceeded` instead of their export. **Fix**: Use `scores.reduce((max, s) => s > max ? s : max, -Infinity)` or a simple loop.

#### Medium

- **`src/app/components/reports/QuizExportCard.tsx:125` (confidence: 75)**: **`tabIndex={0}` on span wrapper creates unexpected tab stop.** The disabled export button's span wrapper (`<span className="inline-block" tabIndex={0}>`) is focusable via keyboard Tab. When a keyboard user tabs through the Reports page, they land on this span (which has no visible focus indicator), then the button inside (which has `tabIndex={-1}`). The span isn't a semantic interactive element. **Fix**: Add a focus-visible ring to the span so keyboard users know where focus is, or use `tabIndex={0}` only with appropriate `role="button"` and `aria-disabled`:
  ```tsx
  <span className="inline-block focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 rounded-xl" tabIndex={0} role="group" aria-label="Export quiz results (disabled)">
  ```

- **`tests/e2e/e18-s10-export-quiz-results.spec.ts:103-106,135-137,166-168,238-240` (confidence: 70)**: **`waitForLoadState('networkidle')` is fragile.** Playwright's `networkidle` waits for no network requests for 500ms. This is a heuristic that can fail if background analytics, service workers, or long-polling connections fire. While this pattern is used across the codebase (85 occurrences in 14 files), it's still an anti-pattern that causes intermittent failures. **Fix**: Replace with specific element assertions:
  ```typescript
  await page.reload()
  await expect(page.getByTestId('quiz-export-card')).toBeVisible()
  ```

- **`src/lib/quizExport.ts:57-62` (confidence: 68)**: **`formatTimeSpent` duplicates existing `formatDuration` utility.** The codebase already has `src/lib/formatDuration.ts` which converts milliseconds to human-readable strings (`"8m 32s"`). `formatTimeSpent` outputs `"M:SS"` format instead. While the CSV/PDF format may benefit from the more compact `M:SS` representation, this is a new time-formatting function that should either extend or delegate to `formatDuration`, or be documented as deliberately different. **Fix**: Consider reusing `formatDuration` for PDF (where human-readability matters) and keeping `formatTimeSpent` only for CSV columns. Or add a JSDoc note explaining why the different format is needed for export.

- **`src/lib/quizExport.ts:57-62` (confidence: 65)**: **`formatTimeSpent` does not handle negative or NaN inputs.** `Math.floor(NaN / 1000)` produces `NaN`, and `Math.floor(-5000 / 1000)` produces `-5`, yielding output like `NaN:NaN` or `-1:-55`. While `QuizAttemptSchema` constrains `timeSpent` to `min(0)`, IndexedDB data isn't validated on read — corrupted or manually-edited data could trigger this. **Fix**: Add a guard: `const totalSeconds = Math.max(0, Math.floor(ms / 1000))` (matching the existing pattern in `formatDuration.ts` line 8).

#### Nits

- **Nit** `src/lib/quizExport.ts:289` (confidence: 60): Question text truncated to 60 characters in PDF (`qText.substring(0, 60)`) with hard-coded limit. This is a reasonable PDF layout decision, but the magic number could be a named constant for clarity. Consider `const PDF_QUESTION_TEXT_MAX_LENGTH = 60`.

- **Nit** `tests/e2e/e18-s10-export-quiz-results.spec.ts:282` (confidence: 55): File is 282 lines, approaching the 300-line E2E spec limit noted by the validator. The four test describes share identical `beforeEach` setup (AC1/AC2/AC3). Consider extracting the seed + reload + waitForLoad into a shared fixture helper to reduce repetition and stay under the limit.

- **Nit** `src/app/components/reports/QuizExportCard.tsx:89-90` (confidence: 55): Pluralization logic (`totalAttempts === 1 ? 'attempt' : 'attempts'`) is inline. The codebase may benefit from a shared `pluralize(count, singular, plural?)` utility if this pattern recurs.

### Recommendations

1. **(Blocker)** Fix CSV formula injection in `escapeCsv` — this is a real security concern for user-generated quiz content.
2. **(High)** Replace `console.error` with user-facing error state in `QuizExportCard` count loading. The misleading disabled state erodes trust.
3. **(High)** Make `new Date()` injectable in export functions for test determinism, or accept the regex-matching trade-off with a comment explaining the choice.
4. **(High)** Replace `Math.max(...scores)` with a reduce/loop to prevent call stack overflow on large datasets.
5. **(Medium)** Add focus styling to the disabled-button span wrapper, or rethink the `tabIndex={0}` approach.
6. **(Medium)** Guard `formatTimeSpent` against negative/NaN inputs to match `formatDuration`'s defensive style.
