## Summary

Story implements dual-format position tracking for linked EPUB/audiobook pairs: `linkedBookId` field on `Book`, pure `chapterSwitchResolver` utility, `updateBookPosition`/`linkBooks` store methods, and "Also available as" badges in BookCard/BookListItem.

## Findings

### MEDIUM

1. **Non-atomic `linkBooks` Dexie writes** (`src/stores/useBookStore.ts:256-262`)
   Two sequential `db.books.update()` calls without `db.transaction()`. If the second fails, book A links to B but B does not link back to A. Recommend wrapping in `db.transaction('rw', db.books, async () => { ... })`.

2. **TypeScript IIFE workaround** (`src/app/components/library/BookCard.tsx:21-25`)
   IIFE used to narrow `ContentPosition` union through computed array index. Works correctly but is unusual. A named helper would improve readability.

### LOW

3. **`new Date()` in store methods** — Not testable without mocking globals. Consider optional `now` parameter.

4. **`as Parameters<>` type assertions** (4 occurrences in useBookStore.ts) — Bypasses Dexie typing for `linkedBookId` field. Indicates schema types may be out of sync.
