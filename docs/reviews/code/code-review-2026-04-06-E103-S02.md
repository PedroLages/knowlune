## Findings

### HIGH — Position not saved before format switch (Story-Related)

**File:** `src/app/pages/BookReader.tsx` lines 592, 628
**AC:** AC2 ("audiobook position is saved before switching"), AC4 ("EPUB reading position (CFI) is saved before switching")

`switchToFormat()` is called without the `savePosition` callback in both wiring points. The hook's design accepts `savePosition` as an optional second argument but it is never provided. The debounced position save on page turns may not have flushed yet.

**Fix:** Pass a `savePosition` callback that flushes the current position to Dexie/BookStore before navigation. For audiobook: save current time from `useAudioPlayer`. For EPUB: save current CFI.

**Severity:** HIGH — acceptance criteria explicitly require position persistence before switch.

### MEDIUM — `setTimeout` for switching guard reset (Story-Related)

**File:** `src/app/hooks/useFormatSwitch.ts` line 108-110

```typescript
setTimeout(() => {
  switchingRef.current = false
}, 500)
```

Hard-coded 500ms timeout to reset the double-click guard. If navigation takes longer, the guard resets prematurely. If the component unmounts during the timeout, it's a no-op (ref, not state) so no React warning, but the pattern is fragile.

**Fix:** Consider resetting via a route change listener or simply not resetting (navigation replaces the component).

**Severity:** MEDIUM — unlikely to cause user-visible bugs but a code smell.

### LOW — `useEffect` with empty deps for startChapter cleanup (Pre-Existing Pattern)

### NIT — Unused `closest` variable check (Story-Related)

**File:** `src/app/hooks/useFormatSwitch.ts` line 98

`targetIndex = closest ? closestIdx : 0` — `closest` is always truthy here since it's initialized to `mapping.mappings[0]` and the loop only reassigns it. The ternary is redundant.

**Severity:** NIT
