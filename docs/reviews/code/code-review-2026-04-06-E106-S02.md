## Findings

### High Priority

1. **[Correctness] [src/services/__tests__/AudiobookshelfService.test.ts:100] (confidence: 95)**: Tests assert CORS-specific error message `"...CORS settings."` but the production code returns `"...try again."`. The service was refactored to use a proxy (`/api/abs/proxy/...`) instead of direct CORS requests, so the error message changed. Six tests fail because of this mismatch.

   **Affected tests:**
   - `:100` — `testConnection` returns CORS error for TypeError
   - `:263` — `fetchLibraryItems` returns CORS error for network failure
   - `:554` — `fetchCollections` returns CORS error for network failure

   **Fix:** Update expected error messages from `"Could not connect to server. Check the URL and CORS settings."` to `"Could not connect to server. Check the URL and try again."` (~3 lines).

2. **[Correctness] [src/services/__tests__/AudiobookshelfService.test.ts:79] (confidence: 95)**: Test asserts `options.headers.Authorization` is `Bearer ${TEST_API_KEY}`, but the production code sends `X-ABS-URL` and `X-ABS-Token` headers (proxy-based auth, not Bearer). The `Authorization` header is not set by the service.

   **Fix:** Assert `options.headers['X-ABS-Token']` equals `TEST_API_KEY` instead (~1 line).

3. **[Correctness] [src/services/__tests__/AudiobookshelfService.test.ts:303,321] (confidence: 95)**: Tests for `getStreamUrl` and `getCoverUrl` assert direct server URLs like `http://abs.local:13378/api/items/item-1/play?token=...` but the production code returns proxy paths like `/api/abs/proxy/api/items/item-1/play?token=...&_absUrl=...&_absToken=...`.

   **Fix:** Update expected URL patterns to match proxy-based routing (~2 assertions).

4. **[Testing] [src/services/__tests__/AudiobookshelfService.test.ts:696] (confidence: 90)**: ESLint error `@typescript-eslint/no-this-alias` at line 696 (`createdWs = this` inside WebSocket constructor mock). This is the only ESLint **error** in the story's changed files. While `no-this-alias` exists for good reasons, this mock context requires capturing `this`. 

   **Fix:** Either add `// eslint-disable-next-line @typescript-eslint/no-this-alias` or restructure the mock to avoid aliasing (e.g., assign to outer variable via `Object.assign`).
