### Findings

#### High Priority

- **`src/lib/analytics.ts:254-261` (confidence: 82)**: Skipped/unanswered questions are counted as incorrect attempts in the P-value calculation. The `analyzeTopicPerformance` function at line 64 explicitly checks `isUnanswered(answer.userAnswer)` and treats skipped answers differently, but `calculateItemDifficulty` blindly trusts `answer.isCorrect` without checking whether the learner actually attempted the question. A skipped answer has `isCorrect: false` and inflates the denominator, making questions appear harder than they actually are. Why: A learner who skips a question due to time pressure (not lack of knowledge) sees a misleadingly low P-value, undermining the "understand which concepts need more practice" goal. Fix: Check `isUnanswered(answer.userAnswer)` and either exclude unanswered responses from both numerator and denominator, or document the intentional choice to treat skips as wrong answers (matching the scoring engine's convention).

- **`src/app/components/quiz/ItemDifficultyAnalysis.tsx:67` / `src/app/pages/QuizResults.tsx:144` (confidence: 85)**: The `ItemDifficultyAnalysis` card inherits `text-center` from the parent container at `QuizResults.tsx:144`. The question text in each `<li>` will be center-aligned, which looks wrong for a left-to-right ranked list. The `flex justify-between items-center` controls horizontal layout but does not override `text-align`. Why: Center-aligned question text in a ranked list breaks visual scanning -- learners expect left-aligned text in data tables/lists. Fix: Add `text-left` to the `<Card>` or `<ul>` element, e.g., `<Card className="text-left">`.

- **`src/app/components/quiz/ItemDifficultyAnalysis.tsx:50-55` (confidence: 78)**: `calculateItemDifficulty` is called directly during render with no memoization. Every parent re-render (e.g., from Zustand store updates, which happen on attempt load) re-runs the aggregation loop across all attempts and questions. Why: With many attempts this is O(attempts * answers) work on every render -- not a concern for 5 attempts, but the component sits in `QuizResults` which re-renders during async loading. Fix: Wrap in `useMemo`:
  ```tsx
  const items = useMemo(() => calculateItemDifficulty(quiz, attempts), [quiz, attempts])
  ```

### Recommendations

1. **First**: Fix the `text-center` inheritance issue -- this is visually broken today and will be immediately visible to learners.
2. **Second**: Decide on the skipped-answer semantics for P-value calculation. Either add the `isUnanswered` check or add a code comment documenting the intentional design choice. This affects data accuracy.
3. **Third**: Add `useMemo` around `calculateItemDifficulty` for consistency with the rest of `QuizResults` (every other derived computation is memoized).
4. **Fourth**: Add `aria-label` to the suggestions list and an E2E test for AC4.
