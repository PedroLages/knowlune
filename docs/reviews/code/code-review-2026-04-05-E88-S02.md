## Story-Related Issues

### MEDIUM

1. **Missing ESLint deps in useEffect (OpdsBrowser.tsx:300-306)** -- The `useEffect` at line 300 references `selectedCatalog`, `fetchFeed`, and implicitly `setBreadcrumbs`/`setCurrentFeedUrl` but the dependency array is `[selectedCatalogId]` only. The `// Only refetch when catalog selection changes` comment acknowledges this is intentional, but `fetchFeed` is a `useCallback` with `[]` deps so it is stable. The real issue is `selectedCatalog` which is derived from `selectedCatalogId` + `catalogs` -- if `catalogs` changes shape without `selectedCatalogId` changing, stale data could be used. This is a minor edge case since catalogs rarely change while browsing.

2. **Redundant source.type check in isAlreadyInLibrary (OpdsBrowser.tsx:87-88)** -- The inner `entry.acquisitionLinks.some()` callback re-checks `b.source.type === 'remote'` which is already guaranteed by the outer condition. Not a bug, but dead code.

3. **Auth credentials stored in Book record (OpdsBrowser.tsx:425-429)** -- When adding a book to library, the catalog auth credentials (username/password) are copied into each Book's `source.auth`. This means credentials are duplicated per-book in IndexedDB. If the user changes catalog credentials, previously-added books retain the old credentials. Consider referencing the catalog ID instead.

### LOW

4. **Component size warning (OpdsBrowser.tsx)** -- ESLint reports 518 non-blank lines vs 500 threshold. The sub-components (EntrySkeletons, NavigationCard, OpdsBookCard, BreadcrumbTrail) are already extracted as local components, which is good. Consider extracting them to separate files if the component grows further.

5. **handleAddToLibrary does not handle importBook errors (OpdsBrowser.tsx:435)** -- `await importBook(book)` is not wrapped in try/catch. If `importBook` throws, the `addingIds` cleanup at lines 438-442 never executes, leaving the button in a permanent loading state. The toast.success would also not fire.

6. **getBookFormat defaults to 'pdf' (OpdsBrowser.tsx:75-80)** -- If an entry has no epub link but also no pdf link (e.g., MOBI only), the format is incorrectly set to 'pdf'. This is a minor edge case since most OPDS catalogs serve EPUB or PDF.

### NITS

7. **Unused parameter prefix convention (OpdsBrowser.tsx:333)** -- `_title` parameter in `handleNavigateToFeed` is prefixed with underscore correctly.

8. **BreadcrumbTrail uses index as key (OpdsBrowser.tsx:236)** -- Using array index as React key is fine here since breadcrumbs are append-only and never reordered.
