### Findings

#### High Priority

- **QuestionGrid.tsx:22-24 (confidence: 90)**: `isAnswered` check only handles `string` answers. The type is `Record<string, string | string[]>`, but the check `answers[questionId] !== undefined && answers[questionId] !== ''` does not catch empty arrays. For `multiple-select` question types (defined at `src/types/quiz.ts:27`), an answer of `[]` passes this check and the bubble appears "answered" (blue styling) when nothing is actually selected. The same flaw exists in `countUnanswered` at `Quiz.tsx:59` where `a === undefined || a === ''` would not catch `[]`, meaning the submit confirmation dialog would not trigger for empty multi-select answers. Why: Learners taking multi-select quizzes see misleading visual feedback and miss the unanswered warning. Fix:
  ```typescript
  const a = answers[questionId]
  const isAnswered = a !== undefined && a !== '' && !(Array.isArray(a) && a.length === 0)
  ```
  Apply the same pattern to `countUnanswered` in `Quiz.tsx`.

- **[Recurring] QuestionGrid.tsx:34 (confidence: 90)**: Uses `w-8 h-8` instead of Tailwind v4 shorthand `size-8`. Recurring pattern flagged since E02-S05. Fix: Replace with `size-8`.

- **QuestionGrid.tsx:34-35 (confidence: 82)**: Conflicting sizing declarations. The button has `w-8 h-8` (32px) and `min-w-[44px] min-h-[44px]` (44px) simultaneously. The `min-*` constraints always win, making `w-8 h-8` dead code. The story design guidance (line 102) calls for 32px visual circles with 44px touch targets achieved "via padding/flex alignment," but the implementation makes the entire button 44px. Why: For quizzes with 15+ questions, the oversized bubbles consume significantly more horizontal space than intended, causing multi-row wrapping earlier than necessary. Fix: Either (a) remove `w-8 h-8` and use `size-11` (44px) to be explicit about the actual rendered size, or (b) use an inner `<span>` at `size-8` for the visual circle with button padding for the touch target.

- **QuizNavigation.tsx:25 (confidence: 78)**: Layout diverges from both the old implementation and the story design guidance. The old nav used `justify-between` (Previous on left, Next/Submit on right). The story design guidance (line 97) specifies `flex items-center justify-between`. The implementation uses `flex-col sm:flex-row items-center gap-4`, which groups action buttons and grid side-by-side rather than spread apart. Why: The Previous and Next/Submit buttons cluster together instead of being at opposite ends, changing the established quiz navigation UX. Fix: Review whether `justify-between` was the intended outer-nav layout and align accordingly.

#### Medium

- **Quiz.tsx:232-237 (confidence: 75)**: Silent fallback masks data corruption. When `questionOrder[index]` returns an ID not matching any question, the code falls through to `currentQuiz.questions[currentProgress.currentQuestionIndex]` -- a potentially different question. A learner could see question X content while the grid highlights question Y's position. Fix: Log a warning when the primary lookup fails and show an explicit error rather than silently displaying a mismatched question.

- **Quiz.tsx:161-165 (confidence: 72)**: `handleQuestionClick` is a trivial pass-through `useCallback` wrapping `navigateToQuestion` with no added logic. Since `navigateToQuestion` is already a stable Zustand selector reference, the wrapper adds indirection without benefit. Fix: Pass `navigateToQuestion` directly as the `onQuestionClick` prop.

- **QuizNavigation.tsx:35 (confidence: 72)**: `progress.questionOrder.length || quiz.questions.length` uses `||` instead of explicit length check. An empty `questionOrder` array (length 0, which is falsy) would silently fall through to `quiz.questions.length`, masking what is actually a data integrity bug. Fix: Use `progress.questionOrder.length > 0 ? progress.questionOrder.length : quiz.questions.length` or assert that `questionOrder` is never empty.

#### Nits

- **Nit** QuestionGrid.tsx:38 (confidence: 60): `aria-current="true"` could use the more semantic `aria-current="step"` value, which ARIA defines specifically for step-based navigation (question 1 of N).

- **Nit** QuizActions.tsx:22 (confidence: 58): The wrapper `<div>` could benefit from `role="group" aria-label="Quiz controls"` to help screen readers distinguish action buttons from question grid buttons within the same `<nav>`.

- **Nit** QuizNavigation.test.tsx:7 (confidence: 55): Uses a five-level relative path `'../../../../../tests/support/fixtures/factories/quiz-factory'`. Consider adding a Vitest path alias for test fixtures to avoid fragile deep relative paths.

### Recommendations

1. **Fix array answer check** (High, two files) -- Correctness issue for multi-select quizzes affecting both visual feedback and submission warning.
2. **Use `size-8` shorthand** (High) -- Recurring Tailwind v4 pattern, one-line fix.
3. **Resolve 32px vs 44px bubble sizing** (High) -- Either align with design guidance or simplify to `size-11` and remove dead code.
4. **Review layout intent** (High) -- Confirm whether `justify-between` was the intended layout per story design guidance.
5. **Fix silent question fallback** (Medium) -- Add logging for mismatched question IDs.
6. **Remove pass-through callback** (Medium) -- Minor cleanup.
7. **Use explicit length guard** (Medium) -- Replace `||` with explicit check.
