## Findings

### [High] `isAnswered` check fails for `multiple-select` answers (arrays)
**File:** `src/app/components/quiz/QuestionGrid.tsx:22-23` | **Confidence:** 90

The type signature declares `answers: Record<string, string | string[]>`, but the check `answers[questionId] !== ''` only handles the `string` case. An empty array `[]` is truthy and not equal to `''`, so a learner who submits zero selections on a multi-select question would see it marked as "answered" (blue styling) even though it is semantically unanswered.

**Fix:**
```typescript
const a = answers[questionId]
const isAnswered = a !== undefined && a !== '' && !(Array.isArray(a) && a.length === 0)
```

### [High] `countUnanswered` has the same array-handling bug
**File:** `src/app/pages/Quiz.tsx:57-60` | **Confidence:** 85

The check `a === undefined || a === ''` does not catch empty arrays `[]`. A multi-select question with `answers[q.id] = []` would be counted as "answered" and the submit confirmation dialog would not trigger, allowing silent submission of incomplete answers.

**Fix:** Add `|| (Array.isArray(a) && a.length === 0)` to the unanswered check.

### [High] `w-8 h-8` should be `size-8` (recurring pattern since E02-S05)
**File:** `src/app/components/quiz/QuestionGrid.tsx:34` | **Confidence:** 85

Additionally, the bubble has both `w-8 h-8` (32px) and `min-w-[44px] min-h-[44px]` (44px) applied simultaneously. The `min-*` values always override `w-8 h-8`, making that declaration dead code. Consider just `size-[44px]` for clarity.

### [High] Manual IndexedDB seeding in E2E tests instead of shared helpers
**File:** `tests/e2e/story-e13-s01.spec.ts:63-98` | **Confidence:** 80

The inline `seedQuizData` function manually opens IndexedDB and writes to the `quizzes` store. This duplicates logic that should live in `tests/support/helpers/indexeddb-seed.ts`. If the DB schema changes, every inline seeder must be updated independently.

**Fix:** Extract to or reuse a shared `seedQuizzes()` helper in the test support directory.

### [Medium] Silent fallback masks data corruption in question resolution
**File:** `src/app/pages/Quiz.tsx:232-237` | **Confidence:** 75

When `questionOrder[index]` returns an ID that doesn't match any question, the code falls through to `currentQuiz.questions[currentProgress.currentQuestionIndex]` — a different question. This means a learner could be shown question X while the grid highlights question Y.

**Fix:** Log a warning and show an error state rather than silently displaying the wrong question.

### [Medium] `||` vs `??` for `questionOrder` length fallback
**File:** `src/app/components/quiz/QuizNavigation.tsx:35` | **Confidence:** 72

`progress.questionOrder.length || quiz.questions.length` treats an empty array (length 0, falsy) the same as undefined/null. An empty `questionOrder` is a bug that should be surfaced, not silently papered over.

### [Medium] "Start Over" vs disabled "Previous" discrepancy with AC spec
**File:** `src/app/components/quiz/QuizActions.tsx:26` | **Confidence:** 70

The story spec mentions `"Start Over" on Q1` but the implementation shows a disabled "Previous" button on Q1 instead. If intentionally simplified, document the deviation in the story file.

### [Nit] Long inline className string in QuizNavigation
**File:** `src/app/components/quiz/QuizNavigation.tsx:25` | **Confidence:** 62

Consider extracting the responsive layout classes into a `cn()` call for consistency with the rest of the codebase.
