## Story-Related Issues

### HIGH

1. **E2E tests AC1 and AC2 fail — welcome wizard blocks CTA button click** — `tests/e2e/story-e91-s01.spec.ts:99,116`
   The "Welcome to Knowlune" wizard dialog overlays the page, blocking `ctaButton.click()`. The test seeds sidebar state but doesn't dismiss the wizard. Fix: seed `localStorage` to skip the wizard (e.g., `knowlune-wizard-completed` or equivalent key).

### MEDIUM

2. **`getLastWatchedLesson` sort does not use `updatedAt` as documented in story spec** — `src/lib/progress.ts:115-121`
   The story spec says to "order by `updatedAt DESC`" but the implementation sorts by `currentTime > 0` then `completionPercentage`. This means if a user watches lesson 5 after lesson 3, the function returns whichever has higher `completionPercentage`, not the most recently watched. Consider sorting by `updatedAt` first if the progress table has that field.

3. **No error handling in `resolveCta` async function** — `src/app/pages/UnifiedCourseDetail.tsx:138-159`
   If `getLastWatchedLesson` or `getFirstLesson` throws (e.g., Dexie error), the promise rejection is unhandled. Wrap in try/catch with `console.error` at minimum.

4. **Prettier formatting violations in story-changed files** — `src/app/pages/UnifiedCourseDetail.tsx`, `src/lib/progress.ts`
   Run `npx prettier --write` on these files.

### LOW

5. **AC6 test is weak — only checks `data-slot` attribute** — `tests/e2e/story-e91-s01.spec.ts:218`
   The test for "brand variant styling" only asserts `data-slot="button"` which every Button has regardless of variant. Consider checking computed background color or a more specific attribute.

6. **`getFirstLesson` falls back to `lessons[0]` which could be a PDF** — `src/lib/progress.ts:140`
   The story spec says "first non-PDF lessonId" but the fallback `?? lessons[0]` may return a PDF if no video lessons exist. The behavior is technically correct (returns something) but contradicts the docstring.
