# Code Review: E07-S02 — Recommended Next Dashboard Section (Round 2)

**Date:** 2026-03-08
**Branch:** feature/e07-s02-recommended-next-dashboard-section
**Reviewer:** Code Review Agent (Adversarial)

## What Works Well

1. Previous blockers addressed: shallow copy fix in `getAllProgress()` prevents cache mutation
2. E2E tests significantly improved: AC1 seeds 4 courses/asserts 3, AC2 asserts exact count, AC3 actually clicks
3. Clean scoring algorithm: dead `thirtyDaysAgo` + `void` code removed

## Findings

### Blockers

**B1 — StorageEvent does not fire for same-tab localStorage changes (confidence: 95)**
`src/app/components/RecommendedNext.tsx:66-71`: The `storage` event handler only fires when *another* tab modifies localStorage. When a learner marks a lesson complete and navigates back to the dashboard (same tab), `saveAllProgress()` calls `localStorage.setItem()` but the StorageEvent never fires. The `progressTick` counter stays at 0, and `useMemo` at line 74 does not re-execute. AC5's primary use case (same-tab) remains broken.

**Fix:** Dispatch a custom event from `saveAllProgress()` (e.g., `window.dispatchEvent(new Event('course-progress-updated'))`) and listen for that in `RecommendedNext` instead of/in addition to `StorageEvent`.

### High Priority

**H1 — AC5 E2E test doesn't test dynamic refresh (confidence: 90)**
`tests/e2e/story-e07-s02.spec.ts:154-208`: Test seeds progress, reloads, verifies cards, reloads *again* with same data, verifies cards still render. No progress changes occur between observations. The test would pass even if `useMemo` were a constant.

**Fix:** Between the two assertions, update `course-progress` in localStorage via `page.evaluate()`, trigger the event, and assert card order/count changes.

**H2 — Missing intermediate responsive breakpoint (confidence: 78)**
`src/app/components/RecommendedNext.tsx:112`: `grid-cols-1 sm:grid-cols-3` jumps from 1 to 3 columns at 640px. At 640-767px, cards are ~180px wide — too narrow for CourseCard content.

**Fix:** Use `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`.

### Medium

**M1 — Hardcoded Tailwind color (recurring) (confidence: 80)**
`src/app/components/RecommendedNext.tsx:39`: `bg-blue-100 dark:bg-blue-900/30` instead of semantic theme token.

**M2 — Skeleton layout duplicated (confidence: 75)**
Loading skeleton defined identically in both `Overview.tsx:122-138` and `RecommendedNextSkeleton`. Changes must be synchronized in both.

**Fix:** Export `RecommendedNextSkeleton` from `RecommendedNext.tsx` and import in `Overview.tsx`.

**M3 — `totalLessons` computed via `modules.reduce()` duplicated (confidence: 72)**
`src/lib/recommendations.ts:27,64`: Both `computeCompositeScore` and `getRecommendedCourses` compute total lessons from modules. The `Course` type has a `totalLessons` field that could diverge.

### Nits

- Skeleton at line 17 should match main grid breakpoints if H2 is fixed
- Empty state CTA says "Start a course" but AC4 specifies "import courses"
- `loadSessionStats()` called without error handling in useEffect

## Verdict

**1 blocker** (StorageEvent same-tab), **2 high** priority items. Fix B1 before shipping.
