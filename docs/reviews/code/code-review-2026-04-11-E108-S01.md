## Findings

### MEDIUM — Unstable object reference in useCallback dependency arrays

**File:** `src/app/components/library/BookImportDialog.tsx:90,200,229`

`bulkImport` is used as a dependency in `useCallback` arrays, but `useBulkImport()` returns a new object on every render. This causes `reset`, `handleDrop`, and `handleFileInput` callbacks to be recreated every render, defeating the purpose of `useCallback`.

**Fix:** Destructure the stable callbacks from `useBulkImport()`:
```tsx
const { phase: bulkPhase, progress: bulkProgress, results: bulkResults, startBulkImport, cancel: cancelBulkImport, reset: resetBulkImport } = useBulkImport()
```
Then use the individual stable function references in dependency arrays.

### MEDIUM — Catch block missing `silent-catch-ok` comment

**File:** `src/app/hooks/useBulkImport.ts:137`

The catch block at line 137 intentionally isolates per-file errors (logging to results array) and the summary toast fires at end. This is correct behavior, but the ESLint rule `error-handling/no-silent-catch` flags it because there's no per-file `toast.error()`. Add `// silent-catch-ok: per-file errors collected in results, summary toast shown at end` to suppress the warning.

### LOW — Unused imports in test file

**File:** `src/app/hooks/__tests__/useBulkImport.test.ts:11,133`

- `waitFor` imported but never used (line 11)
- `progressSnapshots` declared but never used (line 133)

### LOW — Progress bar shows 0% when processing first file

**File:** `src/app/hooks/useBulkImport.ts:61`

`setProgress({ current: i, ... })` where `i` starts at 0. The UI shows `current + 1` for the count text but the progress bar uses `(current / total) * 100`, so the bar shows 0% for the first file. Consider using `i` for "completed" count and showing indeterminate or per-file sub-progress.

### NIT — `bulkImport.progress.total` could be 0 causing division by zero

**File:** `src/app/components/library/BookImportDialog.tsx:405`

`value={((bulkImport.progress.current) / bulkImport.progress.total) * 100}` — if `total` is 0, this yields NaN. Protected by the `phase === 'importing'` guard (which requires files.length > 0), but defensive coding would add a `|| 1` denominator guard.
