#### High Priority

- **[Recurring] [`/Volumes/SSD/Dev/Apps/Elearningplatformwireframes/src/app/pages/LessonPlayer.tsx`:364-368] (confidence: 92)**: `handleNoteChange` calls `saveNote()` fire-and-forget with no error handling. If the IndexedDB write fails (quota exceeded, database closed, version conflict), the error is silently swallowed and the learner's note content is lost without any indication. This pattern has been flagged in E03-S03, E03-S04, E03-S06, and E03-S09 (round 1). Why: Silent note loss is the worst UX failure for a learning platform. Fix: Add `.catch(err => toast.error('Failed to save note'))` or wrap in try/catch.

  ```typescript
  // Current (fire-and-forget):
  const handleNoteChange = (value: string, tags: string[] = []) => {
    setNoteText(value)
    if (courseId && lessonId) {
      saveNote(courseId, lessonId, value, tags) // no error handling
    }
  }
  ```

- **[`/Volumes/SSD/Dev/Apps/Elearningplatformwireframes/src/app/components/notes/frame-capture/FrameCaptureView.tsx`:36-37] (confidence: 88)**: The `.catch(() => { if (!cancelled) setStatus('error') })` block silently swallows IndexedDB read errors. The UI shows "Frame unavailable" but provides no diagnostic information. Why: If the database is corrupted or a migration fails, developers and learners get no actionable information. Fix: Add `console.error('Failed to load frame thumbnail:', error)` before setting status.

- **[`/Volumes/SSD/Dev/Apps/Elearningplatformwireframes/src/app/components/notes/frame-capture/FrameCaptureView.tsx`:46-48] (confidence: 80)**: `handleTimestampClick` calls `editor.storage.frameCapture?.onSeek?.(timestamp)` which silently no-ops when `onSeek` is null. This happens when FrameCaptureView is rendered in a read-only context (e.g., the Notes Dashboard from E03-S08 uses a separate `useEditor` instance that never wires up `onSeek`). The learner clicks "Frame at 0:30" and nothing happens, with no visual feedback. Why: Confusing interaction for learners. Fix: Either (a) visually distinguish the timestamp as non-interactive in read-only contexts (render as `<span>` instead of `<button>`), or (b) check `onSeek` availability and show a toast like "Open in lesson to seek video".
