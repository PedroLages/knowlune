## Findings

### HIGH: E2E tests fail — Zustand isLoaded guard prevents IDB reload after seeding

**File:** `tests/e2e/story-e83-s04.spec.ts:58-70`

The `seedBooks()` helper navigates to `/` (initializing the Zustand store with empty books, setting `isLoaded: true`), seeds IndexedDB, then navigates to `/library`. Since SPA navigation preserves Zustand state, `loadBooks()` returns early. All 4 tests fail.

**Fix:** Use `page.reload()` after seeding, or reset `isLoaded` via page.evaluate.

### MEDIUM: Touch target below WCAG minimum on mobile

**File:** `src/app/components/library/BookContextMenu.tsx:141`

MoreVertical button uses `min-h-[36px] min-w-[36px]` at mobile, only 44px at `sm:`. WCAG 2.5.5 requires 44x44px.

### LOW: getBookCountByStatus() called without memoization

**File:** `src/app/components/library/LibraryFilters.tsx:32`

Creates new object every render.

### LOW: Missing coverUrl in test data

**File:** `tests/e2e/story-e83-s04.spec.ts:16-55`
