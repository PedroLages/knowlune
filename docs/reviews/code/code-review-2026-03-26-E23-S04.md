## Summary

Navigation config restructure from Learn/Connect/Track to Library/Study/Track. Changes are limited to `navigation.ts` (config), `navigation.test.ts` (unit tests), and `story-e23-s04.spec.ts` (E2E tests). Layout.tsx and BottomNav.tsx consume the config generically and require no changes.

## Findings

### Blockers

1. **BLOCKER: E2E test import paths are wrong** (`tests/e2e/regression/story-e23-s04.spec.ts:11-12`)
   - Uses `'../support/fixtures'` and `'../support/helpers/navigation'`
   - Should be `'../../support/fixtures'` and `'../../support/helpers/navigation'`
   - The file is in `tests/e2e/regression/` (3 levels deep from tests/), not `tests/e2e/` (2 levels)
   - All other regression tests use `../../support/` prefix
   - **Impact**: E2E tests cannot run at all (module resolution fails)

2. **BLOCKER: E2E tests don't seed progressive disclosure state** (`tests/e2e/regression/story-e23-s04.spec.ts`)
   - Tests AC2-AC6 assert visibility of items with `disclosureKey` (Authors, Notes, Review, Learning Path, Knowledge Gaps, Retention, Challenges, Session History, all analytics)
   - Progressive disclosure hides these items by default for new users
   - Tests need a `beforeEach` that seeds `knowlune-sidebar-show-all-v1` in localStorage (see `story-e25-s08.spec.ts:14` for the key)
   - **Impact**: Even with correct imports, most assertions would fail

### High Priority

None beyond the blockers.
