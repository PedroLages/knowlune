**Date:** 2026-04-13

## Findings

### MEDIUM

1. **Duplicated utility functions** — `tierBadgeClass()` and `tierLabel()` are identically defined in both `FocusAreasPanel.tsx:18-37` and `KnowledgeMapWidget.tsx:24-43`. Extract to a shared `knowledge/utils.ts` module.

2. **Tier threshold logic duplicated** — The ternary `score >= 70 ? 'strong' : score >= 40 ? 'fading' : 'weak'` appears in `KnowledgeMapWidget.tsx:91` and `KnowledgeMapWidget.tsx:123`. The `knowledgeScore.ts` module likely already has a `getTier()` function — use it instead of inline ternaries to keep thresholds in one place.

### LOW

3. **Index signature on TreemapDataItem is a leaky abstraction** — `TopicTreemap.tsx:12`: `[key: string]: string | number` was added to satisfy Recharts' `TreemapDataType`, but it weakens type safety (any string key now accepted). Consider using `as unknown as TreemapDataType[]` at the call site instead, keeping the interface clean.

4. **`computeRelevanceScore` uses `Date.now()` directly** — `dashboardOrder.ts:172`: Not testable with deterministic time without `vi.useFakeTimers()`. The test at line 198 uses `new Date().toISOString()` (real time) which makes the test non-deterministic. This is mitigated by the test using relative comparisons rather than absolute values.

### NIT

5. **Magic number 14 for name truncation** — `TopicTreemap.tsx:90`: `name.length > 14 ? name.slice(0, 12) + '...'` — consider extracting as a named constant.

## Architecture

- Proper store usage with individual selectors (prevents unnecessary re-renders)
- useEffect cleanup pattern with `ignore` flag is correct

## Verdict

PASS with 2 MEDIUM, 2 LOW, 1 NIT.
