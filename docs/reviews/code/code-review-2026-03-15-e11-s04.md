## High Priority

1. **[src/lib/importService.ts:143-148] (confidence: 90)** — localStorage restoration happens outside the Dexie transaction, creating a split-brain on failure. If IndexedDB succeeds but localStorage throws QuotaExceededError, function returns success but settings are partially restored.

2. **[src/lib/importService.ts:43-53] (confidence: 85)** — Schema migration silently skips versions without registered migration functions. A v14 export imported into a v16 app would apply v15's migration but stop there.

3. **[src/lib/exportService.ts:134] (confidence: 82)** — `new Date().toISOString()` is non-deterministic and not mockable. Date-dependent filenames in Settings.tsx also untestable.

4. **[src/app/pages/Settings.tsx] (confidence: 80)** — `w-4 h-4` used instead of Tailwind v4 `size-4` shorthand (19 instances).

## Medium

1. **[src/lib/exportService.ts:64-77] (confidence: 78)** — `getLocalStorageData()` exports ALL localStorage keys without filtering. Could include sensitive data.

2. **[src/lib/openBadges.ts:148] (confidence: 75)** — `localStorage.getItem('streak-milestones')` parsed without type validation.

3. **[src/app/pages/Settings.tsx:148-151] (confidence: 75)** — Early returns for empty data use `toastError.saveFailed()` for what is a "no data" situation, not a failure.

4. **[src/lib/fileDownload.ts:36-45] (confidence: 72)** — `downloadBlob` synchronously revokes the object URL immediately after `a.click()`. Could cause download to fail on slower machines.

5. **[src/lib/xapiStatements.ts:82-92] (confidence: 70)** — `createActor()` reads localStorage on every statement in bulk export. For 1000+ sessions, this means 1000+ localStorage reads.

## Nits

1. `session.startTime.split('T')[0]` is fragile — use `toLocaleDateString('sv-SE')` per pre-review checklist
2. `bg-warning/10` instead of design token `bg-warning-soft` (inconsistent with other export cards)
3. `yieldToUI` uses `requestAnimationFrame` which stalls in background tabs
