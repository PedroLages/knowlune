## Findings

### 🔴 Blockers (Must Fix)

#### 1. Encryption Key Never Persisted - API Keys Unrecoverable
**Location:** [crypto.ts:34,75](src/lib/crypto.ts#L34)
**Confidence:** 97%

**Problem:**
The encryption key is generated fresh on every `encryptData` and `decryptData` call. When a user saves their API key:
1. `saveAIConfiguration()` calls `encryptData()` → generates random key #1 → encrypts with key #1 → stores encrypted data
2. User refreshes page
3. `getDecryptedApiKey()` calls `decryptData()` → generates random key #2 → tries to decrypt with key #2 → **ALWAYS FAILS**

The stored API key is **write-only** - it can never be recovered after page reload.

**Why This Wasn't Caught:**
E2E test at line 160 only checks `connectionStatus: 'connected'` (stored as plaintext), not the actual decrypted key.

**Impact:**
- All learners' API keys are lost on every page refresh
- Any future story (S02-S07) calling `getDecryptedApiKey()` will always get `null`
- Silent failure - catch block returns `null` with no error

**Fix Options:**

**Option A - Session-scoped key (recommended for S01):**
```typescript
let _sessionKey: CryptoKey | null = null

async function getSessionKey(): Promise<CryptoKey> {
  if (!_sessionKey) {
    _sessionKey = await crypto.subtle.generateKey(
      { name: 'AES-GCM', length: 256 }, true, ['encrypt', 'decrypt']
    )
  }
  return _sessionKey
}

// Use in encryptData and decryptData:
const key = await getSessionKey()
```

**Option B - Persistent key (better long-term):**
```typescript
async function getOrCreateKey(): Promise<CryptoKey> {
  const stored = localStorage.getItem('crypto-key-jwk')
  if (stored) {
    return await crypto.subtle.importKey('jwk', JSON.parse(stored),
      { name: 'AES-GCM' }, true, ['encrypt', 'decrypt'])
  }
  const key = await crypto.subtle.generateKey(...)
  const exported = await crypto.subtle.exportKey('jwk', key)
  localStorage.setItem('crypto-key-jwk', JSON.stringify(exported))
  return key
}
```

### 🟠 High Priority (Should Fix)

#### 1. Error State Immediately Overwritten in handleSave
**Location:** [AIConfigurationSettings.tsx:132-139](src/app/components/figma/AIConfigurationSettings.tsx#L132)
**Confidence:** 92%

**Problem:**
Two consecutive `setSettings()` calls in catch block. The first sets error state, the second immediately overwrites it with `getAIConfiguration()` from localStorage (which has no error). React batches these and uses only the last value.

**Impact:** Users never see connection test errors.

**Fix:**
```typescript
} catch (error) {
  setSettings(prev => ({
    ...prev,
    connectionStatus: 'error',
    errorMessage: error instanceof Error ? error.message : 'Connection test failed',
  }))
  // Remove the second setSettings(getAIConfiguration()) call
}
```

#### 2. Unhandled Promise Rejections in Event Handlers
**Locations:** [AIConfigurationSettings.tsx:174,261](src/app/components/figma/AIConfigurationSettings.tsx#L174)
**Confidence:** 85%

**Problem:**
`handleProviderChange` and `updateConsent` are async but called from event handlers without `.catch()`. If `saveAIConfiguration()` throws (quota exceeded, encryption fails), the error becomes an unhandled rejection.

**Fix:**
```typescript
onValueChange={value => {
  handleProviderChange(value as AIProviderId).catch(err =>
    console.error('Failed to save provider change:', err)
  )
}}

onCheckedChange={checked => {
  updateConsent(key, checked).catch(err =>
    console.error('Failed to save consent:', err)
  )
}}
```

#### 3. AIUnavailableBadge Missing Cross-Tab Sync
**Location:** [AIUnavailableBadge.tsx:32-34](src/app/components/figma/AIUnavailableBadge.tsx#L32)
**Confidence:** 82%

**Problem:**
Badge only listens for `ai-configuration-updated` custom event, not `storage` event. Won't update when AI configuration changes in other tabs.

**Fix:**
```typescript
useEffect(() => {
  function handleStorageUpdate(e: StorageEvent) {
    if (e.key === 'ai-configuration') checkAvailability()
  }
  function handleCustomUpdate() {
    checkAvailability()
  }

  window.addEventListener('storage', handleStorageUpdate)
  window.addEventListener('ai-configuration-updated', handleCustomUpdate)

  return () => {
    window.removeEventListener('storage', handleStorageUpdate)
    window.removeEventListener('ai-configuration-updated', handleCustomUpdate)
  }
}, [])
```

#### 4. Silent Catch Blocks Hide Errors
**Locations:** [aiConfiguration.ts:133,195](src/lib/aiConfiguration.ts#L133)

**Problem:**

- Line 133: `JSON.parse` failure (corrupted localStorage)

} catch (error) {
