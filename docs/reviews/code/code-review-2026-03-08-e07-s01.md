# Code Review: E07-S01 — Momentum Score Calculation & Display

**Date**: 2026-03-08
**Reviewer**: Code Review Agent (Adversarial)

## What Works Well

1. Clean library design — `calculateMomentumScore` is a well-structured pure function with `MomentumInput` interface
2. Thorough unit tests covering all three score components, clamping, tier thresholds, and zero-session default
3. Good retro fixes bundled — `challengeProgress.ts` refactor and `useChallengeStore.ts` snapshot rollback

## Findings

### Blockers

1. **AC5 broken — `study-session-ended` event never dispatched** — `Courses.tsx:69` (confidence: 95)
   - The effect listens for `study-session-ended` but no code in the codebase dispatches this event
   - Only `study-log-updated` events are dispatched in `studyLog.ts`
   - Fix: Either dispatch `study-session-ended` from session recording code or listen to `study-log-updated`

2. **`@ts-nocheck` file shipping** — `fireMilestoneToasts.tsx:1` (confidence: 92)
   - Bypasses all TypeScript safety; dependencies (`ChallengeMilestoneToast`, `challengeMilestones`) don't exist
   - Fix: Remove from branch or complete the dependencies

### High Priority

3. **No `.catch()` on `loadMomentumScores()`** — `Courses.tsx:66` (confidence: 90)
   - Unhandled rejection if IndexedDB fails; badges silently don't render
   - Fix: Add `.catch(err => console.error('[Courses] Failed to load momentum scores:', err))`

4. **No cleanup guard on async effect** — `Courses.tsx:48-71` (confidence: 85)
   - `setMomentumMap` called after unmount; race condition with `handleSessionEnded`
   - Fix: Add `let ignore = false` pattern with cleanup in return

5. **ImportedCourseCard momentum never computed** — `Courses.tsx:248` (confidence: 82)
   - Prop exists but never populated — inconsistent experience
   - Fix: Compute for imported courses too, or remove unused prop

### Medium

6. **`Math.max(...spread)` stack overflow risk** — `momentum.ts:30` (confidence: 75)
   - Large session arrays could exceed call stack size
   - Fix: Use `sessions.reduce()` instead

7. **Hardcoded Tailwind colors** — `MomentumBadge.tsx:16-17` (confidence: 78)
   - Should use theme tokens from `theme.css` for dark mode/theme support

8. **Raw HTML `<select>` instead of shadcn Select** — `Courses.tsx:264-273` (confidence: 72)
   - Breaks design system consistency

9. **O(N*M) complexity building momentum map** — `Courses.tsx:50-61` (confidence: 70)
   - Pre-group sessions by courseId in single pass for scalability

### Nits

10. Uses `h-4 w-4` instead of Tailwind v4 `size-*` shorthand (pre-existing pattern)
11. `aria-label` vs tooltip phrasing inconsistency ("Momentum: Hot (82)" vs "Momentum score: 82/100")
12. E2E sort test doesn't verify actual ordering (also flagged in test coverage review)
