## Findings

### Blocker

- **[`src/stores/useSessionStore.ts:176`] (confidence: 92)**: AC5 broken — `endSession()` persists to IndexedDB but dispatches no event. Courses page listens for `study-log-updated` (from `studyLog.ts`), but the session store uses a different data path. Momentum won't recalculate after completing a study session until page reload. **Fix**: Dispatch `study-log-updated` event at end of `endSession()`.

### High Priority

- **[`src/lib/momentum.ts:30`] (confidence: 85)**: `Math.max(...sessions.map(...))` will throw `RangeError` on large arrays (thousands of sessions). **Fix**: Use `sessions.reduce()` instead.
- **[`src/lib/momentum.ts:36`] (confidence: 82)**: `calculateMomentumScore` documented as "pure" but calls `getCourseCompletionPercent()` which reads `localStorage`. **Fix**: Accept `completionPercent` as parameter in `MomentumInput`.
- **[`src/app/components/figma/MomentumBadge.tsx:14-27`] (confidence: 78)**: Hardcoded Tailwind colors instead of theme tokens from `theme.css`. **Fix**: Define semantic CSS custom properties for momentum tiers.

### Medium

- **[`src/app/pages/Courses.tsx:279-288`] (confidence: 72)**: Native `<select>` instead of shadcn `Select` — breaks visual consistency.
- **[`src/app/pages/Courses.tsx:109-114`] (confidence: 70)**: `sortedCourses` recomputes on every render. **Fix**: Wrap in `useMemo`.
- **[`tests/e2e/story-e07-s01.spec.ts:47-164`] (confidence: 68)**: Sort E2E test relies on reload; doesn't verify AC5.
