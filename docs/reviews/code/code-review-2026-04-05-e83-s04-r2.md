## New R2 Findings

### BLOCKER

1. **All 4 E2E tests fail — onboarding overlay blocks interaction**
   - File: `tests/e2e/story-e83-s04.spec.ts:58-67`
   - `seedBooks()` navigates to `/` and `/library` without dismissing the onboarding overlay. The "Welcome to Knowlune" dialog covers the entire page, blocking all assertions.
   - Fix: Use `navigateAndWait()` from `tests/support/helpers/navigation.ts` or seed onboarding-dismissed localStorage before navigation.

### MEDIUM

2. **View toggle buttons lack `role="tab"` / `role="tablist"` semantics**
   - File: `src/app/components/library/LibraryFilters.tsx:74-99`
   - The grid/list toggle uses `aria-pressed` which is correct for toggle buttons, but is inconsistent with the status pills below (which use `role="tablist"` / `role="tab"`). Consider using `role="toolbar"` on the container for semantic grouping.
   - Not a blocker — `aria-pressed` is valid, just inconsistent with adjacent pattern.

3. **Status filter pills show count only for "All" pill**
   - File: `src/app/components/library/LibraryFilters.tsx:130-132`
   - `getBookCountByStatus()` computes counts for all statuses, but only the "All" pill renders a count badge. Consider showing counts on individual status pills for better UX.
   - This may be intentional design — LOW if so.

### LOW

4. **`setFilter('status', 'all')` stores 'all' string in filters state**
   - File: `src/stores/useBookStore.ts:127-128` and `LibraryFilters.tsx:120`
   - Clicking "All" pill calls `setFilter('status', 'all')` which stores `{ status: 'all' }`. This works because `getFilteredBooks()` checks `!== 'all'`, but it's semantically cleaner to use `setFilter('status', undefined)` to represent "no filter".
   - Functional, just slightly inconsistent with `setFilters({})` used by the "Clear filters" button.

5. **Duplicate menu content between ContextMenu and DropdownMenu**
   - File: `src/app/components/library/BookContextMenu.tsx:77-155`
   - The same menu items (Edit, Change Status submenu, Delete) are duplicated between ContextMenu and DropdownMenu. Consider extracting a shared `MenuItems` component to reduce duplication.
   - Not a correctness issue — maintenance concern only.
