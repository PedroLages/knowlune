## Code Review: E23-S06 — Featured Author Layout For Single Author State

### What Works Well

1. **Clean conditional rendering pattern.** The `allAuthors.length === 1` ternary in `Authors.tsx` is straightforward and easy to follow, with a clear separation between the featured and grid branches.

2. **Good defensive programming.** The empty state guard (`allAuthors.length === 0`) was added proactively, preventing a potential crash if the data source returns an empty array. The specialty badge overflow cap at 5 with a `+N` indicator is thoughtful.

3. **Solid test coverage for a UI-only story.** The test file covers empty state, single author (featured), and multiple authors (grid), including edge cases like singular/plural labels and the optional `featuredQuote` blockquote. The `vi.hoisted` + getter pattern for `allAuthors` is a clean approach to dynamic mock data.

### Findings

#### Blockers

None.

#### High Priority

- **[`src/app/components/figma/FeaturedAuthor.tsx:9`] (confidence: 80)**: `getInitials` is imported from `@/lib/avatarUpload` — a 335-line file whose primary concern is image validation, compression, and format conversion. This creates an invisible coupling: anyone importing `FeaturedAuthor` now transitively depends on the avatar upload module. If `avatarUpload.ts` ever gains browser-only APIs (Canvas, File, Blob — which it almost certainly already uses given its purpose), this import will break SSR or test environments that don't polyfill those APIs. The same import exists in `Authors.tsx:9` and `AuthorProfile.tsx`. **Why**: A future bundler tree-shake failure or test environment change could break the Authors page silently. **Fix**: Extract `getInitials()` to `@/lib/utils.ts` or `@/lib/string-utils.ts` — a pure utility module with no side-effect-heavy neighbors. This is low effort and eliminates the coupling risk.

- **[`src/app/components/figma/FeaturedAuthor.tsx:86`] (confidence: 78)**: `author.shortBio` is rendered directly without a fallback. While `shortBio` is typed as `string` (not optional), the data comes from a static file today. If any future author record omits or has an empty `shortBio`, the paragraph renders as an empty `<p>` tag, which is semantically incorrect and creates a visual gap. **Why**: Learners see a visually broken layout with an empty paragraph between the stats and the CTA button. **Fix**: Add a fallback: `{(author.shortBio || author.bio) && <p ...>{author.shortBio || author.bio}</p>}`. This also leverages the full `bio` as a graceful degradation.

#### Medium

- **[`src/app/components/figma/FeaturedAuthor.tsx:21`] (confidence: 75)**: `StatCard` uses `bg-muted` for its background, but the parent `<Card>` uses the default `bg-card` background. In light mode these are nearly identical, making the stat cards visually indistinct from the card surface. In dark mode, both `bg-muted` and `bg-card` resolve to very similar dark values. The design guidance in the story file specifies `bg-card` for stats (via the `<Card>` wrapper pattern from AuthorProfile). **Why**: The stats strip may lack visual separation from the card body, reducing scanability. **Fix**: Consider using `bg-background` or adding a subtle `ring-1 ring-border/20` to give the stat cards more visual definition against the card surface.

- **[`src/app/pages/Authors.tsx:6`] (confidence: 82)**: `getAuthorStats` is called inline during render for each author in the grid branch (line 46). This function calls `useCourseStore.getState().courses.filter(...)` — accessing Zustand state outside React's reactive cycle. While it works for static data, if courses are ever loaded asynchronously, the grid will render with stale stats because there is no subscription to the store. The `FeaturedAuthor` component has the same pattern (line 30). **Why**: When the course store eventually becomes async (e.g., loading from IndexedDB), the Authors page will silently show stale or zero stats. **Fix**: Consider using `useCourseStore(state => state.courses)` at the page level and passing courses down, ensuring reactivity. This is forward-looking but worth noting now.

- **[`src/app/pages/__tests__/Authors.test.tsx`] (confidence: 72)**: No test covers the specialty badge overflow behavior. `FeaturedAuthor` slices specialties to 5 and shows a `+N` overflow badge, but no test verifies this edge case. A test with 7+ specialties would catch regressions if the cap changes. **Fix**: Add a test case with `specialties: ['A','B','C','D','E','F','G']` and assert that `+2` is rendered and only 5 specialty texts appear.

- **[`src/app/pages/__tests__/Authors.test.tsx`] (confidence: 70)**: No test verifies the empty state subtitle is absent. The empty state branch (lines 12-22 of `Authors.tsx`) omits the subtitle paragraph entirely, but the test only checks for the "No authors available" text. A negative assertion like `expect(screen.queryByText(/meet the/i)).not.toBeInTheDocument()` would ensure the subtitle does not leak into the empty state.

#### Nits

- **Nit** [`src/app/components/figma/FeaturedAuthor.tsx:33`] (confidence: 65): The Card uses `rounded-[24px]` which matches the project convention, but `border-0` removes the card's default border entirely. Other cards in the grid branch (`Authors.tsx:53`) also use `border-0`, so this is consistent — but it means the card relies solely on `shadow-sm` for visual separation, which may be invisible on low-contrast displays or in dark mode.

- **Nit** [`src/app/components/figma/FeaturedAuthor.tsx:52`] (confidence: 60): The blockquote's left border is hidden on mobile (`sm:border-l-2 sm:border-brand sm:pl-3`) but the text remains italic. On mobile, the quote has no visual indicator that it is a quote beyond the curly quotes and italics. Consider keeping a minimal `border-l-2` on mobile as well for consistent visual treatment.

- **Nit** [`src/app/pages/Authors.tsx:1-9`] (confidence: 55): The grid branch imports `BookOpen`, `Clock`, `GraduationCap`, `Badge`, and `Avatar` components that are also imported and used by `FeaturedAuthor`. If the page only has one author (current production state), these imports are dead code for that render path. Vite should tree-shake them, but the import list could be cleaner by moving the grid branch into its own component (e.g., `AuthorGrid`).

### Recommendations

1. **First priority**: Extract `getInitials` to a standalone utility module to eliminate the `avatarUpload` coupling (High).
2. **Second priority**: Add a fallback for empty `shortBio` in `FeaturedAuthor` (High).
3. **Third priority**: Add test coverage for the specialty badge overflow edge case (Medium).
4. **Optional**: Evaluate `StatCard` background token and blockquote mobile styling during design review pass.
