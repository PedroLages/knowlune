#### High Priority

- **[`src/app/pages/AuthorProfile.tsx:21-27`] (confidence: 90)**: `getInitials` is still duplicated as a local function in `AuthorProfile.tsx`, despite the fix commit's purpose being to extract it to `@/lib/textUtils` and consolidate all consumers. Three files now import from `textUtils` (`FeaturedAuthor.tsx`, `Authors.tsx`, `Layout.tsx`) but `AuthorProfile.tsx` still defines its own inline version with different behavior -- the local version does not guard against empty/null names, does not trim whitespace, and does not cap at 2 initials. This means `getInitials("A B C")` returns `"ABC"` in `AuthorProfile` but `"AB"` everywhere else. **Why**: A learner clicking "View Full Profile" from the featured layout will see a different avatar fallback than what they saw on the Authors page if the author has a 3+ word name, creating an inconsistency that erodes trust. **Fix**: Replace the local function in `AuthorProfile.tsx:21-27` with `import { getInitials } from '@/lib/textUtils'`.

- **[`src/app/components/quiz/questions/MultipleChoiceQuestion.tsx`, `TrueFalseQuestion.tsx`] (confidence: 92)**: These two quiz component files are committed under the `fix(E23-S06): address all review findings` commit (`c9709da0`) but are entirely unrelated to the Featured Author Layout story. They add a `handleRadioGroupKeyDown` callback using `requestAnimationFrame` to read `document.activeElement` after Radix arrow key navigation -- a WAI-ARIA selection-follows-focus fix for the quiz system. Bundling unrelated changes into a story commit violates the single-responsibility principle for commits and makes it harder to revert, bisect, or understand the change history. **Why**: If the quiz radio-group fix introduces a regression, reverting `fix(E23-S06)` would also revert the Author page fixes. Conversely, if the Author page changes need to be reverted, the quiz fix is lost. **Fix**: Cherry-pick these changes into their own commit with a message like `fix(quiz): arrow key selection-follows-focus for radio groups`, or at minimum document the scope bleed in the story file.

#### Medium

- **[`src/lib/avatarUpload.ts:311-312`] (confidence: 78)**: The `avatarUpload.ts` module now re-exports `getInitials` from `textUtils` (`export { getInitials } from '@/lib/textUtils'`). While no current file imports `getInitials` from `avatarUpload` anymore (all three consumers were migrated to `textUtils`), this re-export creates a "ghost dependency" -- the import path `@/lib/avatarUpload` still works for `getInitials`, and any future developer or AI agent could accidentally use the old path. This maintains the coupling the fix was supposed to eliminate. **Why**: The re-export is a trap that invites regression to the old import pattern. **Fix**: Remove lines 307-312 from `avatarUpload.ts` (the section header comment and re-export). If any consumer still needs it, the import error will surface immediately at build time.

- **[`src/app/components/figma/FeaturedAuthor.tsx:88`] (confidence: 75)**: The `totalHours` display logic `Math.max(Math.round(stats.totalHours), stats.totalHours > 0 ? 1 : 0)` is correct but convoluted. When `totalHours` is `0.4`, `Math.round(0.4)` returns `0`, then `Math.max(0, 1)` returns `1`, displaying `1h`. When `totalHours` is `0`, `Math.max(0, 0)` returns `0`, displaying `0h`. Technically right, but the nested ternary inside `Math.max` is hard to read and easy to break during future edits. **Why**: The next developer who touches this line will need to trace the logic carefully to avoid introducing a bug. **Fix**: Extract to a named helper: `function formatHours(h: number): string { if (h === 0) return '0h'; return Math.max(1, Math.round(h)) + 'h' }`.

- **[`src/app/components/figma/FeaturedAuthor.tsx:53-55`] (confidence: 72)**: The blockquote always renders with `border-l-2 border-brand pl-3` at all breakpoints. On mobile (where the parent uses `text-center`), this creates a left-aligned blockquote with a left border inside a centered column -- the border visually anchors to the left while the text above (name, title) is centered. The first review's design review flagged this as M1 but the fix only removed the explicit `text-left` class. The border itself still creates an alignment inconsistency on mobile. **Why**: Learners on mobile see the author name centered, then an abrupt left-anchored border on the quote below. **Fix**: On mobile, either hide the border (use `sm:border-l-2 sm:border-brand sm:pl-3` -- border only on `sm+`) or add `text-center sm:text-left` to the blockquote to center its text on mobile.

- **[`src/app/pages/Authors.tsx:46` + `src/app/components/figma/FeaturedAuthor.tsx:30`] (confidence: 73)**: `getAuthorStats(author)` calls `useCourseStore.getState()` outside of React's reactive cycle. Both the grid branch (line 46) and FeaturedAuthor (line 30) call this during render. `getState()` provides a snapshot but does not create a subscription -- if the course store updates after mount, the stats will not re-render. Acknowledged from the prior review as forward-looking. Since the data is static today, no action required. **Fix**: When courses become async, refactor `getAuthorStats` to accept a courses array parameter and subscribe to the store at the page level with `useCourseStore(state => state.courses)`.

#### Nits

- **Nit** [`src/app/components/figma/FeaturedAuthor.tsx:21`] (confidence: 60): `StatCard` uses `rounded-2xl` (`16px`) for its border radius. The project convention specifies `rounded-[24px]` for cards and `rounded-xl` (`12px`) for buttons. A stat card is neither, so `rounded-2xl` is defensible, but `rounded-xl` would align more closely with the button rounding convention for small contained elements.

- **Nit** [`src/app/pages/__tests__/Authors.test.tsx:114`] (confidence: 58): The test `renders no bio paragraph when both shortBio and bio are empty` queries `card.querySelector('p.max-w-prose')` -- a CSS class selector rather than a semantic query. This couples the test to the Tailwind class name. If the class changes, the test silently passes. Using `data-testid="author-bio"` on the paragraph would be more resilient.

- **Nit** [`src/app/pages/Authors.tsx:53`] (confidence: 55): The grid card uses both `transition-shadow` and `transition-transform` as separate classes. Tailwind v4 supports `transition-[shadow,transform]` as a single utility, which is more concise and prevents one from being accidentally removed without the other.
