### OBSERVATIONS (not issues — for awareness)

#### INFO: ReviewRating uses string literals vs ts-fsrs numeric Rating enum

The story defines `ReviewRating = 'again' | 'hard' | 'good' | 'easy'` (strings) while ts-fsrs uses `Rating.Again = 1, Rating.Hard = 2, Rating.Good = 3, Rating.Easy = 4` (numbers). A mapping layer will be needed in subsequent stories. This is appropriate for a type-foundation story — the mapping belongs in the scheduling logic story.

#### INFO: ts-fsrs has `Rating.Manual` (0) not represented in ReviewRating

ts-fsrs includes a `Manual` rating for manual scheduling overrides. The story's `ReviewRating` intentionally excludes it. If manual scheduling is needed later, this can be extended.
