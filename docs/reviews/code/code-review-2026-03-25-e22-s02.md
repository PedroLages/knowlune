## Findings

### STORY-RELATED

#### MEDIUM

1. **Missing async cleanup in useEffect** — `OllamaModelPicker.tsx:86-90`: The `fetchModels()` async call has no abort/cleanup mechanism. If the component unmounts mid-fetch, `setModels()`, `setIsLoading()`, and `setError()` fire on an unmounted component. While React 18+ tolerates this, it's an anti-pattern. Consider using an `AbortController` or a `cancelled` flag in the cleanup function.

2. **Prettier formatting not applied before commit** — `ollama-client.ts` and `ollama-client-models.test.ts` have formatting issues caught by `npx prettier --check`. The working tree has uncommitted prettier fixes.

#### LOW

3. **`onModelSelect` callback identity instability** — `AIConfigurationSettings.tsx:466-471`: The `onModelSelect` prop is an inline arrow function, creating a new reference every render. This causes `fetchModels` callback to be recreated (via dependency array at `OllamaModelPicker.tsx:83`). The `lastFetchedUrl.current` guard prevents spurious network calls, but wrapping the handler in `useCallback` would be cleaner.

4. **`formatModelSize` does not guard negative input** — `aiConfiguration.ts:385-391`: `Math.log(negative)` returns `NaN`. While Ollama won't return negative sizes, a defensive `if (bytes < 0) return '0 B'` guard would be more robust.

### PRE-EXISTING

5. **[WARNING] `server/index.ts` catch blocks** — Lines 55, 90, 245, 298: ESLint `error-handling/no-silent-catch` warnings. These are server-side code where `toast.error()` is not applicable; they correctly log and return HTTP error responses. Pre-existing, not introduced by this story.

6. **[WARNING] E2E test failures** — `overview.spec.ts`, `onboarding.spec.ts`, `courses.spec.ts`: 6 tests fail looking for "Your Library" heading. These tests were not modified by this story and fail on main as well.

## Architecture Assessment

- **SSRF protection**: The `/api/ai/ollama/tags` endpoint correctly reuses `isAllowedOllamaUrl()` to block loopback addresses.
- **Separation of concerns**: Static `fetchModels()` method allows the UI to discover models without a full client instance. Good pattern.
- **Proxy/direct duality**: Both connection modes are properly handled in `fetchModels()` with different URL construction paths.
- **Error taxonomy**: Network, timeout, and HTTP errors are properly classified into distinct `LLMError` types.

## Verdict

No blockers. Two MEDIUM issues (async cleanup, formatting) should be addressed before merge.
