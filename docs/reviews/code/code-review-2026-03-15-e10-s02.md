#### High Priority

- **[Recurring] [src/app/pages/Overview.tsx:302] (confidence: 90)**: Fire-and-forget async `importCourseFromFolder()` without `.catch()`. This is an async function (returns `Promise<ImportedCourse>`) invoked as `onAction={() => importCourseFromFolder()}` with no error handling. If the File System Access API throws (user cancels picker, permission denied, disk error), the rejection is unhandled — it will surface as an unhandled promise rejection in the console but the user sees nothing. Pattern from: E03-S03 and recurring since. Why: Learners who cancel the file picker or encounter a permission error will get no feedback. Fix: Wrap in try/catch or append `.catch()` with a `toast.error()`:
  ```tsx
  onAction={() => importCourseFromFolder().catch(() => {})}
  ```
  Note: the codebase already does `importCourseFromFolder().catch(() => {})` at line 211 of `courseImport.ts` — but even a silent catch is better than an unhandled rejection. Ideally surface a toast.

- **[Recurring] [src/app/components/EmptyState.tsx:42-43] (confidence: 85)**: `w-16 h-16` and `w-8 h-8` instead of Tailwind v4 `size-16` and `size-8` shorthand. Pattern from: E02-S05 and recurring since. Why: Consistency with the rest of the codebase that has adopted `size-*`. Fix: Replace `w-16 h-16` with `size-16` and `w-8 h-8` with `size-8`.

- **[Recurring] [src/app/pages/Challenges.tsx:73] (confidence: 90)**: Hardcoded Tailwind colors `bg-blue-600/10 text-blue-600` and `bg-amber-600/10 text-amber-600`. These bypass the design token system. Pattern from: E03-S05 and recurring since. Why: These colors will not adapt to dark mode theme and break visual consistency. Fix: Use `bg-brand/10 text-brand` for the default state. For amber/completed state, consider adding a `--warning` or `--gold` token, or use the existing `bg-amber-*` if explicitly approved as an exception. Also at line 173: `focus-visible:ring-blue-600` should be `focus-visible:ring-ring`.

- **[Recurring] [src/app/pages/Reports.tsx:330] (confidence: 85)**: String interpolation for className (`className={\`w-2 h-2 rounded-full ${...}\``}) instead of `cn()`. Pattern from: E01-S03 and recurring since. Why: String interpolation does not handle class conflicts correctly; `cn()` (via `twMerge`) is the project standard. Fix: `className={cn('size-2 rounded-full', action.type === 'lesson_complete' ? 'bg-success' : 'bg-brand')}`.

#### Medium

- **[src/app/components/EmptyState.tsx:45] (confidence: 72)**: The design guidance specifies `font-display text-lg font-medium` for the title (using Space Grotesk heading font), but the implementation uses `text-lg font-semibold` without `font-display`. Why: The empty state title will render in DM Sans (body font) instead of Space Grotesk (heading font), creating visual inconsistency with other section headings. Fix: Add `font-display` to the `<h3>` className: `className="font-display text-lg font-medium mb-2"`.

- **[src/app/components/EmptyState.tsx:43] (confidence: 75)**: Design guidance specifies icon color `text-brand-muted` (`#c8c9e4`), but implementation uses `text-brand` (`#5E6AD2`). These are distinctly different colors — `brand-muted` is a subdued pastel while `brand` is the full saturated primary. Why: The icon will be more prominent than designed, potentially competing visually with the CTA button. Fix: Change `text-brand` to `text-brand-muted` on the icon.

- **[src/app/pages/Reports.tsx:140-146] (confidence: 75)**: The `hasActivity` check only considers `totalLessons > 0 || recentActions.length > 0`. It does not account for study session data (from `useSessionStore`). A user who has logged study time via the session timer but has not completed any lessons or logged actions will see the empty state despite having actual study data. Why: Learners who use the study timer but haven't completed lessons will be told to "start studying" — contradicting their experience. Fix: Also check session store data: `const hasActivity = totalLessons > 0 || recentActions.length > 0 || studyNotes > 0`.

- **[src/app/components/EmptyState.tsx:47-56] (confidence: 70)**: If both `onAction` and `actionHref` are provided, `onAction` wins silently (line 52 has `!onAction` guard). There is no TypeScript enforcement via discriminated union — a caller could accidentally pass both and the link would be silently ignored. Why: Future developers may not notice the precedence rule. Fix: Consider a discriminated union type:
  ```tsx
  type EmptyStateAction =
    | { actionLabel: string; onAction: () => void; actionHref?: never }
    | { actionLabel: string; actionHref: string; onAction?: never }
    | { actionLabel?: never; actionHref?: never; onAction?: never }
  ```

- **[src/app/pages/Reports.tsx:180] (confidence: 72)**: `h-5 w-5` should be `size-5` (same recurring `h-* w-*` pattern). Fix: Replace with `size-5`.

#### Nits

- **Nit** [src/app/components/EmptyState.tsx:39] (confidence: 60): The `motion.div` wrapper has both `variants={fadeUp}` and `initial="hidden" animate="visible"` set directly. When the EmptyState is placed inside a parent `motion` container with `staggerContainer` variants (as in Overview.tsx line 295), the child's explicit `initial`/`animate` props override the parent orchestration — meaning the empty state will animate immediately rather than staggering with siblings. Fix: Remove `initial` and `animate` from the component, or make them configurable via props, so parent orchestration works correctly.

- **Nit** [src/app/components/EmptyState.tsx:48] (confidence: 55): Buttons lack the `rounded-xl` class specified in the project design convention for buttons. The shadcn Button default may or may not match. Fix: Add `className="rounded-xl"` to both Button instances if the default variant does not include it.

- **Nit** [src/app/pages/Reports.tsx:328] (confidence: 50): Using array index `i` as the `key` for recent actions list items. If actions are reordered or filtered, this could cause incorrect reconciliation. Fix: Use a composite key like `${action.courseId}-${action.timestamp}-${i}`.
