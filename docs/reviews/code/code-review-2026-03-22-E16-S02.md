## Code Review: E16-S02 — Display Score History Across All Attempts

### What Works Well

1. **Clean responsive design pattern.** The `AttemptHistory` component implements a proper desktop table / mobile card split using `hidden sm:block` and `sm:hidden`, with consistent data across both layouts. This is a good pattern for data-dense components.

2. **Design token compliance.** All colors use semantic tokens (`bg-brand-soft`, `text-success`, `bg-muted`, `text-muted-foreground`, `bg-success-soft`). No hardcoded hex or Tailwind color violations detected.

3. **Sort-order fix is well-tested.** The `loadAttempts` reverse fix is verified at three levels: store unit test, component unit test, and E2E test. The `previousBestPercentage` logic and test data were correctly updated to match the new most-recent-first contract.

### Findings

#### High Priority

- **[Recurring] `src/stores/useQuizStore.ts:168` (confidence: 85)**: **`submitQuiz` appends to `attempts` in wrong order.** After the sort-order fix, `attempts` is now most-recent-first. But `submitQuiz` still does `[...get().attempts, attempt]` which appends the new attempt to the **end** of the array. This means if a user submits a quiz and stays on the results page without a full `loadAttempts` refresh, the newest attempt lands at the end (oldest position) instead of the front. On the QuizResults page, `loadAttempts` runs on mount and overwrites this, masking the bug in most flows. However, any code path that reads `attempts[0]` as "most recent" between `submitQuiz` and the next `loadAttempts` call will get stale data. Why: If a learner submits a quiz and the component reads `attempts` before the effect fires `loadAttempts`, the current attempt is misidentified. Fix: Change line 168 to `attempts: [attempt, ...get().attempts]`.

- **`tests/e2e/story-e16-s02.spec.ts:76-116` (confidence: 80)**: **Manual IndexedDB seeding duplicates shared helper logic.** The `seedIdbStore` function reimplements the retry-based IndexedDB seeding pattern that already exists in `tests/support/helpers/seed-helpers.ts` as `seedQuizAttempts()` and `seedIndexedDBStore()`. This is a known anti-pattern -- when the DB name or retry logic changes, this inline copy will silently diverge. Why: Maintenance burden and flakiness risk for learner-facing test infrastructure. Fix: Import and use `seedQuizAttempts` and `seedIndexedDBStore` from `tests/support/helpers/seed-helpers.ts`. Pattern from: E07-S04, E08-S01, E11-S02 all use the shared helpers.

- **`src/app/components/quiz/AttemptHistory.tsx:28` (confidence: 75)**: **`courseId` and `lessonId` props accepted but unused.** The component destructures only `{ attempts, currentAttemptId }` from props, silently discarding `courseId` and `lessonId`. These were intended for the Review button navigation (E16-S01 dependency). While the toast stub is fine, the silent discard means TypeScript won't catch if these props are accidentally removed from the call site before E16-S01 lands. Why: When E16-S01 is implemented, the developer may not realize these props are needed. Fix: Either destructure all four props (prefixing unused ones with `_`) or add a `// TODO(E16-S01): use courseId, lessonId for review navigation` comment next to the destructuring to make intent explicit.

#### Medium

- **`src/app/components/quiz/AttemptHistory.tsx:36-37` (confidence: 72)**: **No empty-state handling for zero attempts.** If `attempts` is an empty array (defensive edge case -- shouldn't happen from the parent, but the component accepts `QuizAttempt[]` with no minimum), the label renders "(0 attempts)" and the collapsible contains an empty table. Why: A learner could see an empty, confusing table if a race condition or data corruption yields zero attempts. Fix: Either guard with `if (attempts.length === 0) return null` or show a message like "No attempts recorded."

- **`src/app/components/quiz/AttemptHistory.tsx:79` (confidence: 72)**: **`toLocaleString()` produces non-deterministic output across locales and environments.** The date display uses `new Date(attempt.completedAt).toLocaleString()` with no locale or options argument. This renders differently in en-US ("1/20/2025, 2:00:00 PM") vs en-GB ("20/01/2025, 14:00:00") vs test environments. Why: Minor -- users see their local format which is acceptable, but this makes snapshot testing or exact-text E2E assertions fragile. The E2E tests wisely avoid asserting on date text. Fix (optional): Consider passing `{ dateStyle: 'medium', timeStyle: 'short' }` for a more predictable, readable format.

- **`src/app/pages/QuizResults.tsx:33-43` (confidence: 70)**: **`loadAttempts` effect has no cleanup / abort guard.** If the component unmounts before the async `loadAttempts` promise resolves, `setAttemptsLoaded(true)` is called on an unmounted component. In React 18+ concurrent mode this causes a React warning. In React 19 the warning was removed, but the state update still runs pointlessly. Why: Minor -- no user-visible bug, but it indicates a missing cleanup pattern. Fix: Add an `isMounted` ref guard or use an AbortController pattern.

#### Nits

- **Nit** `src/app/components/quiz/AttemptHistory.tsx:49-50` (confidence: 70): Duplicate `aria-label="Quiz attempt history"` on both the wrapper `<div>` and the `<Table>`. The `<div>` wrapper is presentational (`hidden sm:block`) and having two identical labels may confuse screen readers. Remove the `aria-label` from the `<div>` wrapper and keep it only on the `<Table>` element.

- **Nit** `src/app/components/quiz/AttemptHistory.tsx:68` (confidence: 70): `cn(isCurrent ? 'bg-brand-soft' : '')` -- passing an empty string to `cn()` is harmless but noisy. Simplify to `cn(isCurrent && 'bg-brand-soft')` which is the idiomatic pattern used elsewhere in the codebase.

### Recommendations

1. **First:** Fix the `submitQuiz` append order (High) -- this is a latent correctness bug that will surface if any future code path reads `attempts[0]` between submit and reload.
2. **Second:** Replace the manual IDB seeding in the E2E test with shared helpers (High) -- reduces maintenance burden and aligns with established patterns.
3. **Third:** Destructure or annotate the unused `courseId`/`lessonId` props (High) -- prevents silent breakage when E16-S01 lands.
4. **Fourth:** Add empty-state guard (Medium) -- defensive programming for edge cases.
5. **Remaining nits:** Address at developer discretion.
