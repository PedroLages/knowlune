## Summary

The implementation correctly wraps the pre-seeded section in a shadcn `Collapsible` with ARIA support, persists collapse state to `localStorage`, and applies conditional opacity to the Overview gallery. The Radix Collapsible primitive is used correctly, and `prefers-reduced-motion` is respected. 10 findings across 4 severity levels.

---

## What Works Well

1. **Collapsible ARIA is thorough.** `aria-expanded`, `aria-label`, and chevron rotation all update correctly. Keyboard activation (Enter/Space) works. Labels switch correctly between expand/collapse states.
2. **localStorage persistence is well-structured.** `COLLAPSE_KEY` constant, `useState` initializer pattern, and the auto-collapse `useEffect` (only fires when `stored === null`) are clean and correct.
3. **`motion-reduce:transition-none` on ChevronDown** respects `prefers-reduced-motion` — a detail often missed.

## Findings

### High

**H1: `localStorage.getItem()` in `useState` initializer can throw in restricted contexts**
- **Location**: `src/app/pages/Courses.tsx:54-55`
- **Confidence**: 90
- **Impact**: A learner in a restricted browser environment (private browsing on some browsers, storage quota exceeded) gets a white screen with no error feedback — component crashes during render.
- **Fix**: Wrap in try/catch: `const stored = (() => { try { return localStorage.getItem(COLLAPSE_KEY) } catch { return null } })()`

**H2: `localStorage.setItem()` calls lack try/catch**
- **Location**: `src/app/pages/Courses.tsx:229` and useEffect at line 74
- **Confidence**: 85
- **Impact**: If `setItem` throws (quota exceeded), the collapse state silently diverges from what was persisted — the learner collapses the section but it re-expands on next load.
- **Fix**: Wrap both `setItem` calls in try/catch with `console.warn`.

**H3: WCAG AA contrast violation — opacity applied to section heading area**
- **Location**: `src/app/pages/Courses.tsx:376`
- **Confidence**: 82
- **Impact**: At 0.6 opacity on `#FAF5EE` background, `text-muted-foreground` (~#656870) yields approximately 2.2:1 contrast ratio — below the WCAG AA 4.5:1 minimum for normal text. Learners with low vision cannot read the "Sample Courses (N)" heading.
- **Fix**: Apply opacity only to `CollapsibleContent` (the grid), not the header area. Or add `opacity-100` directly on the `h2` to exempt it from the container's opacity.

**H4: Auto-collapse `useEffect` fires on every `importedCourses.length` change, not just initial load**
- **Location**: `src/app/pages/Courses.tsx:70-76`
- **Confidence**: 75
- **Impact**: If a learner imports their first course while browsing (e.g., imports during a session where they manually expanded the section), the effect fires and force-collapses, overriding their explicit preference.
- **Fix**: Track whether auto-collapse has already been attempted this mount with a `useRef` flag.

### Medium

**M1: Touch target on collapse toggle is 32×32px — below the 44×44px minimum**
- **Location**: `src/app/pages/Courses.tsx:385-401`
- **Confidence**: 78
- **Impact**: Mobile learners struggle to tap the small collapse toggle; disproportionately affects users with motor impairments.
- **Fix**: Change `className="p-2"` to `className="p-3"` (12px padding + 20px icon = 44px), or use `size="icon"` variant.

**M2: Non-null assertion on `boundingBox()` results in E2E test**
- **Location**: `tests/e2e/story-e23-s05.spec.ts:81`
- **Confidence**: 80
- **Impact**: If the element is not visible or has zero dimensions, `boundingBox()` returns `null` and the test crashes with an unhelpful error.
- **Fix**: Add explicit null checks: `expect(importedBox).toBeTruthy()` before comparing `.y` values.

**M3: Layout shift when search query hides the imported section header**
- **Location**: `src/app/pages/Courses.tsx:316`
- **Confidence**: 72
- **Impact**: Condition `(importedCourses.length > 0 || !searchQuery.trim())` means when there are zero imported courses AND user types a search query, the entire imported courses section disappears — jarring layout shift.
- **Fix**: Consider always showing the section header when empty, regardless of search state.

**M4: `transition-opacity` on Overview card wrapper is present; same should be verified on Courses wrapper**
- **Location**: `src/app/pages/Courses.tsx:291` (section wrapper)
- **Confidence**: 72
- **Note**: `motion-reduce:transition-none` is present on `Overview.tsx:342` but confirm it's on the `Courses.tsx` section opacity wrapper as well.

### Nits

**N1: Conditional `data-testid` changes based on state, making stable selectors harder**
- **Location**: `src/app/pages/Overview.tsx:340`
- `data-testid` uses ternary `importedCourses.length > 0 ? 'sample-course-card' : 'course-card-${course.id}'`. Consider always including the course ID.

**N2: Template literals vs `cn()` for conditional class strings**
- **Location**: `src/app/pages/Courses.tsx:376` and `Overview.tsx:342`
- **Note**: The implementation plan explicitly chose template literals over `cn()` to avoid adding a new import (Plan Review Notes, point 4). This is a style preference rather than a bug. If `cn()` is later added to these files, convert for consistency.
