## Summary

This story wraps two unguarded `JSON.parse()` calls in `Layout.tsx` with try/catch blocks to prevent corrupted localStorage values from crashing the app. Both `sidebarOpen` and `sidebarCollapsed` state initializers are now protected.

## Findings

### MEDIUM: Silent catch blocks trigger ESLint warning (lines 260, 278)

The catch blocks have no `console.warn()` or user feedback. The ESLint rule `error-handling/no-silent-catch` flags these. Since silently recovering is intentional (corrupted localStorage is not user-actionable), add `// silent-catch-ok` comments inside each catch block to suppress the warning and document the intent.

**File:** `src/app/components/Layout.tsx:260, 278`

### LOW: No type validation on parsed value

`JSON.parse(legacy)` or `JSON.parse(saved)` could return any valid JSON (string, number, object, array), not just a boolean. If someone stores `"hello"` or `42` in localStorage, the state would be set to a non-boolean value. Consider adding `typeof parsed === 'boolean'` guard.

**File:** `src/app/components/Layout.tsx:256, 259, 274, 277`

## Verdict

**PASS** — The core defensive fix is correct and addresses the audit finding. The MEDIUM issue (silent catch) should be fixed before merge. The LOW issue (type validation) is optional but recommended.
