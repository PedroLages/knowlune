## Findings

### [High] clearIndexedDBStore throws on missing store in afterEach cleanup

**File:** `tests/support/helpers/indexeddb-seed.ts:210-212`
**Confidence:** 82

`clearIndexedDBStore` throws after 10 retries when the store doesn't exist. Called in `afterEach`, this masks the real test failure with a confusing "Store not found" error. `indexedDB.open(dbName)` on a non-existent DB creates an empty DB (version 1, no stores), so the store will never appear regardless of retries.

**Fix:** Add `{ throwOnMissing?: boolean }` option defaulting to `false` for cleanup use cases, or silently return when the store doesn't exist on first attempt.

### [High] submitQuiz helper swallows errors + 1s implicit wait per submission

**File:** `tests/e2e/story-e15-s06.spec.ts:109`
**Confidence:** 78

`.catch(() => false)` on the confirmation dialog check doesn't distinguish "dialog not found" (expected) from "dialog found but click failed" (unexpected). Also adds ~1s hard wait to every submission.

**Fix:** Use `if (await confirmBtn.count() > 0)` pattern or `Promise.race` with shorter timeout.

### [Medium] previousAttemptTimeSpent doesn't filter untimed prior attempts

**File:** `src/app/pages/QuizResults.tsx:144-145`
**Confidence:** 75

If a learner took attempt 1 as "untimed" accommodation and attempt 2 as "standard" on a timed quiz, "Previous: Xm Ys" would display for an attempt where time tracking was meaningless.

**Fix:** Add `priorAttempt.timerAccommodation !== 'untimed'` guard to the `previousAttemptTimeSpent` memo.

### [Medium] previousAttemptTimeSpent passes timeSpent=0 through

**File:** `src/app/components/quiz/ScoreSummary.tsx:165`
**Confidence:** 72

`previousAttemptTimeSpent != null` passes `0` through, rendering "Previous: 1s" due to Math.max clamp — misleading for a data integrity issue.

**Fix:** Add `&& previousAttemptTimeSpent > 0` guard.

### [Medium] requestAnimationFrame-based delay unreliable in headless Chromium

**File:** `tests/support/helpers/indexeddb-seed.ts:56-62`
**Confidence:** 70

rAF cadence is not guaranteed at 60fps in headless Playwright. Replace with `setTimeout` for deterministic delay timing.

### [Nit] Double type-cast ceremony in navigateToQuiz

**File:** `tests/e2e/story-e15-s06.spec.ts:80-84`
**Confidence:** 65

`as unknown as Record<string, unknown>` in 4 places. Consider making seed helpers generic or accepting typed objects directly.

### [Nit] text-muted-foreground/70 may drop below contrast on some backgrounds

**File:** `src/app/components/quiz/ScoreSummary.tsx:165`
**Confidence:** 60

The 70% opacity stacks with muted-foreground's already-reduced contrast. Worth spot-checking against `bg-card` in both modes.
