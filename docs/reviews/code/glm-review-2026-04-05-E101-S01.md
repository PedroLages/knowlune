### Findings

#### Blockers
- **[src/services/AudiobookshelfService.ts:56] (confidence: 95)**: **Timeout leak on successful fetch.** `clearTimeout(timeoutId)` is called before `response.json()`. If `response.json()` hangs (e.g., a streaming/malformed response that never completes), the `AbortController` timeout has already been cleared and the AbortController itself goes out of scope, so the promise hangs forever with no cleanup. Fix: Move `clearTimeout(timeoutId)` into a `finally` block (or after `response.json()`) so the timeout covers the full body-read as well, ensuring the abort signal can still cancel a hung body download:
  ```typescript
  try {
    const response = await fetch(...)
    const data = (await response.json()) as T
    return { ok: true, data }
  } catch (err: unknown) {
    ...
  } finally {
    clearTimeout(timeoutId)
  }
  ```

#### High Priority

None.

#### Medium

- **[src/services/AudiobookshelfService.ts:123] (confidence: 80)**: **API key leaked in URL via query parameter.** `getStreamUrl` appends the API key as a `?token=` query parameter. This key will appear in browser history, server access logs, referrer headers, and any proxy logs. While the docstring acknowledges this is needed for `<audio>` elements, the API key grants full user-level access to the ABS server. Fix: Consider using a short-lived, scoped token or single-use session token from the ABS API if available; or at minimum document the exposure tradeoff clearly so future integrators understand the risk. If ABS supports it, prefer `#token` (fragment) which isn't sent to the server in the request URI.

- **[src/services/AudiobookshelfService.ts:44] (confidence: 70)**: **Controller garbage-collected before abort fires in edge case.** The `AbortController` is a local variable inside `absApiFetch`. In environments with aggressive GC (rare but possible in long-idle SPAs), if no other reference to `controller` exists after the `fetch` promise is created, the controller could theoretically be collected before the timeout fires. Fix: This is low risk in practice, but can be made deterministic by storing the controller on the timeout closure explicitly (it already is via the `controller.abort()` closure, so this is borderline). No action required unless issues surface.

*After further review, removing the second medium finding as the closure does capture `controller` — it's safe.*
