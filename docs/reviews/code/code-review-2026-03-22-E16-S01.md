## Findings

### High Priority

**H1: FIB review panel re-derives correctness independently from `mode` prop**
- **File:** `src/app/components/quiz/questions/FillInBlankQuestion.tsx:104-135`
- **Confidence:** 90
- The review panel computes `isCorrect` via its own `trim() + toLowerCase()` comparison instead of trusting `mode === 'review-correct'`. This can diverge from the stored `answerRecord.isCorrect`. A learner could see green "correct" text inside an orange "incorrect" card (or vice versa).
- **Fix:** Replace the internal comparison with `const isCorrect = mode === 'review-correct'`. Keep the case-insensitive note as supplementary info only.

**H2: String interpolation for `className` instead of `cn()`**
- **File:** `src/app/components/quiz/questions/FillInBlankQuestion.tsx:116`
- **Confidence:** 92
- `` `mt-2 rounded-lg p-3 text-sm ${isCorrect ? 'bg-success-soft text-foreground' : 'bg-warning/10 text-foreground'}` `` â€” template literals bypass Tailwind merge and can be purged in production.
- **Fix:** `className={cn('mt-2 rounded-lg p-3 text-sm', isCorrect ? 'bg-success-soft text-foreground' : 'bg-warning/10 text-foreground')}`

**H3: Manual IndexedDB seeding in E2E â€” use shared helpers**
- **File:** `tests/e2e/story-e16-s01.spec.ts:67-99`
- **Confidence:** 85
- Custom `seedStore()` function duplicates retry logic from `tests/support/helpers/seed-helpers.ts`. This is the same anti-pattern flagged in E13-S02.
- **Fix:** Import and use `seedQuizzes` and `seedQuizAttempts` from `tests/support/helpers/seed-helpers.ts`.

**H4: Sorted questions array re-created on every render**
- **File:** `src/app/components/quiz/QuizReviewContent.tsx:26`
- **Confidence:** 78
- `[...quiz.questions].sort(...)` creates a new array reference on every render, causing `ReviewQuestionGrid` to re-render unnecessarily on every state change.
- **Fix:** `const questions = useMemo(() => [...quiz.questions].sort((a, b) => a.order - b.order), [quiz.questions])`
