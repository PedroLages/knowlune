## What Works Well

1. **Solid autosave architecture.** The 3s debounce + 10s max-wait using separate `saveTimeoutRef` and `maxWaitRef` is well-designed. The `doSave` function deduplicates via `lastSavedContentRef`, and unmount cleanup correctly flushes pending saves (in theory ŌĆö see High finding).

2. **Clean Tiptap migration.** Edit/Preview tabs replaced with inline WYSIWYG. The `onSaveRef` latest-ref pattern avoids stale closures. Toolbar uses proper `aria-label` attributes and autosave indicator uses `aria-live="polite"`.

3. **LessonPlayer fixes are correct.** `handleNoteChange` now accepts `tags` and `currentVideoTime` was wired up.

## Findings

### Blockers

**B1: `video://` protocol not registered in Link extension**
- File: `src/app/components/notes/NoteEditor.tsx:102-104`
- Link extension's `isAllowedUri` validates against an allowlist that does not include `video`. When `setContent(initialContent)` is called on lesson re-entry, `video://` links fail validation and are stripped to plain text.
- **Breaks AC3** (timestamp links don't survive navigation) and **AC4** (clicking timestamps to seek).
- Fix: Register `video` protocol: `Link.configure({ protocols: ['video'], ... })` ŌĆö or better, configure through StarterKit (see design review B2).

**B2: `onVideoSeek` stale closure in `useEditor`**
- File: `src/app/components/notes/NoteEditor.tsx:116-131`
- `handleClick` inside `useEditor` captures `onVideoSeek` from initial render. Unlike `onSave` (which uses `onSaveRef`), `onVideoSeek` has no ref indirection. `useEditor` creates the handler once and doesn't recreate it.
- Currently works by accident because `handleVideoSeek` calls the stable `setSeekToTime` setter, but fragile.
- Fix: Add `onVideoSeekRef` parallel to `onSaveRef`.

### High Priority

**H1: Unmount cleanup captures `editor = null`**
- File: `src/app/components/notes/NoteEditor.tsx:170-185`
- The cleanup effect has `[]` deps but references `editor`. On initial render, `useEditor` returns `null`. The empty-deps closure captures `editor = null`, so on unmount the forced save never fires.
- Impact: Content loss if learner navigates away before 3s debounce fires.
- Fix: Store editor in a ref and read from it in the cleanup.

**H2: `handleNoteChange` propagates saved content back as `initialContent`**
- File: `src/app/pages/LessonPlayer.tsx:242-247`
- `setNoteText(value)` causes re-render ŌåÆ `initialContent` changes ŌåÆ `useEffect` in NoteEditor compares `getHTML() !== initialContent` ŌĆö fragile if Tiptap normalizes HTML differently.
- Fix: Remove `setNoteText(value)` from `handleNoteChange` ŌĆö NoteEditor manages its own state.

**H3: Missing AC4 click-to-seek E2E test**
- File: `tests/e2e/story-3-1.spec.ts`
- AC4 requires "clicking the timestamp link seeks the video" but no test verifies this. Given B1 (links stripped on reload), this gap means broken behavior would pass CI.

### Medium

**M2: `window.prompt()` with no URL validation**

**M3: Toolbar buttons 32x32px (below 44px minimum)**
