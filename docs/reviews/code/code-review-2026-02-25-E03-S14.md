## Findings

### High Priority

1. **Context menu has `role="menu"` but no arrow key navigation between items** — `src/app/components/notes/TableContextMenu.tsx:132-169`

   The WAI-ARIA Menu pattern requires `ArrowDown`/`ArrowUp` to move focus between `menuitem` elements, and `Home`/`End` to jump to first/last item. Currently the first button receives focus on open (good), but a keyboard-only user cannot reach subsequent items without Tab — which is non-standard for `role="menu"` and will confuse screen reader users who expect arrow navigation.

   **Fix**: Add a `keydown` handler on the menu container implementing `ArrowDown` (focus next), `ArrowUp` (focus previous), `Home` (first), `End` (last), wrapping at boundaries, skipping separators.

2. **`--table-selected` defined in `:root` but missing from `.dark`** — `src/styles/theme.css:62`

   The token `--table-selected: oklch(0.92 0.05 250)` is a light-mode blue tint. In dark mode, since `--table-selected` has no `.dark` override, it falls through to `:root`. On a dark background this will be a jarring bright rectangle.

   **Fix**: Add `--table-selected: oklch(0.25 0.05 250);` to `.dark` in `theme.css`.

3. **AC2 specifies "Enter creates a new row at the end" but no E2E test verifies Enter behavior** — `tests/e2e/story-3.14.spec.ts`

   The test at line 225 tests Tab, not Enter. TipTap's default Table behavior: Tab at the last cell creates a new row; Enter inside a cell inserts a newline. If the AC literally requires Enter to create a new row, custom code is needed. If it means Tab, the AC text should be updated.

   **Fix**: Either (a) add custom Enter keymap + E2E test, or (b) clarify AC meant Tab and update the AC text.

### Medium

4. **`useLayoutEffect` with `position.y` in deps causes guaranteed double-render on viewport clamp** — `src/app/components/notes/TableContextMenu.tsx:60-67`

   When the menu opens near the bottom, `setPosition` fires, changing `position.y`, which re-triggers the effect. Fix: Remove `position.y` from deps, run clamp only when `visible` transitions to `true`.

5. **Grid picker label shows "col x row" (inverted convention)** — `src/app/components/notes/TableGridPicker.tsx:94, :102`

   Every spreadsheet and the AC itself says "rows x columns." Both the display text and aria-label present columns first. Fix: Swap to `${hoveredRow} x ${hoveredCol}`.

6. **Syntax highlighting comment color ~3.8:1 contrast ratio** — `src/styles/index.css:108-109`

   `oklch(0.45 0.02 250)` still falls short of WCAG AA 4.5:1 minimum for normal text. Pre-existing but persists. Fix: Darken to ~`oklch(0.40 0.02 250)`.

7. **Inline `style={{ position: 'fixed' }}` instead of Tailwind** — `src/app/components/notes/TableContextMenu.tsx:139`

   `position: fixed` can be `className="fixed"`, keeping only dynamic `left`/`top` in `style`.

### Nits

8. `menuItems` array recreated on every render — could memoize with `useMemo`.
9. `grid-cols-[repeat(6,36px)]` uses arbitrary values — `grid-cols-6` with `size-9` cells would avoid brackets.
10. Mixed quote styles between TableGridPicker (double) and TableContextMenu (single).
