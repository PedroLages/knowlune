## New Finding in R2

### BLOCKER (fixed during review)

**[BLOCKER] `useMemo` stale closure in Library.tsx — `filteredBooks` returned empty array**
- File: `src/app/pages/Library.tsx:101`
- `useMemo(() => getFilteredBooks(), [getFilteredBooks, books, filters])` cached an empty array from the initial render. Because `getFilteredBooks` is a stable function reference in Zustand's `create()`, and Zustand's batched state updates could cause the memo to evaluate before `books` was populated, the memo returned stale empty results even after books loaded from IndexedDB.
- **Impact:** All E2E tests failed — books were loaded (visible in storage indicator) but the grid showed "No books match your filters."
- **Fix:** Removed `useMemo` — `getFilteredBooks()` reads from `get()` which always returns latest state. The performance impact is negligible for typical library sizes.
- Commit: `9c458e57`
