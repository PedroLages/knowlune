## Findings

### HIGH

**[HIGH] `loadNextPage` captures stale `state.pagination` in closure (Confidence: 85)**
- File: `src/app/hooks/useAudiobookshelfSync.ts:179`
- `loadNextPage` depends on `state.pagination` in its `useCallback` deps, but `state` is the useState value at render time. When `syncCatalog` updates pagination state, `loadNextPage` may still reference the old pagination object on the same render cycle.
- **Impact:** Could skip loading next page or re-request an already-loaded page
- **Fix:** Use a ref for pagination state, or access it via a callback pattern

### MEDIUM

**[MEDIUM] `new Date().toISOString()` called in `mapAbsItemToBook` and `upsertAbsBook` — non-deterministic (Confidence: 75)**
- File: `src/app/hooks/useAudiobookshelfSync.ts:90` and `src/stores/useBookStore.ts:231`
- Not in test code, but creates non-deterministic `createdAt`/`updatedAt` values that could complicate snapshot testing
- Not a bug — standard pattern in production code — but noted for completeness

**[MEDIUM] `AbsPaginationSentinel` may fire multiple `loadNextPage` calls simultaneously (Confidence: 80)**
- File: `src/app/pages/Library.tsx:391-394`
- When IntersectionObserver fires, it loops through all servers and calls `loadNextPage` for each. No guard against calling `loadNextPage` while a previous page is still loading (the `syncingServers` ref in the hook prevents duplicate syncs for the *same* server but not concurrent page-fetches).
- **Impact:** Low in practice since `syncingServers` Set provides adequate protection per-server

### LOW

**[LOW] `duration` extracted from `metadata.duration` but ABS may also have it at `media.duration` (Confidence: 60)**
- File: `src/app/hooks/useAudiobookshelfSync.ts:68`
- Story notes say duration may be at either `media.metadata.duration` or `media.duration`. Only `metadata.duration` is used.
- **Fix:** `const duration = absItem.media.metadata.duration || absItem.media.duration || undefined`

**[LOW] ESLint disable comment on catch block is correct but misleading (Confidence: 50)**
- File: `src/app/hooks/useAudiobookshelfSync.ts:160`
- Comment says "handled via toast below" — the toast IS on the next line, so the comment is accurate. No issue.

### NITS

**[NIT] `source` filter `'all'` value stored as undefined but typed as `'all' | 'local' | 'audiobookshelf'`**
- When "All" tab is clicked, `setFilter('source', 'all')` is called, which then converts to `undefined` via the `value === 'all' ? undefined : value` logic. This works but the type suggests `'all'` is a stored value when it never is.

## Verdict

**PASS** — 1 HIGH (stale closure in loadNextPage — low practical impact due to IntersectionObserver throttling), 2 MEDIUM, 2 LOW, 1 NIT. No blockers.
