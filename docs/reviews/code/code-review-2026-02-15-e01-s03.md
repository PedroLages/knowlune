## What Works Well

1. **Solid optimistic update pattern with rollback** in `useCourseImportStore.ts` (lines 101-131). The `updateCourseTags` action captures `oldTags` before mutation and rolls back on persistence failure.

2. **Well-structured E2E test suite** in `tests/e2e/story-1-3-organize-by-topic.spec.ts`. The `seedAndReload` helper is clean, tests are organized by acceptance criterion, and the IndexedDB fixture properly handles cleanup. The use of `data-testid` selectors throughout is resilient.

3. **Good accessibility on TopicFilter** (`TopicFilter.tsx`, lines 23-24, 34). Using `role="group"`, `aria-label="Filter by topic"`, and `aria-pressed` on toggle buttons is textbook accessible toggle pattern.

## Findings

### High Priority (4)

1. **`ImportedCourseCard.tsx:14`** — `getAllTags()` called on every render of every card, causing O(N*M) computation. The `Courses.tsx` parent already computes `allTags` via `useMemo` (line 60) but the card ignores it and recomputes independently.

   **Fix**: Lift `allTags` to `Courses.tsx` and pass as a prop to `ImportedCourseCard`.

2. **`useCourseImportStore.ts:46-65`** — `addImportedCourse` does not normalize tags. Tags imported via File System API enter the system un-normalized, which could cause case-sensitivity mismatches with tags created via `updateCourseTags`.

   **Fix**: Normalize tags in `addImportedCourse` before persisting.

3. **`useCourseImportStore.test.ts`** — Missing unit tests for `updateCourseTags` and `getAllTags`. These are the two core store methods for this story, yet neither has a unit test.

   **Fix**: Add test cases for normalization, optimistic update, rollback, and `getAllTags` deduplication.

4. **`Courses.test.tsx:30-41`** — Store mock missing `updateCourseTags` and `getAllTags`. Will cause runtime errors when tag-related code paths execute.

   **Fix**: Add `updateCourseTags: vi.fn()` and `getAllTags: vi.fn(() => [])` to mock.

### Medium (4)

1. **`TagBadgeList.tsx:18`** — Uses string interpolation for className instead of project's `cn()` utility.

2. **`TagEditor.tsx`** — No maximum tag limit enforced. A user can add unlimited tags.

3. **`TagEditor.tsx:51`, `TagBadgeList.tsx:37`** — Using `h-5 w-5` instead of Tailwind v4 `size-5` shorthand.

4. **`TopicFilter.tsx:41`** — Hardcoded `bg-blue-600` / `text-white` bypasses theme system. Acceptable for now since it matches project convention.

### Nits (4)

1. **`ImportedCourseCard.tsx:44-54`** — Lone "+" button with `mb-3` margin when course has no tags.
2. **`TagEditor.tsx:84`** — `create-${trimmed}` value prefix could collide with a tag literally named "create-react".
3. **`ImportedCourseCard.tsx:42`** — `toLocaleDateString()` without explicit format options.
4. **`Courses.tsx:60`** — Non-obvious `importedCourses` dependency in `allTags` memo deserves a comment.
