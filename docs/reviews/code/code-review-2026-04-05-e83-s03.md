# Code Review: E83-S03 Library Grid and List Views

**Date:** 2026-04-05
**Reviewer:** Claude Opus 4.6
**Story:** E83-S03 Library Grid and List Views
**Files Changed:** BookCard.tsx, BookListItem.tsx, BookStatusBadge.tsx, Library.tsx

## Findings

### HIGH

1. **TypeScript error: unused `importBook` import** — Library.tsx:27
   - `const importBook = useBookStore(s => s.importBook)` is declared but never used. This causes `tsc --noEmit` to fail, which blocks CI.
   - **Fix:** Remove line 27.

### MEDIUM

2. **Prettier formatting violations** — BookStatusBadge.tsx, Library.tsx
   - Two files fail `prettier --check`. Must run `npx prettier --write` on them.

3. **`text-white` hardcoded instead of design tokens** — Library.tsx:77,90 and BookStatusBadge.tsx:13-16
   - Should use `text-brand-foreground`, `text-success-foreground`, etc.

4. **Drag-drop handler discards dropped files** — Library.tsx:50-61
   - `handleDrop` extracts EPUB files from `e.dataTransfer.files` but doesn't pass them to `BookImportDialog`. The files are lost when `setImportOpen(true)` opens the dialog.

5. **`Date.now()` in production code** — BookListItem.tsx:32
   - `relativeTime()` uses `Date.now()` which makes it non-deterministic for testing. Consider accepting an optional `now` parameter or using a shared time utility.

### LOW

6. **Inline style for progress bar width** — BookCard.tsx:72, BookListItem.tsx:107
   - `style={{ width: \`\${book.progress}%\` }}` is flagged by `no-inline-styles` ESLint rule. This is a legitimate dynamic value case, but worth noting.

7. **`role="article"` semantic choice** — BookCard.tsx:32, BookListItem.tsx:67
   - `role="article"` is semantically for self-contained compositions (like blog posts). A book card is arguably better as a `link` or generic clickable element. Not a blocker since it works.

### NIT

8. **`React` import not needed for `React.memo`** — BookCard.tsx:9, BookListItem.tsx:9
   - With React 19 JSX transform, `import React` is only needed for `React.memo`. Could destructure: `import { memo } from 'react'`.

## Architecture

- Component decomposition is clean: BookCard, BookListItem, BookStatusBadge are well-separated
- React.memo on list components is good for performance
- Zustand selectors are granular (individual fields) -- good practice
- useCallback on drag handlers prevents unnecessary re-renders

## Verdict

ISSUES FOUND — 1 HIGH (unused import causes TS error), 4 MEDIUM, 2 LOW, 1 NIT.
