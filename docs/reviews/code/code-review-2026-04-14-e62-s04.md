## Summary

This story adds a comprehensive E2E test spec for the Knowledge Map FSRS integration feature. The test file covers 7 acceptance criteria including gradient treemap rendering, decay prediction tooltips, TopicDetailPopover memory decay section, dark mode rendering, and deterministic time verification. No production code changes — test-only.

## Findings

### MEDIUM

1. **Dark mode class lost on navigation** — `tests/e2e/story-e62-s04.spec.ts:352-358`
   The dark mode test sets `document.documentElement.classList.add('dark')` via `page.evaluate()` before `page.goto('/knowledge-map')`. The `goto()` creates a new document, which loses the class. The test relies on `localStorage.setItem('knowlune-theme', 'dark')` persisting, which works IF the app reads localStorage on mount. However, the `page.evaluate()` runs after the initial `beforeEach` `goto('/')`, then `seedAllData` runs, then `goto('/knowledge-map')`. The dark mode class is set on the first page's document but the second navigation destroys it. If the app applies dark mode from localStorage on mount (which it does based on theme system), this works — but the `classList.add('dark')` line is misleading dead code.
   **Recommendation**: Remove the `classList.add('dark')` line or move localStorage seeding to before the `beforeEach` `goto('/')` and use `addInitScript` for the class.

2. **Fragile Radix popover selector** — `tests/e2e/story-e62-s04.spec.ts:314,337`
   `[data-radix-popper-content-wrapper]` is a Radix internal implementation detail. Prefer `data-testid` on the PopoverContent component. Since this is test-only and doesn't modify production code, this is acceptable for now but should be addressed when the component is next modified.

### LOW

3. **AC-1 gradient assertion is weak** — `tests/e2e/story-e62-s04.spec.ts:247-256`
   The test asserts `fills.length >= 2` which would pass even with just 2 discrete colors. The AC says "not just 3 discrete colors" suggesting a gradient, but the assertion only checks for multiple colors. With 3 seed courses this is inherently limited. Acceptable given the seed data constraints.

4. **AC-6 console error filter is narrow** — `tests/e2e/story-e62-s04.spec.ts:381-383`
   Only filters console errors containing "treemap", "color", or "NaN". Other rendering errors (e.g., "Cannot read property", "undefined is not a function") would be missed. Consider asserting `consoleErrors.length === 0` or filtering only known-benign errors.

### NIT

5. **Magic number timeout** — `tests/e2e/story-e62-s04.spec.ts:240,267,280,306,326,362`
   `{ timeout: 10000 }` is repeated 6 times. Extract to a constant like `const TREEMAP_RENDER_TIMEOUT = 10_000`.

## Verdict

No blockers. 2 MEDIUM, 2 LOW, 1 NIT.
