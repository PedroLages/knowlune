## Findings

### Blocker

**B1: `urlTransform` fix only in working tree — committed branch is broken (confidence: 98)**
- File: `src/app/components/notes/NoteEditor.tsx:272-274`
- The `urlTransform` override preserving `video://` URLs exists ONLY as uncommitted working tree change. React-markdown's built-in `defaultUrlTransform` strips `video://` URLs to empty strings.
- Impact: Committed branch renders timestamp links with empty hrefs, breaking AC2 (seek) and AC3 (visual).
- Fix: Commit the working tree changes.

### High Priority

**H1: `handleNoteChange` drops tags — silent data loss (confidence: 92)**
- File: `src/app/pages/LessonPlayer.tsx:306-310`
- NoteEditor's `onSave` provides `(content, tags)` but `handleNoteChange` only accepts `(value: string)`. Tags are silently discarded.
- Fix: Update signature to forward tags to `saveNote()`.

**H2: `createVideoLinkComponent` recreated on every render (confidence: 85)**
- File: `src/app/components/notes/NoteEditor.tsx:279`
- Called inline in JSX, creating new component reference each render. Combined with ~4x/sec re-renders from `videoCurrentTime`, causes unnecessary DOM churn.
- Fix: Memoize with `useMemo(() => createVideoLinkComponent(onVideoSeek), [onVideoSeek])`.

**H3: Alt+T shortcut scope mismatch in overlay (confidence: 82)**
- File: `src/app/components/figma/VideoShortcutsOverlay.tsx:31`
- Alt+T listed in "Controls" but only works when textarea is focused. Video player ignores Alt combos.
- Fix: Move to "Notes" section with focus note, or implement at LessonPlayer level.

### Medium

**M1: Duplicate `formatTimestamp` — 4th copy of time formatting (confidence: 80)**
- Fix: Extract shared `formatTime` utility to `@/lib/time.ts`.

**M2: `videoCurrentTime` re-render cascade (confidence: 78)**
- `setVideoCurrentTime(time)` ~4x/sec re-renders all 4 NoteEditor instances.
- Fix: Use ref + throttled state update.

**M3: No unit tests for NoteEditor logic (confidence: 75)**
- `parseVideoSeconds`, `formatTimestamp`, and `insertTimestamp` are untested.
- Fix: Add unit tests covering both URL formats and edge cases.

### Nits
