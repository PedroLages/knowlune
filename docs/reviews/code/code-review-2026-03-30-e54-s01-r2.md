## New Issues Found (Round 2)

### HIGH: E2E Tests Fail ‚Äî 6 of 9 tests broken (story-e54-s01.spec.ts)

**Root cause:** `simulateVideoEnd()` waits for `page.locator('video')` to be visible, but seeded ImportedVideo records lack `fileHandle` properties. Without a fileHandle, `LocalVideoContent` renders "Video not found" ‚Äî never producing a `<video>` element.

**Failing tests:** AC1, AC2 (2 tests), AC3 (2 tests), AC6 ‚Äî all require `simulateVideoEnd()`.

**Fix options:**
1. Mock the video element at the page level (e.g., inject a stub `<video>` via route interceptor)
2. Seed videos with a blob URL or stub fileHandle
3. Refactor tests to trigger `onEnded` callback directly without needing a real video element (e.g., dispatch a custom event on a known test hook element)

### MEDIUM: Lint warning ‚Äî silent .catch() on line 126 (UnifiedLessonPlayer.tsx:126)

The `.catch()` in the lesson metadata useEffect (line 134) logs to console but doesn't surface a toast to the user. ESLint `error-handling/no-silent-catch` warning. The `console.error` is reasonable for metadata degradation, but adding `// silent-catch-ok` would suppress the warning.

**File:** `src/app/pages/UnifiedLessonPlayer.tsx:126`

### LOW: Implicit dependency on PlayerHeader calling loadCourseProgress

`UnifiedLessonPlayer` uses `getItemStatus` in `checkCourseCompletion` but never calls `loadCourseProgress` itself. It relies on `PlayerHeader` (a child component) to call `loadCourseProgress(courseId)` via its own `useEffect`. This creates a fragile ordering dependency ‚Äî if `PlayerHeader` is ever removed or lazily loaded, course completion detection breaks silently.

**Suggestion:** Add `loadCourseProgress(courseId)` call in `UnifiedLessonPlayer` directly, or document the dependency.
