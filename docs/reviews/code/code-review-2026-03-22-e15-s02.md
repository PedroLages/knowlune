## Code Review: E15-S02 — Configure Timer Duration and Accommodations

### What Works Well

1. **Zod validation on localStorage read** (`Quiz.tsx:62-72`). The `loadSavedAccommodation()` function uses `TimerAccommodationEnum.safeParse()` to guard against corrupted/tampered stored values, falling back to `'standard'`. This is a best practice for client-side persistence.

2. **Local state commit pattern in the modal** (`TimerAccommodationsModal.tsx:51-61`). Using a local `selected` state that only commits on Save prevents premature side effects from radio interactions. The `useEffect` sync on `open` is clean and avoids stale state when reopening.

3. **Untimed guard in timer initialization** (`Quiz.tsx:248-252`). The added `currentProgress.timerAccommodation !== 'untimed'` check correctly prevents the timer from starting on a timed quiz when the user selects untimed accommodation. The lessons-learned note about this edge case shows good testing discipline.

### Findings

#### Blockers

None.

#### High Priority

- **[src/app/pages/QuizResults.tsx:78] (confidence: 92)**: Retake from results page ignores saved accommodation. `handleRetake` calls `retakeQuiz(lessonId)` without passing the accommodation, so retakes initiated from the results page always use `'standard'` time — even if the learner previously selected 150%/200%/untimed. This silently resets the learner's accessibility accommodation, which directly violates AC5 ("my preference persists... I don't need to re-configure on every attempt"). Why: Learners who depend on extended time accommodations will unexpectedly get the standard timer on retake, creating a frustrating and inaccessible experience. Fix: Load the saved accommodation from localStorage in QuizResults and pass it to `retakeQuiz()`:
  ```typescript
  const acc = loadSavedAccommodation(lessonId) // extract to shared util
  await retakeQuiz(lessonId, acc)
  ```

- **[Recurring] [src/stores/useQuizStore.ts:87, src/app/components/quiz/QuizStartScreen.tsx:34, src/app/components/quiz/TimerAccommodationsModal.tsx:38-41] (confidence: 88)**: Multiplier logic is duplicated in three places. The ternary `acc === '150%' ? 1.5 : acc === '200%' ? 2 : 1` appears in the store, `formatTimeBadge`, and the modal's `ACCOMMODATION_OPTIONS` array. If a new accommodation tier is added (e.g., 300%), all three must be updated in sync. Why: Divergent multiplier values would cause the badge to show one time but the timer to initialize with another — a confusing mismatch for learners. Fix: Extract a single `getAccommodationMultiplier(acc: TimerAccommodation): number | null` utility into `src/lib/quiz-utils.ts` or co-locate with the `ACCOMMODATION_OPTIONS` constant and import everywhere.

- **[src/app/components/quiz/TimerAccommodationsModal.tsx:75] (confidence: 82)**: Unsafe `as` cast on radio value. `onValueChange={v => setSelected(v as TimerAccommodation)}` bypasses type checking. If the RadioGroup ever emits a value not in the `TimerAccommodation` union (e.g., from a stale DOM state or Radix edge case), the cast silently passes an invalid value through. Why: Could propagate an invalid accommodation value all the way to the store's multiplier logic, resulting in `NaN` timer values. Fix: Use a runtime guard:
  ```typescript
  onValueChange={v => {
    const parsed = TimerAccommodationEnum.safeParse(v)
    if (parsed.success) setSelected(parsed.data)
  }}
  ```

#### Medium

- **[src/app/components/quiz/QuizStartScreen.tsx:82-87] (confidence: 75)**: Badge className uses string literal instead of `cn()`. The conditional className is a raw ternary between two full class strings with significant duplication (`rounded-full px-3 py-1 text-sm` repeated). This is a recurring pattern (string interpolation instead of `cn()`). Fix: Use `cn()` to compose:
  ```typescript
  className={cn(
    'rounded-full px-3 py-1 text-sm',
    isAdjusted
      ? 'bg-brand-soft text-brand-soft-foreground'
      : 'bg-muted text-muted-foreground'
  )}
  ```

- **[src/app/pages/Quiz.tsx:69] (confidence: 72)**: Empty catch block in `loadSavedAccommodation` silently swallows errors without logging. While `localStorage.getItem` rarely throws, when it does (e.g., SecurityError in private browsing), the learner gets no diagnostic feedback. The adjacent `loadSavedProgress` at line 57 logs via `console.warn` — inconsistent pattern. Fix: Add `console.warn('[Quiz] Failed to load accommodation:', e)` to match existing convention.

- **[src/app/pages/Quiz.tsx:166] (confidence: 70)**: Empty catch block in `handleAccommodationChange` for `localStorage.setItem`. Same pattern — the comment says "best-effort persistence" but doesn't log the failure. If storage is full or blocked, the learner won't know their preference wasn't saved and will be surprised on the next visit. Fix: Add `console.warn` and optionally a `toast.info('Preference may not persist in private browsing.')`.

- **[tests/e2e/story-e15-s02.spec.ts:1042] (confidence: 78)**: Fragile submit confirmation handling. `if (await submitConfirm.isVisible({ timeout: 2000 }).catch(() => false))` uses a timeout-based conditional instead of deterministic state. If the confirmation dialog takes >2 seconds (slow CI, heavy browser), the test proceeds without confirming, causing the quiz to stay in-progress and the `waitForURL(/\/quiz\/results/)` to timeout with an unhelpful error. Fix: Use Playwright's auto-retry pattern — `expect(submitConfirm).toBeVisible()` if the dialog is expected, or check for the specific UI state that indicates whether confirmation is needed.

#### Nits

- **Nit** [src/app/components/quiz/QuizStartScreen.tsx:89] (confidence: 65): Non-null assertion `quiz.timeLimit!` is safe here because `hasTimed` guards the branch, but a cleaner pattern is to capture the narrowed type: `const timeLimit = quiz.timeLimit; if (timeLimit != null) { ... formatTimeBadge(timeLimit, accommodation) }`. This makes the safety self-documenting.

- **Nit** [src/app/components/quiz/TimerAccommodationsModal.tsx:26-31] (confidence: 55): `formatDuration` uses singular/plural logic for "minute(s)" when `seconds === 0` but switches to abbreviated "min" when there are seconds. Consider using consistent formatting (always abbreviated or always spelled out).

- **Nit** [src/app/components/quiz/QuizStartScreen.tsx:102] (confidence: 60): The accommodations button height is set to `h-auto p-0` overriding shadcn defaults. While functional, this may result in a touch target smaller than 44px on mobile. Verify the rendered height meets the 44px minimum specified in the design guidance.

### Recommendations

1. **Fix QuizResults retake accommodation pass-through** (High) — Extract `loadSavedAccommodation` to a shared utility and use it in QuizResults.tsx. This is the only finding that breaks an acceptance criterion.
2. **Extract multiplier logic to shared constant** (High) — Single source of truth prevents future divergence.
3. **Add runtime validation on radio value change** (High) — Defense in depth, matches the Zod validation pattern already used for localStorage reads.
4. **Use `cn()` for conditional badge classNames** (Medium) — Consistency with codebase conventions.
5. **Add logging to empty catch blocks** (Medium) — Two instances in Quiz.tsx.
