#### High Priority

- **[CLAUDE.md:7,46] (confidence: 85)**: CLAUDE.md still references "Instructors" in two places -- the project overview ("six main sections: Overview, My Class, Courses, Instructors, Reports, and Settings") and the file structure listing (`Instructors.tsx`). Why: Stale documentation causes developers and AI agents to reference non-existent files. Fix: Update CLAUDE.md line 7 to say "Authors" and line 46 to say `Authors.tsx`.

- **[tests/e2e/story-e23-s03.spec.ts:12-20] (confidence: 78)**: AC1 test ("no Instructor text appears anywhere in the app") only checks the `/authors` page, not "any page in the application" as the AC states. The sidebar rename could pass while another page (e.g., CourseDetail, AuthorProfile) still contains "Instructor" text. Why: A regression on CourseDetail or AuthorProfile would not be caught by this test. Fix: Extend the test to navigate to at least 2-3 key pages (overview, course detail, author profile) and check each for "Instructor" text, or iterate over an array of routes.

#### Medium

- **[tests/e2e/story-e23-s03.spec.ts:51-59] (confidence: 72)**: AC5 test only verifies no horizontal overflow at 375px width. It does not check that actual content renders (e.g., author cards visible, heading visible). A blank page with no overflow would pass this test. Fix: Add at least one content assertion (e.g., `await expect(page.getByRole('heading', { level: 1 })).toBeVisible()`) to confirm the page actually rendered at mobile width.

- **[~15 files: formatting-only changes] (confidence: 70)**: The Prettier auto-format commit (`44fe152a`) reformatted numerous unrelated test files (quiz-factory.ts, story-e16-s01.spec.ts, story-e15-s06.spec.ts, seed-helpers.ts, etc.), adding noise to the diff. While individually harmless, mixing formatting changes with a rename makes the diff harder to review and increases merge conflict surface. Fix: For future renames, consider running Prettier in a separate PR or at minimum a clearly separated commit that only touches files the rename actually modified.

The `i` was fine when it stood for "instructor" but is now ambiguous. Fix: Rename to `a` or `author` for clarity: `allAuthors.find(author => author.id === id)`.
