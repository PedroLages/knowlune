## Findings

### Blockers

### High Priority

**H1**: `src/app/config/navigation.ts:93-98` — `getPrimaryNav()` filters by `primaryNavPaths.includes(item.path)` without a count constraint (confidence: 82)

The mobile bottom bar depends on exactly 4 items. Nothing validates that count at runtime or in tests. The unit test at `navigation.test.ts:55-58` asserts the expected names but not the length.

Fix: Add `expect(primary).toHaveLength(4)` to the unit test. Add a comment on `primaryNavPaths` documenting the mobile bottom bar's 4-item constraint.

**H2**: `tests/e2e/story-e23-s04.spec.ts:117-122` — AC5 uses `page.evaluate()` to click "More" button (confidence: 78)

This bypasses Playwright's actionability checks (visibility, enabled, pointer-event interception). The comment says "bypassing dev-only toolbar overlays" — this masks a real interactability problem rather than fixing it.

Fix: Use `moreButton.click({ force: true })` or fix the toolbar overlay. At minimum, add a comment with a tracking reference for the underlying issue.

### Medium

**M1**: `tests/e2e/story-e23-s04.spec.ts:84-87` — Group label assertions use `innerText` regex matching on CSS `text-transform: uppercase` output (confidence: 75)

`innerText` reflecting computed `text-transform` is Chromium-specific behavior, not standardized.

Fix: Use `getByText('Learn', { exact: true })` or test the raw text attribute.

**M2**: `tests/e2e/story-e23-s04.spec.ts:140-146` — Separator selector uses Tailwind class `.border-t` (confidence: 72)

Tightly coupled to Tailwind class names — a rename or style change breaks the selector silently.

Fix: Add `data-testid="group-separator"` to separator divs in Layout.tsx and select by test ID.

**M3**: `src/app/config/navigation.ts:93` — `primaryNavPaths` is not exported (confidence: 70)

Makes it impossible for consumers or tests to verify which items are "primary" without inspecting source.

Fix: Export `primaryNavPaths` as a `const`.

**M4**: `tests/e2e/story-e23-s04.spec.ts:63-71` — `getOverflowNav` unit test never validates total count (confidence: 70)

With 14 total items minus 4 primary = 10 expected overflow items. Items could be accidentally dropped and this test would still pass.

Fix: Add `expect(overflow).toHaveLength(10)` to the unit test.

### Nits

**N1**: `tests/e2e/story-e23-s04.spec.ts:16-18` — Viewport constants `DESKTOP`, `MOBILE`, `TABLET` duplicated across specs. Consider extracting to `tests/support/constants.ts`.

**N2**: `src/app/config/navigation.ts:59-60` — Group label `'Review'` collides with nav item name `'Review'`. A comment explaining the intentional overlap would help future readers.
