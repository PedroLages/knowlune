## Issues

### MEDIUM — Consecutive triple-identical-message not re-announced

**File:** `src/hooks/useAriaLiveAnnouncer.ts:30`

The deduplication logic stores `prevRef.current = msg` (raw message) but appends `\u200B` for duplicates. If the same message is announced 3+ times consecutively, the 3rd call produces the same effective value as the 2nd (`msg + '\u200B'`), so React skips the re-render and the screen reader does not re-announce.

**Fix:** Use a counter or toggle to ensure every call produces a unique effective value:
```ts
const effective = msg === prevRef.current ? msg + '\u200B'.repeat(countRef.current++ % 2 + 1) : msg
```

**Impact:** Low in practice — quiz interactions rarely trigger the exact same announcement 3 times in a row. But it's a correctness issue in the reusable hook.
