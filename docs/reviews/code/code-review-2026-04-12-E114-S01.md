## Findings

### HIGH — ReadingRuler blocks tap navigation zones when enabled

**File:** `src/app/components/reader/ReadingRuler.tsx:50`
**Also affects:** `src/app/pages/BookReader.tsx:768`

The ReadingRuler renders at `z-20` with `pointer-events-auto absolute inset-0`, which sits above EpubRenderer's interaction zones at `z-10`. When the ruler is enabled, all tap/click events are captured by the ruler, blocking:
- Left zone (previous page)
- Center zone (toggle header/footer)
- Right zone (next page)

Users cannot navigate pages or toggle UI while the reading ruler is active.

**Fix:** Change the ruler container to `pointer-events-none` and use a dedicated, smaller event-capture zone (e.g., only capture pointermove on the parent `<main>` element via a ref passed from BookReader), or forward click events through the ruler to the underlying zones.

### MEDIUM — ReadingRuler captures pointer events before first move with no visual feedback

**File:** `src/app/components/reader/ReadingRuler.tsx:47-53`

When `yPosition` is null (before the user moves their pointer), the component renders an empty full-viewport div with `pointer-events-auto`. This blocks all interaction with no visual indication that the ruler is active. The user must move their pointer/finger before any ruler UI appears, but clicks are already being swallowed.

**Fix:** Start with `pointer-events-none` until first pointermove, then switch to active.

### LOW — Spacing conditionals only inject when > 0, but reset to 0 doesn't remove prior styles

**File:** `src/app/components/reader/EpubRenderer.tsx:93-98`

When `letterSpacing` or `wordSpacing` are set to a non-zero value and then reset to 0, the `if (letterSpacing > 0)` guard means the property is simply omitted from the next `themes.default()` call. However, `rendition.themes.default()` in epub.js may not remove previously-set CSS properties — it only sets new ones. This could leave stale spacing values in the rendered content.

**Fix:** Explicitly set `letter-spacing: 'normal'` and `word-spacing: 'normal'` when values are 0, rather than omitting them.

### NIT — getSettingsFromState is an identity function for ReaderSettings

**File:** `src/stores/useReaderStore.ts:122-132`

`getSettingsFromState` manually lists every field of `ReaderSettings`. When new settings are added, this function must also be updated (maintenance burden). Consider using a destructure + rest pattern or a constant array of keys.
