# Code Review: E51-S02 — Reduced Motion Toggle with Global MotionConfig

## Summary

Clean implementation of a reduced motion toggle with centralized MotionConfig. The story correctly removes 17 local MotionConfig wrappers, introduces a `useReducedMotion` hook, a `shouldReduceMotion()` utility, and a flash-prevention script. Architecture is sound and follows established patterns (useColorScheme.ts).

## Findings

### HIGH

**1. [useCourseCardPreview.ts:11] — `shouldReduceMotion()` called on every render**
Confidence: 85/100

`const prefersReducedMotion = shouldReduceMotion()` is called at the top level of the hook, meaning every render re-reads localStorage and creates a new matchMedia query. This is not reactive (won't update when the user changes the setting in another tab) and is wasteful on frequent re-renders.

**Recommendation:** Use `useReducedMotion()` hook instead, which is reactive and listens for setting changes. The `shouldReduceMotion()` utility is designed for non-React contexts (event handlers, plain functions).

### MEDIUM

**2. [DisplayAccessibilitySection.tsx:166] — AlertDialogAction uses raw bg-brand classes instead of variant="brand"**
Confidence: 90/100

The AlertDialogAction button uses `className="bg-brand text-brand-foreground hover:bg-brand-hover"` directly. Per styling rules, `<Button>` elements should use `variant="brand"` instead. While AlertDialogAction inherits from Button, the variant approach ensures consistent hover/focus/disabled states.

**3. [reduce-motion-init.js] — No `'system'` + OS check in flash prevention script**
Confidence: 75/100

The init script only checks `s.reduceMotion === 'on'`. When the user has `reduceMotion: 'system'` AND the OS has `prefers-reduced-motion: reduce`, the flash prevention doesn't kick in. The class won't be applied until React hydrates (50-200ms delay). This means system-preference users still get a brief flash of animations.

To fix, add: `else if ((!s || !s.reduceMotion || s.reduceMotion === 'system') && window.matchMedia('(prefers-reduced-motion: reduce)').matches) { document.documentElement.classList.add('reduce-motion'); }`

**4. [EngagementPreferences.tsx] — "animations" toggle semantics may confuse users**
Confidence: 65/100

The animations toggle now controls "celebratory effects like confetti and achievement banners" while the Motion Preference RadioGroup controls CSS/Framer Motion animations. Users may not understand the distinction. The code comment explains the relationship, but the UI doesn't make it clear they're related but distinct controls.

This is a UX concern more than a code concern. Consider adding a helper text linking the two, or a future story to consolidate them.

### LOW / NITS

**5. [App.tsx:94] — MotionConfig `'never'` means "never reduce" (animations always play)**
Confidence: 95/100

The MotionConfig value mapping is correct: `shouldReduceMotion ? 'always' : 'never'` means "always reduce" or "never reduce." This is clear but worth noting the potentially confusing naming convention of the Framer Motion API.

**6. [eslint.config.js] — `public/` added to ESLint ignores**
Confidence: 90/100

Added `public/` to ESLint ignore patterns to avoid linting the plain JS init script. This is correct since the file uses `var` and plain JS syntax intentionally (must run before module system).

## Architecture Assessment

- **Hook pattern**: Follows `useColorScheme.ts` exactly. Reactive to both `settingsUpdated` and `storage` events. Clean.
- **Utility function**: `shouldReduceMotion()` in `settings.ts` is well-placed for non-React contexts. Clear JSDoc.
- **CSS**: Class-based rules mirror the existing `@media (prefers-reduced-motion: reduce)` block exactly. Correct specificity.
- **Flash prevention**: Synchronous script in `<head>` before stylesheets. Correct pattern.
- **MotionConfig centralization**: Root-level in App.tsx, wrapping all children. Correct.

## Verdict

**PASS with 1 HIGH, 3 MEDIUM, 2 LOW findings.** The HIGH finding (useCourseCardPreview using non-reactive utility) should be fixed. The MEDIUM findings are improvements but not blockers.
