## Story-Related Issues

### MEDIUM: Case-insensitive filtering inconsistency for imported courses

**File:** `src/app/pages/Courses.tsx:219`
**Description:** The `filteredImportedCourses` filter uses `c.tags.includes(topic)` (exact match) while `selectedTopics` contains normalized lowercase tags from `mergedTags`. If an imported course has a tag like "Python" (capitalized), the lowercase `selectedTopics` value "python" would not match via `includes()`. The pre-seeded course filter at line 162 correctly uses `.toLowerCase()` comparison.

**Fix:** Change line 219 from:
```typescript
courses = courses.filter(c => selectedTopics.every(topic => c.tags.includes(topic)))
```
to:
```typescript
courses = courses.filter(c => selectedTopics.every(topic => c.tags.some(t => t.toLowerCase() === topic)))
```

### MEDIUM: E2E test `seedAndReload` failed due to localStorage access on about:blank

**File:** `tests/support/helpers/seed-helpers.ts:207`
**Description:** The `seedAndReload` helper called `page.evaluate` to set localStorage before any navigation, causing a SecurityError on `about:blank`. Fixed during review by removing the redundant localStorage call since `goToCourses()` already handles sidebar state via `addInitScript`.
