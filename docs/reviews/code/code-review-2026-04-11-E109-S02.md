## STORY-RELATED Issues

### BLOCKER-1: Dexie query on non-indexed field breaks feature

**File:** `src/app/pages/HighlightReview.tsx:42-47`
**Severity:** BLOCKER

`loadDailyHighlights()` uses `.where('reviewRating').notEqual('dismiss')` but `reviewRating` is not in the Dexie index (version 43 in `src/db/schema.ts` only adds `lastReviewedAt`). Dexie requires fields to be indexed to use `.where()`. This causes the query to return zero results, making the entire feature non-functional.

**Evidence:** All 4 seeded-data E2E tests fail — page shows "No highlights yet" despite valid data.

**Fix options:**
1. Add `reviewRating` to the index: `'id, bookId, color, flashcardId, createdAt, lastReviewedAt, reviewRating'`
2. Or use `.filter()` instead: `db.bookHighlights.filter(h => h.reviewRating !== 'dismiss').limit(...).toArray()`

Option 1 is preferred since it enables efficient index-based filtering.

### MEDIUM-1: Stale closure in handleRate rollback snapshot

**File:** `src/app/pages/HighlightReview.tsx:97-133`
**Severity:** MEDIUM

`handleRate` captures `highlights` and `ratings` via `useCallback` deps. If called rapidly (e.g., user clicks Keep then immediately Dismiss on the same card), `prevStateRef.current` may snapshot stale state because React hasn't re-rendered yet. The rollback would restore incorrect state.

**Fix:** Use `useRef` to always hold the latest `highlights`/`ratings`, or use functional state updaters for the snapshot.
