## Code Review: E23-S01 -- Remove Hardcoded Branding from Courses Page

### What Works Well

1. **Clean two-tier empty state architecture.** The global `EmptyState` component short-circuits the entire page content when no courses exist, while preserving the per-section imported courses empty state for the case where pre-seeded courses exist but no imports do. The hierarchy is well-documented in the story's lessons learned.

2. **Proper reuse of existing `EmptyState` component.** Rather than creating a one-off empty state, the implementation reuses the motion-animated `EmptyState` component already used in Overview.tsx and Reports.tsx, maintaining visual consistency across the platform.

3. **E2E test replaced manual IndexedDB seeding with `clearStore()` fixture helper.** The lessons learned section documents the migration from the anti-pattern, and the test now uses the established fixture pattern.

### Findings

#### Blockers

None.

#### High Priority

- **[FIXED] `src/app/pages/prototypes/HybridCourses.tsx:42` (confidence: 92)**: Hardcoded Tailwind color `text-neutral-500` introduced in changed code. The diff shows the branding text was removed but the replacement still uses `text-neutral-500` instead of the design token `text-muted-foreground`. The same file also has pre-existing violations on lines 47-55 (`bg-white`, `border-neutral-100`, `text-neutral-400`), but line 42 is *new code in this diff*. Why: Hardcoded colors break dark mode for learners and violate AC3 ("design tokens and the existing theme system are used for all remaining styling"). Fix: Replace `text-neutral-500` with `text-muted-foreground` on line 42. Note: The pre-existing violations on lines 47-55 are out of scope for this story but should be tracked.

- **[FIXED] `src/app/pages/Courses.tsx:249` (confidence: 78)**: `w-5 h-5` used on Search icon instead of Tailwind v4 shorthand `size-5`. This is a pre-existing pattern in unchanged code that was re-indented but not fixed. The `size-5` shorthand is the v4 convention used elsewhere in the file (e.g., `size-4` on lines 223, 228). Why: Inconsistent icon sizing conventions make the codebase harder to grep and maintain. Fix: Replace `w-5 h-5` with `size-5`.

#### Medium

- **`tests/e2e/story-23-1.spec.ts:45-61` (confidence: 75)**: The `addInitScript` monkey-patches `IDBObjectStore.prototype.add` to intercept course seeding, creating a fragile fake `IDBRequest` object with manual `setTimeout` dispatch. If the app ever switches to `bulkAdd` (which uses `put` internally in some Dexie versions), or if the seeding logic changes, this test will silently pass while not actually blocking seeding. The fake `IDBRequest` also lacks `addEventListener`/`removeEventListener` methods, which could cause runtime errors if Dexie's internals call those. Fix: Consider a simpler approach -- seed an empty state via the `indexedDB` fixture's `clearStore()` *after* navigation, combined with a flag in localStorage that the app reads to skip seeding, or mock at a higher level (e.g., intercept the network/module rather than low-level IDB primitives). If the current approach is validated by burn-in testing, add a comment noting the fragility.

- **`src/app/pages/Courses.tsx:209` (confidence: 72)**: The expression `allCourses.length + importedCourses.length` is computed twice -- once in the conditional on line 209 and once in the template literal on line 211. While trivially cheap, this is a readability concern in a ~400-line component. Fix: Extract `const totalCourses = allCourses.length + importedCourses.length` before the return and reuse it in both the subtitle conditional and the empty state conditional on line 235.

- **`tests/e2e/story-23-1.spec.ts:68` (confidence: 70)**: The `{ timeout: 10_000 }` on the empty state visibility assertion is 5x the default Playwright timeout. While understandable given the IDB clearing + navigation dance, a 10-second timeout could mask genuine regressions where the empty state never appears (the test would hang for 10 seconds before failing rather than failing fast). Fix: If the test passes reliably at a lower timeout (e.g., 5000ms), reduce it. If 10s is genuinely needed, add a comment explaining why.

#### Nits

- **Nit** `src/app/pages/prototypes/HybridCourses.tsx:41` (confidence: 65): The subtitle only counts `allCourses.length` but does not include `importedCourses`. In `Courses.tsx` the subtitle correctly sums both. While this is a prototype file, the inconsistency could confuse future developers. Fix: Match the `Courses.tsx` pattern if the prototype is still actively referenced.

- **Nit** `tests/e2e/story-23-1.spec.ts:22` (confidence: 60): The AC1 test uses the structural locator `page.locator('main > div > div').first()` to scope to the header area. This is fragile -- any wrapper div change will break it. Fix: Consider using `page.getByRole('heading', { name: 'All Courses' }).locator('..')` or adding a `data-testid="courses-header"` to the header div.

- **Nit** Various test files (confidence: 55): The diff includes ~200 lines of Prettier reformatting across 7 unrelated test files (trailing comma removal, line-length reformats). While harmless, this noise makes the diff harder to review and could cause unnecessary merge conflicts with other in-flight branches. Fix: In future stories, run Prettier in a separate commit or limit reformatting to files touched by the story.

3. **Consider**: Evaluate the `addInitScript` monkey-patch fragility -- if burn-in passes, document; if not, simplify.
4. **Optional**: Fix `w-5 h-5` to `size-5` on the Search icon.
