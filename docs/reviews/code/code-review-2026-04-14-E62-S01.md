## Findings

### MEDIUM — Redundant flashcardRetention/fsrsRetention pass-through (NIT-borderline)

**File:** `src/stores/useKnowledgeMapStore.ts:283-318`

The store sets `flashcardRetention = aggregateRetention` and then also passes `fsrsRetention: aggregateRetention` to `calculateTopicScore()`. Inside `calculateTopicScore()`, `fsrsRetention` takes priority and overrides `flashcardRetention`. This means both parameters always carry the same value in practice, and the `flashcardRetention` field in `TopicScoreInput` is now vestigial in this call site.

**Impact:** No behavioral bug. Minor readability/maintenance concern — a future reader might think `flashcardRetention` and `fsrsRetention` can differ here.

**Recommendation:** Consider passing `flashcardRetention: null` and letting `fsrsRetention` handle it, or document the intentional duplication.

### MEDIUM — AC8 (SM-2 feature detection) partially implemented

**File:** `src/lib/knowledgeScore.ts:168`

AC8 states: "Given a flashcard with SM-2 fields (interval, reviewedAt) and no FSRS fields (no stability), calculateAggregateRetention() processes it via the SM-2 predictRetention path."

The current filter `card.last_review && card.stability > 0` silently drops SM-2-only cards (which have `reviewedAt` instead of `last_review`, and `interval` instead of `stability`). The `RetentionFlashcard` interface declares `interval?` and `reviewedAt?` fields but they are never used in the logic.

**Impact:** SM-2-only flashcards are treated as unreviewed and excluded from retention aggregation. This is acceptable if all cards have been migrated to FSRS fields, but it means AC8 is not fully satisfied as written.

**Recommendation:** Either implement SM-2 card handling (map `reviewedAt` to `last_review`, `interval` to `stability`) or explicitly mark AC8 as deferred/out-of-scope with a TODO comment.

### LOW — `new Date()` default parameter in exported functions

**File:** `src/lib/knowledgeScore.ts:163,201`

`calculateAggregateRetention()` and `calculateDecayDate()` use `now: Date = new Date()` as default. This is fine for production but the ESLint `deterministic-time` rule flags `new Date()` in test files. The functions are always called with explicit `now` in tests, so this is not a problem, but it's worth noting for future callers.

**Impact:** None currently. All call sites pass explicit `now`.

### NIT — Unused `RetentionFlashcard` fields

**File:** `src/lib/knowledgeScore.ts:42-49`

The `RetentionFlashcard` interface defines `interval?` and `reviewedAt?` fields that are never read. These were presumably added for AC8 SM-2 support that was simplified during implementation.

**Recommendation:** Remove unused fields or add a TODO linking to a future SM-2 migration story.
