## Lessons Learned

### 1. Average confidence is the wrong gate for data correctness decisions

When routing between auto-save and manual review, a mean over all mappings masks outlier failures. A book with 95% perfectly matched chapters and 5% catastrophically wrong ones looks like a "high confidence" result. Use minimum confidence or a threshold like "fewer than N% of mappings below floor" for gates that affect data persistence.

### 2. Scroll areas must not contain their own action buttons

If a `ScrollArea` contains interactive controls that must be reachable regardless of scroll position (Save, Cancel, Submit), those controls need to be outside the scroll region. This is a layout constraint, not a preference. The fix is always to restructure — not to assume test data will have short enough content.

### 3. Story "done" gate should verify Testing Notes are implemented, not just that the test runner passes

A story can have aspirational testing notes that describe tests which were never written. The unit-test gate (npm run test:unit exits clean) does not detect this. A lightweight addition to the review checklist — "does the number of test files increase proportionally to Testing Notes?" — would catch this systematically.

### 4. Ghost enum states are future debugging traps

When a state machine has a named state (`'unlink-confirm'`) that no JSX branch explicitly handles, future maintainers will either assume the state is used somewhere they haven't found yet, or incorrectly assume it is safe to remove. Either path wastes time. Clean up dead states before marking a story done.

### 5. Retro action items must be committed as code, not listed as notes

Six action items from E103, zero completed before E104 started. The pattern of documenting action items in a retro file and carrying them forward is not working. The only thing that prevents carry-forward is making the action items concrete — a test file written, a docs file updated, a pattern committed. Aspirational text in a retro doc has a half-life of approximately one epic.
