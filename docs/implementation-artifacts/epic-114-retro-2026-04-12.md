## Lessons Learned

### 1. Full-viewport overlays: pointer-events-none is the default, not an option

A full-viewport overlay at a high z-index with `pointer-events-auto` silently blocks all interaction below — tap zones, navigation buttons, and all clickable elements. The rule is: any UI layer that must coexist with interactive elements below it must use `pointer-events-none` on the overlay container and capture events via document-level listeners. This applies to reading rulers, annotation layers, highlight overlays, and any feature that spans the reading area. The HIGH finding in E114-S01 would have been prevented by applying this rule at design time. (E114-S01, ReadingRuler z-20 blocking)

### 2. epub.js theme API is a merge patch, not a replace

`rendition.themes.default()` accumulates properties: calling it twice with different values merges the results. Removing a property requires sending the explicit CSS reset value (`'normal'`, `'0'`, etc.) — omitting the key has no effect on already-set properties. Any epub.js integration that conditionally applies CSS must explicitly reset properties when they are no longer desired. The same merge behavior applies to font-size, color, background, and any other theme property. (E114-S01, word-spacing and letter-spacing reset)

### 3. epub.js flow switch side effects require explicit follow-up

`rendition.flow()` changes the layout mode but has two side effects: the theme is reset (styles must be re-applied), and the content dimensions are not updated (a `resize()` call is required to re-layout at correct dimensions). The correct sequence for a runtime flow switch is: `flow()` → `resize()` → `applyTheme()`. Any feature that switches epub.js flow mode must include this sequence to avoid visual artifacts. (E114-S02, scrolled-doc to paginated switch)

### 4. Deferred render pattern for position-dependent UI

UI elements that require a position to display correctly (overlay guides, cursor followers, context menus) should initialize their position state as `null` and not render the element until the position is set by the first user interaction. Rendering immediately with an undefined position creates an invisible element in the layout that can block interaction below. The deferred render pattern: `const [y, setY] = useState<number | null>(null); if (y === null) return null;` — ensures the element only appears after the first valid position is known. (E114-S01, ReadingRuler blank overlay on activation)

### 5. Absolute positioning for independent overlay zones

When tap zones or interaction regions are conditionally rendered siblings in a layout (flex row, grid), removing one sibling shifts the others. This makes conditional visibility of one zone affect the position of all remaining zones. The fix is absolute positioning with explicit coordinates for each zone — each zone's position is then independent of its siblings. This applies to all multi-zone tap target layouts where zones can be individually hidden. (E114-S02, center tap zone shifting when prev/next removed)
