## Lessons Learned

### L1: TypeScript type propagation must follow union type expansions

When a shared union type (like `TutorMode`) is expanded, immediately search all consumer types for the old literal union and update them. Use `tsc --noEmit` locally before pushing — don't wait for code review to catch structural type errors.

### L2: Acceptance criteria are hard gates, not checklists

All task checkboxes in a story file represent committed scope. If a task (especially tests) is not done, either complete it or explicitly defer it with a `known-issues.yaml` entry and reviewer sign-off. Silent omission is not acceptable.

### L3: Zod-validated structured LLM output is the standard pattern

The pattern from `courseTagger.ts` — `callOllamaChat` with `format: 'json'` + Zod schema validation before consuming the result — works well and should be the default for all LLM JSON calls. Intermediate type shapes must be explicitly typed to avoid unsafe casts.
