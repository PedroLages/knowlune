## Challenges and Growth Areas

1. **Circular dependency: Zustand store sync ↔ React hook (S01 BLOCKER)** — Writing timer state back to Zustand every 60s caused the component to re-render, recompute `timerInitialSeconds`, and re-initialize the `useQuizTimer` hook — resetting the timer. **Fix:** Capture initial seconds in a `useRef` so store updates don't re-trigger hook initialization. **Root cause:** Hooks that both read from and write to a Zustand store create invisible dependency cycles.

2. **React automatic batching swallows intermediate states (S03 BLOCKER)** — When a short quiz causes all 3 warning thresholds to fire in the same synchronous tick, React batches the state updates and only the last one (`'1min'`) survives. First two warnings silently disappear. **Fix:** Fire toasts imperatively in the callback rather than routing through useState. **Root cause:** React 19's automatic batching (inherited from React 18) drops intermediate state updates in synchronous blocks.

3. **Fix-introduces-anti-pattern cycle (S02)** — v1 code review caught empty catch block in Quiz.tsx. The fix for a different finding (extracting shared logic) introduced the same empty-catch anti-pattern in QuizResults.tsx. **Root cause:** Pattern fixes are applied locally without verifying the fix doesn't re-introduce the same pattern in other files. **Prevention:** When fixing a pattern issue, `grep` for the same anti-pattern in ALL files touched by the fix.

4. **Dirty .gitignore mass file deletions (S04)** — A modified `.gitignore` excluded `docs/` and `.claude/` directories. Running `git add <file> && git commit` tracked the file removals caused by `.gitignore` changes, silently deleting files from the commit. **Prevention:** `git diff --cached --stat` before every commit to verify staged file count matches intent. Restore `.gitignore` with `git checkout HEAD -- .gitignore` before committing.

5. **Persistent toast cleanup (S03)** — Sonner toasts with `duration: Infinity` survive route changes because `<Toaster />` lives in the root layout. Must store the toast ID and call `toast.dismiss(id)` on quiz end. **Root cause:** Any non-auto-dismissing UI element needs an explicit teardown path.

6. **Duplicate ARIA live regions (S03)** — Both QuizTimer (pre-existing minute announcements) and TimerWarnings (new threshold announcements) announced at the 25% threshold, doubling screen reader output at a stressful moment. **Fix:** Consolidate threshold announcements into a single component. **Root cause:** No audit of existing ARIA regions before adding new ones.

7. **Timer test fragility (S01, S03)** — Exact time assertions (`/01:00 remaining/`) fail because 1-3 seconds of real time elapse during test setup (navigation, button clicks). **Fix:** Use flexible regex (`/00:\d+ remaining/`) or `page.clock` API for deterministic control.

8. **Heading hierarchy inversion (S05)** — h3 in PerformanceInsights followed by h2 in AreasForGrowth violates document outline. **Fix:** Normalize heading levels in results page flow.

## Key Insights and Learnings

1. **State management bugs are the automation blind spot** — ESLint catches styling (design tokens), test patterns (deterministic time), imports (correct paths), and async cleanup. But Zustand ↔ hook circular dependencies and React batching edge cases require human code review. These were the only BLOCKERs in the epic. Until we can automate state management validation, code review remains the critical safety net.

2. **"Grep the fix" discipline prevents fix-introduces-fix cycles** — When fixing a pattern issue, always `grep` for the same anti-pattern in ALL files touched by the fix, not just the flagged file. S02 proved that fixing one empty catch while introducing another creates false confidence.

3. **Timer tests need time tolerance by default** — Never assert exact times in E2E tests. Real time always elapses during test setup. Use flexible regex or `page.clock` API for deterministic control.

4. **`useRef` breaks Zustand ↔ hook circular dependencies** — When a hook receives values from a store that the hook also writes to, capture the initial value in a `useRef` on first render. Store updates then can't re-trigger hook initialization.

5. **Imperative calls over useState for multi-fire scenarios** — When multiple state transitions can occur in a single synchronous block (React batching), use refs or imperative calls (like `toast()`) instead of routing through `useState`.

6. **Audit existing ARIA regions before adding new ones** — grep for `aria-live` on the page before introducing new live regions. Duplicate announcements compound cognitive load for screen reader users at their most stressed moments.

7. **`git diff --cached --stat` should be a pre-commit habit** — The .gitignore disaster would have been caught by verifying staged file count. This is the git equivalent of "measure twice, cut once."

8. **Persistent UI elements need explicit teardown paths** — Any element with `duration: Infinity` or that lives outside the component tree (like Sonner toasts in root layout) will survive route changes. Store IDs and call cleanup on unmount.
