## What Went Well

### 1. OpdsService as Living Template

The `AudiobookshelfService` was implemented by reading `OpdsService.ts` as prior art. The abort/timeout/error-message pattern (`AbortController`, `DOMException` vs `TypeError`, discriminated union return) transferred without modification. Having working prior-art in the codebase means architectural decisions are settled before the first line of code.

### 2. Discriminated Union Error Propagation

`AbsResult<T> = { ok: true; data: T } | { ok: false; error: string }` propagated through all 6 stories. TypeScript forced callers to handle error branches at compile time. Zero silent failures in shipped code.

### 3. `addInitScript` Fetch Mock Pattern (E101-S02 Discovery)

Playwright's `page.route()` cannot intercept requests from `addInitScript`-injected code when the app makes cross-origin calls to user-supplied ABS server URLs. Overriding `window.fetch` via `addInitScript` solved this cleanly and became the standard pattern for all subsequent ABS stories. This is a project-wide pattern applicable to every integration story (E102, E103, E84-EPUB, E88-OPDS).

### 4. Batch IDB Writes (E101-S03)

Writing 50 books as a single `bulkPut` instead of 50 individual `put` calls brought catalog sync under the 1-second NFR1 threshold. This is the correct pattern for any sync hook that operates on sets of records.

### 5. Zero-Infrastructure Learning Loop (E101-S06)

Session tracking, streak counting, and Reports integration all worked without changes because ABS books emit the same `book_listened` event type as local books with an empty `courseId` sentinel. The entire existing infrastructure (XP, streaks, Reports, achievement analytics) treated ABS books identically. This is the architecture dividend from the event-driven session model.

### 6. Typed Stubs for Future Epic (E101-S01)

Shipping `fetchProgress()` and `updateProgress()` as fully-typed stubs in S01 means E102-S01 (Bidirectional Progress Sync) can start on LWW sync logic without any service layer work. The right pattern for sequencing integration epics: lay the typed interface first, fill the implementation in the dependent epic.

### 7. Three-Mode State Machine Pattern (E101-S02)

The `'list' | 'add' | 'edit'` state machine from `OpdsCatalogSettings.tsx` transferred cleanly. It handled the ABS-specific differences (post-test library checkboxes, three-state status badges) with minimal additions. Pattern is proven and reusable for any future settings dialog with CRUD.

## What Didn't Go Well

### 1. Incomplete Rename/Fix Pattern (E101-S01)

Three separate commits were needed to properly centralize trailing-slash normalization. Each commit fixed the shared helper but missed that the pure URL builders (`getStreamUrl`, `getCoverUrl`) bypass `absApiFetch` and needed their own normalization. The root cause: no checklist item confirming "all paths that construct URLs are covered." Future fix-round stories should verify the complete call graph, not just the primary path.

### 2. Detached Audio Element E2E Failures (E101-S04)

When Playwright navigates away mid-playback, the HTMLAudioElement gets detached from the DOM but continues emitting `timeupdate` events into unmounted React state, causing `act()` warnings and flaky tests. The fix — explicit `removeEventListener` in `useEffect` cleanup — is correct and standard, but was a knowledge gap. The pattern was not documented and had to be rediscovered.

### 3. Challenges & Lessons Learned Left Blank (E101-S04, S05)

S04 and S05 shipped with placeholder text in the Challenges and Lessons Learned section. Hard-won patterns (detached Audio cleanup, bookmark-to-flashcard IDB write ordering) were not captured at story time. Retrospectives become harder and future stories start from scratch on problems already solved.

### 4. Recurring Stale Closures in Review (E101-S02, S03)

Stale closures on Zustand state in async callbacks were flagged in both S02 (`selectedLibraryIds.length` in `handleTestConnection`) and S03 (stale `useMemo` on `filters.source` in Library.tsx). Same category of issue in consecutive stories indicates a gap: stale closures are not caught by ESLint or type-checking and must be caught by code review or author checklist.

### 5. IDB Method Selection Error (E101-S02)

`addServer` used Dexie `put()` (upsert — silently overwrites on duplicate ID) when `add()` (insert — throws on duplicate) was the correct intent. This was flagged as MEDIUM in code review. The distinction matters: `put()` masks duplicate UUID generation bugs; `add()` surfaces them. Pattern: prefer `add()` for new records, `update()` for modifications, `put()` only when upsert semantics are explicit.

### 6. Architectural Decision Not Documented at Code Site (E101-S01)

GLM's cross-model review flagged `clearTimeout` before `response.json()` as a blocker, requiring a round-trip explanation that this is a known pattern (AbortController only aborts `fetch()`, not `.json()` parsing). The decision exists in engineering-patterns.md but not at the code site. An inline comment would have pre-empted this false positive and saved a review round.
