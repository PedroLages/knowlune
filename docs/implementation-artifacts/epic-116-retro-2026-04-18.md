## What Went Well

1. **CE orchestrator epic-loop ran end-to-end in a single day.** Three stories, three PRs, all merged. No thrash.
2. **Primitive-first decomposition paid off.** S01 and S02 shipped review-clean. The only substantive review finding across the epic landed on S03 — and it was an integration-level concern (mock data), not a primitive-level one.
3. **Barrel export landed early.** S02 established `src/app/components/library/index.ts`, and S03 imported exclusively through it. Zero deep-import coupling in the integration story.
4. **Existing internal helper extraction was disciplined.** A `SectionHeading` inside `SmartGroupedView` became the template for `LibraryShelfRow`'s heading rather than being copy-pasted. Documented in `docs/solutions/best-practices/extract-shared-primitive-on-second-consumer-2026-04-18.md`.
5. **Type-level safety on dynamic heading tag.** `headingLevel: 'h2' | 'h3' | 'h4'` with `const Tag = headingLevel` kept the API both flexible and safe — no runtime string checks needed.

## What Didn't Go Well

1. **Mock data almost shipped to production (S03 R1, MEDIUM).** `<LibraryShelves />` was unconditionally mounted with placeholder content on the Library route. The review agent caught it. Fix was a deferred-mount pattern; component preserved, mount removed until real data wires in.
2. **Aria-label contract for `ShelfSeeAllLink` is incomplete (S02 NIT).** When `label` differs from the default "See all", consumers must pass `aria-label` — but the types do not enforce this. Deferred to a future epic.
3. **No explicit acceptance criterion for "do not ship with mock data."** This is a systemic gap: UI primitive stories rarely encode the "real data must arrive before mount" constraint. Review caught it, but it's the kind of thing an AC should prevent upstream.

## Key Insights

1. **Primitive-first is cheap when CE orchestrator is doing the heavy lifting.** The "three PRs instead of one" cost is negligible when PR review is automated and parallel-dispatched.
2. **Review-time catches are still the dominant safety net against mock-data leaks.** ESLint can't flag "this placeholder content is headed to production." Human/agent review did.
3. **Barrel exports are lightweight and pay back immediately.** One file, three lines, and every downstream import becomes future-proof.
