## What Went Well

- **OPFS foundation was solid:** S01's thorough upfront documentation (directory structure, async API constraints, Dexie migration gate) carried all 7 dependent stories without rediscovery friction.
- **Cascade deletion pattern established:** S06 introduced and documented the first multi-table cascade + OPFS file cleanup pattern in Knowlune — reusable for audiobooks, notes, and future content types.
- **Learning curve effect worked:** S05 and S08 each had only 3 issues vs. S03's 10 — pattern absorption across the epic was real and measurable.
- **No new dependencies for metadata:** epub.js metadata extraction used `epubjs` (already a dependency via `react-reader`), not a new package.
- **100% delivery, no production incidents, no blockers.**

## What Could Be Improved

- **Visual stories need design-token pre-flight:** S03 (grid/list views) had 10 issues partly because it was the first high-surface-area visual story in a new domain. A design-token pre-flight checklist before first review would reduce round-trip cost.
- **ESLint blind spot in dynamic Tailwind classes:** `text-white` inside a conditional badge variant evaded the `no-hardcoded-colors` rule. The plugin has a known limitation with dynamic/conditional class construction.
- **`relativeTime()` non-determinism evaded test ESLint scope:** The `test-patterns/deterministic-time` rule fires on test files, not utility functions. A utility with `Date.now()` called from a test bypasses detection.
- **OPFS browser fallback (NFR23) may be underimplemented:** The fallback path was specified in S01 but not audited in S02–S07. Risk to E84 launch.
- **Infrastructure-first sequencing:** S08 (PWA offline shell) delivered infrastructure before users have a reader. Consider feature-first sequencing for future book-track epics.

## Lessons Learned

1. **Foundation documentation is a force multiplier.** Thorough upfront notes in S01 (OPFS API constraints, directory structure, Dexie migration gate) meant 7 subsequent stories had zero rediscovery cost. Time invested in S01 documentation paid dividends across the entire epic.

2. **Visual stories carry disproportionate review load early in a new domain.** The first high-surface-area visual story in a new codebase area (S03) tends to accumulate more review findings than later stories — not because of lower quality, but because patterns haven't been established yet. Expect this and build in pre-flight discipline for the first visual story.

3. **ESLint static analysis has blind spots that require complementary patterns.** Dynamic/conditional Tailwind class construction and date utilities in non-test files are two confirmed gaps. These require documentation and manual discipline — not just tool reliance.
