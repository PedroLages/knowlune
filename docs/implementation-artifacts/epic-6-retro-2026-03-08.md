## Challenges and Growth Areas

1. **S01 required 3 review rounds.** While review rounds produce quality, 3 rounds indicate that certain categories of issues are predictable and could be caught earlier — error swallowing, timezone bugs, and test gaps are recurring themes.

2. **E2E test AC coverage remains shallow (S02: 2/6 ACs fully covered).** `refreshAllProgress` had zero unit tests. The 100% progress cap was untested at every layer. A streak unit test was a no-op. This is a carry-over problem from Epic 5.

3. **Code review found architectural issues in S02.** String date comparisons instead of `new Date()` parsing, missing useEffect cleanup, and optimistic UI updates before DB writes. These are patterns that should be caught during implementation, not review.

4. **Accessibility violations persist.** S02: heading level skips (H2), missing focus-visible rings on CollapsibleTrigger, generic progress bar aria-labels, loading state missing role="status". Design review catches these, but they should be prevented by convention.

5. **S02 streak progress not scoped to challenge creation date.** Contradicted AC4. Core business logic error caught by code review rather than tests — a gap in the test-first approach.

6. **S02 full table scan for completion progress.** No `updatedAt` index on the relevant table. Performance concern flagged by review that should have been considered during schema design.

7. **S02 E2E afterEach IDB cleanup swallows errors.** Same issue as S01 — the lesson from S01 wasn't applied to S02's test setup. Pattern: lessons documented in one story's notes don't propagate to the next story.

8. **Overview page redesign ripple (S01).** Changing the Overview page layout broke smoke test selectors across multiple specs. Shared pages need more careful change management.

## Key Insights and Learnings

1. **Lessons in story files don't propagate.** S01 documented "E2E IDB cleanup needs await" but S02 repeated the fire-and-forget pattern. Story-level documentation is valuable for reference but doesn't prevent repetition. Need a shared patterns document or pre-story checklist.

2. **Three recurring review-catchable patterns need pre-implementation checklist:**
   - Error swallowing (caught in S01 R1, S02)
   - Missing useEffect cleanup (caught in S02)
   - Optimistic UI before DB write (caught in S02)
   These are checklistable — add to a "before submitting for review" checklist.

3. **Type guard edge cases with form state.** When form state allows empty `type`, validation messages referencing `UNIT_LABELS[type]` produce "undefined" strings. Always guard with a type check before lookup. (From S01)

4. **`addInitScript` runs on every page load including reloads.** Using `localStorage.clear()` inside it wipes seeded test data on `page.reload()`. Use sessionStorage flag to run setup only once per test. (Carry-forward from E05, validated in E06)

5. **Design review catches accessibility regressions that unit tests miss.** Stale `aria-invalid` attributes, heading level skips, missing focus-visible rings — all caught by design review, invisible to unit/E2E tests.

6. **Optimistic rollback needs full state snapshot.** Snapshot the full state array before mutation. Reconstructing state after failure (e.g., appending a deleted item) loses ordering. (From S01)

7. **Schema design matters for query performance.** S02's completion progress required a full table scan because there was no `updatedAt` index. Think about query patterns during schema design, not as an afterthought.
