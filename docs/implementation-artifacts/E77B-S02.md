# Story E77B-S02: Drive Course Import — Metadata and Schema

## Story

As a user,
I want to import a course from Google Drive without downloading any files,
so that I can stream Drive-hosted content directly in Knowlune.

## Acceptance Criteria

1. **Given** a Google Drive folder is selected **When** the user confirms the import **Then** an `ImportedCourse` record is created with `sourceDriveId` set to the Drive folder ID, and each file becomes an `ImportedVideo` record with a `driveFileRef` pointing to its Drive file ID.
2. **Given** a Drive-imported course **When** its video lessons are rendered **Then** the player uses `driveFileRef.fileId` to stream content on-demand from Google Drive via the access token.
3. **Given** a Drive-imported course **When** the course metadata is displayed **Then** `sourceDriveId` identifies it as a Drive-native course (not local, not YouTube).

## Dev Notes

- Adds `DriveFileRef` type and `sourceDriveId` to `ImportedCourse`, `driveFileRef` to `ImportedVideo`
- Adds v66 Dexie migration adding these fields
- `importCourseFromDrive()` in `courseImport.ts` creates courses with no file download
- No .tsx/UI changes — pure data layer and migration

## Challenges and Lessons Learned

- **Minimal schema addition**: Drive-imported courses reuse `ImportedCourse`/`ImportedVideo` — no new table needed. The `sourceDriveId` and `driveFileRef` fields are optional and only populated for Drive-sourced courses.
- **No file download at import**: Drive files are streamed on-demand using the Drive API's media endpoint, avoiding local storage usage at import time. The `path` field is intentionally empty for Drive videos.
- **Format detection deferred**: Video format defaults to `'mp4'` at import time because the file isn't downloaded. Format detection happens on first play when the media element reads the actual stream.
