# Story E77B-S04: Drive Source Management UI and Sync Validation

## Story

As a user with Drive-imported courses,
I want to see which courses are Drive-sourced, reconnect folders when files change, and manage Drive configuration in Settings,
so that I have full visibility and control over my Drive integration.

## Acceptance Criteria

1. **Given** a Drive-imported course on the course card **When** displayed **Then** it shows a "Drive" badge with HardDrive icon next to file metadata.
2. **Given** a Drive-imported course detail page **When** opened **Then** it shows a Google Drive source banner with a "Reconnect" button.
3. **Given** the Reconnect button is clicked **When** a new Drive folder is selected **Then** `handleReconnectFolder` maps new file IDs to existing lessons by filename and updates `sourceDriveId` on the course.
4. **Given** a Drive folder reconnect **When** files are matched **Then** a toast shows the match count (e.g., "X of Y files matched").
5. **Given** the Settings page Integrations & Data section **When** opened **Then** it shows a Google Drive configuration card with connected account email, read scope status, and disconnect action.
6. **Given** the Drive configuration card **When** the read scope is not granted **Then** a "Grant Access" button is available to trigger `requestDriveReadScope()`.
7. **Given** the Drive configuration card **When** "Disconnect Google Drive" is clicked **Then** a confirmation dialog appears and on confirm signs out via `signOut()`.
8. **Given** the course import store's `updateCourseDetails` **When** called with `sourceDriveId` **Then** it updates the field on the database record.
