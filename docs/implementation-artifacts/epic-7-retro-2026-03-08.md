4. **Documentation quality correlates with developer investment** — S02 and S04 have excellent lessons learned because devs invested time. S03 and S05 have placeholder text. The difference: **enforcement**. Without a gate requiring completion, documentation is optional.

7. **Design tokens prevent review churn** — S02 learned this lesson, but S01, S04, S05 repeated hardcoded color violations. The insight is correct; the **enforcement** is missing.

8. **Sophisticated algorithms delivered successfully** — Hamilton allocation (S05), composite scoring (S02), tag overlap with tiebreaking (S03) all worked. Team can handle complex algorithmic work when given clear requirements.

9. **Infrastructure reuse accelerates delivery** — 5 stories in 1 day was possible because Epic 7 leveraged momentum calculations, study log patterns, and IndexedDB schemas from prior epics.

10. **Zero production incidents across 5 epics** — Review process is **effective** at preventing shipped bugs, even if it requires 2-3 rounds per story.
