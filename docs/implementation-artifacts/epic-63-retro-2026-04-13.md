## Lessons Learned

1. **Test-as-own-story structure works — with one caveat.** Separating tests into a dedicated story (S04) produced excellent coverage quality (97.82%, 48 tests, clean review). The caveat: integration points between the code story and the test story need explicit AC coverage. When S03 wired a new parameter, no AC in S04 said "test this wire." Add an explicit integration-path AC to test stories whenever the code story touches inter-component contracts.

2. **Pure-function constraint is a quality multiplier.** No UI, no Dexie, no state — E63 had zero UI review rounds, zero migration risk, and dramatically smaller review surface. When new capabilities can be delivered as pure lib functions, prefer that decomposition. It compounds across review efficiency, testability, and future replaceability.

3. **Forward-declared exports accumulate debt fast.** `SIGNAL_PRIORITY` and `ProfileSignal` were both exported "for future use" before their consumers existed. This is a habit to curb: export when consumed, not when anticipated. Dead exports signal-drift and mislead future readers.
