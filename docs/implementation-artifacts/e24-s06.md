# Story 24.6: Video Drag-and-Drop Reorder

## Story

As a learner,
I want to reorder videos within an imported course via drag-and-drop,
so that I can arrange the viewing sequence to match my preferred study order.

## Acceptance Criteria

- AC1: A "Video Order" tab appears in the Edit Course dialog alongside the existing "Details" tab.
- AC2: The Video Order tab displays all course videos in their current order with drag handles.
- AC3: Users can reorder videos by dragging rows up/down. Keyboard reordering is supported via arrow keys.
- AC4: Order changes persist immediately to IndexedDB on drop with optimistic UI and rollback on failure.
- AC5: Empty state shown when course has no videos.
- AC6: No E2E spec required (drag-and-drop interaction tested via unit tests).

## Implementation Notes

- Uses `@dnd-kit/sortable` (already in project from E9B-S03) with keyboard sensor support
- `persistWithRetry` wrapper provides resilient IndexedDB persistence
- Optimistic UI with rollback on failure follows established patterns
- Tab UI in EditCourseDialog cleanly separates details editing from video reordering
- `SortableVideoRow` renders with `role="listitem"` for screen reader compatibility

## Testing Notes

- Unit tests cover: empty state rendering, video list rendering with order/duration, drag handle ARIA labels, drag-and-drop interaction via DndContext mock, persist failure rollback with toast.error
- E2E spec waived per AC6 (drag-and-drop tested in unit tests)
- Test factory uses deterministic counter-based IDs (not Math.random)

## Challenges and Lessons Learned

- @dnd-kit drag interaction testing in jsdom requires mocking DndContext's onDragEnd callback directly rather than simulating pointer events, since jsdom lacks real layout/positioning
- Optimistic UI rollback testing needs careful mock setup: the persist function must reject to trigger the rollback path, and assertions must verify both the toast.error call and the state reverting to original order
- The `useEffect` for loading videos from IndexedDB needs a cancelled flag pattern to prevent stale promise resolution when the dialog closes and reopens with a different course
