## Challenges and Growth Areas

1. **HNSW rabbit hole cost 3 hours of zero-value work** — Should have started with brute force (the simplest viable solution) and only escalated to HNSW if brute force failed performance targets. Research scores differed by <12% (HNSW 8.90 vs brute force 7.85) but complexity risk was vastly different.

2. **E2E test mocking for LLM streaming was initially painful** — Time burned trying network-level mocking (intercepting fetch calls) before discovering that LLM client-layer mocking was simpler and more reliable for streaming responses. Pattern should have been documented earlier.

3. **IndexedDB state persistence timing issues** — In-memory caches needed explicit refresh after seeding test data (E09B-S04). The `about:blank` localStorage SecurityError (E09B-S06) required navigating to `/` before accessing browser storage APIs in tests.

4. **Zod v4 type inference fighting** — E09B-S03 had complex generic type inference issues requiring pragmatic type guards instead of precise types. Works but is a code smell.

5. **CSP configuration for external APIs** — CSP violations fail silently in browser but clearly in E2E tests, creating debugging confusion. CSP should be configured as part of infrastructure stories before external API integration.

6. **Story file metadata inconsistency** — E09-S02 story file still shows IN-PROGRESS status despite being marked done in sprint-status.yaml. Indicates story files aren't always updated after completion.

7. **No formal criteria for when to split epics** — The 9/9B split happened organically, which worked, but there's no documented criteria for when infrastructure and features deserve separate epics. Important for quiz epics (12-18).

## Key Insights and Learnings

1. **Automation-first process changes work; manual compliance doesn't** — Epic 7 retro's automation mandate achieved 100% follow-through (up from 0%). The ESLint design-token rule eliminated the #1 recurring review issue. This validates the principle permanently.

2. **The Boring Technology Principle is real and valuable** — Custom HNSW (700+ lines, 6.2% recall, 3 hours wasted) vs. brute force k-NN (200 lines, 100% recall, 10x performance margin). When research scores differ by <5%, always choose the lower-risk option. Start simple, prove it's insufficient before escalating.

3. **LLM client-layer mocking > network-layer mocking for streaming** — Mock at the AI client level, not the fetch/network level. Gives deterministic streaming behavior, avoids timing issues, easier to maintain.

4. **Split infrastructure from features in AI epics** — The 9/9B split created a stable foundation. Infrastructure stories (config, workers, embeddings) should complete before feature stories begin.

5. **Graceful degradation is non-negotiable for AI features** — Every AI feature needs a <2 second timeout with rule-based or tag-based fallback. AI enhances but never gates functionality.

6. **CSP must be configured before external API integration begins** — Silent browser failures + clear E2E failures = debugging confusion. Add CSP allowlists as part of infrastructure stories.

7. **`about:blank` is hostile to browser APIs** — Always navigate to a real URL before accessing IndexedDB or localStorage in tests.

8. **Fire-and-forget patterns need explicit error boundaries** — Auto-analysis features must never throw. Wrap in try/catch with toast reporting. Errors in analytics should never break user workflows.

9. **Sunk cost awareness saves significant time** — The HNSW pivot (3 hours invested, 12-20 more estimated) was a textbook cut-losses decision. Document decision frameworks for future use.

10. **Pattern compounding accelerates velocity** — Cross-tab sync, streaming patterns, mock injection, error boundary patterns — each reusable pattern reduces future story effort.
