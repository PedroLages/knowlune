# Story 84.4: Reading Position Persistence and Progress

Status: done

## Story

As a learner,
I want my reading position saved automatically and my progress shown as a percentage,
so that I always resume where I left off and can see how far I have progressed through a book.

## Acceptance Criteria

1. **Given** the user is reading an EPUB **When** the page location changes (via navigation, page turn, or chapter jump) **Then** the current CFI and progress percentage are saved to the Book record in Dexie via `useReaderStore.savePosition()` **And** the save is debounced (500ms) to avoid excessive writes

2. **Given** the user reopens a previously-read book **When** the BookReader page loads **Then** the reader resumes at the exact CFI position stored in `Book.currentPosition` **And** the progress percentage displays in the footer progress bar

3. **Given** the user is reading an EPUB **When** they look at the reader footer **Then** they see a progress bar with brand-color fill, percentage text (e.g., "34%"), and page indicator (e.g., "Page 127 of 372") **And** the progress bar and percentage update in real-time as pages turn

4. **Given** the user returns to the Library page **When** they view their book cards **Then** each card shows the updated reading progress percentage and progress bar

## Dev Notes

### Architecture Constraints

- **ContentPosition type** from architecture: `{ type: 'cfi', cfi: string, percent: number }`. This is the polymorphic position type shared across epub (CFI), audiobook (time), and video (time). Always use the `type: 'cfi'` discriminator.
- **Debounce is critical** — epub.js fires `locationChanged` frequently (every page turn, scroll adjustment). Without 500ms debounce, Dexie writes would be excessive.
- **Dexie write pattern**: Follow existing optimistic update pattern — update Zustand state first, then persist to Dexie. But for position persistence, the Dexie write is the primary concern (Zustand is ephemeral per session).
- **epub.js location object**: The `locationChanged` callback receives a CFI string. To get the percentage, use `book.locations.percentageFromCfi(cfi)`. Note: `book.locations.generate()` must be called once after book load to enable percentage calculation — this can take 1-2 seconds for large books. Call it in `getRendition` and show approximate progress until complete.
- **Page count estimation**: epub.js can estimate total pages via `book.locations.total` after `locations.generate()`. This is an estimate based on viewport size, not a fixed number.

### Key Dependencies (from E84-S01, S02)

- EpubRenderer with `locationChanged` callback and rendition ref
- ReaderFooter component with progress bar
- useReaderStore with `currentCfi`, `readingProgress` state
- useBookStore with `books` array and Dexie persistence
- Book type with `currentPosition: ContentPosition` and `readingProgress: number`

### Performance Considerations

- `book.locations.generate()` runs async and can take 1-2s for large books. Do NOT block rendering on it — show approximate progress from CFI until locations are ready.
- Debounce position save at 500ms — do not save on every render frame
- Dexie `books.update(bookId, { currentPosition, readingProgress, lastOpenedAt })` — single update call, not three separate calls

### Error Handling

- If Dexie write fails: log error, do NOT show toast (would be disruptive during reading). Position will be re-saved on next page turn.
- If CFI restoration fails (corrupted position): log warning, fall back to beginning of book, clear stored position.
