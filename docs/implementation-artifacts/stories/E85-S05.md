# Story 85.5: Flashcard-to-Book Back-Navigation

Status: done

## Story

As a learner,
I want to navigate from a flashcard in my review queue back to its source highlight in the book,
so that I can re-read the surrounding context when I need to refresh my understanding.

## Acceptance Criteria

1. **Given** a flashcard with `sourceType: 'book'` appears during FSRS review **When** the user views the flashcard **Then** a "View in Book" link/button appears showing the book title **And** tapping it navigates to `/library/{sourceBookId}` and the reader opens at the highlight's CFI position **And** the highlight is visually emphasized (brief pulse animation) to draw attention

2. **Given** a flashcard with `sourceType: 'book'` exists but the source book has been deleted **When** the user views the flashcard **Then** the "View in Book" link shows "Book removed" in muted text and is not clickable **And** the flashcard itself continues to function normally in the FSRS queue

## Dev Notes

### Architecture Constraints

- **Existing flashcard review UI**: Find the flashcard review component in the codebase and extend it. Do NOT create a separate review UI for book flashcards. The "View in Book" link is an addition to the existing review card layout.
- **Navigation via query param**: Pass highlight CFI as a URL query parameter to the BookReader page. This avoids tight coupling between the flashcard review and reader state stores. The BookReader reads the param, opens the book, and navigates to the CFI.
- **URL encoding**: CFI strings contain special characters (parentheses, slashes). Use `encodeURIComponent()` for the query param.
- **Flashcard type extension is backward compatible**: `sourceType`, `sourceBookId`, `sourceHighlightId` are all optional. Existing course flashcards have them as undefined ŌĆö no migration needed.

### Key Dependencies

- Existing flashcard review component (from prior epics ŌĆö find in codebase)
