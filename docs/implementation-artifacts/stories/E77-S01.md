# Story 77.1: Google OAuth Connection & Premium Gate

Status: ready-for-dev

## Story

As a premium user,
I want to connect my Google account to Knowlune via OAuth,
so that I can authorize Knowlune to store backups in my personal Google Drive.

## Acceptance Criteria

1. **Given** a premium user is on the Settings page **When** the Google Drive Backup section renders **Then** the section displays a "Connect Google Drive" button with the Google Drive logo SVG and a privacy note: "Knowlune only requests access to files it creates. We can't see your other Drive files."

2. **Given** a premium user clicks "Connect Google Drive" **When** GIS `requestAccessToken()` triggers **Then** the consent screen requests only the `drive.file` scope, and the button shows a loading spinner with "Connecting..." text while OAuth is in progress.

3. **Given** the user approves the OAuth consent screen **When** the GIS callback returns an access token **Then** the token is stored in `useGoogleDriveStore` (memory only, never localStorage/cookies/IndexedDB), the UI transitions to CONNECTED state showing "Connected as {email}" with a green `bg-success-soft` status bar, and a success toast confirms "Connected as {email}".

4. **Given** the user cancels the OAuth consent screen **When** the GIS callback returns with no token **Then** the UI remains in DISCONNECTED state with no error toast.

5. **Given** a connected user clicks "Disconnect" **When** the confirmation AlertDialog appears **Then** clicking "Disconnect" clears the token from memory, invalidates cached folder IDs, disables any active backup schedule, and the UI returns to DISCONNECTED state with toast "Google Drive disconnected".

6. **Given** the access token has expired (~1 hour) **When** any Drive API call returns a 401 **Then** the connection status bar changes to warning state (`bg-warning/10`) with "Session expired. Reconnect to continue backups." and a "Reconnect" button triggers `requestAccessToken()`.

7. **Given** a free-tier user visits the Settings page **When** the Google Drive Backup section renders **Then** a `PremiumGateOverlay` is displayed with Lock icon, feature description (4 bullet points), and "Upgrade to Premium" button with `variant="brand"`, with `aria-label="Google Drive Backup is a premium feature"`.

8. **Given** `backupToDrive()` or `restoreFromDrive()` service functions are called **When** the caller is not premium **Then** the function throws `PremiumRequiredError` before executing any Drive API calls (defense-in-depth, Adversarial Blocker #2).

9. **Given** a premium user downgrades to free tier **When** entitlement state changes **Then** the backup UI is replaced with the premium gate overlay, existing backups in Drive are preserved, and any active scheduled backup is disabled.

## Dev Notes

### Architecture Constraints

- **No npm dependencies** for Drive integration -- GIS loaded via CDN `<script>` tag
- **Token storage**: Access token in Zustand store ONLY (in-memory). Never persist to localStorage, cookies, or IndexedDB. Token lost on page refresh by design (re-auth is quick since consent is remembered).
- **Scope**: `https://www.googleapis.com/auth/drive.file` -- non-sensitive, only accesses files created by the app
- **GIS library**: `google.accounts.oauth2.initTokenClient()` + `requestAccessToken()` -- implicit token model, no backend needed
- **Environment variable**: `VITE_GOOGLE_CLIENT_ID` -- stored as env var, never hardcoded

### Key Files to Create

| File | Purpose |
|------|---------|
| `src/types/google-drive.d.ts` | GIS type declarations (no `@types` package exists) |
| `src/stores/useGoogleDriveStore.ts` | Zustand store for Drive connection state |
| `src/services/errors/driveErrors.ts` | Drive-specific error class hierarchy |
| `src/app/components/settings/GoogleDriveBackupPanel.tsx` | Main backup panel with state machine |

### Key Files to Modify

| File | Change |

### Existing Patterns to Follow

- **Zustand stores**: See `src/stores/useAuthStore.ts`, `src/stores/useNotificationPrefsStore.ts` for patterns -- use individual selectors, never destructure
- **Settings page cards**: Follow existing Card + CardHeader + icon pattern in Settings.tsx
- **Design tokens**: Use `bg-brand-soft`, `text-success`, `bg-success-soft`, `bg-warning/10`, `text-destructive` -- never hardcode colors (ESLint `design-tokens/no-hardcoded-colors` enforced)
- **Button variants**: Use `variant="brand"` for primary CTAs, `variant="brand-outline"` for secondary, `variant="ghost"` for tertiary
- **AlertDialog**: Already imported in Settings.tsx -- reuse same pattern
- **Toast**: Use `toast.success()` / `toast.error()` from sonner (per ESLint `error-handling/no-silent-catch`)
- **Touch targets**: All interactive elements `min-h-[44px]` per WCAG

### Premium Entitlement

- E35 provides the entitlement system. Use `useEntitlementStore(state => state.isPremium)` to check.
- If E35 is not yet implemented, stub with a simple boolean flag that defaults to `true` for development.
- `assertPremium()` is defense-in-depth: even if UI hides features, service functions must independently verify.

### CSP Configuration

- GIS script loads from `https://accounts.google.com` -- add to `script-src` in CSP if configured
- Token info endpoint: `https://www.googleapis.com/oauth2/v3/userinfo` -- add to `connect-src`

### Accessibility Requirements

- Connection status: `role="status" aria-live="polite"` for state changes
- Token expired alert: `role="alert" aria-live="assertive"`
- Premium gate: `aria-label="Google Drive Backup is a premium feature"`
- All buttons keyboard navigable, Enter/Space activates
- Tab order: Connection status -> toggle -> frequency pills -> Back Up Now -> history items

### Responsive Design

- < 640px: Full-width layout, buttons become full-width, connection status stacks vertically
- 640-1024px: Standard layout
- > 1024px: Full layout as designed

### Project Structure Notes

- New files align with existing `src/stores/`, `src/services/errors/`, `src/app/components/settings/`, `src/types/` directories
- Store follows Zustand v5 pattern used throughout the project
- Component placed in `settings/` subdirectory matching existing Settings sub-components
