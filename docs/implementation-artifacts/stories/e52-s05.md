# Story 52.05: Recommendation Engine

## Story

As a learner,
I want the system to recommend courses similar to ones I have completed,
so that I can discover relevant learning content without manual searching.

## Acceptance Criteria

**Given** a learner has completed 3 or more courses with embeddings
**When** the recommendation engine is invoked
**Then** it queries the `courseEmbeddings` table and loads vectors into `BruteForceVectorStore`
**And** computes cosine similarity against each completed course's embedding
**And** over-fetches 3x candidates (15 for a top-5 result)

**Given** the raw similarity scores are computed
**When** the threshold filter is applied
**Then** only courses with similarity in the range 0.40-0.85 are retained
**And** already-completed courses are excluded from results

**Given** the filtered candidate set
**When** MMR diversity reranking is applied with lambda=0.5
**Then** the final list balances relevance and diversity
**And** iteratively selects items maximizing `lambda * relevance - (1-lambda) * max_similarity_to_selected`

**Given** a learner has completed 1-2 courses (cold start)
**When** the recommendation engine is invoked
**Then** it falls back to tag-based matching (overlap of auto-generated course tags)
**And** fills remaining slots with recently imported courses

**Given** a learner has 0 completed courses (full cold start)
**When** the recommendation engine is invoked
**Then** it returns recently imported courses sorted by import date
**And** includes the message "Complete a few courses to get personalized recommendations"

**Given** the recommendation engine is invoked
**When** it computes results
**Then** the total computation time is <10ms (in-browser, no network)
**And** it works fully offline using cached Dexie data

**Given** the embedding pipeline has failed for some courses
**When** the recommendation engine runs
**Then** it gracefully degrades to tag-based matching for courses without embeddings
**And** does not throw errors or show empty results

## Tasks / Subtasks

- [ ] Task 1: Recommendation Service Core (AC: 1, 2, 6)
  - [ ] 1.1 Create `src/ai/courseRecommendationService.ts` with `getRecommendations(completedCourseIds, options?)` function
  - [ ] 1.2 Load all `courseEmbeddings` from Dexie
  - [ ] 1.3 Load completed course embeddings, compute average or per-course similarities
  - [ ] 1.4 Use `BruteForceVectorStore` for cosine similarity computation
  - [ ] 1.5 Over-fetch 3x candidates for diversity reranking
  - [ ] 1.6 Ensure pure client-side computation (no network calls)

- [ ] Task 2: Threshold Filter (AC: 2)
  - [ ] 2.1 Filter results to similarity range 0.40-0.85
  - [ ] 2.2 Exclude courses already in `completedCourseIds`
  - [ ] 2.3 Return `{ courseId, similarity, sourceCourseId }` tuples

- [ ] Task 3: MMR Diversity Reranking (AC: 3)
  - [ ] 3.1 Implement MMR algorithm: `lambda * relevance - (1-lambda) * max_similarity_to_selected`
  - [ ] 3.2 Set lambda=0.5 (balanced relevance-diversity)
  - [ ] 3.3 Iteratively select top items from candidate set
  - [ ] 3.4 Return final top-5 (dashboard) or top-4 (course detail) results

- [ ] Task 4: Cold Start Cascade (AC: 4, 5)
  - [ ] 4.1 Detect cold start condition: `completedCourseIds.length < 3`
  - [ ] 4.2 For 1-2 completed courses: tag-based matching using course tags from `importedCourses`
  - [ ] 4.3 For 0 completed courses: return recently imported courses sorted by `importedAt`
  - [ ] 4.4 Fill remaining slots with recently imported courses if tag matches are insufficient
  - [ ] 4.5 Include `explanation` field: "Because you completed [X]" or "Recently imported"

- [ ] Task 5: Graceful Degradation (AC: 7)
  - [ ] 5.1 Handle missing embeddings: fall back to tag matching for those courses
  - [ ] 5.2 Handle empty courseEmbeddings table: use cold start cascade
  - [ ] 5.3 Never throw — always return a result (even if fallback)

- [ ] Task 6: useRecommendations Hook (AC: 6)
  - [ ] 6.1 Create `src/hooks/useRecommendations.ts`
  - [ ] 6.2 Return `{ recommendations, isLoading, error, refresh }` shape
  - [ ] 6.3 Query completed course IDs from Dexie `contentProgress`
  - [ ] 6.4 Call `getRecommendations()` on mount and on `refresh()`
  - [ ] 6.5 Support `mode` parameter: `'dashboard'` (top 5) or `'courseDetail'` (top 4, exclude current)

- [ ] Task 7: Unit Tests (AC: all)
  - [ ] 7.1 Test cosine similarity computation with known vector pairs
  - [ ] 7.2 Test threshold filter (0.40-0.85 range, excluded courses)
  - [ ] 7.3 Test MMR reranking produces diverse results
  - [ ] 7.4 Test cold start: 0 completed → recently imported
  - [ ] 7.5 Test cold start: 1-2 completed → tag-based + recent
  - [ ] 7.6 Test graceful degradation: missing embeddings → tag fallback
  - [ ] 7.7 Test performance: <10ms for 100 courses with embeddings

## Implementation Notes

**New files:**
- `src/ai/courseRecommendationService.ts` — Recommendation engine (embeddings + MMR + cold start)
- `src/hooks/useRecommendations.ts` — React hook for recommendation UI

**Existing files to reuse (not modify):**
- `src/lib/vectorSearch.ts` — `BruteForceVectorStore` for cosine similarity
- `src/db/schema.ts` — `courseEmbeddings` table (created in S01/S04)

**Key patterns:**
- Pure client-side computation — no server calls
- `BruteForceVectorStore` handles cosine similarity (never implement custom)
- MMR reranking: ~30 lines of implementation on top of vector store
- Cold start cascade: 0 → recently imported; 1-2 → tags; 3+ → embeddings + MMR
- Stateless service function (same pattern as `courseTagger.ts`)

**Dependencies:**
- E52-S04 (Course Embedding Aggregation — needs `courseEmbeddings` table populated)
