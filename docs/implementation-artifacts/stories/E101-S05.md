# Story 101.5: Audio Bookmarks & Learning Loop

## Story

As a Knowlune user listening to an Audiobookshelf audiobook,
I want to create timestamped bookmarks with a single tap and convert them to flashcards,
so that I can retain key concepts from audiobooks through spaced repetition.

## Acceptance Criteria

**Given** an ABS audiobook is streaming in the AudiobookRenderer
**When** the playback controls are visible
**Then** a floating bookmark FAB button is visible
**And** the FAB meets the 44x44px minimum touch target (NFR13)
**And** the FAB has sufficient contrast against the player background (NFR14)

**Given** audio is playing
**When** the user taps the bookmark FAB
**Then** a timestamped AudioBookmark is created within 200ms (NFR3)
**And** the bookmark captures: current chapter index, timestamp (seconds), book ID
**And** a brief visual confirmation appears (pulse animation or toast) without pausing playback
**And** the bookmark count badge on the FAB updates

**Given** multiple bookmarks are created during a session
**When** the user stops or finishes playback
**Then** a "Review Bookmarks" panel appears showing all bookmarks from this session (FR23)
**And** each bookmark shows: chapter title, formatted timestamp (e.g., "Chapter 3 — 14:32"), and a text input for notes

**Given** the user is reviewing a bookmark in the post-session panel
**When** they type a note and save
**Then** the bookmark's `note` field is updated in Dexie (FR24)
**And** the note persists across sessions (FR27)

**Given** a bookmark with a note
**When** the user taps "Create Flashcard"
**Then** the ClozeFlashcardCreator opens with the note text
**And** the user can tap words to blank them for cloze deletion (FR25)
**And** saving creates a Flashcard with `sourceType: 'book'`, `sourceBookId: bookId`
**And** the flashcard is added to the FSRS spaced repetition queue (FR26)

**Given** a bookmark without a note
**When** the user taps "Create Flashcard"
**Then** a prompt asks the user to add a note first (can't create cloze from empty text)

**Given** the existing AudiobookRenderer bookmark functionality from E87
**When** this story ships
**Then** the same BookmarkButton and BookmarkListPanel components work for both local and ABS audiobooks
**And** no E87 functionality is broken (bookmark creation, listing, note editing)

**Given** the user is interacting with the bookmark panel
**When** using keyboard navigation
**Then** all bookmark items, note fields, and action buttons are tab-navigable
**And** Enter activates buttons, Escape closes the panel

## Design Guidance

**Floating FAB placement:**
- Position within the `AudiobookRenderer` player container, not fixed to viewport
- Bottom-right quadrant of the controls area, visually distinct from playback controls
- Use `variant="brand"` or `variant="ghost"` with explicit `bg-brand/90` for contrast on dark player background
- FAB: `Bookmark` icon (lucide), `min-h-[44px] min-w-[44px]`, `rounded-full`
- Badge: small pill showing count, top-right corner of FAB button, `bg-destructive text-destructive-foreground` (red) or `bg-brand text-brand-foreground`

**Post-session review panel:**
- Use `<Sheet side="bottom">` — consistent with `BookmarkListPanel` and `ClozeFlashcardCreator`
- `max-h-[80vh] rounded-t-2xl` — same as `ClozeFlashcardCreator` styling
- Each bookmark row: `flex flex-col gap-1 px-4 py-3` — chapter/time in `text-xs text-muted-foreground`, note textarea below
- "Create Flashcard" button: `variant="brand-outline"` size="sm"

**Inline note textarea:**
- `<textarea>` with `resize-none`, auto-height via rows or CSS
- `placeholder="Add a note about what you heard…"`
- Save on blur or "Save" button (both trigger the Dexie update)

**Contrast / accessibility:**
- FAB must meet 3:1 contrast against player background (dark bg → use `bg-brand` button)
- All interactive elements: visible focus ring via Tailwind `focus-visible:ring-2 focus-visible:ring-brand`
- ARIA labels on FAB: `aria-label="Add bookmark"` + `aria-live` region for count badge

### Key Architectural Constraints

1. **FAB uses `BookmarkButton`** — do not create a new bookmark creation component. If `BookmarkButton` needs FAB styling, extend via props or wrapper — never duplicate logic.

2. **Post-session review is a NEW component** `PostSessionBookmarkReview.tsx` — it's separate from `BookmarkListPanel` because it adds note editing + flashcard creation (BookmarkListPanel only has seek + delete).

3. **No new Dexie tables** — `AudioBookmark` already lives in `db.audioBookmarks` (added in E87). No schema bump needed for this story.

4. **No new dependencies** — NFR5: zero new npm packages. All via existing Dexie, Zustand, Tailwind, shadcn/ui.

5. **ClozeFlashcardCreator link** — the component currently links flashcard ↔ highlight via `highlightId`. For audio bookmarks, omit `highlightId` (leave undefined). The flashcard `sourceBookId` still connects it to the book. This is intentional and already supported by the component.

6. **Flashcard FSRS queue** — `ClozeFlashcardCreator` already sets `state: 0, due: now`. Flashcard will appear in the next review session automatically. No additional queue wiring needed.

### Post-Session Trigger Logic

```typescript
// In AudiobookRenderer: detect isPlaying → false transition with bookmarks present
const [postSessionOpen, setPostSessionOpen] = useState(false)
const [sessionBookmarkCount, setSessionBookmarkCount] = useState(0)
const prevIsPlayingRef = useRef(false)

useEffect(() => {
  if (prevIsPlayingRef.current && !isPlaying && sessionBookmarkCount > 0) {
    setPostSessionOpen(true)
  }
  prevIsPlayingRef.current = isPlaying
}, [isPlaying, sessionBookmarkCount])
```

Track bookmarks created in the current session via local state in `AudiobookRenderer`. Increment on each `BookmarkButton` success (add `onBookmarkCreated` callback prop to `BookmarkButton` or subscribe to Dexie liveQuery).
