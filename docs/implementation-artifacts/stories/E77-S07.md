# Story 77.7: Storage Management & Backup Cleanup

Status: ready-for-dev

## Story

As a connected premium user,
I want to see my Google Drive storage usage and manage my backups,
so that I can monitor Drive space and keep my backup history clean.

## Acceptance Criteria

1. **Given** a connected premium user views the backup panel **When** the StorageUsageBar renders **Then** it displays "Drive storage: {used} of {total}" with a Progress bar, the bar color is `bg-brand` normally, `bg-warning` at >80% usage, `bg-destructive` at >95% usage, and below the bar: "Knowlune backups: {size} ({count} backups)" in `text-xs text-muted-foreground`, with `aria-label="Google Drive storage usage" aria-valuenow={percent}`.

2. **Given** the user's Google Workspace account has unlimited storage **When** the quota check returns no `limit` field **Then** the display shows "Drive storage: {used} (Unlimited)" with no progress bar.

3. **Given** Drive storage exceeds 90% capacity **When** the warning threshold is crossed **Then** a warning indicator appears below the storage bar: "Your Drive is almost full. Free up space to ensure backups succeed."

4. **Given** the storage information is stale **When** the panel opens or a backup/delete operation completes **Then** the storage info is refreshed from the `about.get()` API.

5. **Given** a user deletes a backup from the history list **When** the deletion completes **Then** the Knowlune backup size and count update immediately, and the Drive storage usage refreshes.

6. **Given** the auto-prune setting is configurable **When** the maximum backup count needs adjustment **Then** the setting is available in the backup settings section (default: 10, range: 3-20).

## Dev Notes

### Architecture Constraints

- **Storage quota** fetched via `GET /drive/v3/about?fields=storageQuota` (already in googleDriveService from Story 77.2)
- **Knowlune backup size**: calculated client-side by summing `size` from backup file list (not a separate API call)
- **Refresh triggers**: panel mount, after backup, after delete -- avoid excessive API calls
- **Max backup count** stored in IndexedDB settings (same pattern as other preferences)

### Storage Quota Response Shape

```typescript
interface StorageQuota {
  limit?: string    // Total quota in bytes (missing = unlimited)
  usage: string     // Used storage in bytes
  usageInDrive: string  // Used in Drive specifically
}
```

Note: Values come as strings from the API -- parse with `parseInt()`.

### Format Bytes Helper

```typescript
function formatBytes(bytes: number): string {
  if (bytes === 0) return '0 B'
  const k = 1024
  const sizes = ['B', 'KB', 'MB', 'GB', 'TB']
  const i = Math.floor(Math.log(bytes) / Math.log(k))
  return `${(bytes / Math.pow(k, i)).toFixed(1)} ${sizes[i]}`
}
```
