# Story 114.01: Reading Ruler and Letter/Word Spacing Controls

## Story

As a reader with dyslexia or reading difficulties,
I want a visual reading ruler and adjustable letter/word spacing,
so that I can reduce visual crowding and track my position on the page more easily.

## Acceptance Criteria

- **AC-1**: Reader Settings panel includes a Reading Ruler toggle (on/off switch).
- **AC-2**: When enabled, a horizontal band follows the pointer/touch position across the page, providing a visual guide for the current reading line.
- **AC-3**: The ruler band is `pointer-events-none` and does not block tap zones (previous page, next page, header/footer toggle).
- **AC-4**: The ruler band does not appear until after the first pointer move (no blank overlay intercepting clicks on activation).
- **AC-5**: Reader Settings panel includes a Letter Spacing slider (0–0.3em, step 0.02em).
- **AC-6**: Reader Settings panel includes a Word Spacing slider (0–0.5em, step 0.02em).
- **AC-7**: Letter and word spacing are applied to EPUB content via `rendition.themes.default()`.
- **AC-8**: Resetting spacing to 0 explicitly clears any stale `letter-spacing`/`word-spacing` values (uses `'normal'` rather than omitting the property).
- **AC-9**: All three settings (ruler enabled, letter spacing, word spacing) are persisted to localStorage and survive app reload.
- **AC-10**: The ruler band has `aria-hidden="true"` so screen readers skip it.

## Implementation Notes

- `ReadingRuler` uses a document-level `pointermove` listener (passive) so the component itself has `pointer-events-none` — tap navigation zones in EpubRenderer remain accessible at all times.
- `yPosition` state starts as `null`; the ruler band is only rendered after the first `pointermove`, preventing blank overlay blocking on activation.
- `getSettingsFromState` in `useReaderStore` is now data-driven via `Object.keys(DEFAULT_SETTINGS)` — new settings are automatically included without manual maintenance.
- Spacing values are clamped server-side: `letterSpacing` to `[0, 0.3]`, `wordSpacing` to `[0, 0.5]`.
- epub.js `rendition.themes.default()` does not remove previously-set properties; explicit `'normal'` strings are required when resetting to zero.

## Challenges and Lessons Learned

- **z-index + pointer-events interaction**: A full-viewport overlay at z-20 with `pointer-events-auto` silently blocks all interaction below. The fix is to keep the overlay `pointer-events-none` and use document-level event listeners instead of capturing events in the component.
- **epub.js theme accumulation**: `rendition.themes.default()` merges styles rather than replacing them — stale CSS properties persist until explicitly overridden. When resetting to zero, always send the explicit reset value (`'normal'`) rather than omitting the key.
- **Deferred render pattern**: Showing a UI element immediately on activation when it has no initial position creates an invisible interaction blocker. Deferring the visual render to after the first user interaction avoids this pitfall.
