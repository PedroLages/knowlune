# Story 108.3: Keyboard Shortcuts

## Story

As a reader,
I want keyboard shortcuts for common library and reader actions,
so that I can navigate and control the app efficiently without using the mouse.

## Acceptance Criteria

- **AC-1**: Pressing `?` anywhere in the library opens a keyboard shortcuts help dialog listing all available shortcuts
- **AC-2**: Library page shortcuts: `N` to open import dialog, `/` to focus search, `G` then `L` to toggle grid/list view, `Escape` to close dialogs
- **AC-3**: EPUB reader shortcuts: `Left`/`Right` arrows for page navigation, `Space` to toggle scroll/page mode, `T` to open TOC, `H` to open highlights, `B` to add bookmark, `S` to open settings, `Escape` to return to library
- **AC-4**: Audiobook player shortcuts: `Space` to play/pause, `Left`/`Right` arrows to skip back/forward 30s, `Up`/`Down` arrows for volume, `[`/`]` for speed decrease/increase, `M` to mute, `Escape` to return to library
- **AC-5**: Shortcuts are disabled when a text input or textarea is focused (prevent interference with typing)
- **AC-6**: All shortcuts are documented in the `KeyboardShortcutsDialog` component with appropriate groupings (Library, Reader, Audiobook)

## Design Guidance

- **ShortcutsDialog**: Extend existing `KeyboardShortcutsDialog` component with sections
- **Kbd component**: Use existing `<Kbd>` from `src/app/components/ui/kbd.tsx`
- **Visual hints**: Consider adding small shortcut hints to toolbar buttons (e.g., tooltip showing "Space" on play button)
- **Scope**: Shortcuts are scoped to their page context — library shortcuts only active on `/library`, reader shortcuts only in `/library/:id/read`

## Implementation Notes

**Key files to modify:**
- `src/app/components/figma/KeyboardShortcutsDialog.tsx` — add library/reader/audiobook sections
- `src/app/pages/Library.tsx` — register library shortcuts
- `src/app/pages/BookReader.tsx` — extend existing keyboard handling
- `src/app/components/audiobook/AudiobookRenderer.tsx` — add audiobook shortcuts

**New files:**
- `src/app/hooks/useKeyboardShortcuts.ts` — reusable shortcut registration hook

**Existing patterns:**
- `BookReader.tsx` already handles `ArrowLeft`, `ArrowRight`, `Space`, `Escape` via `useEffect` + `keydown` (see lines 130-170 approx)
- `KeyboardShortcutsDialog` uses `Kbd` component for key display
- `useReadingMode.ts` uses `keydown` for reading mode toggle

**Important**: The `BookReader` page is a full-viewport layout (NOT inside standard Layout), so its keyboard handlers don't conflict with global shortcuts.

**Chord patterns** (e.g., `G` then `L`): Implement with a timeout-based state machine — store first key, wait 500ms for second key, reset if timeout or wrong key.

## Testing Notes

- Unit tests: mock `document.activeElement` to test input-focus guard
- Unit tests: test chord pattern state machine (G → L sequence, G → timeout, G → wrong key)
- E2E: test shortcuts via Playwright `page.keyboard.press()`
- Edge case: shortcuts should not fire when browser's native shortcuts conflict (e.g., Cmd+K is already search)
- Verify no shortcut fires during IME composition (e.g., typing in search with Japanese keyboard)
