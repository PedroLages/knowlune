# Story E119-S13: Annual Review Checklist + ICO External Review + Beta User Re-ack

## Story

As Pedro (controller/operator) and as beta users (data subjects),
I want an annual review checklist, an ICO SME checklist walk completing E119, and a re-acknowledgement flow for beta users with ack rate measurement,
so that the epic closes with verified compliance and users are aware of their new rights.

## Acceptance Criteria

- **AC-1**: `docs/compliance/annual-review.md` — one-page checklist: re-read notice, verify retention job ran every day, verify sub-processor list, refresh DPA addenda, count consent records, review breach register.
- **AC-2**: `docs/compliance/ico-sme-checklist-2026.md` — each ICO SME checklist item recorded as pass/fail/NA with evidence link. All blockers resolved before closing E119.
- **AC-3**: `docs/compliance/README.md` created; cross-links all nine compliance docs.
- **AC-4**: A new notice version (e.g., `2026-05-01.1`) is pushed; `LegalUpdateBanner` fires for all existing users via `useNoticeAcknowledgement` (S02).
- **AC-5**: `scripts/compliance/ack-rate-report.ts` reads `notice_acknowledgements` + auth user list; outputs ack% with user count. Runs weekly during the 30-day beta window.
- **AC-6**: E2E test: beta user opens app → banner → reads notice → acknowledges → counted correctly in ack report.
- **AC-7**: User dismissing banner without ack is still counted as unacked; re-prompted next session.
- **AC-8**: After 30 days, unacked users enter soft-block mode (already built in S02); `annual-review.md` notes the follow-up email process.
- **AC-9**: End-to-end lifecycle E2E `tests/e2e/compliance/lifecycle.spec.ts` covers: signup → notice ack → use AI → export data → withdraw consent → delete account → verify zero rows.
