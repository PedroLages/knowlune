# Story 86.2: Daily Highlight Review Notifications

Status: ready-for-dev

## Story

As a learner,
I want to receive daily notifications surfacing random highlights from my books,
so that I can revisit important passages and reinforce my learning through spaced exposure.

## Acceptance Criteria

**Given** the user has 10 or more highlights across all books
**When** the configured daily review time arrives
**Then** a notification is created: "You have {count} highlights to review. Revisit 5 random passages from your books."
**And** the notification uses the existing notification system from E58

**Given** the user taps the daily highlight review notification
**When** the review screen opens
**Then** it displays 5 randomly-selected highlights as Readwise-style quote cards
**And** each card shows: quoted text (large, italic), book title, author, chapter, user note (if exists), and dot indicators showing position (e.g., 3 of 5)
**And** the user can swipe left/right (mobile) or use arrow buttons to navigate between cards

**Given** a highlight review card is showing
**When** the user sees the action buttons
**Then** three actions are available: "Open in Book" (navigates to highlight in reader), "Create Flashcard" (opens ClozeFlashcardCreator), and "Next" (advances to next card)
**And** if the highlight already has a linked flashcard, "Create Flashcard" changes to "Review Flashcard"

## Dev Notes

### Architecture Patterns

- **Notification system**: `src/services/NotificationService.ts` (E58) — existing service with trigger patterns. Add a new `HighlightReviewTrigger` following the same pattern as existing triggers (e.g., SRS due reminder from latest commit).
- **Notification preferences**: `src/stores/useNotificationPrefsStore.ts` — add a preference for daily highlight review time.
- **Event bus**: `src/lib/eventBus.ts` — notification system already uses event bus for trigger coordination.
- **Carousel/swipe**: Project uses Embla carousel via shadcn. Consider reusing `Carousel` component from `src/app/components/ui/carousel.tsx` for card navigation.

### UX Design Requirements (UX-DR17)

- Readwise-style quote cards: large, italic text with book metadata
- 5 cards per session with dot indicators
- Swipe on mobile, arrow buttons on desktop
- Three actions: Open in Book, Create Flashcard (or Review Flashcard), Next

### Dependencies

- **Hard dependency on E85**: BookHighlight records must exist in Dexie
- **Hard dependency on E58**: Notification system and NotificationService must exist
- **Soft dependency on E85-S04**: ClozeFlashcardCreator for "Create Flashcard" action
