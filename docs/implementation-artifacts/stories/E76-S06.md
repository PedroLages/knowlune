# Story 76.6: Email Digest Preferences UI

Status: ready-for-dev

## Story

As a user,
I want to enable or disable email digests and choose my preferred frequency in Settings,
so that I have full control over whether and how often I receive learning summaries.

## Acceptance Criteria

1. **Given** the NotificationPreferencesPanel component
   **When** the user scrolls past the Quiet Hours section
   **Then** they see an "Email Digest" section with:
   - Mail icon (lucide-react) with heading "Email Digest"
   - Description: "Receive a weekly summary of your learning progress via email."
   - Toggle switch (shadcn Switch component), default OFF
   **And** the section is separated from Quiet Hours by a Separator component

2. **Given** the email digest toggle is OFF
   **When** the user toggles it ON
   **Then** a frequency selector animates in (fade-in + slide-in-from-top, 300ms, matching Quiet Hours pattern)
   **And** the selector shows options: "Weekly" (default), "Daily"
   **And** the "Send Test Email" button appears below the selector
   **And** the preference is persisted immediately to the database (optimistic update)

3. **Given** the email digest toggle is ON
   **When** the user toggles it OFF
   **Then** the frequency selector and test button animate out
   **And** `email_digest_enabled` is set to `false` in the database immediately

4. **Given** the frequency selector
   **When** the user changes the frequency
   **Then** `digest_frequency` is updated in the database immediately
   **And** the UI confirms the change (no separate save button needed)

5. **Given** the Email Digest section
   **When** navigated via keyboard
   **Then** Tab moves through toggle, select, and button in order
   **And** all controls have appropriate `aria-label` attributes
   **And** the conditional section uses `aria-live="polite"` for screen reader announcements

6. **Given** the toggle state changes
   **When** the database update fails
   **Then** the UI reverts to the previous state
   **And** a toast notification displays the error message

## Dev Notes

### Design Guidance

**Layout**: Matches existing NotificationPreferencesPanel patterns exactly. The Email Digest section follows the same structure as Quiet Hours:
1. Separator
2. Section header (icon + title + description)
3. Toggle switch
4. Conditional reveal (frequency selector + test button)

**Component reuse**: Use existing shadcn components — `Switch`, `Select`, `SelectContent`, `SelectItem`, `SelectTrigger`, `SelectValue`, `Separator`, `Label`.

**Animation**: `animate-in fade-in-0 slide-in-from-top-1 duration-300` — this is the exact same animation class used for the Quiet Hours reveal. Provided by `tw-animate-css`.

**Styling rules** (from styling.md):
- Use design tokens, NOT hardcoded colors: `text-muted-foreground` (not `text-gray-500`), `bg-brand` for accents
- Button variant for test email: `variant="brand-outline"` with `Send` icon from lucide-react
- Touch targets >= 44px on all interactive elements

### Existing Code Context

The `NotificationPreferencesPanel.tsx` at `src/app/components/settings/NotificationPreferencesPanel.tsx` already has:
- Notification type toggles (course-complete, streak-milestone, etc.)
- Quiet Hours section with progressive disclosure pattern
- Uses `useNotificationPrefsStore` for state management

The `useNotificationPrefsStore.ts` at `src/stores/useNotificationPrefsStore.ts` needs:
- `emailDigestEnabled` and `digestFrequency` in DEFAULTS (added in E76-S01)
- `setEmailDigest` action (added in E76-S01)
- This story only CONSUMES the store — it does not modify the store itself

### Critical Constraints

- Do NOT create a new component file — the Email Digest section is inline in `NotificationPreferencesPanel.tsx` (30-40 lines, following existing pattern).
- Do NOT use hardcoded Tailwind colors — use design tokens per styling.md.
- The "Send Test Email" button is rendered here but functionality is wired in E76-S07.
- Respect `prefers-reduced-motion` — `tw-animate-css` already handles this.

### Dependencies

- E76-S01 (schema + store) — `emailDigestEnabled`, `digestFrequency` fields and `setEmailDigest` action must exist
