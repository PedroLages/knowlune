# Story 70.1: Scheduling Types and Free/Busy Slot Merger

Status: ready-for-dev

## Story

As a learner,
I want the system to know when I'm free and when I'm busy,
so that study suggestions only recommend times that actually work for me.

## Acceptance Criteria

1. **Given** the scheduling types module exists at `src/data/scheduling-types.ts`, **When** the types are imported by any consumer, **Then** `TimeWindow`, `ScoredSlot`, `SchedulingSuggestion`, `WeekPlan`, and `WeekAllocation` types are available with all fields defined per architecture ADR-1.

2. **Given** the user has study blocks in the E50 `studyBlocks` Dexie table and optionally Google Calendar events cached locally, **When** the `mergeSlots` function is called with a date range (default: next 7 days, 7am-11pm daily), **Then** all busy intervals from Knowlune study blocks, Google Calendar events, and quiet hours from NotificationPreferences are collected, sorted by start time, and merged (overlapping intervals combined). **And** the complement (free windows) is computed within the scheduling horizon. **And** any free window crossing midnight is split into two windows. **And** windows shorter than 15 minutes (`MIN_SESSION_MINUTES`) are filtered out. **And** each returned `TimeWindow` includes `start` (ISO 8601), `end` (ISO 8601), `durationMinutes`, and `source` (`'knowlune'` | `'google-calendar'` | `'merged'`).

3. **Given** no Google Calendar data is available (Phase 3 not complete or never synced), **When** `mergeSlots` is called, **Then** computation proceeds using only Knowlune study schedule data -- no error, no warning (this is normal for Tier 1-2).

4. **Given** the scheduling engine at `src/lib/scheduling/`, **When** any module in the scheduling directory is inspected, **Then** it contains zero React imports and zero Dexie imports -- pure TypeScript computation only.

5. **Given** the `mergeSlots` module has unit tests at `src/lib/scheduling/__tests__/mergeSlots.test.ts`, **When** tests are run, **Then** they cover: overlapping intervals, adjacent intervals, disjoint intervals, midnight-crossing windows, sub-15-minute filtering, empty input, and quiet hours integration. **And** all tests use `FIXED_DATE` pattern for deterministic time.

### Architecture Compliance

**Pure computation boundary (CRITICAL):** `src/lib/scheduling/` must have ZERO React and ZERO Dexie imports. All data must be passed in as plain objects. The Zustand store (E70-S03) will handle Dexie reads and pass data to the engine.

**Type definitions (ADR-1):** Copy these exactly into `src/data/scheduling-types.ts`:

```typescript
export interface TimeWindow {
  start: string        // ISO 8601
  end: string          // ISO 8601
  durationMinutes: number
  source: 'knowlune' | 'google-calendar' | 'merged'
}

export interface ScoredSlot {
  window: TimeWindow
  score: number        // 0-100 composite score
  signals: {
    productivityScore: number    // 0-1
    gapUrgency: number           // 0-1
    deadlineProximity: number    // 0-1
    fsrsUrgency: number          // 0-1
    recommendationWeight: number // 0-1
  }
  reasoning: string
}

export interface SchedulingSuggestion {
  courseId: string
  courseName: string
  requestedDuration: number  // minutes
  slots: ScoredSlot[]
  generatedAt: string        // ISO 8601
}

export interface WeekPlan {
  weekStart: string          // ISO 8601 (Monday)
  allocations: WeekAllocation[]
  unscheduledCourses: string[]
  totalStudyMinutes: number
}

export interface WeekAllocation {
  courseId: string
  courseName: string
  slot: ScoredSlot
  sessionType: 'new-content' | 'review' | 'mixed'
  estimatedDuration: number  // minutes
}
```

### Interval Merge Algorithm (ADR-5)

1. Collect all busy intervals from inputs (study blocks, Google Calendar events, quiet hours)
2. Sort by start time
3. Merge overlapping intervals: if `current.end >= next.start`, extend `current.end = max(current.end, next.end)`
4. Compute complement: free windows between merged busy intervals within the daily horizon (7am-11pm)
5. Split any window crossing midnight into two separate windows
6. Filter out windows < 15 minutes

**mergeSlots function signature:**

```typescript
export function mergeSlots(params: {
  studyBlocks: Array<{ start: string; end: string }>
  googleCalendarEvents?: Array<{ start: string; end: string }>
  quietHours?: { start: string; end: string } | null  // e.g., "22:00"-"08:00"
  horizonDays?: number   // default: 7
  now?: string           // ISO 8601, for testing with FIXED_DATE
}): TimeWindow[]
```
