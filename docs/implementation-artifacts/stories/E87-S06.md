# Story 87.6: Audiobook Session Tracking and Streak

Status: ready-for-dev

## Story

As a learner,
I want my audiobook listening time tracked and counted toward my study streak,
so that listening to educational audiobooks contributes to my learning habits.

## Acceptance Criteria

**Given** the user opens an audiobook and presses play
**When** audio playback begins
**Then** a listening session starts with a timestamp recorded

**Given** the user pauses, closes the player, or navigates away
**When** playback stops
**Then** a `studySession` record is created with content type 'audiobook', bookId, start/end timestamps, and listening duration
**And** the streak service processes the session event
**And** listening time appears in Reports alongside reading and course time

## Tasks / Subtasks

- [ ] Task 1: Track listening session start/stop (AC: 1, 2)
  - [ ] 1.1 In `useAudioPlayer` hook, record `sessionStartTime` when `play()` is called
  - [ ] 1.2 On `pause()`, tab visibility change (`visibilitychange`), or component unmount: calculate duration and create session record
  - [ ] 1.3 Minimum duration threshold: only create session if duration >= 30 seconds (prevent accidental plays)

- [ ] Task 2: Create studySession record (AC: 2)
  - [ ] 2.1 Write to `studySessions` Dexie table with: `contentType: 'audiobook'`, `contentId: bookId`, `startedAt`, `endedAt`, `durationMinutes`
  - [ ] 2.2 Follow same pattern as E85-S06 reading session tracking (reuse existing studySession type)

- [ ] Task 3: Emit session event for streak service (AC: 2)
  - [ ] 3.1 Emit `listening:session-ended` event on event bus after session record is created
  - [ ] 3.2 Ensure streak service (existing from prior epics) processes this event type
  - [ ] 3.3 If streak service doesn't handle `listening:session-ended`, extend it to handle both `reading:session-ended` and `listening:session-ended`

- [ ] Task 4: Integrate listening time into Reports (AC: 2)
  - [ ] 4.1 Extend `ReadingStatsService` (from E86-S01) to also query sessions with `contentType: 'audiobook'`
  - [ ] 4.2 Or create a unified "media time" aggregation that combines course, reading, and listening sessions
  - [ ] 4.3 Ensure Reports "total study time" includes audiobook listening

- [ ] Task 5: Handle edge cases (AC: 1, 2)
  - [ ] 5.1 Sleep timer pause: when sleep timer pauses audio, end the current session
  - [ ] 5.2 Tab/window close: use `beforeunload` or `visibilitychange` to persist session before page closes
  - [ ] 5.3 Multiple play/pause cycles: each creates a separate session (don't merge)

- [ ] Task 6: Tests (AC: 1, 2)
  - [ ] 6.1 Test session created on pause with correct timestamps and duration
  - [ ] 6.2 Test minimum duration threshold (< 30s creates no session)
  - [ ] 6.3 Test event bus emission for streak
  - [ ] 6.4 Test session included in Reports aggregation

## Dev Notes

### Architecture Patterns

- **Session tracking pattern**: Identical to E85-S06 reading session tracking. Reuse the same `studySessions` table with `contentType: 'audiobook'` instead of `'book'`.
- **Streak service**: Existing streak service processes study session events. May need to register new event type `listening:session-ended`.
- **Event bus**: `src/lib/eventBus.ts` — add `listening:session-ended` event type if not present.

### Session Lifecycle

```
play() → record sessionStartTime
  ↓
pause() / unmount / visibility hidden / sleep timer
  ↓
if (duration >= 30s) → create studySession record → emit event → streak processes
```

### Edge Case: beforeunload

```typescript
// Persist session on page close
window.addEventListener('beforeunload', () => {
  if (sessionStartTime) {
    // Use navigator.sendBeacon or synchronous Dexie write
    // Note: Dexie.js writes are async — may not complete on beforeunload
    // Fallback: store session in localStorage, reconcile on next load
  }
})
```

Consider storing in-progress session data in localStorage as a safety net, then reconciling on next app load.
