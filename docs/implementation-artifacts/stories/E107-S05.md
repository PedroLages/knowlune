# Story 107.5: Sync Reader Themes

## Story

As a reader,
I want the EPUB reader's theme (background color, text color, font styles) to automatically match the app's active color scheme,
so that switching between light/dark mode or color schemes provides a consistent reading experience without manual adjustment.

## Acceptance Criteria

- **AC-1**: When the app theme changes (light/dark mode toggle or color scheme switch), the EPUB reader iframe background and text colors update to match the active theme tokens
- **AC-2**: The reader's theme state is derived from the app's theme system (CSS custom properties from `theme.css`), not from a separate hardcoded theme config
- **AC-3**: Theme transitions in the reader are smooth (no flash of wrong colors when opening a book or switching themes)
- **AC-4**: All three color schemes (Professional, Vibrant, Clean) render correctly in the reader with appropriate contrast ratios (WCAG AA 4.5:1 for body text)

## Implementation Notes

- Created `readerThemeConfig.ts` as single source of truth bridging app color schemes (Professional/Vibrant/Clean) with reader themes (Light/Sepia/Dark)
- Used static hex color map instead of runtime CSS property reads because epub.js renders in an isolated iframe where CSS custom properties don't cascade
- Pre-computed Tailwind arbitrary-value classes as literal strings for JIT scanning compatibility
- Added `useAppColorScheme()` hook that reactively tracks settings changes via `settingsUpdated` custom event
- Updated all 5 reader chrome components (EpubRenderer, ReaderHeader, ReaderFooter, ReaderSettingsPanel, TtsControlBar) to derive colors from shared config

## Testing Notes

- 100 unit tests for readerThemeConfig covering all theme x scheme combinations and fallback behavior
- Updated existing ReaderHeader and EpubRenderer unit tests for shared theme config integration
- 8 E2E tests covering AC-1 through AC-4: background color verification per scheme, header/footer color derivation, settings panel pill reflection, and runtime scheme switching
- Sepia theme intentionally ignores app color scheme (same colors across all 3 schemes) - verified in unit tests

## Challenges and Lessons Learned

1. **epub.js iframe isolation requires resolved hex values**: CSS custom properties from the host document don't cascade into the epub.js iframe. The static `THEME_COLORS` map resolves all values upfront, avoiding runtime `getComputedStyle()` calls that would be fragile across color scheme transitions.

2. **Tailwind v4 JIT needs literal class strings**: Arbitrary values like `bg-[#faf5ee]` must appear as literal strings in source code for Tailwind's scanner to detect them. Dynamic string interpolation (e.g., `` `bg-[${hex}]` ``) produces classes Tailwind never sees, resulting in missing styles. The `BG_CLASSES` and `TEXT_CLASSES` lookup maps ensure all class strings are scannable.

3. **Sepia as reader-only theme simplifies the matrix**: Rather than trying to tint sepia differently per color scheme, keeping sepia identical across Professional/Vibrant/Clean avoids combinatorial complexity while matching user expectations (sepia is an explicit reader preference, not an app theme derivative).
