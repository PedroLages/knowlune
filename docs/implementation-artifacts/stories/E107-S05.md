# Story 107.5: Sync Reader Themes

## Story

As a reader,
I want the EPUB reader's theme (background color, text color, font styles) to automatically match the app's active color scheme,
so that switching between light/dark mode or color schemes provides a consistent reading experience without manual adjustment.

## Acceptance Criteria

- **AC-1**: When the app theme changes (light/dark mode toggle or color scheme switch), the EPUB reader iframe background and text colors update to match the active theme tokens
- **AC-2**: The reader's theme state is derived from the app's theme system (CSS custom properties from `theme.css`), not from a separate hardcoded theme config
- **AC-3**: Theme transitions in the reader are smooth (no flash of wrong colors when opening a book or switching themes)
- **AC-4**: All three color schemes (Professional, Vibrant, Clean) render correctly in the reader with appropriate contrast ratios (WCAG AA 4.5:1 for body text)

## Tasks / Subtasks

- [ ] Task 1: Audit current reader theme implementation (AC: 1, 2)
  - [ ] 1.1 Identify where EpubRenderer applies theme/background styles
  - [ ] 1.2 Identify how epub.js rendition themes are configured
  - [ ] 1.3 Map current theme values to app theme tokens
- [ ] Task 2: Connect reader theme to app theme system (AC: 2)
  - [ ] 2.1 Read active theme from CSS custom properties or settings store
  - [ ] 2.2 Apply theme tokens to epub.js rendition via `rendition.themes.default()`
  - [ ] 2.3 Ensure reader background matches `--background` token
- [ ] Task 3: Handle dynamic theme changes (AC: 1, 3)
  - [ ] 3.1 Listen for theme/color scheme changes (MutationObserver on `<html>` class or settings store subscription)
  - [ ] 3.2 Re-apply epub.js rendition theme when app theme changes
  - [ ] 3.3 Prevent flash by applying theme before rendering content
- [ ] Task 4: Validate all color schemes (AC: 4)
  - [ ] 4.1 Test Professional scheme in light/dark modes
  - [ ] 4.2 Test Vibrant scheme in light/dark modes
  - [ ] 4.3 Test Clean scheme (light mode only)
- [ ] Task 5: Add tests for theme sync behavior (AC: 1, 2, 3)
  - [ ] 5.1 Unit test: reader theme derived from app theme tokens
  - [ ] 5.2 Unit test: theme update triggers rendition re-theme
