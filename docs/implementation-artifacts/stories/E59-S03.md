# Story 59.03: Dexie v29 Schema Migration with Data Transformation

## Story

As a learner with existing flashcards and review records,
I want my SM-2 data automatically migrated to FSRS fields when the app upgrades,
so that the new FSRS scheduling algorithm can operate on my existing learning data without data loss.

## Acceptance Criteria

**Given** existing `flashcards` records with SM-2 fields (`easeFactor`, `interval`, `reviewCount`, `nextReviewAt`, `reviewedAt`)
**When** Dexie upgrades from v30 to v31
**Then** each flashcard record gains FSRS fields (`stability`, `difficulty`, `reps`, `lapses`, `state`, `elapsed_days`, `scheduled_days`, `due`, `last_review`) computed from SM-2 data
**And** legacy SM-2 fields are removed from each record
**And** the `flashcards` index changes from `nextReviewAt` to `due`

**Given** existing `reviewRecords` with SM-2 fields (`easeFactor`, `interval`, `reviewCount`, `nextReviewAt`, `reviewedAt`)
**When** Dexie upgrades from v30 to v31
**Then** each review record gains FSRS fields (`stability`, `difficulty`, `reps`, `lapses`, `state`, `elapsed_days`, `scheduled_days`, `due`, `last_review`) computed from SM-2 data
**And** legacy SM-2 fields are removed from each record
**And** the `reviewRecords` index changes from `nextReviewAt, reviewedAt` to `due, last_review`

**Given** a fresh install (no existing data)
**When** the app creates the database
**Then** the checkpoint schema (v31) is used directly without running migration callbacks

**Given** the migration runs on existing data
**When** SM-2 fields are converted
**Then** `easeFactor` maps to `difficulty` (inverted scale), `interval` maps to `stability`, `reviewCount` maps to `reps`, `nextReviewAt` maps to `due`, `reviewedAt` maps to `last_review`
**And** cards with `reviewCount === 0` get state=0 (New), cards with `interval < 1` get state=1 (Learning), all others get state=2 (Review)

**Given** the schema-checkpoint test
**When** comparing migration-built vs checkpoint-built schemas
**Then** both produce identical table structures at v31
