# Story 86.1: Reading Statistics in Reports Dashboard

Status: done

## Story

As a learner,
I want to see my reading statistics alongside my course study statistics,
so that I can understand my complete learning habits across all content types.

## Acceptance Criteria

**Given** the user has reading session data from E85
**When** the user navigates to the Reports dashboard
**Then** a "Reading" section appears showing: time read today, books currently in progress (count), total books finished, average pages per session, and reading time trend (daily/weekly chart)
**And** reading time is integrated into the existing total study time metrics (not shown separately)
**And** the Reports page does not break or error if the user has no book data

## Dev Notes

### Architecture Patterns

- **Reports page**: `src/app/pages/Reports.tsx` — existing page with tabs (study, quizzes, ai). Reading stats integrate into the "study" tab, not as a separate tab.
- **Session store**: `src/stores/useSessionStore.ts` — existing Zustand store for study sessions. Reading sessions use the same `studySessions` Dexie table with `contentType: 'book'`.
- **Chart components**: Reports page uses recharts via shadcn Chart component. Follow the same pattern for reading time trend chart.
- **Event bus**: `src/lib/eventBus.ts` — `reading:session-ended` events are emitted by E85. This story consumes that data from Dexie, not directly from events.

### Dependencies

- **Hard dependency on E85-S06**: Reading session tracking must exist (studySession records with `contentType: 'book'`). If E85 is not yet implemented, this story can still be built with mock data and the Dexie query infrastructure.
- **Soft dependency on E83**: Book records must exist for "books in progress" and "books finished" counts. Queries against `books` table by `status` field.
