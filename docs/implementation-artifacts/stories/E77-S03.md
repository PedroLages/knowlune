# Story 77.3: Manual Backup to Google Drive

Status: ready-for-dev

## Story

As a connected premium user,
I want to back up all my learning data to Google Drive with one tap,
so that I have a cloud copy of my courses, progress, notes, and flashcards.

## Acceptance Criteria

1. **Given** a user with a valid Drive token clicks "Send to Drive" **When** the button is clicked **Then** a multipart upload sends the exported JSON blob to the user's Google Drive root folder, with a progress indicator showing "Exporting data..." and "Uploading to Google Drive..." phases.

2. **Given** the upload completes successfully **When** the Drive API returns `{ id, webViewLink }` **Then** a success toast shows "Saved to Drive" with a "View" link to `webViewLink`, the progress indicator reaches 100%, and the button returns to its enabled state.

3. **Given** the export phase encounters an error **When** `exportAllAsJson()` throws **Then** an "Export failed" error toast is shown, distinct from upload-phase errors.

4. **Given** the upload encounters a 403 with a quota/storage-related message **When** mapped through `mapDriveError()` **Then** a `DriveQuotaError` throws and the UI shows a quota-exceeded toast.

5. **Given** the upload encounters a 403 with a permission-related message (scope revoked) **When** mapped through `mapDriveError()` **Then** a `DrivePermissionError` throws, the reconnect dialog opens, and the user is prompted to re-authenticate with the Drive scope.

6. **Given** the upload encounters a 401 (token expired mid-upload) **When** caught **Then** the token is refreshed via `refreshDriveToken()` and the upload is retried once. If the refresh also fails, a `DrivePermissionError` is thrown.

7. **Given** the upload encounters a network error **When** `fetch()` throws **Then** the upload is retried once after a 1-second backoff. If both attempts fail, a `DriveNetworkError` is thrown and a "Upload failed. Try again?" toast is shown.

8. **Given** a user without a Drive token (token helper returns `null`) **When** "Send to Drive" is clicked **Then** a `ReconnectGoogleDialog` opens with an explanation that Drive scope is needed and a button to trigger `signInWithOAuth`.

9. **Given** any unexpected error occurs during the backup flow **When** the error is not a recognized error type **Then** a generic "Upload failed. Try again?" toast is shown and the error is logged to the console.

## Dev Notes

### Architecture Constraints

- **Reuse existing `exportAllAsJson()`** from `src/lib/exportService.ts` -- do NOT reimplement data collection
- **`KnowluneExport` interface** already handles non-serializable field stripping (FileSystemHandles, Blobs)
- **`CURRENT_SCHEMA_VERSION = 14`** is embedded in the export automatically
- **Progress callback pattern**: `(percent: number, phase: string) => void` -- same signature as `ExportProgressCallback` in exportService.ts
- **Filename format**: `YYYY-MM-DD_HHmmss.knowlune.json` -- sortable, human-readable, unique per second
- **5 MB threshold**: Matches Google's documented limit for multipart uploads
- **Post-upload verification**: Fetch metadata after upload, compare `size` field (Adversarial #4)

### Key Files to Create

| File | Purpose |
|------|---------|
| `src/services/googleDriveBackupService.ts` | Backup orchestrator: `backupToDrive()`, `pruneOldBackups()` |

### Performance Target

- NFR-1: Backup (data collection + upload) must complete in <30 seconds for typical usage (<2 MB)
- Typical backup size: 500 KB - 2 MB
