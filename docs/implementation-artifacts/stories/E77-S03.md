# Story 77.3: Manual Backup to Google Drive

Status: ready-for-dev

## Story

As a connected premium user,
I want to back up all my learning data to Google Drive with one tap,
so that I have a cloud copy of my courses, progress, notes, and flashcards.

## Acceptance Criteria

1. **Given** a connected premium user clicks "Back Up Now" **When** the backup process starts **Then** the button becomes disabled and shows a spinner with "Collecting data..." text, and `exportAllAsJson()` is called to collect all IndexedDB data (0-60% progress).

2. **Given** the export data is serialized as JSON **When** the serialized size is <5 MB **Then** the file is uploaded via multipart upload as `YYYY-MM-DD_HHmmss.knowlune.json` to `Knowlune/backups/` and progress text changes to "Uploading to Google Drive..." (60-90%).

3. **Given** the serialized size is >=5 MB **When** the upload strategy is selected **Then** a resumable upload is initiated with proper `X-Upload-Content-Length` header and the content is uploaded to the session URI.

4. **Given** the upload completes successfully **When** post-upload verification runs **Then** the service fetches the uploaded file's metadata and verifies `size` matches the expected byte count (Adversarial #4), and if size mismatch is detected the file is deleted and an error is shown.

5. **Given** the backup count exceeds the maximum (default: 10) **When** auto-prune runs after successful backup **Then** the oldest backups are deleted until count equals the maximum, progress text shows "Finalizing..." (90-100%), and pruning failures are logged but do not block the backup success notification.

6. **Given** the entire backup flow completes **When** the success state is reached **Then** a toast confirms "Backup complete -- {size} saved to Google Drive", the backup history list refreshes, and `lastBackupAt` timestamp updates in the store.

7. **Given** the pre-upload quota check detects insufficient space **When** the backup is blocked **Then** a toast error shows "Not enough Drive storage. Your backup needs {X} MB but only {Y} MB is available." and the button returns to normal enabled state.

8. **Given** a token expires mid-backup (401 during upload) **When** the error is caught **Then** any partial resumable upload session is abandoned, the token expired state is shown with "Reconnect" prompt and a "Retry" option.

9. **Given** a network error occurs during backup **When** retries are exhausted **Then** a toast error shows "Backup failed. Check your connection and try again."

## Dev Notes

### Architecture Constraints

- **Reuse existing `exportAllAsJson()`** from `src/lib/exportService.ts` -- do NOT reimplement data collection
- **`KnowluneExport` interface** already handles non-serializable field stripping (FileSystemHandles, Blobs)
- **`CURRENT_SCHEMA_VERSION = 14`** is embedded in the export automatically
- **Progress callback pattern**: `(percent: number, phase: string) => void` -- same signature as `ExportProgressCallback` in exportService.ts
- **Filename format**: `YYYY-MM-DD_HHmmss.knowlune.json` -- sortable, human-readable, unique per second
- **5 MB threshold**: Matches Google's documented limit for multipart uploads
- **Post-upload verification**: Fetch metadata after upload, compare `size` field (Adversarial #4)

### Key Files to Create

| `src/services/googleDriveBackupService.ts` | Backup orchestrator: `backupToDrive()`, `pruneOldBackups()` |
