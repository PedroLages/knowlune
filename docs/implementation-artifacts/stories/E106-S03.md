# Story 106.3: Hook Coverage & Threshold Raise

## Story

As a developer,
I want unit tests for zero-coverage hooks and the coverage threshold raised to 70%,
So that the coverage gap is permanently closed with a CI gate.

## Acceptance Criteria

**AC1: Hook test files created**
**Given** the 6 target hooks have 0% or very low coverage
**When** unit tests are written for each hook's behavior
**Then** test files exist for each target hook

**AC2: Additional low-coverage files tested**
**Given** 4 additional files have coverage below 20%
**When** unit tests are written
**Then** test files exist for each target

**AC3: Global coverage reaches 70%**
**Given** all E106 stories are complete
**When** `npm run test:unit -- --coverage` is executed
**Then** global statement coverage is 70% or higher

**AC4: Threshold raised**
**Given** coverage reaches 70%+
**When** `vitest.config.ts` is updated
**Then** the coverage threshold is set to 70%

**AC5: KI-036 resolved**
**Given** the threshold is raised and coverage meets it
**When** `docs/known-issues.yaml` is updated
**Then** KI-036 status is `fixed` with `fixed_by: E106-S03`

**AC6: Suite passes**
**Given** all new tests and config change are applied
**When** `npm run test:unit` is executed
**Then** zero test failures reported
