## Challenges and Lessons Learned

1. **70% coverage target was aspirational, not achievable through unit tests alone.** Analysis showed 6280 uncovered statements in .tsx component/page files. These require React Testing Library component rendering (`render()`, not `renderHook()`), which is fundamentally different from unit testing hooks and lib modules. The realistic ceiling for hooks/lib/store unit testing is ~62-63%.

2. **vi.mock() hoisting requires vi.hoisted() for mock state.** Multiple test files initially failed because `const mockFn = vi.fn()` declared at module scope is not available inside `vi.mock()` factory functions (which are hoisted above all declarations). Using `vi.hoisted()` is the correct pattern.

3. **Fake timers and async hooks don't mix.** The useQuizGeneration hook uses `setInterval` for health checks. Combining `vi.useFakeTimers()` with `await waitFor()` causes timeouts because the async resolution never triggers. Solution: use real timers and let the hook's async effects resolve naturally.

4. **Three non-existent hook files in the story spec.** `useFlashcardPreview.ts`, `useCoverPreview.ts`, and `useFocusModeEvents.ts` do not exist in the codebase. Substituted with `useCourseCardPreview`, `useHoverPreview`, and `useDashboardOrder` which were actual 0%-coverage hooks.

5. **jsdom lacks scrollIntoView.** Tests for `scrollIntoViewReducedMotion` needed to manually assign `el.scrollIntoView = vi.fn()` before spying, since jsdom doesn't implement this method.
