# Story 87.2: Audiobook Player Controls and Playback

Status: ready-for-dev

## Story

As a learner,
I want to play audiobooks with standard controls including play, pause, seek, and skip,
so that I can listen to my audiobooks with a familiar playback experience.

## Acceptance Criteria

**Given** the user opens an audiobook from the Library
**When** the BookReader page loads at `/library/{bookId}` for an audiobook
**Then** the AudiobookRenderer displays (lazy-loaded via React.lazy()) with: large cover art, chapter title, progress scrubber (Slider component), time labels (current/total), play/pause button (size-16), skip back 15s and skip forward 30s buttons

**Given** the audiobook player is loaded
**When** the user taps play
**Then** the HTML5 `<audio>` element plays the current chapter's MP3 file
**And** the progress scrubber updates in real-time
**And** the user can drag the scrubber to seek within the chapter
**And** seek operations complete in <200ms (NFR4)

**Given** the user taps skip forward 30s or skip back 15s
**When** the current position updates
**Then** the audio seeks to the new position
**And** if skipping past chapter end, the next chapter loads and plays from the beginning
**And** if skipping before chapter start, the previous chapter loads and plays from its end minus the remaining skip amount

**Given** playback has been paused for more than 30 seconds
**When** the user resumes playback
**Then** playback starts 5 seconds before the paused position (clamped to chapter start if less than 5s into the chapter)
**And** a brief "Rewound 5s" toast appears for 2 seconds

### UX Design (UX-DR14)

- Expanded player: large cover art, chapter title, progress scrubber, large play button, skip buttons
- Controls: Play/Pause (center, largest), Skip Back 15s (left), Skip Forward 30s (right)
- Time labels: `currentTime / totalDuration` below scrubber
- Touch targets: all buttons >= 44x44px

### Auto-Rewind on Resume

Standard audiobook UX pattern (Overcast "Smart Resume", BookPlayer, Smart AudioBook Player). After a pause >30s, the listener has lost context and needs a few seconds of replay. The 30s threshold avoids rewinding on quick pause/play toggles (e.g., someone talking to you). The 5s rewind duration is the industry standard.

### NFR Compliance

- NFR4: Seek operations < 200ms
- NFR14: Controls accessible via keyboard and screen reader
- NFR18: Audiobook code adds < 200KB gzipped
- NFR20: Lazy-loaded, not in initial bundle
