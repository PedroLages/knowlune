# Story 71.2: ActionCard and SuggestedActionsPanel UI Components

Status: ready-for-dev

## Story

As a learner viewing the Knowledge Map,
I want to see a panel of action suggestion cards with topic details, urgency indicators, and one-click CTAs,
so that I can quickly navigate to the right remediation activity for my weakest topics.

## Acceptance Criteria

1. **Given** an `ActionSuggestion` with `actionType: 'flashcard-review'`, score 35, and trend 'declining', **When** `ActionCard` renders, **Then** it displays: the `Layers` icon, topic name, a score badge with `bg-destructive/10 text-destructive` styling, a `TrendingDown` icon in `text-destructive`, the action description "Review 5 flashcards on [topic]", a time estimate badge "5 min review", and a CTA button with `variant="brand"` text "Start Review".

2. **Given** an `ActionSuggestion` with `actionType: 'quiz-refresh'` and score 55 (fading), **When** `ActionCard` renders, **Then** the score badge uses `bg-warning/10 text-warning` styling, the trend icon is `Minus` in `text-warning`, the action type icon is `Brain`, and the CTA button says "Take Quiz".

3. **Given** an `ActionSuggestion` with `actionType: 'lesson-rewatch'`, **When** `ActionCard` renders, **Then** the action type icon is `RotateCcw` and the CTA button says "Watch Lesson".

4. **Given** the CTA button on an ActionCard with `actionRoute: '/flashcards?topic=body-language'`, **When** the learner clicks it, **Then** React Router navigates to `/flashcards?topic=body-language` (via `<Link>`, not `useNavigate`).

5. **Given** the ActionCard renders with a topic name longer than the card width, **When** displayed, **Then** the topic name is truncated with ellipsis and a `title` attribute shows the full name on hover.

6. **Given** a `SuggestedActionsPanel` with 3 action suggestions, **When** rendered on desktop (>= 1024px), **Then** the panel appears as a vertical sidebar (`w-80`) with cards stacked in a `flex flex-col gap-3` layout.

7. **Given** a `SuggestedActionsPanel` with 3 action suggestions, **When** rendered on tablet (640-1023px), **Then** the panel renders below the treemap as a 2-column grid (`grid grid-cols-2 gap-3`).

8. **Given** a `SuggestedActionsPanel` with 3 action suggestions, **When** rendered on mobile (< 640px), **Then** the panel renders as a horizontal scroll row with `snap-x snap-mandatory` and each card at `min-w-[280px]`.

9. **Given** no action suggestions exist (all topics strong), **When** `SuggestedActionsPanel` renders, **Then** it shows the empty state with a `CheckCircle2` icon and "All topics looking strong!" message.

10. **Given** 7 action suggestions with `maxVisible: 5`, **When** `SuggestedActionsPanel` renders, **Then** only 5 cards are visible initially with a "Show 2 more suggestions" link below.

11. **Given** the "Show more" link is clicked, **When** all 7 suggestions are revealed, **Then** the link text changes to "Show less" with a `ChevronUp` icon.

12. **Given** the panel renders, **When** inspecting accessibility, **Then** the panel uses `<section aria-labelledby="suggested-actions-title">` with `role="region"`, and each card uses `<article role="listitem">` with a descriptive `aria-label`.

13. **Given** the panel header renders, **When** displayed, **Then** it shows a `Sparkles` icon in `text-brand`, the title "Suggested Actions" in `text-lg font-semibold`, and subtitle "Topics that need your attention" in `text-sm text-muted-foreground`.

## Dev Notes

### Architecture Compliance

**Component location:** `src/app/components/knowledge/` ŌĆö this directory may not exist yet. Create it. Components in this directory are feature-specific to the Knowledge Map.

**shadcn/ui components used:**
- `Card`, `CardContent` from `@/app/components/ui/card` ŌĆö card container
- `Badge` from `@/app/components/ui/badge` ŌĆö score display, time estimate
- `Button` with `variant="brand"` from `@/app/components/ui/button` ŌĆö CTA buttons (NEVER use `className="bg-brand"` on Button)

**Navigation:** Use `<Link>` from `react-router` for CTA buttons, NOT `useNavigate()`. This makes routes bookmarkable and follows project convention.

### Design Token Enforcement

All colors MUST use design tokens. ESLint `design-tokens/no-hardcoded-colors` will block hardcoded colors at save-time.

| Use Case | Token |
|----------|-------|
| Weak score badge | `bg-destructive/10 text-destructive` |
| Fading score badge | `bg-warning/10 text-warning` |
| Strong score badge | `bg-success/10 text-success` |
| Left border (weak) | `border-l-destructive` |
| Left border (fading) | `border-l-warning` |
| Left border (strong) | `border-l-success` |
| CTA button | `variant="brand"` (not manual `bg-brand`) |
| Panel header icon | `text-brand` |
| Secondary text | `text-muted-foreground` |
| Card background | `bg-card` |
| Card border | `border-border` |

### Responsive Layout Strategy

Mobile-first approach (Tailwind v4):
- **Default (< 640px):** Horizontal scroll with snap ŌĆö `flex overflow-x-auto gap-3 snap-x snap-mandatory`
- **sm (640px+):** 2-column grid ŌĆö `sm:grid sm:grid-cols-2 sm:gap-3 sm:overflow-visible`
- **lg (1024px+):** Vertical stack sidebar ŌĆö `lg:flex lg:flex-col lg:w-80 lg:overflow-visible`

### Accessibility Requirements (WCAG 2.1 AA)

- Panel: `<section aria-labelledby="suggested-actions-title" role="region">`
- Card list: `<div role="list">`
- Each card: `<article role="listitem" aria-label="...">`
- CTA buttons: 44px min height via `h-11` on Button
- Hover animations: `motion-safe:` prefix ŌĆö no animation with `prefers-reduced-motion: reduce`
- Focus visible: `focus-visible:ring-2 focus-visible:ring-brand/50 focus-visible:ring-offset-2`
- Non-color indicators: trend arrows + score numbers supplement color coding
