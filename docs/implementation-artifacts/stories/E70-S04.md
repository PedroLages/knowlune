# Story 70.4: Ambient Scheduling Nudges (Tier 1)

Status: ready-for-dev

## Story

As a learner,
I want to receive a notification when I have free time that's perfect for studying a specific course,
so that I can seize study opportunities I might not have noticed -- without opening any scheduling UI.

## Acceptance Criteria

1. **Given** new EventBus event types are added to `src/lib/eventBus.ts`, **When** the `AppEvent` union is inspected, **Then** it includes `{ type: 'schedule:suggestion-ready'; courseId: string; courseName: string; slotStart: string; durationMinutes: number }` and `{ type: 'schedule:block-created'; courseId: string; courseName: string; slotStart: string }`.

2. **Given** new `NotificationType` values are added to `src/data/types.ts`, **When** the `NotificationType` union is inspected, **Then** it includes `'study-suggestion'` and `'schedule-created'`.

3. **Given** the `SchedulingNudgeService` exists at `src/services/SchedulingNudgeService.ts`, **When** the app mounts (initialized in `App.tsx` alongside `NotificationService`), **Then** it checks if the user has active courses with FSRS cards due or stale progress. **And** computes the single best slot for the highest-priority course using the scheduling engine. **And** if a good slot exists (score > 60), emits `schedule:suggestion-ready` on the EventBus. **And** rate-limits to max 1 suggestion notification per day (same dedup pattern as `hasReviewDueToday`).

4. **Given** `NotificationService` handles the `schedule:suggestion-ready` event, **When** the event is received, **Then** a notification is created using the appropriate copy template: FSRS cards due, stale course, good productivity window, or streak protection. **And** the notification respects quiet hours from `NotificationPreferences`.

5. **Given** a new `studySuggestion` toggle is added to `NotificationPreferences`, **When** the user has `studySuggestion` set to false, **Then** no scheduling nudge notifications are generated. **And** the toggle defaults to true for new users.

6. **Given** the user taps "Schedule it" on an ambient nudge notification, **When** the action is triggered, **Then** a study block is created in the Dexie `studyBlocks` table for the suggested time. **And** a toast confirms "Scheduled! {Course} at {time}". **And** the user navigates to the course page.

7. **Given** no active courses exist or no good slots are found (score < 60), **When** the nudge service evaluates on app mount, **Then** no notification is emitted -- this is normal and logged at debug level only.

### Notification Copy Templates (UX-DR7)

4 variants based on context:

| Variant | Condition | Title | Message |
|---------|-----------|-------|---------|
| FSRS cards due | `fsrsDueCount > 0` | "Time for {courseName}" | "You have {duration} free at {time} -- perfect for reviewing {N} flashcards" |
| Stale course | `daysSinceLastSession > 3` | "Pick up {courseName}" | "It's been {days} days since you studied {courseName}. You have {duration} free at {time}" |
| Productivity window | high productivity score | "Your best focus time" | "Your best focus time is coming up at {time} -- great for {courseName}" |
| Streak protection | streak at risk + packed tomorrow | "Protect your streak" | "Tomorrow is packed -- schedule a 15-min review at {time} to keep your {N}-day streak alive" |

Priority: FSRS > Stale > Streak > Productivity (use first matching)

### Rate Limiting Pattern

Follow existing `hasReviewDueToday` dedup pattern from `NotificationService`:

```typescript
const NUDGE_KEY = 'scheduling-nudge-last-date'
function hasNudgedToday(): boolean {
  const last = localStorage.getItem(NUDGE_KEY)
  return last === new Date().toLocaleDateString('sv-SE')
}
function markNudgedToday(): void {
  localStorage.setItem(NUDGE_KEY, new Date().toLocaleDateString('sv-SE'))
}
```
