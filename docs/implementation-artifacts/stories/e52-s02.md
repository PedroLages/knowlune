# Story 52.02: Quiz Generation UI

## Story

As a learner,
I want a "Generate Quiz" button on the lesson page with clear loading and error states,
so that I can trigger quiz generation with confidence and understand what is happening.

## Acceptance Criteria

**Given** the learner is on a lesson detail page that has a transcript
**When** the page renders
**Then** a "Generate Quiz" button is displayed using `variant="brand"`
**And** the button meets WCAG 2.1 AA (4.5:1 contrast, keyboard navigable, `aria-label`)

**Given** the Ollama server is unavailable (health check fails)
**When** the lesson page renders
**Then** the "Generate Quiz" button is disabled
**And** a tooltip explains "Quiz generation unavailable -- Ollama server is offline"
**And** the disabled state is accessible (aria-disabled, tooltip readable by screen reader)

**Given** the learner clicks "Generate Quiz"
**When** the generation is in progress (15-40 seconds)
**Then** a loading skeleton is displayed with "Generating quiz from transcript..." text
**And** an ARIA live region announces the loading state for screen readers

**Given** quiz generation completes successfully
**When** the quiz is displayed
**Then** it shows an "AI-generated" badge (using `QuizBadge` component)
**And** a Bloom's level picker is available before generation (dropdown or radio group, defaults to Remember)

**Given** quiz generation fails (timeout, malformed output, all chunks failed)
**When** the error is caught
**Then** a sonner toast notification displays a user-friendly error message
**And** the "Generate Quiz" button returns to its ready state

**Given** a quiz was previously generated for this lesson
**When** the lesson page loads and the LLM provider is offline
**Then** the cached quiz loads instantly from Dexie
**And** the "Generate Quiz" button shows "Regenerate" instead
