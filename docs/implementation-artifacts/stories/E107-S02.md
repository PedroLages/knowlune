# Story 107.2: Fix EPUB Reader Rendering

## Story

As a reader,
I want EPUB books to render correctly in the reader with proper sizing, theme support, and responsive behavior,
so that I can read comfortably without visual glitches, overflow, or clipped content.

## Acceptance Criteria

- **AC-1**: EPUB content fills the entire reader viewport without overflow or clipping on all screen sizes (mobile, tablet, desktop)
- **AC-2**: Reader viewport resizes correctly when the browser window is resized (rendition.resize() is called)
- **AC-3**: EPUB iframe background matches the active reader theme (light/sepia/dark) — no white flash on theme change or page turn
- **AC-4**: epub.js uses single-page layout (spread: 'none') to avoid two-page rendering on wide screens
- **AC-5**: Interaction zones (prev/center/next) correctly overlay the full epub content area without gaps

## Tasks / Subtasks

- [x] Task 1: Add ResizeObserver to EpubRenderer for viewport resize handling (AC: 1, 2)
- [x] Task 2: Fix epubOptions to set spread: 'none' for consistent single-page layout (AC: 4)
- [x] Task 3: Apply iframe background color via rendition.themes to match reader theme (AC: 3)
- [x] Task 4: Add CSS to override react-reader default white background (AC: 3)
- [x] Task 5: Add unit tests for resize and theme behavior (AC: 1, 2, 3)
- [x] Task 6: Verify interaction zones fill the full content area (AC: 5)

## Implementation Notes

- **ResizeObserver pattern**: Added a ResizeObserver on the epub container div. When the container size changes, `rendition.resize(width, height)` is called with the new dimensions. This ensures the epub.js iframe always matches the container size.
- **Theme background fix**: Applied the reader theme background directly to the epub container div, and added `rendition.themes.default({ body: { background, color } })` which propagates into the iframe.
- **Spread mode**: Added `spread: 'none'` to `epubOptions` to force single-page rendering on all screen sizes.
- **Container sizing**: The EpubView container now uses explicit `h-full w-full` with `relative` positioning, ensuring the epub.js iframe fills the available space.

## Testing Notes
