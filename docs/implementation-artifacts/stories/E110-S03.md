# Story 110.03: Reading Queue

## Story

As a reader,
I want to add books to a "Read Next" queue and reorder them,
so that I always know what to read next and can maintain a continuous learning flow.

## Acceptance Criteria

- **AC-1**: A "Reading Queue" section is visible on the Library page, showing an ordered list of queued books
- **AC-2**: Users can add any book to the reading queue via the book context menu ("Add to Queue")
- **AC-3**: Users can remove a book from the reading queue via the queue item's context menu or a remove button
- **AC-4**: Users can reorder books in the queue via drag-and-drop
- **AC-5**: The queue persists across sessions via IndexedDB (Dexie migration: `readingQueue` table)
- **AC-6**: The queue shows each book's cover, title, author, and current reading progress
- **AC-7**: Completing a book (100% progress) auto-removes it from the queue and surfaces the next book
