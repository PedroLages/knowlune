# Story 108.5: Genre Detection and Pages Goal

## Story

As a reader,
I want my imported books to be automatically categorized by genre and my daily reading goal to support pages-per-day tracking,
so that I can browse my library by genre and measure reading progress in pages when that suits my reading style better than minutes.

## Acceptance Criteria

- **AC-1**: When a book is imported (EPUB or PDF), the Open Library subjects are automatically mapped to a primary genre stored on the Book record
- **AC-2**: A predefined genre taxonomy is used (Fiction, Non-Fiction, Science, Technology, History, Biography, Fantasy, Mystery, Romance, Self-Help, Philosophy, Science Fiction, Other) — Open Library subjects are matched to the closest genre via keyword mapping
- **AC-3**: Books can be filtered by genre in the Library page via a genre filter dropdown or sidebar section
- **AC-4**: The user can manually override the auto-detected genre from the Book details/metadata editor
- **AC-5**: When the daily reading goal is set to "pages" mode, the Library daily goal ring and streak tracking use pages read per day instead of minutes
- **AC-6**: Page progress is tracked from the EPUB reader (`currentPage` from the reading position) and PDF reader, contributing to the daily pages goal
- **AC-7**: Genre detection runs at import time (not retroactively) — existing books without a genre show "Unset" and can be manually categorized

## Design Guidance

- **Genre filter**: Use shadcn `Select` component in the Library toolbar, positioned next to the search bar
- **Genre badge on cards**: Small text label below the author name on `BookCard` and `BookListItem`, styled with `text-muted-foreground text-xs`
- **Genre in metadata editor**: `Select` dropdown with all genre options + "Unset" option
- **Pages goal ring**: Reuse `DailyGoalRing` component, swap label from "min" to "pages" when `dailyType === 'pages'`

## Implementation Notes

**Key files to modify:**
- `src/data/types.ts` — add `BookGenre` type and `genre` field to `Book`
- `src/services/OpenLibraryService.ts` — already returns `subjects[]`, no change needed
- `src/app/components/library/BookImportDialog.tsx` — wire genre detection after metadata fetch
- `src/app/components/library/BookMetadataEditor.tsx` — add genre override dropdown
- `src/app/pages/Library.tsx` — add genre filter to toolbar
- `src/stores/useReadingGoalStore.ts` — add pages-per-day goal support
- `src/app/components/library/DailyGoalRing.tsx` — support pages display

**New files:**
- `src/services/GenreDetectionService.ts` — genre taxonomy and detection logic
- `src/app/hooks/usePagesReadToday.ts` — compute pages read today from reading sessions

**Existing patterns to follow:**
- `BookMetadataEditor.tsx` — pattern for adding fields to the metadata form
- `DailyGoalRing.tsx` — already handles minutes, extend for pages
- `useReadingGoalStore` — already has `dailyType: 'minutes' | 'pages'` — the 'pages' mode needs wiring

**Genre mapping strategy**: Create a keyword-to-genre map. Each genre has an array of keywords. For each Open Library subject, check if any keyword appears (case-insensitive). The genre with the most keyword matches wins. If no match, default to "Other".

**Pages tracking**: The `Book.currentPosition` already stores page/CFI info. For pages-per-day tracking, compare `currentPage` values from reading sessions within today. The `useBookStore` already tracks position updates.

**Dexie indexing**: Add a `genre` index to the `books` table for efficient filtering. Requires a version bump in `src/data/db.ts`.
