# Story 85.4: Cloze-Deletion Flashcard Creator

Status: done

## Story

As a learner,
I want to create cloze-deletion flashcards by tapping words to blank them within highlighted text,
so that I can turn key passages into active-recall cards that enter my spaced repetition review queue.

## Acceptance Criteria

1. **Given** the user taps the flashcard button in the HighlightPopover or on an existing highlight **When** the ClozeFlashcardCreator opens **Then** it appears as an inline panel below the reader content (reader viewport shrinks to make room) **And** the panel shows: "Create Flashcard" header with close button, instruction text "Tap words to create blanks:", the highlighted text rendered word-by-word as tappable tokens, a real-time preview area, a deck selector, and Cancel/Create Card buttons

2. **Given** the ClozeFlashcardCreator shows word tokens **When** the user taps a word **Then** the word toggles to "blanked" state: `bg-brand text-brand-foreground rounded-md px-1.5 py-0.5` **And** tapping again un-blanks it **And** the preview updates in real-time showing `[___]` for each blanked word **And** each word token has `role="checkbox"` with `aria-checked` and `aria-label="Blank the word '{word}'"` **And** keyboard navigation works: Tab through words, Space/Enter to toggle

3. **Given** the user has blanked at least one word **When** the user taps "Create Card" **Then** a BookHighlight record is created in Dexie (if not already saved from a previous highlight action) **And** a Flashcard record is created with `type: 'cloze'`, `sourceType: 'book'`, `sourceBookId: bookId`, `sourceHighlightId: highlightId`, and the cloze text **And** the flashcard enters the existing FSRS spaced repetition queue **And** the BookHighlight's `flashcardId` field is set to the new flashcard ID **And** the panel closes with `toastSuccess("Flashcard created -- it'll appear in your next review")` **And** a card-fly-up animation (400ms) confirms creation

4. **Given** this is the user's first time using the ClozeFlashcardCreator **When** the word tokens render **Then** the first word gently pulses once to draw attention **And** a one-time tooltip appears: "Tap any word to blank it for your flashcard" (stored in localStorage, shown only once)

5. **Given** no words are blanked **When** the user looks at the Create Card button **Then** the button is disabled with `opacity-50 cursor-not-allowed`

## Dev Notes

### Architecture Constraints

- **Existing Flashcard type**: Book-sourced flashcards go into the SAME `flashcards` Dexie table as course flashcards. The `Flashcard` type is extended with optional `sourceType: 'course' | 'book'`, `sourceBookId`, `sourceHighlightId` fields (architecture decision 6). Do NOT create a separate book-flashcard table.
- **Existing FSRS integration**: Flashcards created here enter the existing `ts-fsrs` spaced repetition queue. They are indistinguishable from course flashcards in the review UI (E85-S05 adds back-navigation context). Use the existing flashcard creation utilities.
- **Cloze text format**: Store the cloze text with `{{c1::word}}` notation (standard cloze format) or `[___]` — check existing flashcard cloze format in the codebase for consistency.
- **Bidirectional linking**: BookHighlight stores `flashcardId` (FK to Flashcard), and Flashcard stores `sourceHighlightId` (FK to BookHighlight). Both must be set atomically.

### Key Dependencies

- HighlightPopover flashcard button (E85-S01)
- HighlightMiniPopover "Create Flashcard" button (E85-S02)
- HighlightListPanel flashcard icon trigger (E85-S03)
- BookHighlight type with `flashcardId` field (E83-S01)
- Existing Flashcard type and FSRS queue (prior epics)
- useHighlightStore for highlight data

### Existing Flashcard Infrastructure to Reuse

Before implementing, scan the codebase for:
- Existing Flashcard type definition in `src/data/types.ts`
- Existing flashcard creation functions/store actions
- FSRS scheduling integration (ts-fsrs)
- Flashcard deck management
- Do NOT reinvent any of these — extend them with the `sourceType: 'book'` field

### Mobile UX (UX-DR19)

- Full-screen bottom sheet on mobile for the cloze creator
- Word tokens should wrap naturally and be large enough for tap targets (`min-h-[44px]` per row)
- Preview area scrollable if text is long

### Accessibility (UX-DR22)

- Word tokens: `role="checkbox"` with `aria-checked` and descriptive `aria-label`
- Keyboard navigation: Tab through words, Space/Enter to toggle
- Focus management: panel receives focus on open, returns to reader on close
- Tooltip: dismissed by Escape key, auto-dismissed after 5 seconds
- `prefers-reduced-motion`: skip pulse animation and card-fly-up
