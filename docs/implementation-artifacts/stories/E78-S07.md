# Story 78.07: @knowlune/plugin-types Package

Status: ready-for-dev

## Story

As a plugin author,
I want a published TypeScript types package for all plugin APIs,
so that I get full autocomplete, type checking, and documentation in my editor.

## Acceptance Criteria

1. **Given** a plugin author adds `@knowlune/plugin-types` as a dev dependency **When** they import types **Then** they have access to: `PluginManifest`, `PluginCapability`, `PluginSlotType`, `PluginDataAPI`, `PluginComponent` interface, and hook type signatures.

2. **Given** the `packages/plugin-types/` directory **When** built **Then** it produces `dist/index.js` and `dist/index.d.ts` with all type exports **And** the package.json declares `name: "@knowlune/plugin-types"`, `types: "dist/index.d.ts"`.

3. **Given** the types package is versioned **When** the plugin system evolves **Then** the types package version is updated in sync with breaking changes **And** plugin manifests declare `minKnowluneVersion` for compatibility.

4. **Given** a plugin author imports from `@knowlune/plugin-types` **When** they define their plugin component **Then** they can type it as `PluginComponent<Props>` and get type errors for incorrect prop shapes.

## Dev Notes

### Architecture Decisions

- **npm workspace package**: Use npm workspaces (not Lerna, not yarn workspaces) since the project already uses npm. Add `"workspaces": ["packages/*"]` to root `package.json`.
- **Type mirroring vs re-export**: The types in `packages/plugin-types/` should be the **canonical** source. `src/plugins/types/index.ts` (from E78-S01) should re-export from the workspace package, OR the two should be kept manually in sync with a note. Re-export is preferred to avoid drift.
- **Build tool**: Use `tsc` directly for building. No bundler needed for a types-only package. Configure `tsconfig.json` with `declaration: true`, `declarationDir: "dist"`.
- **No runtime code**: This package is primarily type definitions. The `dist/index.js` can be empty or contain type guards/validators if useful.

### PluginComponent Interface

```typescript
import type { FC } from 'react'

/**
 * Base interface for all Knowlune plugin components.
 * Plugins export a default component conforming to this type.
 */
export interface PluginComponent<Props = Record<string, never>> extends FC<Props> {
  /** Optional display name for debugging */
  displayName?: string
}
```

### PluginDataAPI Interface

```typescript
export interface PluginDataAPI {
  /** Read plugin-namespaced data by key */
  getPluginData<T = unknown>(key: string): Promise<T | undefined>
  /** Write plugin-namespaced data */
  setPluginData<T = unknown>(key: string, value: T): Promise<void>
  /** Delete plugin-namespaced data */
  deletePluginData(key: string): Promise<void>
}
```
