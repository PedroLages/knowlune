# Story 111.2: Skip Silence and Speed Memory

## Story

As an audiobook listener,
I want silence in audiobooks to be automatically skipped when enabled and my playback speed to be remembered per book,
so that I can listen more efficiently and each book resumes at my preferred speed.

## Acceptance Criteria

- **AC-1**: Given skip silence is enabled in audiobook settings, when the audiobook is playing and a silence segment is detected (audio level below threshold for >500ms), then playback automatically skips past the silence segment
- **AC-2**: Given skip silence is enabled, when a silence segment is skipped, then a subtle visual indicator briefly appears showing the skip occurred (e.g., "Skipped 2.3s silence")
- **AC-3**: Given skip silence is active, when the user disables it in settings, then silence detection stops immediately and playback continues normally
- **AC-4**: Given the skip silence toggle already exists in the audiobook settings panel (E108-S04), when skip silence is toggled, then it is wired to the actual Web Audio API silence detection (replacing the non-functional placeholder)
- **AC-5**: Given a user changes playback speed for a specific book, when playback speed is changed, then the speed is persisted per-book (not globally) in the book's record
- **AC-6**: Given a user returns to a book after listening to a different book, when the audiobook loads, then the previously-set speed for that book is restored automatically
- **AC-7**: Given no per-book speed has been set, when a book is opened for the first time, then it uses the global default speed from audiobook preferences
- **AC-8**: The skip silence and speed memory features are accessible via keyboard, have proper ARIA labels, and do not degrade the listening experience for screen reader users
