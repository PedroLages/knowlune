# Story 85.6: Reading Session Tracking and Streak Integration

Status: done

## Story

As a learner,
I want my reading time automatically tracked and counted toward my study streak,
so that book reading contributes to my overall learning habits alongside course study.

## Acceptance Criteria

1. **Given** the user opens the EPUB reader for a book **When** the reader page loads and the EPUB renders **Then** a reading session starts with a timestamp recorded

2. **Given** the user is actively reading (reader is open and visible) **When** the user closes the reader (back button, navigate away, or close tab) **Then** the reading session ends and a `studySession` record is created in Dexie with: content type 'book', bookId, start/end timestamps, and duration in minutes **And** the session emits a `reading:session-ended` event on the event bus **And** the streak service processes the event and updates the study streak if the reading session meets minimum duration thresholds

3. **Given** the user has read books today **When** they check their streak or overview dashboard **Then** reading time is included in the daily study total alongside course study time

## Tasks / Subtasks

- [ ] Task 1: Start reading session on reader open (AC: 1)
  - [ ] 1.1 In BookReader page, record `sessionStartTime = Date.now()` when the EPUB successfully renders (after `getRendition` callback fires)
  - [ ] 1.2 Store `sessionStartTime` in a ref (not state — no re-render needed)
  - [ ] 1.3 Do NOT start session if EPUB fails to load

- [ ] Task 2: End reading session on reader close (AC: 2)
  - [ ] 2.1 Register cleanup in a `useEffect` cleanup function that fires when BookReader unmounts
  - [ ] 2.2 Also handle `beforeunload` event for tab close/reload
  - [ ] 2.3 Calculate duration: `(Date.now() - sessionStartTime) / 60000` (minutes)
  - [ ] 2.4 Minimum session threshold: only save if duration >= 0.5 minutes (30 seconds) — ignore accidental opens
  - [ ] 2.5 Create `studySession` record in existing Dexie `studySessions` table with:
    - `contentType: 'book'`
    - `contentId: bookId`
    - `startTime: new Date(sessionStartTime).toISOString()`
    - `endTime: new Date().toISOString()`
    - `durationMinutes: duration`

- [ ] Task 3: Emit event and integrate with streak service (AC: 2, 3)
  - [ ] 3.1 Emit `reading:session-ended` event on the event bus (`src/lib/eventBus.ts`) with `{ bookId, durationMinutes }`
  - [ ] 3.2 The existing streak service should already listen for study session events — verify it processes `contentType: 'book'` sessions
  - [ ] 3.3 If streak service filters by content type: add 'book' to the accepted types
  - [ ] 3.4 Reading sessions count toward the daily study total alongside course study time

- [ ] Task 4: Handle edge cases (AC: 2)
  - [ ] 4.1 Tab visibility change: if user switches away for >5 minutes, pause the session timer (use `document.visibilitychange` event)
  - [ ] 4.2 Multiple books: if user navigates from one book to another, end the first session before starting the second
  - [ ] 4.3 If Dexie write fails: log error, do NOT show toast (session tracking is non-critical to the reading experience)

## Dev Notes

### Architecture Constraints

- **Reuse existing `studySessions` table**: Reading sessions go in the SAME Dexie table as course study sessions (architecture decision 1). The `contentType` field distinguishes book sessions from course sessions. Do NOT create a separate reading-sessions table.
- **Event bus pattern**: Use `reading:session-ended` event name — kebab-case with namespace prefix, consistent with existing events in `src/lib/eventBus.ts`. Check existing event names for exact pattern.
- **Streak service integration**: The existing streak service processes study session events. Verify how it's wired — it may listen on the event bus or poll the Dexie table. Ensure `contentType: 'book'` sessions are counted.
- **`beforeunload` for tab close**: This is the standard pattern for saving session data on tab close. Use a synchronous Dexie write or `navigator.sendBeacon` pattern if async writes don't complete in time.

### Key Dependencies

- BookReader page (E84-S01) — add session tracking logic
- Existing `studySessions` Dexie table (from prior epics)
- Existing streak service (from prior epics — find in codebase)

### Codebase Investigation Required

### Visibility API for Idle Detection

Use `document.addEventListener('visibilitychange', ...)`:
- When page becomes hidden: record pause time
- When page becomes visible: if paused > 5 minutes, subtract idle time from session
- This prevents inflated reading times from forgotten tabs

### Error Handling

- Dexie write failure: log error only, do NOT disrupt reading experience with toast
- `beforeunload`: keep the handler lightweight — Dexie operations in `beforeunload` may not complete. Consider writing session data periodically (every 5 minutes) as a safety net.
- Zero-duration sessions (<30 seconds): discard silently
