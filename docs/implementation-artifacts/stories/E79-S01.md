# Story 79.01: Custom Page Route Injection

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a plugin author,
I want to register custom page routes from my plugin manifest,
so that my plugin can provide full-page experiences within Knowlune.

## Acceptance Criteria

1. **Given** a plugin manifest declares `slots: ["custom-page"]` and `capabilities: ["add:routes"]`, **When** the plugin is activated, **Then** the declared route is registered in React Router and accessible via URL, and the route is lazy-loaded via `React.lazy` + `Suspense` (same pattern as PluginSlot).

2. **Given** a plugin declares route `/plugins/study-timer`, **When** a user navigates to that URL, **Then** the plugin's page component renders within the app Layout (sidebar + header remain visible), and the route is wrapped in an error boundary.

3. **Given** a plugin with a custom route is deactivated, **When** a user navigates to the plugin's route, **Then** a 404-style fallback is shown: "This plugin is currently disabled".

## Dev Notes

### Architecture Decisions

- **Route injection strategy**: React Router v7 uses `createBrowserRouter` with static route config. Plugin routes must be injected at router creation time or via a dynamic re-render pattern. Since `createBrowserRouter` is called once, the recommended approach is to use a wrapper component inside the Layout that renders plugin routes via an `<Outlet>`-like pattern, or to recreate the router when plugins change (consider performance implications).
- **Alternative approach**: Use a catch-all route `plugins/*` inside the Layout children that delegates to PluginRouteInjector, which internally matches sub-paths. This avoids router recreation.
- **Error isolation**: Each plugin page route MUST be wrapped in PluginErrorBoundary (from E78-S04) so a crashing plugin page does not take down the app.
- **Lazy loading**: Plugin page components are loaded via dynamic `import()` from the plugin's registered module path. Use the same `React.lazy(() => import(...))` pattern as existing routes.

### Dependencies (from E78)

- `PluginManager` singleton (E78-S02) for querying active plugins and their manifests
- `PluginErrorBoundary` (E78-S04) for error isolation
- EventBus plugin lifecycle events: `plugin:activated`, `plugin:deactivated` (E78-S05)
- Plugin manifest types and ajv validation (E78-S01)

- Lazy loading: `React.lazy(() => import('./pages/X').then(m => ({ default: m.X })))` pattern
- Navigation items: `src/app/config/navigation.ts` exports `NavigationItem` interface with `name`, `path`, `icon` -- plugin sidebar items (E79-S02) will extend this
