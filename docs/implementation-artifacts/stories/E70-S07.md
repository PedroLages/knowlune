# Story 70.7: Weekly Planner View with Auto-Fill (Tier 3)

Status: ready-for-dev

## Story

As a power-scheduling learner,
I want a weekly planning view where I can auto-fill my week with optimized study sessions and adjust as needed,
so that I can batch-plan my entire study week in under 30 seconds.

## Acceptance Criteria

1. **Given** the `WeekPlannerView` component exists at `src/app/components/scheduling/WeekPlannerView.tsx`, **When** accessed from "Plan my week" button in Calendar Settings, **Then** it renders a 7-day grid (Mon-Sun) with time slots from 7am to 11pm. **And** existing E50 study blocks appear as non-draggable gray blocks. **And** the planner uses `role="grid"` with `role="row"` and `role="gridcell"`. **And** on mobile (<640px): scrollable horizontal day view (1 day at a time with swipe navigation and day toggle). **And** on tablet (640-1023px): compressed 7-column grid with abbreviated day labels. **And** on desktop (1024px+): full-width 7-column grid with time labels.

2. **Given** the user clicks "Auto-fill" button (`variant="brand"`), **When** the `AutoFillDialog` is presented, **Then** it lists active courses with urgency indicators and confirms with "Fill my week" (brand) and "Cancel" (outline). **And** has `aria-label="Auto-fill your study week"`. **And** on confirmation, calls `useWeekPlan.generate()`. **And** allocations appear on the grid with slide-in animation (respects `prefers-reduced-motion`). **And** auto-filled allocations are brand-colored and draggable.

3. **Given** auto-filled allocations are displayed on the grid, **When** the user drags an allocation block (`WeekPlannerAllocation` component), **Then** the block snaps to a 15-min grid and re-validates fit. **And** keyboard users can grab with Space, move with arrow keys, drop with Space, cancel with Escape. **And** draggable blocks have `aria-grabbed` and `aria-dropeffect` attributes. **And** a screen reader announcement fires: "{course} moved to {day} at {time}". **And** a toast confirms "{Course} moved to {day} at {time}" (2s).

4. **Given** the summary panel below the grid, **When** auto-fill is complete, **Then** it shows: total sessions, total hours, courses included, and any unscheduled courses with reasons. **And** action buttons: "Clear" (outline), "Adjust manually" (brand-outline), "Approve all" (brand).

5. **Given** the user clicks "Approve all", **When** the action fires, **Then** all allocations are written to Dexie `studyBlocks` table. **And** a toast confirms "Week plan approved -- {N} sessions scheduled" (3s). **And** all blocks become solid (same style as existing E50 blocks).

6. **Given** the user clicks "Clear", **When** the action fires, **Then** all auto-filled blocks are removed (manual E50 blocks remain). **And** a toast confirms "Auto-fill cleared" (2s).

7. **Given** the user clicks "Adjust manually", **When** the action fires, **Then** the E50-S05 schedule editor opens for direct manipulation.

8. **Given** the planner grid has no active courses, **When** the view renders, **Then** the empty state shows "No active courses to schedule. Start a course to use the planner." with a "Browse courses" link.

9. **Given** the week is fully packed (no available windows), **When** auto-fill is attempted, **Then** the message shows "No available windows found. Consider shorter sessions or removing some commitments." with a "Try 15-min sessions" button.

## Dev Notes

### Design Guidance (UX-DR4 through UX-DR6, UX-DR8, UX-DR9)

**WeekPlannerView (UX-DR4):**
- 7-day grid (Mon-Sun), 7am-11pm time slots
- Existing E50 blocks: non-draggable, `bg-muted` (gray), lower opacity
- Auto-filled allocations: draggable, `bg-brand-soft` with `text-brand-soft-foreground`
- Use CSS Grid: `grid-template-columns: auto repeat(7, 1fr)` for desktop
- Time labels in first column

**AutoFillDialog (UX-DR5):**
- Use shadcn `AlertDialog` component
- Course list with urgency indicators using `Badge` component
- Two buttons: "Fill my week" (`variant="brand"`), "Cancel" (`variant="outline"`)

**WeekPlannerAllocation (UX-DR6):**
- Draggable card: course name, duration badge, session type badge
- Snap to 15-min grid increments
- CSS: `height` proportional to duration (e.g., 30min = 2 grid rows if each row is 15min)
- Consider using native HTML Drag and Drop API or a lightweight library (check existing deps first)

### Responsive Design Strategy

| Breakpoint | Layout | Navigation |
|------------|--------|------------|
| Mobile <640px | 1 day visible, full height | Swipe left/right or day toggle buttons |
| Tablet 640-1023px | 7 columns compressed | Scroll within each column |
| Desktop 1024px+ | 7 columns full width | Scroll vertical for time slots |

### Drag-and-Drop Implementation

For keyboard accessibility, implement custom drag logic:
1. Space on focused allocation: enter drag mode (`aria-grabbed="true"`)
2. Arrow keys: move allocation preview (up/down = time, left/right = day)
3. Space: drop at new position, re-validate fit
4. Escape: cancel drag, return to original position
5. Live region announces position changes for screen readers

### Toast Feedback (UX-DR9)

| Action | Message | Duration |
|--------|---------|----------|
| Block moved | "{Course} moved to {day} at {time}" | 2s |
| Week plan approved | "Week plan approved -- {N} sessions scheduled" | 3s |
| Auto-fill cleared | "Auto-fill cleared" | 2s |

### Design Token Usage

- Auto-filled blocks: `bg-brand-soft`, `text-brand-soft-foreground`, `border-brand`
- Existing E50 blocks: `bg-muted`, `text-muted-foreground`
- Grid lines: `border-border`
- Time labels: `text-muted-foreground`
- "Approve all" button: `variant="brand"`
- "Adjust manually": `variant="brand-outline"`
- "Clear": `variant="outline"`
- Never use hardcoded colors

### Accessibility Requirements

- `role="grid"`, `role="row"`, `role="gridcell"` on the planner
- `aria-grabbed` and `aria-dropeffect` on draggable allocations
- `aria-live="polite"` region for screen reader announcements
- `aria-label="Auto-fill your study week"` on dialog
- Focus management: focus returns to trigger button when dialog closes
- Touch targets >= 44x44px for mobile day toggle buttons
