# Story 84.1: EPUB Reader with epub.js Integration

Status: done

## Story

As a learner,
I want to open a book and read it in a paginated in-browser reader,
so that I can read EPUB books directly in Knowlune without needing a separate reading app.

## Acceptance Criteria

1. **Given** the user clicks a book on the Library page **When** the BookReader page loads at `/library/{bookId}` **Then** the EPUB file is loaded from OPFS via `BookContentService.getEpubContent()` and rendered using `ReactReader` (react-reader wrapper around epub.js) **And** the reader uses a full-screen layout with the sidebar hidden for immersive reading **And** the reader shows: a header bar with back button, book title, chapter indicator, and menu button; the epub.js content area; a footer with progress bar and page/chapter indicator **And** epub.js uses the default (single-section) manager for performance **And** the first page renders in <500ms (NFR1) **And** a loading skeleton with centered spinner and "Loading book..." text displays while the EPUB loads **And** if the EPUB fails to load, an error boundary displays with a retry option and toast.error

2. **Given** the reader is displaying EPUB content **When** the user is idle for 3 seconds during reading **Then** the header and footer auto-hide **And** tapping/clicking the center zone (34% width) toggles header/footer visibility

## Dev Notes

### Architecture Constraints

- **react-reader** is the chosen epub.js wrapper per architecture decision 5. It exposes the underlying `Rendition` object via `getRendition` callback. Do NOT use epub.js directly — the wrapper handles React lifecycle.
- **Code-splitting is mandatory** (architecture decision 12): `EpubRenderer` MUST be lazy-loaded via `React.lazy()`. epub.js + react-reader add ~100KB gzipped — this must NOT be in the initial bundle.
- **BookReader page hides the sidebar** — this is a full-viewport layout, NOT nested inside the standard `Layout` component. The reader has its own header/footer. Check how existing pages like LessonPlayer handle full-screen mode if available.
- **BookContentService** (from E83) provides `getEpubContent(book: Book): Promise<ArrayBuffer>`. It abstracts OPFS/IDB/remote source reading. The EpubRenderer receives a Blob URL created from this ArrayBuffer.
- **Design tokens only** — never hardcode Tailwind colors. The reader chrome uses theme-aware backgrounds but these are reader-specific hex values (`#FAF5EE`, `#F4ECD8`, `#1a1a1a`) applied to the epub.js iframe, not to Knowlune UI elements. Knowlune UI elements around the reader must use design tokens (`bg-popover`, `text-foreground`, etc.).

### Key Dependencies (from E83)

This story depends on E83 being complete. It requires:
- `Book` type and `useBookStore` (E83-S01)
- `BookContentService.getEpubContent()` (E83-S01)
- `OpfsStorageService.readBookFile()` (E83-S01)
- `/library` route and at least one importable book (E83-S02)

### Performance Requirements

- First page render <500ms on modern hardware (NFR1)
- epub.js default (single-section) manager — do NOT use continuous scroll manager
- Lazy-load react-reader via dynamic import — zero initial bundle impact
- Total Phase 1-2 bundle addition must stay <150KB gzipped (NFR17)

### Accessibility (UX-DR22)

- Keyboard navigation: Tab to move through header controls, Escape to go back
- `aria-label` on back button, menu button, and navigation zones
- `prefers-reduced-motion`: instant transitions instead of fade animations
- Focus management: when reader loads, focus the content area

### Browser Compatibility

- Chrome 86+, Firefox 111+, Safari 15.2+ (NFR21)
- epub.js is well-supported across these browsers

### Project Structure Notes

- Route pattern: `{ path: 'library/:bookId', element: <BookReader /> }` — matches existing route patterns in `src/app/routes.tsx`
- Store pattern: follows `use[Domain]Store` naming (e.g., `useReaderStore`) consistent with `useNotificationPrefsStore`
- Component folder: `src/app/components/reader/` — new folder for all reader-related components (architecture decision 4)
