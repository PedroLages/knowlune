# Story 106.2: Lib & Service Coverage — Utilities and Services

## Story

As a developer,
I want unit tests for low-coverage lib utilities and services,
So that shared logic is protected against regressions.

## Acceptance Criteria

**AC1: Lib test files created**
**Given** the 9 target lib files have coverage below 30%
**When** unit tests are written for each file's exported functions
**Then** test files exist at `src/lib/__tests__/` for each target

**AC2: Service test files created**
**Given** the 3 target services have coverage below 50%
**When** unit tests are written for each service's public API
**Then** test files exist at `src/services/__tests__/` for each target

**AC3: Coverage targets met**
**Given** the new tests are written
**When** `npm run test:unit -- --coverage` is executed
**Then** each of the 12 files reaches at least 60% statement coverage

**AC4: Suite passes**
**Given** all new tests are added
**When** `npm run test:unit` is executed
**Then** zero test failures reported

**AC5: Test-only changes**
**Given** this is a coverage improvement story
**When** all changes are reviewed
**Then** no production source files are modified (test files only)
