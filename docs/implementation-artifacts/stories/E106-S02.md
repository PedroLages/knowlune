## Challenges and Lessons Learned

- **3 of 9 lib files in the story spec did not exist** (`guidGenerator.ts`, `autoResolver.ts`, `obfuscationPiercing.ts`). Substituted `notificationPiercing.ts` (closest name match at 30% coverage) and skipped the missing ones. Lesson: verify target file existence before writing story specs.
- **2 service names differed from spec** (`StorageService` -> `OpfsStorageService`, `StatsService` -> `ReadingStatsService`). The actual files had more specific names. Lesson: use exact filenames from `coverage-final.json`, not abbreviated names.
- **Canvas/Image APIs not available in jsdom** — `avatarUpload.ts` functions like `loadImageToCanvas`, `canvasToBlob`, `compressAvatar`, `cropImage` require real browser DOM. Statement coverage limited to 35% for this file. The pure functions (validate, color, crop region) are fully tested.
- **WebSocket mocking requires `function` keyword** — `vi.fn(() => obj)` doesn't work as a constructor mock for `new AudioContext()` or `new WebSocket()`. Must use `vi.fn(function(this) { Object.assign(this, ...) })` pattern.
- **Pre-existing test bug fixed** — `AudiobookshelfService.test.ts` `fetchCollections` test asserted `result.data.length` but the actual API returns `{ results, total }`. Fixed to use `result.data.results`.
- **Overall statement coverage**: 57.07% -> 60.31% (3.24pp increase from this story alone).
