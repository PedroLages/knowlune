# Story 70.5: Find Study Time Button and Slot List (Tier 2)

Status: ready-for-dev

## Story

As a learner,
I want to click a "Find study time" button on a course page and see ranked time slot suggestions,
so that I can quickly schedule a study session for a specific course at an optimal time.

## Acceptance Criteria

1. **Given** the `FindStudyTimeButton` component exists at `src/app/components/figma/FindStudyTimeButton.tsx`, **When** rendered on the course detail page, **Then** it appears next to the "Continue Learning" button with a Clock icon and "Find study time" text. **And** uses `variant="brand-outline"`. **And** has `aria-label="Find study time for {courseName}"`. **And** clicking it opens the `SuggestionSlotList` sheet.

2. **Given** the `SuggestionSlotList` component exists at `src/app/components/scheduling/SuggestionSlotList.tsx`, **When** opened, **Then** it renders as a Sheet (bottom slide-up on mobile <640px, right slide on desktop 1024px+). **And** the header shows "Find study time for {courseName}" and "Next 7 days - Based on your schedule + habits". **And** it calls `useStudySuggestions(courseId, defaultDuration)` on mount. **And** shows loading skeleton (3 pulsing card placeholders) while computing. **And** has `aria-label="Study time suggestions for {courseName}"` and focus trapped within the sheet.

3. **Given** 3-5 suggestions are returned, **When** the slot list renders, **Then** each slot is rendered as a `SuggestionSlotCard`. **And** the top slot (rank 0) has `bg-brand-soft` background, "Best match" badge with `aria-label="Recommended -- best match"`, and `variant="brand"` Schedule button. **And** other slots have `bg-card` background and `variant="brand-outline"` Schedule buttons. **And** each card shows: day/date/time, available duration, 1-2 reasoning lines (truncated with tooltip), and session type badge (Review=warning, New content=success, Mixed=brand-soft). **And** the slot list uses `role="list"` with `role="listitem"` children.

4. **Given** the user clicks "Schedule" on a slot, **When** the action fires, **Then** a study block is written to Dexie `studyBlocks` table for the selected time. **And** the slot card animates out (fade + slide, respects `prefers-reduced-motion`). **And** remaining slots shift up. **And** a toast appears: "Scheduled! {Course} at {time} {day}". **And** the user can schedule additional slots or close the sheet.

5. **Given** 1-2 suggestions are returned, **When** the slot list renders, **Then** available slots are shown with the message "Your week is tight -- these are your best options".

6. **Given** 0 suggestions are returned, **When** the slot list renders, **Then** the empty state shows "No available windows in the next 7 days that fit a {duration}-min session." **And** a "Try a shorter session?" button retries computation at 15 minutes.

7. **Given** Google Calendar data is stale (>24h old), **When** the slot list renders, **Then** a warning banner appears at the bottom: "Calendar data last synced {N}h ago. Some time slots may have changed." using `text-warning` and `role="alert"`.

8. **Given** the Schedule button on each card, **When** it receives keyboard focus, **Then** it has `aria-label="Schedule {courseName} at {time} on {day}"`. **And** the Sheet can be closed with Escape key.

## Dev Notes

### Design Guidance (UX-DR1 through UX-DR3)

**FindStudyTimeButton (UX-DR1):**
- Clock icon (`lucide-react` Clock) + "Find study time" text
- `variant="brand-outline"` (secondary action paired with primary "Continue Learning")
- Placed adjacent to "Continue Learning" button on course detail page

**SuggestionSlotList (UX-DR2):**
- Uses shadcn `Sheet` component
- Mobile (<640px): `side="bottom"` (slide-up bottom sheet)
- Desktop (1024px+): `side="right"` (slide-right side sheet)
- Use `useMediaQuery` or Tailwind responsive to switch `side` prop
- Loading skeleton: 3 `Skeleton` components mimicking card shape

**SuggestionSlotCard (UX-DR3):**
- Top slot differentiation: `bg-brand-soft` + `text-brand-soft-foreground` (not `text-brand` -- contrast rule)
- "Best match" uses `Badge` component
- Session type badges: Review=`variant` with warning color, New content=success, Mixed=brand-soft
- Reasoning lines: max 2 lines, use `line-clamp-2` with `Tooltip` for full text
- Schedule button triggers `useSchedulingStore.accept()` which writes to Dexie

### Component Hierarchy

```
FindStudyTimeButton
  └── Sheet (SuggestionSlotList)
        ├── SheetHeader (title + subtitle)
        ├── ScrollArea
        │   └── div[role="list"]
        │       ├── SuggestionSlotCard (rank 0, bg-brand-soft)
        │       ├── SuggestionSlotCard (rank 1)
        │       └── SuggestionSlotCard (rank N)
        ├── StaleCalendarWarning (conditional, role="alert")
        └── EmptyState (conditional)
```

### Animation Pattern

Slot card exit animation on schedule:
```css
/* Respect reduced motion */
@media (prefers-reduced-motion: no-preference) {
  .slot-exit { animation: slotExit 300ms ease-out forwards; }
}
@keyframes slotExit {
  to { opacity: 0; transform: translateX(100px); height: 0; margin: 0; padding: 0; }
}
```

Use Tailwind `motion-safe:` prefix or CSS media query. Consider `tw-animate-css` utilities.

### Toast Feedback (UX-DR9)

- Schedule confirmation: "Scheduled! {Course} at {time} {day}" (3s duration)
- Use `toast.success()` from Sonner

### Design Token Usage

- Top slot background: `bg-brand-soft`
- Top slot text: `text-brand-soft-foreground` (NOT `text-brand`)
- Other slot background: `bg-card`
- Warning banner: `text-warning`
- Session badges: use semantic colors (warning, success, brand-soft)
- Never use hardcoded colors (ESLint enforced)

### Accessibility Requirements

- `role="list"` on container, `role="listitem"` on each card
- `aria-label` on Sheet, "Best match" badge, and each Schedule button
- Focus trapped within Sheet when open
- Escape key closes Sheet (built into shadcn Sheet)
- Touch targets >= 44x44px on mobile for Schedule buttons
