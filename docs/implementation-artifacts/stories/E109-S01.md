# Story E109-S01: Vocabulary Builder

## Story

As a reader,
I want to save words and phrases from my EPUB reader to a personal vocabulary list,
so that I can review and master unfamiliar terms encountered during reading.

## Acceptance Criteria

- **AC-1**: From a text selection in the EPUB reader, the user can tap "Add to Vocabulary" in the highlight popover to save the selected word/phrase with context
- **AC-2**: A dedicated /vocabulary page lists all saved vocabulary items with word, definition placeholder, source book, and mastery level
- **AC-3**: The vocabulary page shows an empty state when no items exist
- **AC-4**: Items can be edited inline (word and definition) and deleted with undo support
- **AC-5**: Mastery level (New → Learning → Familiar → Mastered) can be advanced or reset
- **AC-6**: The vocabulary page is accessible via the sidebar navigation
- **AC-7**: Flashcard-style review mode allows cycling through vocabulary items

## Challenges and Lessons Learned

- **Dexie schema migration discipline**: Adding `vocabularyItems` in v42 required updating the schema checkpoint test. The hardcoded version and table count in `schema.test.ts` should be kept in sync with every migration — best enforced by running unit tests before review.
- **Optimistic update consistency**: `updateItem`, `advanceMastery`, and `resetMastery` initially lacked the try/catch + rollback pattern used by `addItem`/`deleteItem`. Applying the pattern uniformly across all mutating store actions prevents silent state divergence on Dexie failures.
- **`now()` helper for store timestamps**: Multiple call sites using `new Date().toISOString()` were consolidated into a shared `now()` helper, improving consistency and making the pattern testable.
- **EditDialog shadcn migration**: Initial implementation used a custom overlay div. Migrating to shadcn `<Dialog>` added focus trapping, Escape-to-close, and scroll-lock for free — worth the refactor cost.
- **Shared E2E helper**: `tests/helpers/dismiss-onboarding.ts` extracted from repeated inline logic — reusable across subsequent E109 stories.
