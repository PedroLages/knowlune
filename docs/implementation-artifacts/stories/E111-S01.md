# Story 111.1: Audio Clips

## Story

As an audiobook listener,
I want to clip moments from audiobooks by marking start and end times,
so that I can save, replay, and manage memorable passages or important sections.

## Acceptance Criteria

- **AC-1**: Given an audiobook is playing, when the user taps a "Start Clip" button, then the current playback time is captured as the clip start time and a visual indicator shows recording is in progress
- **AC-2**: Given a clip is being recorded, when the user taps "End Clip", then the clip is saved to the `audioClips` Dexie table with start time, end time, book ID, chapter ID, and a timestamp
- **AC-3**: Given an audiobook is open, when the user opens the clips panel, then all clips for that book are listed in chronological order showing start/end times, chapter name, and optional title
- **AC-4**: Given a list of clips, when the user taps a clip, then playback jumps to the clip start time and plays through to the clip end time
- **AC-5**: Given a list of clips, when the user edits a clip, then they can update the clip title/label
- **AC-6**: Given a list of clips, when the user deletes a clip, then the clip is removed from the database and the list updates immediately
- **AC-7**: Given multiple clips exist, when the user reorders them via drag-and-drop, then the new order persists across sessions (using @dnd-kit)
- **AC-8**: The audio clips feature is accessible via keyboard, has proper ARIA labels, and touch targets meet 44x44px minimum
