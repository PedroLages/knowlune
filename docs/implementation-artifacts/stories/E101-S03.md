# Story 101.3: Library Browsing & Catalog Sync

## Story

As a Knowlune user with a connected Audiobookshelf server,
I want to browse my audiobook library with covers and metadata,
so that I can find and select audiobooks to listen to without leaving Knowlune.

**FRs:** FR8, FR9, FR10, FR11, FR12, FR13, FR14
**NFRs:** NFR1 (catalog fetch <1s), NFR4 (covers don't block rendering), NFR11 (screen reader support), NFR17 (graceful offline)

## Acceptance Criteria

**Given** a connected Audiobookshelf server with selected libraries
**When** the Library page loads
**Then** the system fetches the catalog from ABS using `AudiobookshelfService.fetchLibraryItems()`
**And** creates or updates Book records in Dexie via `useBookStore` with `source: { type: 'remote', url, auth }`
**And** deduplicates by matching `absServerId + absItemId` on existing books
**And** the catalog fetch completes in less than 1 second for 50 items on LAN (NFR1)

**Given** audiobooks are loaded from ABS
**When** they appear in the Library grid
**Then** each book card shows: cover image, title, author, narrator, duration, chapter count
**And** ABS books show a subtle server badge icon distinguishing them from local books

**Given** the Library page is rendered with ABS books
**When** the page loads
**Then** cover images lazy-load from ABS server URLs (`/api/items/{id}/cover`)
**And** cover loading does not block the grid rendering — placeholder skeletons show first (NFR4)
**And** all cover images have ARIA alt text with the book title (NFR11)

**Given** the Library page has both local and ABS books
**When** source filter tabs are visible
**Then** three tabs appear: "All" (default), "Local", "Audiobookshelf"
**And** selecting "Audiobookshelf" shows only books with `source.type === 'remote'`
**And** selecting "Local" shows only books with `source.type === 'local'` or `source.type === 'fileHandle'`

**Given** the user types in the Library search bar
**When** the search query is submitted
**Then** results include both local and ABS books matching by title, author, or narrator (FR10)
**And** the search respects the current source filter tab

**Given** a large ABS library with 200+ audiobooks
**When** the user scrolls to the bottom of the grid
**Then** the next page of results loads (pagination at 50 items per page) (FR14)
**And** a loading indicator appears during page fetch

**Given** the ABS server becomes unreachable after initial sync
**When** the user opens the Library page
**Then** cached book metadata from the last sync is displayed (FR13)
**And** the ABS server card shows "Offline" status
**And** cover images that were previously loaded remain visible from browser cache
**And** a subtle toast informs: "Audiobookshelf server is offline. Showing cached library."

**Given** book metadata displayed in the Library
**When** a screen reader reads the book cards
**Then** each card has proper ARIA labels: "Book: {title} by {author}, narrated by {narrator}, {duration}" (NFR11)

### ABS Book → Book Type Mapping

The `AbsLibraryItem` shape (from Audiobookshelf API):
```typescript
// From ABS GET /api/libraries/{id}/items response
interface AbsLibraryItem {
  id: string                        // ABS item ID
  media: {
    metadata: {
      title: string
      authors: { id: string; name: string }[]
      narrators?: { id?: string; name: string }[] | string[]   // NOTE: older ABS versions may return string[]
      series?: { id: string; name: string; sequence?: string }[] | null
      duration?: number             // total seconds
      description?: string
      isbn?: string
    }
    chapters?: { id: number; start: number; end: number; title: string }[]
    duration?: number               // may also be at media level
    coverPath?: string
  }
  libraryId: string
  addedAt: number                   // Unix timestamp (ms)
  updatedAt: number                 // Unix timestamp (ms)
}
```

**Key mapping decisions:**
- `narrator`: handle both `narrators: string[]` (old ABS) and `narrators: {name: string}[]` (new ABS) — use type guard
- `chapters`: map `{ id: String(ch.id), bookId: book.id, title: ch.title, order: index, position: { type: 'time', seconds: ch.start } }` — use the chapter `start` time as the position
- `coverUrl`: `AudiobookshelfService.getCoverUrl(server.url, absItem.id) + '?token=' + encodeURIComponent(server.apiKey)` — token via query param because `<img>` cannot set Authorization header
- `source.url`: `${server.url}/api/items/${absItem.id}` — base item URL (not the streaming URL — that's constructed per-chapter in S04)

### Source Type Filtering

`ContentSource` in `types.ts` currently has three variants:
- `{ type: 'local'; opfsPath: string }` — OPFS-stored files
- `{ type: 'remote'; url: string; auth?: { username: string; password: string } }` — remote (OPDS + ABS)
- `{ type: 'fileHandle'; handle: FileSystemFileHandle }` — FileSystem Access API

For the source filter:
- "Local" tab = `source.type === 'local' || source.type === 'fileHandle'`
- "Audiobookshelf" tab = `source.type === 'remote' && book.absServerId != null`

**Note:** OPDS books also have `source.type === 'remote'`. The `absServerId` field distinguishes ABS books from OPDS books. The "Audiobookshelf" source tab should filter to `source.type === 'remote' && !!book.absServerId` to avoid showing OPDS catalog books in the ABS tab.

### `auth` Field on `ContentSource.remote`

Currently `ContentSource.remote` has `auth?: { username: string; password: string }` (Basic auth for OPDS). For ABS, the Bearer token is not stored in `source.auth` — it's in `useAudiobookshelfStore(server.apiKey)`. When the streaming URL is needed, `useAudioPlayer` will call `AudiobookshelfService.getStreamUrl(server.url, itemId, server.apiKey)` using the `absServerId` to look up the server. Do NOT put the API key in `source.auth` — that field is shaped for Basic auth only.

### Preserving Progress on Re-sync

`upsertAbsBook` must preserve user progress when re-syncing:
```typescript
const existing = get().books.find(b => b.absServerId === book.absServerId && b.absItemId === book.absItemId)
if (existing) {
  return {
    ...book,
    id: existing.id,          // preserve stable UUID
    status: existing.status,  // preserve reading status
    progress: existing.progress,
    currentPosition: existing.currentPosition,
    lastOpenedAt: existing.lastOpenedAt,
    createdAt: existing.createdAt,
    updatedAt: new Date().toISOString(),
  }
}
```

### Performance: Sub-1s Catalog Fetch (NFR1)

- Fetch all pages in parallel if multiple libraries selected: `Promise.all(server.libraryIds.map(libId => fetchLibraryItems(..., libId, { page: 0, limit: 50 })))`
- Do NOT await each library sequentially
- The 1s target is for LAN — assume direct network access. No optimization beyond parallelism needed.

### Offline Degradation (NFR17)

When sync fails (service returns `{ ok: false }`):
1. Check if failure is network error (server unreachable) vs auth error (401)
2. Network error → `toast.warning('Audiobookshelf server is offline. Showing cached library.')` + update server status to `'offline'` via `useAudiobookshelfStore.updateServer(id, { status: 'offline' })`
3. Auth error → update server status to `'auth-failed'` — toast message from S01/S02
4. Books already in Dexie from previous sync remain in `useBookStore.books` — no deletion on sync failure

### Search Across Local + ABS (FR10)

Update `getFilteredBooks` search to include `narrator`:
```typescript
result = result.filter(b =>
  b.title.toLowerCase().includes(q) ||
  b.author.toLowerCase().includes(q) ||
  (b.narrator?.toLowerCase().includes(q) ?? false)
)
```
Search is applied AFTER source filter — so "Audiobookshelf" tab + search = search within ABS books only.

## Challenges and Lessons Learned

- **IndexedDB seeding for new tables**: The `audiobookshelfServers` table needed to exist in the Dexie schema before E2E tests could seed it. The seed helper retries up to 10 times, but the table must be registered in the schema migration. This pattern is consistent with how other stores (books, courses) are seeded.

- **Source filter ordering matters**: Applying source filter before status filter in `getFilteredBooks` ensures that status pill counts reflect the currently selected source tab. Without this ordering, counts would show totals across all sources, confusing users who filtered to "Audiobookshelf" but saw counts including local books.

- **Narrator field handling for ABS API variants**: Audiobookshelf API returns narrators as either `string[]` (older versions) or `{name: string}[]` (newer versions). A type guard normalizes both formats into a single string, preventing runtime errors when mapping ABS items to the Book type.

- **Stale useMemo dependency in Library page**: The initial implementation had a stale closure in the `useMemo` for filtered books. Adding the correct dependencies to the memo (including `filters.source`) fixed the issue where changing source tabs did not update the displayed books.

- **Cover URL authentication via query param**: Since `<img>` elements cannot set Authorization headers, ABS cover URLs append `?token=` as a query parameter. This is a common pattern for image authentication in web apps and avoids the complexity of a service worker proxy.
