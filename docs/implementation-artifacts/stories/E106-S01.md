# Story 106.1: Store Coverage — Low-Coverage Zustand Stores

## Story

As a developer,
I want unit tests for the 6 lowest-coverage Zustand stores,
So that store business logic is protected against regressions.

## Acceptance Criteria

**AC1: Store test files created**
**Given** the 6 target stores have coverage below 40%
**When** unit tests are written for each store's primary actions
**Then** test files exist at `src/stores/__tests__/` for each target store

**AC2: Coverage targets met**
**Given** the new tests are written
**When** `npm run test:unit -- --coverage` is executed
**Then** each of the 6 stores reaches at least 70% statement coverage

**AC3: Suite passes**
**Given** all new tests are added
**When** `npm run test:unit` is executed
**Then** zero test failures reported

**AC4: Test-only changes**
**Given** this is a coverage improvement story
**When** all changes are reviewed
**Then** no production source files are modified (test files only)
