# Story 101.6: Progress Tracking & Streaks

## Story

As a Knowlune user who listens to Audiobookshelf audiobooks,
I want my listening time to count toward my study streak and appear in Reports,
so that my audiobook learning is tracked alongside courses and books.

## Acceptance Criteria

**Given** a user is streaming an ABS audiobook
**When** they play for at least 60 seconds and pause or close the player
**Then** a listening session is recorded via `useAudioListeningSession` (FR28)
**And** the session includes: book ID, start time, end time, duration, chapter progress

**Given** a listening session is recorded for an ABS audiobook
**When** the streak system evaluates daily activity
**Then** the ABS listening session counts toward the study streak (FR29)
**And** the streak behaves identically to local audiobook and course sessions

**Given** a user navigates to the Reports dashboard
**When** learning time is displayed
**Then** audiobook listening time (both local and ABS) appears alongside course and book time (FR30)
**And** the time breakdown distinguishes between content types

**Given** a user has listened to part of an ABS audiobook
**When** the Library page displays the book card
**Then** a progress indicator shows the listening progress percentage (FR32)
**And** the current chapter title is visible
**And** the total duration and time remaining are displayed

**Given** a user has listened to an ABS audiobook
**When** the system updates the Book record
**Then** `progress` reflects the percentage complete (FR31)
**And** `currentPosition` stores `{ type: 'time', seconds: currentTime }`
**And** `lastOpenedAt` is updated to the current ISO timestamp

**Given** the ABS server is unreachable
**When** the user views their progress for an ABS audiobook
**Then** the last-known progress from Dexie is displayed (NFR17)
**And** no error is shown — progress data is local, not dependent on server

## Design Guidance

This story is primarily a data/logic story with no new UI components. The focus is ensuring existing infrastructure correctly handles ABS books:

- Progress bars on book cards are already part of the Library page — verify they work for ABS books (source: `book.progress` 0-100)
- Chapter title display (if added): use existing `BookChapter.title` from `book.chapters` array — same pattern used in AudiobookRenderer chapter list
- Reports page: no new section needed — ABS listening time merges into existing "Reading" section via `ReadingStatsService` sentinel pattern
- If adding time-remaining to book card: format as `formatAudioTime()` already exported from `useAudioPlayer.ts`

**NFR17 (graceful degradation):** Progress display must never trigger a network request to the ABS server. All progress data lives in Dexie `books` table. Library renders from store (pre-loaded from Dexie). No conditional ABS fetch in render path.

## Implementation Notes

### Critical Architecture Context

**This story is ZERO new infrastructure** — `useAudioListeningSession`, `studyLog.ts`, and `ReadingStatsService` already handle ABS books via the `book_listened` event type. The only gaps are:
1. Persisting `currentPosition` and `progress` on the `Book` record during playback (not currently done by `useAudioPlayer`)
2. Restoring position when an ABS book is reopened
3. Verifying Library book cards display progress for audiobooks

### Session Tracking (Already Working)

- `AudiobookRenderer.tsx:82` — `useAudioListeningSession({ bookId: book.id, isPlaying })` is already called, regardless of `book.source.type`
- `useAudioListeningSession` fires `logStudyAction({ type: 'book_listened', courseId: bookId, ... })`
- `studyLog.ts:222` — `book_listened` is already included in `studyDaysFromLog()` for streak counting
- **Conclusion:** Sessions and streaks work automatically — no changes needed

### Progress Persistence Gap

`useAudioPlayer` currently does NOT save `currentPosition`, `progress`, or `lastOpenedAt` to Dexie. This is the only missing piece. The pattern to follow is `BookReader.tsx:330-358`:
- Debounced save (5s) on `timeupdate`
- Immediate save on session end (pause/close/unmount)
- Dual write: `useBookStore.setState` (in-memory) + `db.books.update` (Dexie)

The `saveProgress` callback should be added inside `useAudioPlayer.ts` and called from `useEffect` monitoring `currentTime`. For ABS books, `totalDuration` comes from `book.totalDuration` set during catalog sync.

```typescript
// Pattern for progress save (add to useAudioPlayer.ts):
const saveProgress = useCallback(
  (currentTime: number) => {
    if (!book || book.format !== 'audiobook') return
    const totalDuration = book.totalDuration ?? 0
    if (totalDuration <= 0) return
    const progressInt = Math.min(100, Math.round((currentTime / totalDuration) * 100))
    const position: ContentPosition = { type: 'time', seconds: Math.floor(currentTime) }
    const now = new Date().toISOString()

    useBookStore.setState(state => ({
      books: state.books.map(b =>
        b.id === book.id
          ? { ...b, currentPosition: position, progress: progressInt, lastOpenedAt: now }
          : b
      ),
    }))

    // silent-catch-ok: progress save is non-critical — never disrupt playback
    db.books.update(book.id, { currentPosition: position, progress: progressInt, lastOpenedAt: now }).catch(err => {
      console.error('[useAudioPlayer] Failed to save progress:', err)
    })
  },
  [book]
)
```

### Position Restoration for ABS Books

In `useAudioPlayer.ts` `loadChapter`, after `audio.src = streamUrl` for `source.type === 'remote'`:
```typescript
// Restore position after audio is ready
const handleCanPlay = () => {
  if (book.currentPosition?.type === 'time') {
    audio.currentTime = book.currentPosition.seconds
  }
  audio.removeEventListener('canplay', handleCanPlay)
}
audio.addEventListener('canplay', handleCanPlay)
```

### Anti-Patterns to Avoid

- **Do NOT** create a separate progress tracking service for ABS — reuse existing `useAudioListeningSession` and `studyLog.ts`
- **Do NOT** fetch progress from the ABS server on Library page render — all progress is local (Dexie)
- **Do NOT** add a separate "Audiobook" section to Reports — ABS listening merges into existing "Reading" stats via the sentinel pattern
- **Do NOT** create a new `ContentSource` type — ABS books already use `{ type: 'remote', url, auth }` (Decision 1)
- **Do NOT** add `isPlaying` guards in `studyDaysFromLog` — `book_listened` events count regardless of source type (this is the desired behavior)

### Dependencies

- Hard dependency on **E101-S03** (Library Browsing): ABS books must be in Dexie `books` table with `totalDuration` set and `source.type === 'remote'`
- Hard dependency on **E101-S04** (Streaming Playback): `useAudioPlayer` must support `source.type === 'remote'` and have `isPlaying` state available
- Soft dependency on **E101-S01** (Data Foundation): Dexie v40 schema with `audiobookshelfServers` table
