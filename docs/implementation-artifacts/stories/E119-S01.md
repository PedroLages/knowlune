# Story E119-S01: Privacy Notice Source-of-Truth + Versioning

## Story

As Pedro (controller/operator),
I want a single canonical `docs/compliance/privacy-notice.md` as the source of truth wired to the in-app `PrivacyPolicy.tsx`,
so that the notice never drifts from what the code actually renders and versioning is deterministic.

## Acceptance Criteria

- **AC-1**: `docs/compliance/privacy-notice.md` exists, covers all Art 13/14 required elements: controller identity/contact, purposes + lawful basis per purpose, data categories, sub-processor names, retention summary (link to matrix), rights + how to exercise, complaint path to supervisory authority, AI processing disclosure.
- **AC-2**: `src/lib/compliance/noticeVersion.ts` exports `CURRENT_NOTICE_VERSION` in `YYYY-MM-DD.N` format (e.g. `2026-05-01.1`).
- **AC-3**: `src/app/pages/legal/PrivacyPolicy.tsx` renders content that matches or mirrors the markdown source; a drift test asserts equality or hash match.
- **AC-4**: `src/app/pages/legal/LegalUpdateBanner.tsx` consumes `noticeVersion.ts` constants for `documentId` and `effectiveDate`.
- **AC-5**: Drift test `src/app/pages/legal/__tests__/PrivacyPolicy.notice-sync.test.tsx` is green in CI.
- **AC-6**: `noticeVersion.test.ts` validates version format (`YYYY-MM-DD.N`).
- **AC-7**: Banner displays `Effective YYYY-MM-DD (rev N)` to the user.

## Tasks / Subtasks

- [ ] Task 1: Write `docs/compliance/privacy-notice.md` with all Art 13 elements (AC: 1)
- [ ] Task 2: Create `src/lib/compliance/noticeVersion.ts` with version constant (AC: 2)
- [ ] Task 3: Update `src/app/pages/legal/PrivacyPolicy.tsx` to mirror notice source; write drift test (AC: 3, 5)
- [ ] Task 4: Update `LegalUpdateBanner.tsx` to consume `noticeVersion` constants (AC: 4, 7)
- [ ] Task 5: Write unit tests for `noticeVersion.ts` format validation (AC: 6)

## Context

- Existing `PrivacyPolicy.tsx` already has a 10-section structure aligned with Art 13. Extend it rather than replacing.
- `LegalUpdateBanner.tsx` already has `documentId` + `effectiveDate` props. Wire them to `noticeVersion.ts` constants.
- Version scheme: ISO date + integer revision (`2026-05-01.1`). Bumping the constant signals a material change.
- Drift test approach: normalize both rendered React content and markdown source to plain text; assert equality or hash match.
- This story is a foundation for S02 (notice ack), S05 (export manifest), S07 (consent references notice), S12 (ROPA/DPA link).

## Out of Scope

- Implementing consent records (S07).
- Multi-locale translations (deferred per plan N scope).
