# Story 79.04: Plugin Scaffold CLI

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a plugin author,
I want a CLI command to scaffold a new plugin project,
so that I can start building a plugin in under 5 minutes with correct structure and types.

## Acceptance Criteria

1. **Given** a developer runs `npx create-knowlune-plugin my-widget`, **When** the scaffold completes, **Then** a new directory `knowlune-plugin-my-widget/` is created with:
   - `package.json` (name, version, main, types, peerDependencies on `@knowlune/plugin-types`)
   - `knowlune-plugin.json` (template manifest with placeholder values)
   - `tsconfig.json` (configured for React + TypeScript)
   - `src/index.tsx` (template PluginComponent with TODO comments)
   - `.gitignore`

2. **Given** the scaffolded project, **When** the developer runs `npm install && npm run build`, **Then** it compiles successfully with zero errors, and produces `dist/index.js` and `dist/index.d.ts`.

## Dev Notes

### Architecture Decisions

- **Monorepo location**: `packages/create-knowlune-plugin/` -- follows convention of `packages/` for publishable packages. `@knowlune/plugin-types` (E78-S07) is also in `packages/`.
- **No heavy CLI framework**: Use Node.js `process.argv` directly or lightweight `commander`/`meow` for argument parsing. Avoid `inquirer` or interactive prompts -- the scaffold should work non-interactively for CI/automation.
- **Template approach**: Static template files with `{{variable}}` placeholders, not AST manipulation. Simple, maintainable, and debuggable.
- **TypeScript for CLI source**: The CLI itself is written in TypeScript, compiled to JS. The scaffolded project also uses TypeScript.
- **npx compatibility**: Package name `create-knowlune-plugin` enables `npx create-knowlune-plugin my-widget` (npm convention: `npx create-X` runs `create-X` package).

### Dependencies

- `@knowlune/plugin-types` (E78-S07) -- scaffolded project depends on this as peerDependency
- Node.js `fs`, `path` built-ins for file operations
- Optional: `commander` for CLI argument parsing (lightweight, widely used)

### Scaffolded Plugin Structure

```
knowlune-plugin-my-widget/
  package.json
  knowlune-plugin.json
  tsconfig.json
  .gitignore
  src/
    index.tsx
```

### Template Content Guidelines

- `src/index.tsx` should include:
  - A functional React component named after the plugin (PascalCase)
  - Import of `PluginContext` type from `@knowlune/plugin-types`
  - Usage of design token classes (`bg-card`, `text-card-foreground`, `rounded-[24px]`)
  - TODO comments guiding the developer
- `knowlune-plugin.json` should include all required manifest fields with sensible defaults
- `package.json` must have `"type": "module"` for ESM compatibility

### Project Structure Notes

- CLI package lives in `packages/` (separate from main app `src/`)
- Does NOT modify the main Knowlune app codebase
- Will need workspace configuration if using npm/pnpm workspaces (check if monorepo is already set up)
