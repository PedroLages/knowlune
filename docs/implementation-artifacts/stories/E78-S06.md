# Story 78.06: Sample Plugin — Daily Learning Quote Widget

Status: ready-for-dev

## Story

As a Knowlune user,
I want a sample "Daily Learning Quote" widget on my dashboard,
so that I see the plugin system working end-to-end and have a reference implementation for plugin authors.

## Acceptance Criteria

1. **Given** the `knowlune-plugin-daily-quote` package is installed and activated **When** the user visits the Overview page **Then** a quote widget renders in the `dashboard-widget` PluginSlot **And** the widget displays a motivational learning quote with author attribution **And** the widget uses Knowlune design tokens (`bg-card`, `text-foreground`, `text-muted-foreground`, border radius `rounded-[24px]`).

2. **Given** the plugin declares a `settingsSchema` with category (enum: Motivation, Science, Philosophy) and refreshInterval (number, hours) **When** the user configures settings in Settings > Plugins **Then** the auto-generated form shows a Select for category and Input[number] for refresh interval **And** changing the category filters displayed quotes.

3. **Given** the plugin manifest `knowlune-plugin.json` **When** inspected **Then** it declares: `capabilities: ["write:plugin-data"]`, `slots: ["dashboard-widget"]`, `settingsSchema` with category and refreshInterval.

4. **Given** a developer wants to build their own plugin **When** they examine `plugins/knowlune-plugin-daily-quote/` **Then** the package structure matches the architecture spec (package.json, knowlune-plugin.json, src/index.tsx, dist/) **And** it serves as a complete, working reference implementation.

## Dev Notes

### Architecture Decisions

- **Reference implementation**: This plugin is the canonical example for plugin authors. Every pattern used here (manifest, hooks, settings, design tokens) should be exemplary.
- **No network requests**: Quotes are bundled as static data. The plugin declares `write:plugin-data` (to persist last-shown quote), NOT `access:network`.
- **Design token usage is CRITICAL**: This plugin must demonstrate correct token usage. No hardcoded colors. Use `bg-card`, `text-foreground`, `text-muted-foreground`, `rounded-[24px]`.
- **Plugin data for state**: The plugin uses `setPluginData` / `getPluginData` to remember which quote was last shown and when, so the user doesn't see the same quote on every page load within the refresh interval.

### Manifest Example

```json
{
  "name": "knowlune-plugin-daily-quote",
  "version": "1.0.0",
  "displayName": "Daily Learning Quote",
  "description": "Displays a motivational learning quote on your dashboard, refreshed daily.",
  "author": "Knowlune Team",
  "minKnowluneVersion": "0.0.1",
  "capabilities": ["write:plugin-data"],
  "slots": ["dashboard-widget"],
  "icon": "Quote",
  "settingsSchema": {
    "type": "object",
    "properties": {
      "category": {
        "type": "string",
        "enum": ["Motivation", "Science", "Philosophy"],
        "default": "Motivation",
        "title": "Quote Category"
      },
      "refreshInterval": {
        "type": "number",
        "default": 24,
        "minimum": 1,
        "maximum": 168,
        "title": "Refresh Interval (hours)"
      }
    }
  }
}
```
