# Story 88.4: M4B Audiobook Import

Status: ready-for-dev

## Story

As a learner with M4B audiobook files,
I want to import M4B files and have chapters automatically extracted,
so that I can listen to single-file audiobooks with full chapter navigation in Knowlune.

## Acceptance Criteria

**Given** the user has an M4B audiobook file
**When** the user selects it via the Import dialog (which now accepts `.m4b` files in addition to `.epub` and MP3 folders)
**Then** the `music-metadata` library (~200KB, lazy-loaded on first M4B import) parses the file
**And** chapter markers are extracted from the M4B metadata (chapter titles and start times)
**And** the M4B file is stored in OPFS as a single file at `/knowlune/books/{bookId}/book.m4b`
**And** a Book record is created with `format: 'audiobook'`, chapters array populated from M4B chapter metadata, and `totalDuration` from the file

**Given** the user opens an imported M4B audiobook
**When** the AudiobookRenderer loads
**Then** the HTML5 `<audio>` element plays the M4B file (single file, not per-chapter)
**And** chapter navigation seeks to the chapter's `startTime` within the single file (no file switching)
**And** all audiobook features (speed, sleep timer, bookmarks, Media Session API) work identically to MP3-folder audiobooks

## Dev Notes

### music-metadata Library

- Package: `music-metadata` (npm)
- Browser-compatible: uses `parseBlob()` for File/Blob input
- Extracts: common metadata (title, artist, album, picture), format info (duration, codec), and chapter markers
- M4B is an MP4 container with AAC audio — music-metadata handles MP4/M4A/M4B formats
- **Must be lazy-loaded**: `const mm = await import('music-metadata')` — do NOT add to main bundle

### M4B Chapter Format

M4B files store chapters in the MP4 container's chapter track. music-metadata exposes these via:
```typescript
const metadata = await parseBlob(file)
// Chapters may be in metadata.chapters (newer versions)
// Or in metadata.native['iTunes'] as chapter markers
// Duration: metadata.format.duration (seconds)
// Cover: metadata.common.picture[0] (if present)
```

### Single-File vs Multi-File Playback

The useAudioPlayer hook needs a playback mode flag:
- `'multi-file'` (MP3 folder): each chapter is a separate file, chapter switch = load new file
- `'single-file'` (M4B): one file, chapter switch = seek to `startTime` within file

This impacts: chapter navigation, skip forward/back (cross-chapter boundary), progress calculation, "end of chapter" sleep timer.
