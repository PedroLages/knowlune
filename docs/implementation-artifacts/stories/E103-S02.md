# Story E103-S02: Format Switching UI

## Story

As a Knowlune user reading an EPUB or listening to an audiobook,
I want a "Switch to Listening/Reading" button,
So that I can seamlessly move between formats at the matching chapter.

## Acceptance Criteria

**Given** a user is listening to an ABS audiobook that has a linked EPUB
**When** a chapter mapping exists (created by E103-S01)
**Then** a "Switch to Reading" button is visible in the playback controls

**Given** the user taps "Switch to Reading"
**When** they are at chapter 5 of the audiobook
**Then** the EpubRenderer opens at the matching EPUB chapter (FR40)
**And** the audiobook position is saved before switching

**Given** a user is reading an EPUB that has a linked audiobook on ABS
**When** a chapter mapping exists (created by E103-S01)
**Then** a "Switch to Listening" button is visible in the reader toolbar

**Given** the user taps "Switch to Listening"
**When** they are at chapter 5 of the EPUB
**Then** the AudiobookRenderer opens streaming the matching audiobook chapter from ABS (FR41)
**And** the EPUB reading position (CFI) is saved before switching

**Given** no chapter mapping exists for a book pair
**When** the user views the book
**Then** the format switch button is not displayed

## Design Guidance

**Button placement:**

- **AudiobookRenderer** ("Switch to Reading"): Insert between the chapter title and the playback scrubber, or as a secondary button in the control bar row. Use `variant="brand-outline"` with `BookOpen` icon (16px). Keep it compact — do not disrupt the primary play/pause controls.
- **ReaderHeader** ("Switch to Listening"): Add as an icon+text button in the header action area (right side, before the `MoreHorizontal` menu). Use `variant="ghost"` with `Headphones` icon. Match existing header button styles (`size="sm"`).

**Conditional visibility:**
- Buttons render ONLY when `hasMapping === true`. When not rendered, the layout must not shift (no placeholder gap). Use conditional rendering, not `visibility: hidden`.

**Accessibility:**
- Both buttons need explicit `aria-label` since icon-only buttons are prohibited (WCAG SC 1.1.1).
- Add `title` attribute as tooltip: "Switch to reading" / "Switch to listening".
- Keyboard navigation: buttons must be in natural tab order; no tabindex manipulation.
- WCAG AA contrast required (4.5:1) — use design tokens, not hardcoded colors.

**Touch targets:** Minimum 44x44px on mobile (NFR13 precedent from bookmark FAB).

## Implementation Notes

**Prerequisite:** E103-S01 (Chapter Title Matching Engine) must be complete. This story consumes the `chapterMappings` Dexie table and `ChapterMapping` type created in S01. Do NOT implement the matching algorithm here.

**Data contract from E103-S01:**

```typescript
// Expected shape from E103-S01 (read from Dexie `chapterMappings` table):
export interface ChapterMapping {
  id: string                    // UUID
  epubBookId: string            // FK to Book (epub format)
  audiobookId: string           // FK to Book (audiobook format)
  mappings: ChapterPair[]       // ordered by epubChapterIndex
  confidence: number            // overall mapping confidence 0-1
  createdAt: string
}

export interface ChapterPair {
  epubChapterHref: string       // EPUB chapter href (TOC href)
  epubChapterIndex: number      // 0-based index in EPUB TOC
  audioChapterIndex: number     // 0-based index in ABS chapter list
  confidence: number            // individual match confidence 0-1
}
```

**useFormatSwitch hook design:**

```typescript
// src/app/hooks/useFormatSwitch.ts
export function useFormatSwitch(bookId: string, format: BookFormat) {
  // Returns:
  //   hasMapping: boolean
  //   linkedBook: Book | null
  //   switchToFormat: (currentChapterIndex: number) => void
}
```

- Query Dexie: `db.chapterMappings.where('epubBookId').equals(bookId).or('audiobookId').equals(bookId).first()`
- Loads linked Book from `useBookStore` by the opposing ID
- `switchToFormat(currentChapterIndex)`:
  1. Save current position (call `useBookStore.updateBookPosition` before navigating)
  2. Resolve target chapter index from `ChapterPair[]` lookup
  3. `navigate(`/library/${linkedBook.id}/read?startChapter=${targetIndex}`, { replace: false })`

**BookReader integration points:**

- `AudiobookRenderer` → add `onSwitchToReading?: () => void` prop (optional, backward-compatible)
- `ReaderHeader` → add `onSwitchToListening?: () => void` prop (optional, backward-compatible)
- Both props are undefined when no mapping exists → buttons not rendered

**EPUB chapter index derivation:**

When the user is reading an EPUB and needs to switch, derive the chapter index from the current CFI location:
```typescript
// BookReader already has: currentChapter (string title), toc (NavItem[])
// Derive index by finding toc item whose href matches the chapter the user is on
const currentEpubChapterIndex = toc.findIndex(item => item.href === currentChapterHref)
```
`BookReader` already tracks `currentChapter` (string) and `toc` (NavItem[]) — use these to derive the index.

## Testing Notes

**Seeding approach for E2E tests:**

```typescript
// In test beforeEach — seed two linked books and a chapter mapping:
await db.books.bulkPut([
  makeBook({ id: 'epub-id', format: 'epub', title: 'Dune', author: 'Frank Herbert' }),
  makeBook({ id: 'abs-id', format: 'audiobook', title: 'Dune', author: 'Frank Herbert',
    source: { type: 'remote', url: 'http://abs.local/api/items/abc123', auth: 'Bearer token' }
  }),
])
await db.chapterMappings.put({
  id: 'mapping-1',
  epubBookId: 'epub-id',
  audiobookId: 'abs-id',
  mappings: [
    { epubChapterHref: 'chapter1.xhtml', epubChapterIndex: 0, audioChapterIndex: 0, confidence: 0.95 },
    { epubChapterHref: 'chapter2.xhtml', epubChapterIndex: 1, audioChapterIndex: 1, confidence: 0.91 },
  ],
  confidence: 0.93,
  createdAt: new Date().toISOString(),
})
```

**Key edge cases to test:**
- Book has NO chapter mapping → buttons absent, layout intact
- `startChapter` index out of range → clamp to 0 (no crash)
- EPUB and audiobook have different chapter counts → graceful fallback to chapter 0
- Rapid double-tap "Switch to Reading" → navigation fires only once (debounce or disabled state)
- Position saved correctly: after switching, returning to original book resumes from correct position

**Do NOT test audio output.** Assert component state changes only (chapter loaded, progress updated). Use `data-testid` attributes for button selectors.

**MSW:** The audiobook book record uses `source.type === 'remote'` — ensure MSW handlers from E101-S01 are active in the test context to avoid fetch errors when ABS streaming URLs are constructed.
