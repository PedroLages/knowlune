## Challenges and Lessons Learned

- **Root cause:** The `StatusFilter` component gained a "not-started" status option (index 0) during earlier epics, shifting Active from index 0 to 1 and Completed from index 1 to 2. The test mocks were not updated at the time.
- **Fix approach:** Updated 3 `statusButtons[N]` index references and added inline comments documenting the full button order (`not-started, active, completed, paused`) to prevent future confusion.
- **Pattern worth remembering:** When adding new items to a toggle group or tab bar, search for `getAllByTestId` usage in tests — index-based selectors silently break when the order changes. Consider using `getByRole('button', { name: ... })` for more resilient selectors.
