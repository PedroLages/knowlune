# Story E110-S01: Smart Shelves

## Story

As a reader,
I want to organize my books into named shelves,
so that I can group and filter my library by custom categories (e.g., Favorites, Currently Reading, Want to Read).

## Acceptance Criteria

- **AC-1**: Three default shelves ("Favorites", "Currently Reading", "Want to Read") are auto-created on first library load with fixed IDs for stability
- **AC-2**: Users can create custom shelves with a user-defined name via the Shelf Manager dialog
- **AC-3**: Custom shelves can be renamed in-place within the Shelf Manager dialog
- **AC-4**: Custom shelves can be deleted with a confirmation step; default shelves cannot be deleted
- **AC-5**: Books can be added to or removed from shelves via the book context menu
- **AC-6**: The Filter Sidebar shows a "Shelves" section; selecting a shelf filters the library grid to only books on that shelf
- **AC-7**: Shelf and book-shelf associations are persisted in IndexedDB (Dexie v44 migration: `shelves` + `bookShelves` tables)

## Challenges and Lessons Learned

- **Deterministic shelf IDs**: Using fixed string IDs (`shelf-favorites`, `shelf-currently-reading`, `shelf-want-to-read`) for default shelves rather than `crypto.randomUUID()` ensures E2E tests can reliably target shelves without seeding — a pattern worth applying to other default-data scenarios.
- **Context menu submenu timing**: Playwright requires explicit waiting for submenu visibility after hovering the parent item; `page.getByRole('menuitem')` must be scoped to the visible submenu container.
- **`now()` helper extraction**: Extracting `const now = () => new Date().toISOString()` into a module-level helper (instead of inlining `new Date().toISOString()` everywhere) makes the store easier to test and consistent — flagged by the `test-patterns/deterministic-time` ESLint rule.
