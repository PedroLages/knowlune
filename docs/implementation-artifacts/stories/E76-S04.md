# Story 76.4: Send Weekly Digest Edge Function

Status: ready-for-dev

## Story

As a system,
I want an Edge Function that orchestrates the weekly digest pipeline — querying users, aggregating data, rendering emails, and sending via Resend,
so that opted-in users receive their learning digest on schedule.

## Acceptance Criteria

1. **Given** the `send-weekly-digest` Edge Function at `supabase/functions/send-weekly-digest/index.ts`
   **When** invoked via HTTP POST with service_role_key authorization
   **Then** it:
   1. Queries `notification_preferences` for users where `email_digest_enabled = true`
   2. For each user, calls `get_weekly_digest_data` via RPC
   3. Skips users where `has_activity = false` (FR-10)
   4. Skips users where `last_digest_sent_at` is within the current digest period (idempotency)
   5. Generates a signed JWT unsubscribe token per user (90-day expiry, HS256)
   6. Renders the email via `renderWeeklyDigest()`
   7. Sends via Resend API with dynamic subject line
   8. Updates `last_digest_sent_at` after successful send
   **And** returns JSON summary: `{ sent: N, skipped: N, failed: N, errors: [...] }`

2. **Given** the Resend email send call
   **When** an email is sent
   **Then** it includes both RFC 8058 headers:
   - `List-Unsubscribe: <https://<project>.supabase.co/functions/v1/unsubscribe?token=<jwt>>`
   - `List-Unsubscribe-Post: List-Unsubscribe=One-Click`
   **And** the `from` address is `Knowlune <digest@knowlune.com>`
   **And** the email resolves the recipient address from `auth.users` (FR-9)

3. **Given** a user's digest send fails (Resend API error)
   **When** the error is caught
   **Then** the error is logged with `user_id` and the function continues to the next user (does not abort batch)

4. **Given** the `RESEND_API_KEY` and `UNSUBSCRIBE_SECRET` environment variables
   **When** the Edge Function initializes
   **Then** secrets are accessed via `Deno.env.get()` (never hardcoded)

5. **Given** the dynamic subject line
   **When** generated
   **Then** it follows the format: "Your Week in Learning: {hours}h studied{streakSuffix}" with priority: streak (if > 7 days) > achievements (if any) > flashcards due

6. **Given** the shared utilities
   **When** the Edge Function imports them
   **Then** `supabase-client.ts` from `_shared/` provides the Supabase client factory
   **And** `jwt-utils.ts` from `_shared/` provides `generateUnsubscribeToken()` and `verifyUnsubscribeToken()`

## Dev Notes

### Architecture Decisions

- **AD-5**: Service role key for digest sends — pg_cron invokes with service_role_key (not anon key). Stored in Supabase Vault, accessed via `Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')`.
- **AD-6**: Two separate Edge Functions — `send-weekly-digest` (batch sender) and `unsubscribe` (user-facing). This function is the batch sender.
- **AD-7**: Every email includes `List-Unsubscribe` and `List-Unsubscribe-Post` headers (RFC 8058, required by Gmail/Yahoo).
- **AD-4**: JWT unsubscribe tokens signed with HS256 via `jose`, 90-day expiry.

### Dynamic Subject Line Logic

```
Format: "Your Week in Learning: {hours}h studied{suffix}"

Priority for suffix:
1. If currentStreak > 7 → ", {streak}-day streak"
2. Else if achievementsEarned.length > 0 → ", {count} achievement(s) earned"
3. Else if cardsDueForReview > 0 → ", {count} cards due"
4. Else → no suffix
```

### Idempotency Logic

Before sending to a user, check `last_digest_sent_at`:
- Weekly: skip if `last_digest_sent_at >= date_trunc('week', NOW() - INTERVAL '1 week')`
- Daily: skip if `last_digest_sent_at >= date_trunc('day', NOW() - INTERVAL '1 day')`
This prevents duplicate emails on pg_cron retry or manual trigger.

### Critical Constraints

- **Never hardcode secrets**: `RESEND_API_KEY`, `UNSUBSCRIBE_SECRET`, `SUPABASE_SERVICE_ROLE_KEY` all via `Deno.env.get()`.
- **Error isolation**: One user's failure must NOT abort the batch. Catch per-user, log, continue.
- **From address**: `Knowlune <digest@knowlune.com>` — requires domain verification (SPF/DKIM/DMARC).
- **Edge Function timeout**: 150s max. With 200ms delay per user, supports ~750 users per batch.
- **Resend import**: Use `npm:resend` for Deno compatibility.
- **jose import**: Use `npm:jose` for JWT operations in Deno.
