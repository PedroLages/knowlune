# Story 52.04: Course Embedding Aggregation

## Story

As a learner,
I want the system to automatically compute course embeddings when I import courses,
so that the recommendation engine can find similar courses based on content.

## Acceptance Criteria

**Given** the Dexie database is at version 27
**When** the application loads with the v28 migration
**Then** a new `courseEmbeddings` table is created with primary key `courseId`
**And** existing v27 databases upgrade transparently without data loss

**Given** a learner imports a new YouTube course
**When** the course import completes
**Then** a course metadata embedding is generated from `title + " " + description + " " + tags.join(", ")`
**And** the embedding is 384 dimensions (same `all-MiniLM-L6-v2` model)
**And** it is stored in the `courseEmbeddings` table with `courseId`, `embedding`, `generatedAt`, and `sourceHash`

**Given** the embedding generation pipeline
**When** it generates a course embedding
**Then** it uses the existing `generateEmbeddings` Web Worker
**And** completes in <2 seconds
**And** requires zero new npm dependencies

**Given** a course's metadata changes (tags updated, description edited)
**When** the system detects the `sourceHash` (SHA-256 of title+description+tags) has changed
**Then** the course embedding is regenerated and the old one replaced

**Given** the embedding pipeline fails during course import
**When** the error is caught
**Then** the course import still succeeds (embedding generation is non-blocking)
**And** the missing embedding is logged for future retry
**And** recommendations fall back to tag-based matching for that course
