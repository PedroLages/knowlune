## Challenges and Lessons Learned

**KI-021 root cause (CSP upgrade-insecure-requests):** The real root cause was NOT a Courses page component issue — it was the CSP `upgrade-insecure-requests` and `block-all-mixed-content` directives in index.html causing WebKit (Mobile Safari) to upgrade all `http://localhost` module requests to `https://localhost`, producing TLS errors and a blank page. Fix: added `testModeCspPlugin` to vite.config.ts that strips those CSP directives when `PLAYWRIGHT_TEST=1` is set.

**Pattern — WelcomeWizard blocks E2E interactions:** Tests that used `page.goto()` directly (without the `navigateAndWait` helper) would encounter the WelcomeWizard dialog blocking interactions. The fix is to seed `knowlune-welcome-wizard-v1` alongside `knowlune-onboarding-v1` via `addInitScript` before any navigation. Updated `local-storage-fixture.ts` to auto-seed both dismissal keys for all tests using the localStorage fixture.

**DEFAULT_SECTION_ORDER drift:** The `dashboard-reordering.spec.ts` test had a hard-coded 7-item `DEFAULT_SECTION_ORDER` that no longer matched `src/lib/dashboardOrder.ts` (which grew to 10 items). Tests also hardcoded `toBe(7)` counts. When `DEFAULT_ORDER` changes in the future, the test constant must be updated too — consider importing the constant from the source file directly.

**skill-proficiency is conditional:** The `section-skill-proficiency` element only renders when `skillProficiencyData.length > 0`. Without seeded quiz data, count is 9 (not 10). The customizer panel shows all 10 drag handles regardless.
