# Story 102.2: Series Browsing

## Story

As a Knowlune user with audiobook series in Audiobookshelf,
I want to browse my audiobooks organized by series with reading order,
so that I can follow a series sequentially and track my progress through it.

## Acceptance Criteria

**Given** a connected ABS server with audiobooks in series
**When** the user selects a "Series" view on the Library page
**Then** audiobooks are grouped by series name (FR36)
**And** within each series, books are ordered by series sequence number

**Given** a series group is displayed
**When** the user views the series
**Then** series progress is shown: "{completed} of {total} books" (FR37)
**And** each book shows its individual listening progress

**Given** a series with partially listened books
**When** the user taps a series card
**Then** the series expands to show all books in order
**And** the next unfinished book is highlighted

## Design Guidance

**Layout approach:** Series view replaces the flat grid with a list of expandable series cards. Each card is a full-width row in collapsed state, expanding inline (accordion pattern) to reveal the book list. Do NOT navigate to a separate series page — this is an inline expansion.

**Component structure:**
```
Library.tsx
  └── SeriesView (new, conditional on view mode)
        └── SeriesCard[] (one per series, accordion)
              └── BookCard[] or condensed BookRow[] (within expanded series)
```

**Series cover:** Use first book's cover image (same lazy-load pattern as regular BookCard: `getCoverUrl(server.url, books[0].absItemId)`). If no cover available, use a placeholder icon.

**Progress label:** `"{completed} of {total} books complete"` — use `text-muted-foreground` for the count, emphasis on series name. Do not use raw numbers without label.

**"Continue" highlight:** Apply `bg-brand-soft` and `text-brand-soft-foreground` to the next unfinished book row. Add a small "Continue" badge using `variant="brand"` Badge component. Do not use `bg-blue-*` or hardcoded colors.

**Accessibility:**
- SeriesCard toggle button: `aria-expanded={isExpanded}` + `aria-controls={seriesId}`
- Book list: `role="list"`, each book `role="listitem"`
- Series cover image: `alt="{seriesName} series cover"`
- Progress text: included in aria-label on the series card

## Implementation Notes

### Prerequisite: E102-S01 must be complete
This story builds on the bidirectional progress sync from E102-S01. The `readingProgress` field on Book records must reflect synced ABS progress for the "completed/total" calculation to be accurate.

### ABS Series API
The Audiobookshelf API endpoint for series:
```
GET /api/libraries/{libraryId}/series
  ?page=0&limit=50&sort=name&desc=0&filter=
Authorization: Bearer {apiKey}
```

Response shape:
```typescript
interface AbsSeriesResponse {
  results: AbsSeries[]
  total: number
  page: number
  limit: number
}

interface AbsSeries {
  id: string
  name: string
  nameIgnorePrefix: string
  nameIgnorePrefixSort: string
  type: 'series'
  books: AbsSeriesBook[]  // Books already in series order
  totalDuration: number
  addedAt: number
  updatedAt: number
}

interface AbsSeriesBook {
  id: string                    // ABS item ID
  sequence: string | null       // e.g. "1", "2", "2.5" — may be null if unset
  // Extends AbsLibraryItem with series context:
  media: AbsMediaMetadata
  // ... other AbsLibraryItem fields
}
```

**Important:** `sequence` is a string (not number) in ABS — sort with `parseFloat(sequence ?? '0')` to handle decimals like "2.5" correctly.

**Pagination:** Series endpoint is paginated. If a library has 100+ series, load all pages. Use the same loop pattern as `fetchLibraryItems()` — fetch `total` from first page, then load remaining pages.

### Service layer — follow AudiobookshelfService pattern exactly
```typescript
// ADD to src/services/AudiobookshelfService.ts (do NOT create new file)

/** Fetch all series from a library */
export async function fetchSeriesForLibrary(
  url: string,
  apiKey: string,
  libraryId: string,
  options?: { page?: number; limit?: number }
): Promise<AbsResult<{ results: AbsSeries[]; total: number }>> {
  const params = new URLSearchParams({
    page: String(options?.page ?? 0),
    limit: String(options?.limit ?? 50),
    sort: 'name',
  })
  return absApiFetch(url, apiKey, `/api/libraries/${libraryId}/series?${params}`)
}
```

This is the ONLY file that should call ABS API endpoints. Do NOT add ABS fetch calls elsewhere.

### Store layer — follow useAudiobookshelfStore pattern exactly
Series data lives in `useAudiobookshelfStore` alongside server connection data — NOT in `useBookStore`. The Zustand + Dexie pattern:
```typescript
// Extend existing useAudiobookshelfStore state:
interface AudiobookshelfState {
  // ... existing fields ...
  series: AbsSeries[]
  isLoadingSeries: boolean
  seriesLoaded: boolean  // isLoaded guard
}
```

Series data is NOT persisted to Dexie (it's derived/cached from ABS, refreshed on demand). Store it only in Zustand state (in-memory). This matches how catalog browsing works — it's fetched fresh, not stored permanently.

### Progress calculation — use existing Book records
Do NOT call `fetchProgress()` from AudiobookshelfService per book to calculate series progress. The `readingProgress` field on Book records in Dexie (synced via E102-S01) is the source of truth.

```typescript
// In useSeriesProgress or computed inside SeriesCard:
const booksInKnowlune = useBookStore(s =>
  series.books.map(absBook =>
    s.books.find(b => b.absItemId === absBook.id)
  )
)
const completed = booksInKnowlune.filter(b => b && b.readingProgress >= 1.0).length
const total = series.books.length
```

**Next unfinished book:** First book in sequence order where `readingProgress < 1.0` (or no local record found — treat as unstarted).

### File locations — follow architecture structure
- Service function: add to `src/services/AudiobookshelfService.ts` (existing file)
- Store: add fields/actions to `src/stores/useAudiobookshelfStore.ts` (existing file from E101)
- New component: `src/app/components/library/SeriesCard.tsx`
- View integration: modify `src/app/pages/Library.tsx` (existing, modified in E101 for source tabs)
- Types: add `AbsSeries`, `AbsSeriesBook` to `src/data/types.ts` (existing, extended in E101)
- MSW handler: add to `tests/support/msw/audiobookshelf-handlers.ts` (existing from E101)
- E2E spec: `tests/e2e/audiobookshelf/series.spec.ts` (NEW)

**Anti-pattern:** Do NOT create a separate `SeriesService.ts` — ABS API calls belong in `AudiobookshelfService.ts` per NFR18 and Architecture Decision 4.

### Library page view integration
The Library page currently has `LibrarySourceTabs` (All/Local/Audiobookshelf) from E101. The "Series" view is a sub-mode within the Audiobookshelf tab — it does NOT add a 4th source tab. Suggested pattern:

```
[All] [Local] [Audiobookshelf ▾]
                  │
           [Grid] [Series]  ← view toggle within ABS tab
```

Or alternatively, add "Series" as a segment control below the source tabs when Audiobookshelf is selected. The exact UX is flexible — pick the pattern that integrates cleanest with existing E101 tab code without refactoring.

### No new npm dependencies
Zero new packages. This feature uses:
- Native `fetch` (via existing `absApiFetch` helper)
- Existing Zustand, Dexie, Tailwind patterns
- No sorting library needed (`parseFloat()` for sequence sort)

### E2E test patterns
- Use `FIXED_DATE` for any date operations (ESLint `test-patterns/deterministic-time`)
- Navigate to `/` before library operations
- Seed a connected ABS server via Dexie seeding helpers before each test
- Do NOT use `waitForTimeout()` — use `waitFor` with selectors instead
- `afterEach` cleanup must use `await` (not fire-and-forget)

### Edge cases to test
1. Series with `sequence: null` — should still appear, sorted to end of list
2. Series where no books are in Dexie (never synced) — `completed: 0 of {total}` (no crash)
3. Series with a single book — no "Continue" highlight needed (just show the book)
4. Empty series list (ABS library has no series) — show empty state message
