# Story E77B-S01: Drive read-only scope and folder browser

## Story

As a user,
I want to connect my Google Drive with read-only access and browse folders in the import wizard,
so that I can import courses stored in my Google Drive.

## Acceptance Criteria

1. **Given** the user opens the import wizard **When** the Drive tab is selected **Then** Drive folder browser opens with a scope check.
2. **Given** the user has not granted Drive read scope **When** the Drive folder browser opens **Then** an incremental OAuth grant flow is initiated.
3. **Given** the user grants Drive read scope **When** the token is obtained **Then** Drive folder contents are listed with folder navigation.
4. **Given** a Drive folder is selected **When** the user confirms **Then** files (videos, PDFs, EPUBs) are mapped to a scanned course structure and imported.

## Dev Notes

- Uses `drive.readonly` OAuth scope (not bundled with initial sign-in)
- localStorage flag `knowlune_drive_read_granted` as fast-path cache
- Premium-gated behind subscription check
- `hasDriveReadScope()` function in googleDriveToken.ts
- DriveFolderBrowser component with scope check → grant → browse → select flow

## Challenges and Lessons Learned

- **State cleanup on unmount**: Initial implementation had React state updates on unmounted component. Fixed by adding an `ignore` cleanup flag in the `DriveFolderBrowser` effect.
- **Premium gating**: Drive import is premium-only. Initial implementation lacked the gate — added in R1.
- **Keyboard navigation**: The folder list initially lacked keyboard handlers (arrow keys, Enter/Space). Fixed in R1 to comply with WCAG 2.1 AA.
- **Video format detection**: Drive-imported videos initially hardcoded to 'mp4'. Fixed to use `getVideoFormat()` from fileSystem lib.
