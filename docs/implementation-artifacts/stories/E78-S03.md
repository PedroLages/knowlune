# Story 78.03: Plugin Settings UI and Management Panel

Status: ready-for-dev

## Story

As a Knowlune user,
I want to view, enable/disable, configure, and uninstall my plugins from the Settings page,
so that I have full control over my installed plugins and their behavior.

## Acceptance Criteria

1. **Given** the user navigates to the Settings page **When** plugins are installed **Then** a "Plugins" card section appears (4th position, after Subscription) showing a list of PluginCard components **And** each card shows: plugin icon (Lucide, fallback to Puzzle), display name, author, version, status badge, enable/disable toggle, capability badges.

2. **Given** no plugins are installed **When** the user views the Plugins section **Then** an empty state is shown: "No plugins installed yet. Browse available plugins to enhance your learning experience." with a muted Puzzle icon.

3. **Given** a plugin declares a `settingsSchema` in its manifest **When** the user clicks the Settings icon on a PluginCard **Then** an accordion panel expands below the card showing an auto-generated form **And** the form maps JSON Schema types correctly: string->Input, string+enum->Select, boolean->Switch, number->Input[number], string+maxLength>100->Textarea.

4. **Given** the user changes plugin settings and clicks Save **When** the save completes **Then** settings are persisted in the `plugins` Dexie table `settings` field **And** a `plugin:settings-changed` event is emitted on the EventBus **And** a success toast appears.

5. **Given** the user toggles a plugin's switch **When** the toggle completes **Then** the plugin state transitions (active<->inactive) with optimistic UI update **And** a toast confirms: "{Plugin name} enabled" or "{Plugin name} disabled".

6. **Given** the user clicks Uninstall on a plugin **When** they confirm in the AlertDialog **Then** the plugin is uninstalled (data cleaned, record removed) and a success toast appears.

7. **Given** the PluginsSection is rendered **When** a screen reader reads the plugin cards **Then** toggles have `aria-label="Enable {pluginName}"`, status badges have `aria-label="Status: {state}"`, and capability badges are in a `role="list"`.

## Dev Notes

### Architecture Decisions

- **Auto-generated forms from JSON Schema**: The PluginSettingsRenderer reads the plugin's `settingsSchema` (JSON Schema 7) and generates shadcn/ui form controls. No custom form builder needed — just a switch/case over schema property types.
- **Optimistic UI on toggle**: Update Zustand store immediately on toggle, then persist to Dexie. Rollback on failure. Follow the project's established optimistic update pattern.
- **EventBus integration**: The `plugin:settings-changed` event type must be added to `AppEvent` union in `src/lib/eventBus.ts`. This story depends on E78-S05 for the EventBus extension — either implement a minimal addition here or coordinate.

### UX Design Specs

From the UX design document:
- **PluginCard**: Horizontal card with icon, name, author, version, status badge, enable/disable toggle, capability badges, settings/uninstall actions, render time display
- **PluginsSection**: Card section in Settings page, 4th position after Subscription
- **Empty state**: Muted Puzzle icon with encouraging text
- **Capability badges**: `bg-brand-soft` for reads, `bg-warning/10` for writes/network
- **Status badge**: Active=green, Inactive=gray, Error=red

### shadcn/ui Components to Use

- `Card`, `CardHeader`, `CardContent` — section wrapper
- `Switch` — plugin enable/disable toggle
- `Badge` — status and capability indicators
- `Button` (variant: `brand`, `destructive`, `outline`) — actions
- `AlertDialog` — uninstall confirmation
- `Tooltip` — capability explanations
- `Input`, `Select`, `Textarea` — settings form controls
- `Accordion` or collapsible pattern — settings panel expansion
- Sonner `toast` — success/error feedback

### Design Token Usage (CRITICAL — no hardcoded colors)

| Use Case | Token |
|----------|-------|
| Active badge | `bg-success-soft`, `text-success` |
| Error badge | `bg-destructive/10`, `text-destructive` |
| Read capability | `bg-brand-soft`, `text-brand-soft-foreground` |
| Write/network capability | `bg-warning/10`, `text-warning` |
| Card background | `bg-card` |
| Primary actions | `variant="brand"` on Button |
| Destructive actions | `variant="destructive"` on Button |
