# Story 101.2: Server Connection & Authentication UI

## Story

As a Knowlune user with an Audiobookshelf server,
I want to connect my server by entering a URL and API key,
so that Knowlune can access my audiobook library.

## Acceptance Criteria

**Given** the user navigates to Library settings
**When** the Audiobookshelf section is visible
**Then** it shows a form with fields: Server Name (optional), Server URL (required), API Key (required)
**And** a "Test Connection" button
**And** a help link explaining where to find the API key in ABS settings

**Given** the user enters a URL and API key and clicks "Test Connection"
**When** the connection succeeds
**Then** a success message shows the server version and library count (e.g., "Connected — ABS v2.17.3, 3 libraries found")
**And** checkboxes appear for each library allowing the user to select which to sync
**And** a "Save" button becomes active

**Given** the user clicks "Save" after a successful test
**When** the server is persisted
**Then** it is stored in Dexie via `useAudiobookshelfStore.addServer()`
**And** it appears in the settings UI as a server card with status "connected"
**And** the credentials persist across browser sessions (FR4)

**Given** the server URL uses HTTP (not HTTPS)
**When** the form is submitted
**Then** a warning badge appears: "Credentials will be sent unencrypted over HTTP"
**And** the user can still proceed (warning, not blocker)

**Given** a connection test fails with CORS error
**When** the error is displayed
**Then** it shows: "Could not connect to server. Check the URL and CORS settings."
**And** includes a collapsible "Troubleshooting" section explaining how to add Knowlune's URL to ABS CORS allowed origins

**Given** a connection test fails with auth error (401)
**When** the error is displayed
**Then** it shows: "Authentication failed. Check your API key."

**Given** a saved server with status "connected"
**When** the user clicks the edit button on the server card
**Then** the form pre-fills with the existing URL, name, and a masked API key
**And** the user can update any field and re-test the connection

**Given** a saved server
**When** the user clicks the remove button
**Then** a confirmation dialog appears
**And** on confirm, the server is removed from Dexie and the settings UI
**And** associated cached book metadata is NOT deleted (preserving offline reference)

**Given** a saved server whose API key has expired (ABS returns 401 on catalog fetch)
**When** the system detects the auth failure
**Then** the server status updates to "auth-failed"
**And** the server card shows a red "Authentication Failed" badge with icon + text (NFR12)
**And** a "Re-authenticate" button appears

**Given** any server
**When** its status is displayed
**Then** the status indicator uses both icon and text: green checkmark + "Connected", orange cloud-off + "Offline", red alert + "Auth Failed" (NFR12)
**And** all status indicators meet WCAG 2.1 AA contrast (NFR14)

**Given** the connection settings form
**When** interacted with via keyboard only
**Then** all form fields, buttons, and server cards are fully tab-navigable with visible focus indicators

## Design Guidance

### Component Structure

The UI is a Dialog with three views, mirroring `OpdsCatalogSettings.tsx` architecture exactly:

```
AudiobookshelfSettings (Dialog root)
├── AudiobookshelfServerListView (mode='list')
│   └── AudiobookshelfServerCard (×N)
├── AudiobookshelfServerForm (mode='add' | 'edit')
└── DeleteServerDialog (confirmation, portal)
```

### Key Differences from OpdsCatalogSettings

- Auth: Bearer token (single `apiKey` field) instead of Basic auth (username + password)
- Post-test: library checkboxes appear (unique to ABS — not in OPDS)
- Server cards: status badge with 3 states (not in OPDS which has no connection status)
- Remove behavior: cached books are preserved (OPDS catalogs remove imported books on delete)

### Status Badge Design

Use Tailwind design tokens — no hardcoded colors:

| Status | Icon | Token |
|--------|------|-------|
| connected | `CheckCircle2` | `text-success` |
| offline | `CloudOff` | `text-warning` (or `text-muted-foreground`) |
| auth-failed | `AlertTriangle` | `text-destructive` |

Icon + text always together (NFR12: never color-alone status indicators).

### HTTP Warning

Trigger condition: `isInsecureUrl(url)` returns `true` (already exported from `OpdsService.ts`).

Warning shown as soon as URL is entered (not just on submit):

```tsx
import { isInsecureUrl } from '@/services/OpdsService'
const showHttpWarning = url.trim() && isInsecureUrl(url.trim())
```

Styling: `bg-warning/10 text-warning-foreground` (matches CatalogForm.tsx pattern).

### CORS Troubleshooting

Show a `<details>` element below the error when error text contains "CORS settings":

```tsx
<details className="text-xs text-muted-foreground mt-2">
  <summary className="cursor-pointer hover:text-foreground">Troubleshooting</summary>
  <p className="mt-1">In Audiobookshelf → Settings → Security, add <code>{window.location.origin}</code> to Allowed Origins.</p>
</details>
```

### Accessibility (NFR14)

- All `<Input>` fields: `<Label>` association via `htmlFor` / `id`
- API key field: `type="password"` with show/hide toggle (Eye/EyeOff icons, aria-label)
- Test result: `role="status"` + `aria-live="polite"`
- HTTP warning: `role="alert"`
- Server list: `role="list"` on `<ul>` with `aria-label="Connected Audiobookshelf servers"`
- All icon-only buttons: `aria-label` describing the action

### Store Pattern (Critical — Follow Exactly)

`useAudiobookshelfStore` must mirror `useOpdsCatalogStore` exactly:

```typescript
// src/stores/useAudiobookshelfStore.ts
import { create } from 'zustand'
import { toast } from 'sonner'
import type { AudiobookshelfServer } from '@/data/types'
import { db } from '@/db/schema'

interface AudiobookshelfStoreState {
  servers: AudiobookshelfServer[]
  isLoaded: boolean
  loadServers: () => Promise<void>
  addServer: (server: AudiobookshelfServer) => Promise<void>
  updateServer: (id: string, updates: Partial<Omit<AudiobookshelfServer, 'id'>>) => Promise<void>
  removeServer: (id: string) => Promise<void>
  getServerById: (id: string) => AudiobookshelfServer | undefined
}

export const useAudiobookshelfStore = create<AudiobookshelfStoreState>((set, get) => ({
  servers: [],
  isLoaded: false,
  loadServers: async () => {
    if (get().isLoaded) return
    try {
      const servers = await db.audiobookshelfServers.toArray()
      set({ servers, isLoaded: true })
    } catch (err) {
      console.error('[AudiobookshelfStore] Failed to load servers:', err)
      toast.error('Failed to load Audiobookshelf servers.')
    }
  },
  // ... addServer, updateServer, removeServer follow same catch+toast.error pattern
}))
```

**Anti-pattern: No class, no singleton, no stored URL/apiKey in store.**

### Service Calls in Connection Form

The test connection flow calls two service functions in sequence:

```typescript
// 1. Test connection to verify URL + API key
const connResult = await testConnection(url.trim(), apiKey.trim())
if (!connResult.ok) {
  setTestResult({ ok: false, message: connResult.error })
  return
}

// 2. Fetch libraries so user can select which to sync
const libResult = await fetchLibraries(url.trim(), apiKey.trim())
if (!libResult.ok) {
  setTestResult({ ok: true, message: `Connected — ABS v${connResult.data.serverVersion}. Could not fetch libraries.` })
  return
}

setTestResult({
  ok: true,
  message: `Connected — ABS v${connResult.data.serverVersion}, ${libResult.data.length} ${libResult.data.length === 1 ? 'library' : 'libraries'} found`,
  libraries: libResult.data,
})
```

### Save Server Object Construction

When saving, construct the full `AudiobookshelfServer` object:

```typescript
const newServer: AudiobookshelfServer = {
  id: crypto.randomUUID(),
  name: name.trim() || url.trim(), // fallback to URL if name empty
  url: url.trim(),
  apiKey: apiKey.trim(),
  libraryIds: selectedLibraryIds,
  status: 'connected',
  lastSyncedAt: undefined,
  createdAt: new Date().toISOString(),
  updatedAt: new Date().toISOString(),
}
await addServer(newServer)
```

On update, never expose raw apiKey in the form. Pre-fill `apiKey` state with empty string, show placeholder `'••••••••'`. Only send a new `apiKey` to `updateServer()` if the user typed a new one; otherwise retain existing.

### CSP Configuration

The `AudiobookshelfService.ts` calls user-supplied URLs (self-hosted ABS servers). These are dynamic origins and cannot be pre-listed in CSP. Verify the existing CSP policy allows `connect-src *` or user-provided origins. Check `engineering-patterns.md § CSP Configuration` and the existing OPDS implementation for the established pattern.

### Design Tokens — No Hardcoded Colors

ESLint `design-tokens/no-hardcoded-colors` will block at save-time. Always use tokens:

| Use | Wrong | Correct |
|-----|-------|---------|
| Success status | `text-green-600` | `text-success` |
| Auth-failed status | `text-red-500` | `text-destructive` |
| Offline status | `text-orange-400` | `text-warning-foreground` |
| Warning bg | `bg-yellow-50` | `bg-warning/10` |
| Error bg | `bg-red-50` | `bg-destructive/10` |
