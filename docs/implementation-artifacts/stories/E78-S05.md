# Story 78.05: Plugin Data Access Hooks and EventBus Extension

Status: ready-for-dev

## Story

As a plugin author,
I want typed hooks to read Knowlune data and subscribe to learning events,
so that my plugin can display real user data and react to activity.

## Acceptance Criteria

1. **Given** a plugin declares `read:courses` capability **When** the plugin calls `useCourses()` hook **Then** it receives a live-updating array of `ImportedCourse` objects from Dexie.

2. **Given** a plugin declares `write:plugin-data` capability **When** the plugin calls `setPluginData('myKey', { count: 42 })` **Then** the data is stored in the `pluginData` table with compound key `[pluginId+myKey]` **And** `getPluginData('myKey')` retrieves `{ count: 42 }` **And** `deletePluginData('myKey')` removes the entry.

3. **Given** a plugin declares `subscribe:events` capability **When** the plugin calls `useEventBus('course:completed', callback)` **Then** the callback fires when a `course:completed` event is emitted **And** the subscription is tracked by PluginManager for clean teardown.

4. **Given** a plugin is deactivated **When** PluginManager transitions it to `inactive` **Then** all EventBus subscriptions for that plugin are automatically unsubscribed.

5. **Given** a plugin calls `emitEvent('plugin:daily-quote:refreshed', payload)` **When** the event is emitted **Then** other plugins subscribed to that event type receive the payload **And** the event name includes the plugin's ID prefix.

6. **Given** the EventBus `AppEvent` union type is extended **When** a developer inspects `src/lib/eventBus.ts` **Then** it includes `plugin:activated`, `plugin:deactivated`, `plugin:error`, `plugin:settings-changed` event types.

## Dev Notes

### Architecture Decisions

- **Dexie `useLiveQuery` for reactive data**: Read hooks use Dexie's `useLiveQuery()` which returns live-updating results. This is the same pattern used throughout Knowlune for reactive IndexedDB access.
- **Compound key for plugin data isolation**: `pluginData` table uses `[pluginId+key]` compound primary key. Each plugin's data is automatically namespaced. On uninstall (E78-S02), `db.pluginData.where('pluginId').equals(id).delete()` removes all data.
- **Trust-based capability enforcement**: In Phase 1, hooks are exported and available to all plugins regardless of declared capabilities. The manifest is documentation, not enforcement. Phase 4 may add runtime checks.
- **EventBus extension**: The existing EventBus in `src/lib/eventBus.ts` uses a typed `AppEvent` discriminated union. Add new event types to this union — the pattern is already established.

### Existing EventBus Pattern

The current `AppEvent` union (from `src/lib/eventBus.ts`):
```typescript
export type AppEvent =
  | { type: 'course:completed'; courseId: string; courseName: string }
  | { type: 'streak:milestone'; days: number }
  | { type: 'import:finished'; courseId: string; courseName: string; lessonCount: number }
  | { type: 'achievement:unlocked'; achievementId: string; achievementName: string }
  | { type: 'review:due'; dueCount: number }
  | { type: 'srs:due'; dueCount: number }
```

Add these new types:
```typescript
  | { type: 'plugin:activated'; pluginId: string }
  | { type: 'plugin:deactivated'; pluginId: string }
  | { type: 'plugin:error'; pluginId: string; error: string }
  | { type: 'plugin:settings-changed'; pluginId: string; settings: Record<string, unknown> }
  | { type: 'plugin:custom'; pluginId: string; eventName: string; payload?: unknown }
```

### Dexie Live Query Pattern

```typescript
import { useLiveQuery } from 'dexie-react-hooks'
import { db } from '@/db/schema'

export function useCourses() {
  return useLiveQuery(() => db.importedCourses.toArray(), [], [])
}
```
