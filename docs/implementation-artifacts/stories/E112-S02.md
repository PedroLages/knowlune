# Story 112.02: Genre Distribution & Reading Reports

## Story

As a reader,
I want to see a breakdown of genres I've read and a reading summary report,
so that I can understand my reading habits and discover any genre gaps.

## Acceptance Criteria

**AC1 вҖ” Genre Distribution Chart**
- Given a user has at least 2 books with genre set (status 'reading' or 'finished')
- When they view the Reports page (Study tab, Reading section)
- Then a "Genre Distribution" card shows a donut/pie chart of book counts per genre
- And genres with < 5% of total are grouped into "Other"
- And each slice shows the genre name and count in a tooltip
- And when fewer than 2 books have genres, the card renders null (no empty state clutter)

**AC2 вҖ” Genre Distribution shows library + finished mix**
- Given a user has books across multiple statuses
- When the genre chart is computed
- Then it includes books with status 'reading', 'finished', and 'want-to-read'
- And it excludes 'abandoned' books from the distribution
- And the chart legend shows up to 8 genres (rest grouped as "Other")

**AC3 вҖ” Reading Summary Card**
- Given a user has finished at least 1 book this year
- When they view the Reports page
- Then a "Reading Summary" section shows:
  - Books finished this year vs yearly goal (if set)
  - Average pages per session (all-time)
  - Longest reading session (all-time, formatted as "Xh Ym")
  - Most read author (the author with the most finished books)
- And each metric shows "вҖ”" if insufficient data

**AC4 вҖ” Reading Summary zero state**
- Given a user has no finished books
- When they view the Reading Summary card
- Then the card renders null (no empty state clutter)

**AC5 вҖ” Reports page Reading section ordering**
- After E112-S01 and E112-S02, the Reports page Reading section renders in this order:
  1. `ReadingStatsSection` (reading time stats + trend chart)
  2. `ReadingPatternsCard` (time-of-day patterns) [from E112-S01]
  3. `ReadingGoalsCard` (streaks + yearly progress)
  4. `GenreDistributionCard` (genre donut chart) [new in this story]
  5. `ReadingSummaryCard` (summary metrics) [new in this story]

**Design tokens:**
- Donut chart: use `var(--chart-1)` through `var(--chart-5)`, then cycle back for > 5 genres
- Cards: follow existing Card/CardHeader/CardContent pattern from ReadingGoalsCard
- Stat pills: follow `StatPill` pattern in ReadingStatsSection (bg-brand-soft, text-brand-soft-foreground)

**Recharts PieChart pattern:**
```tsx
<PieChart width={200} height={200}>
  <Pie data={data} dataKey="count" nameKey="genre" cx="50%" cy="50%" innerRadius={60} outerRadius={90}>
    {data.map((_, i) => <Cell key={i} fill={`var(--chart-${(i % 5) + 1})`} />)}
  </Pie>
  <ChartTooltip content={<ChartTooltipContent />} />
</PieChart>
```

**Accessibility:**
- Chart: `aria-label="Genre distribution chart"`
- Legend items: use `<ul>` with `<li>` items (already accessible in Recharts Legend)

## Implementation Notes

**Genre distribution query:**
```ts
const books = await db.books
  .where('status')
  .anyOf(['reading', 'finished', 'want-to-read'])
  .toArray()
```
Group by `book.genre ?? 'Other'`. Sort by count desc. Compute threshold at 5% of total. Merge below-threshold genres into a single "Other" entry.

**Most read author:**
Count finished books per author (use `book.author` field). Return the author with the most books, break ties alphabetically.

**Avg pages per session:**
Use all book study sessions with `duration > 0`. Compute avg pages per session as `(totalPagesEstimated / sessionCount)` using the speed from E112-S01 (or fall back to 2 min/page heuristic).
