# Story 82.03: xAPI Event Emission

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a Knowlune user,
I want my learning events to be emitted as xAPI statements,
so that my learning data is interoperable with external Learning Record Stores and enterprise platforms.

## Acceptance Criteria

1. **Given** xAPI emission is enabled in Settings, **When** a learning event occurs (course.completed, quiz.submitted, session.completed), **Then** an xAPI 1.0.3 statement is constructed with: actor (user), verb (completed/attempted/experienced), object (course/quiz/session), and timestamp.

2. **Given** an xAPI statement is constructed, **When** the statement is emitted, **Then** it is sent to the configured LRS endpoint via HTTP POST with Basic auth, **And** the statement conforms to the xAPI 1.0.3 specification (actor-verb-object structure).

3. **Given** the user configures xAPI in Settings, **When** they provide an LRS endpoint URL and credentials, **Then** the configuration is validated by sending a test statement, **And** a success/failure toast indicates the result.

4. **Given** xAPI emission is disabled (default), **When** learning events occur, **Then** no xAPI statements are emitted (zero performance impact).

## Dev Notes

- **xAPI 1.0.3 Specification**: Statements follow the actor-verb-object triple. Key IRIs:
  - Verbs: `http://adlnet.gov/expapi/verbs/completed`, `http://adlnet.gov/expapi/verbs/attempted`, `http://adlnet.gov/expapi/verbs/experienced`
  - Activity types: `http://adlnet.gov/expapi/activities/course`, `http://adlnet.gov/expapi/activities/assessment`, `http://adlnet.gov/expapi/activities/lesson`
  - API endpoint: `POST {lrs_endpoint}/statements` with `X-Experience-API-Version: 1.0.3` header
- **NFR18**: xAPI 1.0.3 conformance required. Statements must validate against the spec.
- **Default disabled**: xAPI is OFF by default. Zero performance impact when disabled -- the EventBus listener short-circuits immediately.
- **Error handling**: xAPI emission must NEVER crash the app. All network errors are caught and logged as warnings. Failed statements are silently dropped (no retry queue -- that would be a future enhancement).
- **Basic auth**: LRS endpoints typically use Basic auth (`Authorization: Basic base64(username:password)`). Store credentials securely.
- **Dependency on DeveloperPanel**: The xAPI settings section is added to the DeveloperPanel (E80-S01). If DeveloperPanel doesn't exist yet, create a standalone section.
- **EventBus integration**: Subscribe to existing events in `src/lib/eventBus.ts`. The EventBus already has typed events for course completion, quiz submission, and session completion.
