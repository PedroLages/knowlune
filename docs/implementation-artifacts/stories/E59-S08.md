# Story 59.08: E2E Tests and Test Factory Updates

## Story

As a developer,
I want E2E test factories and existing specs updated for FSRS fields,
so that all test data matches the FSRS schema and E2E tests validate the new scheduling behavior end-to-end.

## Acceptance Criteria

**Given** `tests/support/fixtures/factories/review-factory.ts` uses SM-2 fields (`nextReviewAt`, `interval`, `easeFactor`, `reviewCount`)
**When** this story is implemented
**Then** the review factory uses FSRS fields (`stability`, `difficulty`, `reps`, `lapses`, `state`, `elapsed_days`, `scheduled_days`, `due`, `last_review`)
**And** `createDueReviewRecord()` creates a record with `due` in the past
**And** `createFutureReviewRecord()` creates a record with `due` in the future

**Given** there is no flashcard test factory
**When** this story is implemented
**Then** `tests/support/fixtures/factories/flashcard-factory.ts` exists with `createFlashcard()`, `createDueFlashcard()`, `createFutureFlashcard()`
**And** flashcard factory creates objects matching the `Flashcard` interface from `src/data/types.ts`
**And** all factories use `FIXED_DATE` and `getRelativeDate()` from `tests/utils/test-time`

**Given** existing E2E tests (E11-S01, E11-S02, E11-S05) seed review records with SM-2 fields
**When** this story is implemented
**Then** all inline review record creation in those specs uses FSRS fields
**And** all existing E2E tests still pass

**Given** the FSRS migration is complete
**When** this story adds a new E2E spec `tests/e2e/story-e59-s08.spec.ts`
**Then** the spec validates: flashcard creation with FSRS defaults, review rating updates FSRS fields, due date filtering works with FSRS `due` field
