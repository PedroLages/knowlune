# Story 59.08: E2E Tests and Test Factory Updates

## Story

so that all test data matches the FSRS schema and E2E tests validate the new scheduling behavior end-to-end.

## Challenges and Lessons Learned

- FSRS `stability` can approximate SM-2 `interval` for retention calculations in test data -- a card with `stability: N` retains ~90% after N days, making it a reasonable proxy for setting up retention-level test scenarios.
- The E11-S02 test inline factory used `intervalDays` as both the scheduled interval and an implicit stability proxy. With FSRS, making `stability ~ intervalDays` preserves the same retention behavior for the dashboard's strong/fading/weak categorization.
- Factory `state` field needs `as const` or explicit type annotation (`0 | 1 | 2 | 3`) to satisfy TypeScript's CardState type -- plain `number` literals don't narrow automatically in object spreads.
