# Story 76.2: Weekly Digest Data Aggregation Function

Status: ready-for-dev

## Story

As a system,
I want a PostgreSQL function that aggregates weekly learning metrics for a given user,
so that the Edge Function can retrieve all digest data in a single database call.

## Acceptance Criteria

1. **Given** a user with study activity in the past week
   **When** `get_weekly_digest_data(p_user_id UUID)` is called
   **Then** it returns a JSON object containing:
   - `weekly_study_minutes` (integer): total minutes from study_sessions in the past 7 days
   - `current_streak` (integer): current streak count from user_profiles
   - `best_streak` (integer): best streak count from user_profiles
   - `achievements_earned` (JSON array): achievements completed this week with `title` and `earned_at`
   - `cards_due_for_review` (integer): active flashcards where `next_review_at <= NOW()`
   - `course_progress` (JSON array): courses with progress changes this week, each with `name`, `percent`, `week_change`
   - `lessons_completed` (integer): count of lessons completed this week
   - `has_activity` (boolean): true if any metric is non-zero
   **And** the function uses `SECURITY DEFINER` with explicit `SET search_path = public`

2. **Given** a user with zero activity in the past week
   **When** `get_weekly_digest_data(p_user_id UUID)` is called
   **Then** `has_activity` is `false` and all numeric metrics are 0 / empty arrays

3. **Given** the function is called
   **When** the week boundary is calculated
   **Then** it uses `date_trunc('week', NOW() - INTERVAL '1 week')` to `date_trunc('week', NOW())` for consistent Monday-to-Sunday periods

## Dev Notes

### Architecture Decisions

- **AD-1**: Database-first aggregation â€” single PostgreSQL function returns all metrics as JSON. One database round-trip per user vs. 5+ queries from Edge Function. Leverages PostgreSQL's aggregation strengths.
- **AD-3**: `has_activity` flag enables the Edge Function to skip empty digests without additional logic.

### Tables Queried

| Table | Data Extracted |
|-------|---------------|
| `study_sessions` | `weekly_study_minutes` â€” SUM of duration where session falls in week window |
| `user_profiles` | `current_streak`, `best_streak` |
| `challenges` | `achievements_earned` â€” filtered by completion date in week window |
| `flashcards` | `cards_due_for_review` â€” COUNT where `next_review_at <= NOW()` |
| `content_progress` | `course_progress` â€” courses with progress changes, `lessons_completed` count |

### Critical Constraints

- **SECURITY DEFINER** is required because the Edge Function calls this via RPC with service_role_key. Must include explicit `SET search_path = public` to prevent search_path injection.
- **Week boundary**: Use `date_trunc('week', ...)` for consistent Monday-to-Sunday periods (PostgreSQL default week starts on Monday).
- **COALESCE everything**: All numeric fields must default to 0, arrays to `'[]'::jsonb` â€” the function must never return NULL for any field.
- **Return type**: JSONB (not JSON) for efficient storage and indexing.
