# Story 78.04: PluginSlot Component with Error Boundary and Health Monitoring

Status: ready-for-dev

## Story

As a Knowlune user,
I want plugins to render safely in designated areas of the app without crashing the host,
so that a buggy plugin never takes down my learning dashboard.

## Acceptance Criteria

1. **Given** the Overview page contains a `<PluginSlot slotType="dashboard-widget">` **When** active plugins target the `dashboard-widget` slot **Then** each plugin is loaded via `React.lazy(() => import(modulePath))` wrapped in `<Suspense>` with a skeleton fallback **And** each plugin is individually wrapped in a `<PluginErrorBoundary>`.

2. **Given** a plugin throws a render error **When** the error boundary catches it **Then** the PluginSlot shows "Plugin encountered an error" with Disable and Retry buttons **And** the crash is logged with context (plugin name, error type, timestamp) **And** `crashCount` is incremented in the plugin's Dexie record **And** a `plugin:error` event is emitted on the EventBus.

3. **Given** a plugin crashes 3 times within 1 hour **When** the 3rd crash occurs **Then** the plugin transitions to `error` state (auto-disabled) **And** a warning toast appears: "{Plugin name} disabled due to repeated errors" **And** the crash window resets.

4. **Given** a React `<Profiler>` wraps each plugin render **When** the plugin re-renders **Then** the `actualDuration` is recorded and a running average is maintained in memory **And** the average is accessible to the PluginCard for display (shown as "Render: {N}ms").

5. **Given** one plugin crashes in a multi-plugin slot **When** the error boundary catches it **Then** other plugins in the same slot continue rendering normally.

## Dev Notes

### Architecture Decisions

- **Error boundaries MUST be class components**: React does not support error boundaries with hooks. Use `componentDidCatch` and `getDerivedStateFromError`.
- **Per-plugin isolation**: Each plugin gets its own `<PluginErrorBoundary>` instance. One crash does NOT affect other plugins in the same slot.
- **Auto-disable threshold**: 3 crashes in 1-hour window. The window is tracked via `crashWindowStart` on the `PluginRecord`. If the current time minus `crashWindowStart` exceeds 1 hour, reset crashCount before incrementing.
- **Render time in memory only**: The `Profiler` running average is NOT persisted to Dexie — it resets on page reload. This avoids write amplification on every render.
- **React.lazy for plugin loading**: Each plugin module is loaded via dynamic `import()`. The module path comes from the plugin registry (E78-S02).

### Crash Detection Logic

```
onError(pluginId):
  record = getPluginRecord(pluginId)
  now = Date.now()
  windowStart = new Date(record.crashWindowStart).getTime()

  if (now - windowStart > ONE_HOUR_MS):
    // Reset window
    record.crashCount = 1
    record.crashWindowStart = new Date().toISOString()
  else:
    record.crashCount += 1

  if record.crashCount >= 3:
    record.state = 'error'
    record.crashCount = 0
    record.crashWindowStart = new Date().toISOString()
    toast.warning("{displayName} disabled due to repeated errors")

  persist record to Dexie
  emit plugin:error event
```
