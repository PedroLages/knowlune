# Story 52.01: Quiz Generation Pipeline

## Story

As a learner,
I want the system to generate quiz questions from my lesson transcripts using a structured AI pipeline,
so that I can test my retention on the exact content I studied without manually creating questions.

## Acceptance Criteria

**Given** a lesson with a transcript in the `youtubeTranscripts` table
**When** the quiz generation service is invoked for that lesson
**Then** the transcript is chunked by YouTube chapters from `youtubeChapters` table
**And** each chunk produces 3-5 quiz questions via `callOllamaChat` structured output

**Given** a lesson whose video has no YouTube chapters
**When** the quiz generation service chunks the transcript
**Then** it falls back to fixed 5-minute time windows
**And** each window targets 500-1,500 words per chunk

**Given** the LLM returns a structured JSON response
**When** the response is parsed
**Then** it validates against the `QUIZ_RESPONSE_SCHEMA` using `QuestionSchema.safeParse()`
**And** invalid responses trigger up to 2 retries before marking the chunk as failed

**Given** a quiz is successfully generated
**When** it is stored in the `quizzes` Dexie table
**Then** it includes `source: 'auto-generated'`, `transcriptHash` (SHA-256), `bloomsLevel`, `generatedAt`, and `modelId`
**And** the Dexie v28 migration has created the necessary schema support

**Given** a quiz already exists for a lesson with the same `transcriptHash`
**When** the user requests quiz generation for that lesson
**Then** the cached quiz is returned immediately without calling the LLM

**Given** the user specifies a Bloom's Taxonomy level (Remember, Understand, or Apply)
**When** the system prompt is constructed
**Then** it includes level-specific instructions and 2 few-shot examples for the selected level
**And** generates questions of types: multiple-choice, true/false, and fill-in-blank

**Given** the quiz generation is invoked
**When** the system selects the LLM provider
**Then** it respects the `aiConfiguration` settings (consent, provider, model)
**And** routes through the existing `/api/ai/ollama/chat` Express proxy

## Implementation Notes

**New files:**
- `src/ai/quizGenerationService.ts` — Pipeline orchestrator (stateless async functions)
- `src/ai/quizChunker.ts` — Transcript chunking (chapters + fallback)
- `src/ai/quizPrompts.ts` — Bloom's Taxonomy prompt templates + QUIZ_RESPONSE_SCHEMA
- `src/db/schema.ts` — v28 migration (modify existing)
- `src/db/checkpoint.ts` — Update checkpoint version (modify existing)

**Existing files to reuse (not modify):**
- `src/ai/llm/ollama-client.ts` — `callOllamaChat` for structured output
- `src/ai/courseTagger.ts` — Reference pattern for structured output
- `src/types/quiz.ts` — `QuestionSchema`, `QuizSchema`, `QuestionTypeEnum`
- `src/lib/aiConfiguration.ts` — Provider selection, consent, BYOK keys

**Key patterns:**
- Stateless service functions (same as `courseTagger.ts`)
- `callOllamaChat` with `format` param for constrained JSON decoding
- `QuestionSchema.safeParse()` for validation (never skip Zod)
- SHA-256 hash for `transcriptHash` cache key
- Zero new npm dependencies

**Dependencies:**
- None (first story in epic)
