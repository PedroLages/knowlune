# Story 79.02: Sidebar Navigation Injection

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a plugin author,
I want my plugin to add items to the Knowlune sidebar,
so that users can navigate to my plugin's pages from the main navigation.

## Acceptance Criteria

1. **Given** a plugin manifest declares `capabilities: ["add:sidebar"]` and provides a sidebar item config (label, icon, route), **When** the plugin is activated, **Then** a new item appears in the sidebar navigation below the existing items, and the item uses the Lucide icon specified in the manifest.

2. **Given** a plugin's sidebar item is clicked, **When** navigation occurs, **Then** the sidebar item shows active state styling (matching existing active state patterns), and the plugin's custom page renders in the main content area.

3. **Given** a plugin is deactivated, **When** the sidebar re-renders, **Then** the plugin's sidebar item is removed.

## Dev Notes

### Architecture Decisions

- **Sidebar injection point**: Modify `SidebarContent` in `src/app/components/Layout.tsx` to render an additional "Plugins" navigation group. Do NOT modify the `navigationGroups` constant in `navigation.ts` -- instead, compose plugin items at render time via the hook.
- **Icon resolution strategy**: Lucide has 1500+ icons. Bundling all would bloat the app. Use a curated map of ~50 common icons (matching what Knowlune already imports) with lazy-load fallback for others. Plugin authors declare icon by name string (e.g., `"Timer"`, `"BookOpen"`).
- **NavigationItem compatibility**: Plugin sidebar items must conform to the existing `NavigationItem` interface (`name`, `path`, `icon: LucideIcon`). The hook maps manifest config to this interface.
- **Reactive updates**: The hook subscribes to EventBus events so the sidebar updates immediately when a plugin is activated/deactivated without requiring a page refresh.

### Dependencies (from E78 and E79-S01)

- `PluginManager` singleton (E78-S02) for querying active plugins
- EventBus plugin lifecycle events (E78-S05)
- Custom page routes (E79-S01) -- sidebar items link to plugin routes
- Plugin manifest types (E78-S01)

### Existing Patterns to Follow

- Navigation structure: `src/app/config/navigation.ts` defines `NavigationItem` (`name`, `path`, `icon: LucideIcon`) and `NavigationGroup` (`label`, `items[]`)
- `NavLink` component in Layout.tsx renders each item with icon, label, and active state via `getIsActive()`
- Progressive disclosure: `useProgressiveDisclosure` hook filters groups -- plugin group should bypass disclosure (always visible when plugin is active)
- Sidebar renders groups with `group.items.map(item => <NavLink>)` pattern

### Design Token Usage

- Plugin sidebar items use existing nav styling (no custom styles needed)
- Active state: existing `bg-brand-soft text-brand-soft-foreground` pattern from NavLink
- No hardcoded colors

### Project Structure Notes

- Hook in `src/plugins/hooks/` following existing hook organization
- Utility in `src/plugins/utils/` for icon resolver
- Do NOT modify `src/app/config/navigation.ts` directly -- plugin items are composed at runtime
