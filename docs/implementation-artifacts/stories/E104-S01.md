# Story 104.1: Link Formats Dialog — Book Pairing Entry Point

## Story

As a learner with both EPUB and audiobook versions of a book,
I want to link them as paired formats from my library,
So that I can switch between reading and listening at matching chapters.

## Acceptance Criteria

**AC1: Link Format entry point**
**Given** I have an unlinked EPUB in my library
**When** I open its context menu or detail view and tap "Link Format"
**Then** a dialog opens showing available unlinked audiobooks as pairing targets
**And** the dialog is keyboard-navigable and meets WCAG AA contrast

**AC2: Target list filtering**
**Given** I open the Link Format dialog
**When** the target list loads
**Then** only unlinked books of the opposite format are shown (EPUB sees audiobooks, audiobook sees EPUBs)
**And** books already linked to another book are excluded

**AC3: Auto chapter mapping with confidence**
**Given** I select an audiobook to pair with my EPUB
**When** the pairing is confirmed
**Then** `computeChapterMapping()` runs automatically and shows a confidence score
**And** if confidence > 0.85, the mapping auto-saves after a confirmation step
**And** if confidence <= 0.85, `ChapterMappingEditor` opens for manual review

**AC4: Post-pairing badges**
**Given** a successful pairing
**When** I return to my library
**Then** both books show "Also available as [Audiobook/EPUB]" badge
**And** the "Switch to Reading/Listening" button renders on both book views

**AC5: Unlink**
**Given** I have a linked book pair
**When** I open the Link Format dialog and tap "Unlink"
**Then** the pairing is removed, badges disappear, and format switch button is hidden

**AC6: Bidirectional entry**
**Given** I am on the audiobook detail/context menu
**When** I tap "Link Format"
**Then** the same dialog opens with unlinked EPUBs as targets
