# Story 63.1: Learner Profile Data Aggregation Layer

## Story

As the AI tutor system,
I want to aggregate quiz scores, knowledge map scores, flashcard weakness data, and study session summaries from existing data sources,
So that a comprehensive learner profile can be assembled for each tutor interaction.

## Acceptance Criteria

**Given** quiz attempt data exists for a course with 5 attempts where 3 have percentage < 70% on topics "CSS Grid" and "React Hooks"
**When** `aggregateQuizScores(courseId)` is called
**Then** it returns `{ avgPercentage: <number>, failedCount: 3, weakTopics: ["CSS Grid", "React Hooks"] }`

**Given** the `db.quizAttempts` table is empty (E52 not yet implemented)
**When** `aggregateQuizScores(courseId)` is called
**Then** it returns `null` without throwing an error

**Given** the Knowledge Map store reports 2 weak topics ("TypeScript Generics", "Async Patterns") and 1 fading topic ("State Management") for a course
**When** `aggregateKnowledgeScores(courseId)` is called
**Then** it returns `{ weakTopics: ["TypeScript Generics", "Async Patterns"], fadingTopics: ["State Management"] }`

**Given** the `useKnowledgeMapStore` state has no data for the given courseId
**When** `aggregateKnowledgeScores(courseId)` is called
**Then** it returns `null` without throwing an error

**Given** 8 flashcards for a course where 3 have FSRS fields with `lapses > 2` and `stability < 5` and 2 have `nextReviewAt` in the past
**When** `aggregateFlashcardWeakness(courseId)` is called
**Then** it returns `{ weakCardCount: 3, overdueCount: 2, weakTopicHints: [<first 3-5 words from weak card fronts>] }`

**Given** flashcards using SM-2 fields only (no `stability` field — E59 not yet implemented)
**When** `aggregateFlashcardWeakness(courseId)` is called
**Then** it detects weakness using SM-2 heuristics (`easeFactor < 1.8` AND `reviewCount > 3`) without errors

**Given** a course has no flashcards
**When** `aggregateFlashcardWeakness(courseId)` is called
**Then** it returns `null`

**Given** a course has 6 study sessions in the last 7 days totaling 4.5 hours with average quality score 72
**When** `aggregateStudySessions(courseId)` is called
**Then** it returns `{ totalHours: 4.5, sessionCount: 6, avgQuality: 72, daysSinceLastSession: <number> }`

**Given** a course has no study sessions in the last 7 days
**When** `aggregateStudySessions(courseId)` is called
**Then** it returns `null`

## Dev Notes

### Architecture Pattern
Follow `src/lib/qualityScore.ts` pattern exactly:
- Pure function module — types at top, constants next, exported functions below
- Each function independently testable with no side effects
- Functions compose — small utilities build up to orchestrator (in S02)
- No React hooks, no component code

### Critical Data Model Details

**QuizAttempt** (`src/types/quiz.ts`): Has `quizId`, `percentage`, `passed`, `completedAt`, `answers[]`. Does NOT have `courseId` — requires join through Quiz → Lesson → Course. Quiz schema has `lessonId` field.

**Flashcard** (`src/data/types.ts` line 477): Current SM-2 fields: `interval`, `easeFactor` (default 2.5, min 1.3), `reviewCount`, `nextReviewAt`. E59 will add FSRS fields: `stability`, `difficulty`, `reps`, `lapses`, `state`. Must detect which set exists at runtime.

**StudySession** (`src/data/types.ts` line 255): Has `courseId`, `startTime` (ISO 8601), `duration` (active seconds), `qualityScore` (0-100, optional), `qualityFactors`.

**Lesson** (`src/data/types.ts` line 52): Has `keyTopics: string[]` — this is how quiz topics are resolved.

### Dexie Access Pattern
- Use `db.flashcards.where('courseId').equals(courseId).toArray()` for indexed queries
- `db.studySessions` has `courseId` index
- `db.quizAttempts` does NOT have `courseId` index — must query all attempts and filter, or join through quizzes table
- All queries wrapped in try/catch returning `null` on failure

### useKnowledgeMapStore
Exists at `src/stores/useKnowledgeMapStore.ts` (implemented in E56). Import directly:

```typescript
import { useKnowledgeMapStore } from '@/stores/useKnowledgeMapStore'
const state = useKnowledgeMapStore.getState()
```
Still wrap in try/catch for resilience if store state is empty or malformed.
