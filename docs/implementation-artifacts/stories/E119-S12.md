# Story E119-S12: Operator Artifacts: ROPA, DPA, Breach Runbook, Sub-processors

## Story

As Pedro (controller/operator),
I want the full set of operator compliance documents — ROPA, DPA, sub-processor addenda, breach runbook, breach register — present and coherent,
so that a DPA inquiry, breach event, or external audit has documented answers.

## Acceptance Criteria

- **AC-1**: `docs/compliance/ropa.md` — table-driven, one row per processing activity: authentication, learning content storage, AI tutoring, billing, telemetry, sync. Columns per Art 30.
- **AC-2**: `docs/compliance/dpa-supabase.md` — real contract (markdown, signed date) covering: parties, instructions, confidentiality, Art 32 security measures, sub-processor list, breach notification SLA (24h processor→controller), return/deletion on termination.
- **AC-3**: `docs/compliance/subprocessors.md` — Cloudflare, Stripe, OpenAI, Anthropic DPAs accepted by reference with version links captured. Ollama + Whisper noted as first-party infra (not sub-processors).
- **AC-4**: `docs/compliance/breach-runbook.md` — detection signals, triage steps, severity classification, 72h DPA notification template (Art 33), user notification template (Art 34 for high-risk), decision tree for notify-or-not. Includes tabletop rehearsal notes.
- **AC-5**: `docs/compliance/breach-register.md` — empty table template with pseudonymised schema.
- **AC-6**: `src/lib/compliance/subprocessorRegistry.ts` lists all sub-processors with name, role, DPA link, data transferred.
- **AC-7**: `scripts/compliance/verify-subprocessors.ts` scans `package.json` + env var references; compares against `subprocessorRegistry.ts` + `subprocessors.md`; exits non-zero on unlisted packages.
- **AC-8**: CI runs `verify-subprocessors.ts` on PRs; new dependency without sub-processor registration fails CI.
- **AC-9**: A tabletop breach rehearsal walk-through is recorded in `breach-runbook.md` appendix.
