# Story 86.3: Cross-Highlight Search

Status: done

## Story

As a learner,
I want to search across all my book highlights using text and semantic search,
so that I can find relevant passages across my entire reading history.

## Acceptance Criteria

**Given** the user has highlights across multiple books
**When** the user searches for a term in the existing Knowlune search
**Then** book highlights matching the search term appear in results alongside course notes and other content
**And** each highlight result shows: matched text with search term highlighted, book title, chapter, and highlight color indicator
**And** tapping a result navigates to that highlight in the EPUB reader

**Given** the existing vector search infrastructure (from prior epics)
**When** a new book highlight is created
**Then** the highlight text is indexed in the existing embedding infrastructure for semantic search
**And** semantic search queries return relevant book highlights alongside course content

## Tasks / Subtasks

- [ ] Task 1: Add book highlights to text search (AC: 1)
  - [ ] 1.1 Extend the existing search service/hook to query `bookHighlights` table by `text` field
  - [ ] 1.2 Join results with `books` table for book title and author metadata
  - [ ] 1.3 Format highlight results: matched text with `<mark>` highlighting, book title, chapter href label, color dot indicator
  - [ ] 1.4 Merge highlight results into the unified search results list, interleaved with course notes by relevance

- [ ] Task 2: Create HighlightSearchResult component (AC: 1)
  - [ ] 2.1 Create `src/app/components/search/HighlightSearchResult.tsx`
  - [ ] 2.2 Display: color dot (size-3 rounded-full matching highlight color), matched text preview (2-line clamp with search term bolded), book title, chapter name
  - [ ] 2.3 Click handler: navigate to `/library/{bookId}?cfi={encodedCfi}` to open book at highlight location

- [ ] Task 3: Index highlights in vector search (AC: 2)
  - [ ] 3.1 When a BookHighlight is created, emit event or call embedding service to index the highlight text
  - [ ] 3.2 Use existing embedding infrastructure from prior epics (E68 EmbeddingProvider)
  - [ ] 3.3 Store embedding with metadata: `{ type: 'book-highlight', bookId, highlightId }`
  - [ ] 3.4 When a BookHighlight is deleted, remove its embedding from the vector store

- [ ] Task 4: Semantic search integration (AC: 2)
  - [ ] 4.1 Extend existing semantic search to include `book-highlight` type in results
  - [ ] 4.2 Format semantic results same as text search results (HighlightSearchResult component)
  - [ ] 4.3 Merge semantic results with text results, deduplicating by highlightId

- [ ] Task 5: Tests (AC: 1, 2)
  - [ ] 5.1 Test text search returns matching highlights with correct metadata
  - [ ] 5.2 Test highlight creation triggers embedding indexing
  - [ ] 5.3 Test highlight deletion removes embedding
  - [ ] 5.4 Test navigation from search result to book reader at correct position

## Dev Notes

### Architecture Patterns

- **Search**: The existing search may be in the Layout header component or a dedicated search page. Check `src/app/components/Layout.tsx` header search bar implementation.
- **Vector search**: E68 introduced `EmbeddingProvider` abstraction in `src/services/`. Highlights should use the same provider to generate embeddings.
- **Event bus**: `highlight:created` event (defined in architecture doc) should trigger embedding indexing. Subscribe in the embedding service.
