# Story 77.5: Restore Flow with Backup History

Status: ready-for-dev

## Story

As a connected premium user,
I want to see my backup history and restore from any backup,
so that I can recover from data loss or set up a new device with my learning progress intact.

## Acceptance Criteria

1. **Given** a connected premium user views the backup panel **When** the BackupHistorySection renders **Then** all `.knowlune.json` files in `Knowlune/backups/` are listed sorted newest first, each entry shows date/time, file size, and backup type badge ("Auto" or "Manual"), and skeleton loading (3 items) is shown while fetching.

2. **Given** no backups exist in Drive **When** the backup history renders **Then** an empty state shows "No backups yet. Tap 'Back Up Now' to create your first backup."

3. **Given** a user clicks "Restore" on a backup **When** the RestoreConfirmationDialog opens **Then** it displays the backup date, time, and size; shows "This will replace ALL your current data:" with current data counts (courses, notes, flashcards, sessions) fetched from IndexedDB; a pre-restore safety backup checkbox (checked by default, labeled "Create a safety backup before restoring (Recommended)") (Adversarial Blocker #1); warning text "This action cannot be undone."; and actions "Cancel" and "Restore Backup" (`variant="destructive"`).

4. **Given** the user confirms restore with safety backup checkbox checked **When** the restore process begins **Then** a safety backup is created first (uploaded as `{timestamp}-pre-restore-safety.knowlune.json`), and the dialog shows progress: "Creating safety backup..." -> "Downloading backup..." -> "Validating data..." -> "Restoring...", with dialog buttons disabled during the process.

5. **Given** the user confirms restore with safety backup checkbox unchecked **When** the restore begins **Then** the safety backup step is skipped and the dialog proceeds directly to "Downloading backup..."

6. **Given** the restore completes successfully **When** the success state is reached **Then** the dialog closes, a success toast shows "Restored successfully from {date} backup", all Zustand stores are refreshed from IndexedDB (app state reload), and the backup history list refreshes.

7. **Given** the restore fails (Dexie transaction rolled back) **When** the error is caught **Then** a toast error shows "Restore failed. Your data has not been changed." and the dialog returns to its initial state (buttons re-enabled).

8. **Given** the backup's schema version is newer than the current app **When** the validation step detects mismatch **Then** a toast error shows "This backup was made with a newer version of Knowlune. Please update the app to restore." and the dialog closes without attempting restore.

9. **Given** a user opens the overflow menu on a backup item **When** they click "Delete backup" **Then** a DeleteBackupDialog confirms: "This will permanently remove the backup from {date} ({size}) from your Google Drive." and confirming deletes the file from Drive, refreshes the list, and shows toast "Backup deleted".

## Dev Notes

### Architecture Constraints

- **Pre-restore safety backup** is Adversarial Blocker #1 -- the checkbox must default to checked
- **Safety backup filename**: `{timestamp}-pre-restore-safety.knowlune.json` for identification in history
- **Full data replace** (not merge) -- Dexie transaction clears all tables then inserts (Decision 6)
- **Schema validation** runs before any IndexedDB writes -- reject newer versions, migrate older ones
- **App state refresh**: After restore, all Zustand stores must reload from IndexedDB

### Backup Type Detection

Determine "Auto" vs "Manual" badge from filename:
- Contains `-pre-restore-safety`: show "Safety" badge
- Contains `-auto`: show "Auto" badge (set by Story 77.6)
- Default: show "Manual" badge

### Current Data Counts for Dialog

Fetch via Dexie before showing dialog:
```typescript
const counts = {
  courses: await db.importedCourses.count(),
  notes: await db.notes.count(),
  flashcards: await db.challenges.count(),
  sessions: await db.studySessions.count(),
}
```
