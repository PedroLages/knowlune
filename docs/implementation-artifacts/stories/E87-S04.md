# Story 87.4: Chapter Navigation and Bookmarks

Status: ready-for-dev

## Story

As a learner,
I want to navigate between audiobook chapters and create bookmarks with notes at specific timestamps,
so that I can jump to specific sections and mark important moments for later reference.

## Acceptance Criteria

**Given** the expanded audiobook player is showing
**When** the user views the chapter list (scrollable section below controls)
**Then** completed chapters show a `Check` icon in `text-success`, the current chapter shows `Play` icon in `text-brand` with `font-medium`, and upcoming chapters show default text
**And** each row shows chapter title, duration, and a border-bottom separator
**And** tapping a chapter loads and plays that chapter's MP3 file

**Given** the audiobook player is active
**When** the user taps the bookmark button
**Then** an AudioBookmark record is created in Dexie at the current timestamp
**And** an optional note input slides down with "Add a note..." placeholder
**And** a toast confirms: "Bookmark saved at {time}"
**And** bookmarks are viewable in the header menu under "Bookmarks" (similar to highlight list pattern)

## Dev Notes

### Architecture Patterns

- **Chapter list**: Similar to TOC in EPUB reader (E84). Same Sheet pattern from `src/app/components/ui/sheet.tsx`.
- **Bookmark storage**: Uses `audioBookmarks` Dexie table (added in E87-S01).
- **Auto-advance**: When `audio.ended` fires on current chapter, automatically load and play next chapter. At last chapter, stop playback.

### UX Design (UX-DR14)

- Chapter list: scrollable section below main controls in expanded player view
- Each row: title + duration + status icon
- Bookmark button in action bar alongside speed/sleep controls

### Dependencies

- **Hard dependency on E87-S01**: `audioBookmarks` Dexie table
- **Hard dependency on E87-S02**: AudiobookRenderer, useAudioPlayer hook
