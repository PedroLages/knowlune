# Story 114.02: Continuous Scroll Mode

## Story

As a reader who prefers a flowing reading experience,
I want to toggle continuous scroll mode in the EPUB reader,
so that I can scroll through content seamlessly instead of turning pages one at a time.

## Acceptance Criteria

- **AC-1**: Reader Settings panel includes a "Continuous Scroll" toggle (on/off switch).
- **AC-2**: When enabled, the EPUB reader switches to `scrolled-doc` flow so content scrolls vertically without page breaks.
- **AC-3**: When disabled, the reader returns to `paginated` flow with the standard left/right page-turn navigation.
- **AC-4**: In scroll mode, left/right tap navigation zones are hidden — native scroll replaces page turning.
- **AC-5**: The center tap zone (header/footer toggle) remains active in scroll mode.
- **AC-6**: The scroll mode setting is persisted to localStorage and survives app reload.
- **AC-7**: Theme styles are re-applied after flow change (some epub.js versions reset theme on flow switch).
- **AC-8**: The rendition is resized after flow change to force re-layout of content at the correct dimensions.

## Implementation Notes

- `scrollMode` added to `ReaderSettings` interface alongside existing accessibility fields; persisted via `loadSettings`/`saveSettings`.
- `epubOptions` now depends on `scrollMode` via `useMemo([scrollMode])` — ensures correct initial flow on mount and when toggling.
- A dedicated `useEffect` calls `rendition.flow()` at runtime when `scrollMode` changes, followed by `rendition.resize()` to re-layout and `applyTheme()` to restore styles.
- `onTouchStart`/`onTouchEnd` handlers are set to `undefined` when `scrollMode` is true — native scroll events are not intercepted.
- Prev/next tap zones conditionally rendered: only shown when `!scrollMode`.
- Center tap zone remains always rendered at its fixed 34% width positioned from `left-[33%]`.

## Challenges and Lessons Learned

- **epub.js flow switching at runtime**: `rendition.flow()` changes the layout but does not automatically resize or re-apply the theme. A sequence of `flow()` → `resize()` → `applyTheme()` is required to produce a clean switch without visual artifacts.
- **Tap zone layout in conditional rendering**: When prev/next zones are conditionally removed, the center zone positioning must use absolute coordinates (not flex-based siblings) to remain correctly centered at all times.
- **Touch handler passthrough**: Setting event handlers to `undefined` on the container is the correct pattern to let native scroll events pass through to the epub.js iframe in scroll mode.
