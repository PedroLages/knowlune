# Story 88.2: OPDS Catalog Browsing and Import

Status: ready-for-dev

## Story

As a power user,
I want to browse my Calibre-Web library from within Knowlune and add books to my library,
so that I can select books for reading without manually downloading and importing files.

## Acceptance Criteria

**Given** the user has a connected OPDS catalog
**When** the user selects "Browse Catalog" on the Library page
**Then** the OPDS catalog entries display in a browsable grid/list showing: cover image, title, author, format, and summary for each entry
**And** pagination handles large catalogs (OPDS `next` link navigation)

**Given** the user is browsing an OPDS catalog
**When** the user taps "Add to Library" on a book entry
**Then** a Book record is created with `source: { type: 'remote', url: downloadUrl, auth: authIfConfigured }`
**And** no file is copied to OPFS (the book streams on demand from the remote URL)
**And** metadata and cover are populated from the OPDS entry data
**And** the book appears in the Library alongside locally-imported books with a "Remote" badge

## Dev Notes

### OPDS Entry Parsing

```xml
<entry>
  <title>The Art of War</title>
  <author><name>Sun Tzu</name></author>
  <summary>Ancient Chinese military treatise...</summary>
  <link rel="http://opds-spec.org/acquisition" href="/get/epub/42" type="application/epub+zip"/>
  <link rel="http://opds-spec.org/image" href="/cover/42.jpg" type="image/jpeg"/>
  <link rel="http://opds-spec.org/image/thumbnail" href="/thumb/42.jpg" type="image/jpeg"/>
  <id>urn:uuid:some-uuid</id>
</entry>
```

Key parsing rules:
- Acquisition link: `rel="http://opds-spec.org/acquisition"` — the download URL
- Cover: `rel="http://opds-spec.org/image"` — full-size cover
- Thumbnail: `rel="http://opds-spec.org/image/thumbnail"` — smaller cover for grid
- Navigation: `rel="subsection"` — link to sub-feed (category/author browse)
- Pagination: `rel="next"` — next page of results

### Remote Source Books

Books added from OPDS have `source.type: 'remote'`. They:
- Do NOT use OPFS storage (no local file)
- Stream content from `source.url` when opened (handled by E88-S03)
- Store auth credentials from the catalog connection
- Show a cloud/remote badge in the Library
