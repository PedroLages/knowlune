# Story 77.6: Scheduled Automatic Backups

Status: ready-for-dev

## Story

As a connected premium user,
I want to schedule automatic daily or weekly backups,
so that my data is backed up without manual intervention.

## Acceptance Criteria

1. **Given** a connected premium user views the BackupSettingsSection **When** the auto-backup toggle is off (default) **Then** only the Switch toggle is shown with label "Automatic backups".

2. **Given** the user enables the auto-backup toggle **When** the toggle transitions to on **Then** a frequency selector appears with slide-down animation (200ms), showing pill-style RadioGroup buttons "Daily" | "Weekly" (default: Weekly), with active pill using `bg-brand text-brand-foreground rounded-full` and inactive using `bg-muted text-muted-foreground rounded-full`.

3. **Given** the user selects a backup frequency **When** the preference is saved **Then** the schedule is persisted in IndexedDB settings table, and the Service Worker registers a periodic background sync (if supported).

4. **Given** the browser supports Periodic Background Sync (Chrome PWA) **When** a scheduled backup time arrives **Then** the Service Worker triggers `backupToDrive()`, and if the token is expired the backup is skipped with a notification "Reconnect to resume automatic backups".

5. **Given** the browser does NOT support Periodic Background Sync **When** the app is open at scheduled time **Then** a `setInterval` fallback triggers the backup, and the Settings page shows a note: "Automatic backups run when the app is open".

6. **Given** a scheduled backup fails **When** the error is caught **Then** a toast notification shows "Scheduled backup failed" with a "Retry" action, the failure does not block app functionality, and the next scheduled attempt proceeds normally.

7. **Given** the user views the last backup timestamp **When** a backup (manual or scheduled) has completed **Then** the display shows relative time (e.g., "Last backup: 2 hours ago") with the full timestamp in a tooltip.

8. **Given** a user disconnects Google Drive **When** the schedule is active **Then** the schedule is automatically disabled and the toggle resets to off.

## Dev Notes

### Architecture Constraints

- **No "Monthly" option** -- removed per Adversarial Review #6 (Periodic Background Sync cannot reliably sustain monthly intervals)
- **Service Worker** must already exist for PWA support -- extend it, don't create a new one
- **Periodic Background Sync** requires sufficient engagement score in Chrome -- not guaranteed
- **Fallback**: `setInterval` when Periodic Background Sync unavailable -- check `lastBackupAt` against frequency
- **Scheduled backup reuses `backupToDrive()`** from Story 77.3 -- same function, different trigger
- **Schedule persisted in IndexedDB settings** using existing `getSettings()`/`saveSettings()` pattern

### Auto-Backup Filename Convention

Scheduled backups use filename: `YYYY-MM-DD_HHmmss-auto.knowlune.json`
- The `-auto` suffix distinguishes from manual backups in backup history (Story 77.5 reads this for badge display)
