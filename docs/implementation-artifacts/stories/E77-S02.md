## Story

As a premium user,
I want Knowlune to obtain a Google Drive scoped access token,
so that subsequent Drive API calls can be authenticated.

## Acceptance Criteria

1. **Given** a signed-in user has granted the `https://www.googleapis.com/auth/drive.file` OAuth scope **When** `getDriveToken()` is called **Then** it returns the `provider_token` from the current Supabase session.

2. **Given** the Supabase session has no `provider_token` **When** `getDriveToken()` is called **Then** it attempts a session refresh via `supabase.auth.refreshSession()` and, if a `provider_token` is now present, returns it.

3. **Given** refreshing the session yields no `provider_token` **When** `getDriveToken()` is called **Then** it returns `null`.

4. **Given** Supabase is not configured (`supabase` is null) **When** `getDriveToken()` is called **Then** it returns `null`.

5. **Given** no session exists **When** `getDriveToken()` is called **Then** it returns `null`.

6. **Given** a Drive API call returns 401 **When** the caller needs a fresh token **Then** `refreshDriveToken()` refreshes the session and returns the new `provider_token`, or `null` if still unavailable.

7. **Given** `supabase.auth.refreshSession()` throws (network error) **When** `getDriveToken()` or `refreshDriveToken()` is called **Then** the error is caught and `null` is returned gracefully.

## Dev Notes

### Architecture Constraints

- **`getDriveToken()` reads the session from `useAuthStore`** (Zustand) for synchronous access
- **`refreshDriveToken()` is the 401 recovery path** — callers should use it instead of re-invoking `getDriveToken()` on expired tokens
- **No Drive API calls in this story** — this is purely the token acquisition layer
