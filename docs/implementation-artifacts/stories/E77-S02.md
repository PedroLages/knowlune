# Story 77.2: Google Drive Service Layer

Status: ready-for-dev

## Story

As a premium user,
I want Knowlune to manage folders and files in my Google Drive,
so that my backups are organized in a `Knowlune/backups/` folder structure I can browse.

## Acceptance Criteria

1. **Given** a connected user initiates any Drive operation **When** the `Knowlune/` folder does not exist in Drive root **Then** the service creates it via `POST /drive/v3/files` with `mimeType: application/vnd.google-apps.folder` and caches the folder ID in `useGoogleDriveStore.knowluneFolderId`.

2. **Given** the `Knowlune/` folder exists **When** the `backups/` subfolder does not exist **Then** the service creates it as a child of `Knowlune/` and caches the folder ID in `useGoogleDriveStore.backupsFolderId`.

3. **Given** a Drive API call returns a 5xx or network error **When** `withRetry()` executes **Then** the call is retried with exponential backoff (1s, 2s, 4s) up to 3 retries with jitter, and after 3 failures the error is thrown with a user-friendly message.

4. **Given** a Drive API call returns a 401 **When** the error handler processes the response **Then** the token is cleared from the store and a `DriveAuthError` is thrown immediately (no retry).

5. **Given** a Drive API call returns a 403 with `storageQuotaExceeded` reason **When** the error handler processes **Then** a `DriveQuotaError` is thrown immediately with "Google Drive storage is full".

6. **Given** a Drive API call returns a 403 without quota reason **When** the error handler processes **Then** a `DrivePermissionError` is thrown with "Access denied. Your organization may restrict third-party apps."

7. **Given** the `DELETE` endpoint returns 204 No Content **When** `driveRequest()` processes the response **Then** it returns `undefined` without attempting `response.json()` (Adversarial #10 fix).

8. **Given** the download endpoint is called with `alt=media` **When** the response contains raw file content **Then** `downloadFile()` uses `response.text()` instead of `response.json()` (Adversarial #10 fix).

9. **Given** a user disconnects and reconnects with a different Google account **When** the reconnection completes **Then** cached folder IDs (`knowluneFolderId`, `backupsFolderId`) are cleared and re-resolved (Adversarial #11 fix).

## Dev Notes

### Architecture Constraints

- **No npm dependencies** -- raw `fetch()` to Drive REST API v3
- **All methods accept token as parameter or read from `useGoogleDriveStore`**
- **Folder IDs cached in Zustand store** for session (avoid redundant lookups, NFR-3)
- **`driveRequest()` must handle 204 No Content and non-JSON responses correctly** (Adversarial #10)

### Drive REST API v3 Endpoints

| Operation | Method | URL |
|-----------|--------|-----|
| Create folder | POST | `https://www.googleapis.com/drive/v3/files` |
| Upload (multipart) | POST | `https://www.googleapis.com/upload/drive/v3/files?uploadType=multipart` |
| Upload (resumable init) | POST | `https://www.googleapis.com/upload/drive/v3/files?uploadType=resumable` |
| Upload (resumable put) | PUT | `{sessionUri}` |
| List files | GET | `https://www.googleapis.com/drive/v3/files?q=...` |
| Download file | GET | `https://www.googleapis.com/drive/v3/files/{id}?alt=media` |
| Delete file | DELETE | `https://www.googleapis.com/drive/v3/files/{id}` |
| Get quota | GET | `https://www.googleapis.com/drive/v3/about?fields=storageQuota` |

### Multipart Upload Format

```
--boundary
Content-Type: application/json; charset=UTF-8

{"name": "filename.knowlune.json", "parents": ["folderId"]}
--boundary
Content-Type: application/json

{...backup data...}
--boundary--
```

Content-Type header: `multipart/related; boundary={boundary}`

### Error Hierarchy (from Story 77.1)

Uses `DriveError`, `DriveAuthError`, `DriveQuotaError`, `DrivePermissionError`, `DriveApiError` from `src/services/errors/driveErrors.ts`.

### Retry Pattern

```typescript
// Exponential backoff with jitter
const delay = baseDelayMs * Math.pow(2, attempt) + Math.random() * 500
// Skip retry for auth and quota errors (throw immediately)
```
