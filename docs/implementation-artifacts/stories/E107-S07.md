# Story 107.07: Fix M4B Cover Art Preview

## Story

As a reader,
I want to see the cover art preview for M4B audiobooks in the import form,
so that I can confirm the correct book is being imported before adding it to my library.

## Acceptance Criteria

- Given an M4B file with embedded cover art, when I open the audiobook import form, then the cover image is displayed in the form
- Given an M4B file without cover art, when I open the import form, then a placeholder is shown instead
- Given I close the import form, when it unmounts, then the blob URL for the cover image is properly revoked to prevent memory leaks

## Tasks / Subtasks

- [x] Task 1: Extract cover art from parsed M4B metadata and display in AudiobookImportFlow form
  - [x] 1.1 Add cover preview section with image element and placeholder fallback
  - [x] 1.2 Manage blob URL lifecycle via useEffect cleanup
  - [x] 1.3 Add data-testid attributes for testability

## Implementation Notes

- Modified `src/app/components/library/AudiobookImportFlow.tsx`
- Cover art extracted from `parsedBook.cover` (Uint8Array) and converted to blob URL
- Blob URL created on mount / cover change, revoked on unmount via useEffect return
- Design tokens used throughout (`border-border`, `bg-muted/30`, `text-muted-foreground`)
- Accessible markup: `alt` attribute on img, `aria-label` on placeholder

## Testing Notes
