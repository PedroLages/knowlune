# Story 107.3: Fix TOC Loading and Fallback

## Story

As a reader,
I want the table of contents to load reliably with proper fallbacks when unavailable,
so that I can navigate chapters confidently without encountering broken or missing navigation.

## Acceptance Criteria

- **AC-1**: TOC loading state is tracked and displayed in the TableOfContents panel (loading indicator while `tocChanged` callback is pending)
- **AC-2**: Empty TOC (length === 0) displays a user-friendly message in the TableOfContents panel
- **AC-3**: TOC that fails to load or times out (e.g., corrupted EPUB navigation) gracefully falls back to empty state
- **AC-4**: Chapter tracking in BookReader works even when TOC is unavailable (falls back to displaying progress percentage instead of chapter name)
- **AC-5**: TableOfContents panel button in ReaderHeader remains enabled but shows empty state when TOC is unavailable

## Implementation Notes

- **TOC Loading State**: The current implementation doesn't track whether TOC is loading. The `tocChanged` callback from `react-reader` may not fire for EPUBs with no navigation, or it may fire with an empty array. We need explicit loading state to distinguish "loading" from "loaded (empty)" from "error".

- **Timeout Consideration**: Some EPUBs may have large or complex navigation structures that take time to parse. A 5-second timeout provides a balance between allowing legitimate loads and preventing indefinite loading states.

- **Chapter Tracking Fallback**: When TOC is unavailable, the ReaderHeader currently shows `currentChapter` which may be undefined. The fallback should display progress percentage (e.g., "45%") instead of chapter name.

- ** epub.js Behavior**: The `tocChanged` callback is called by epub.js when navigation is loaded. For EPUBs without navigation, this may still fire with an empty array. We should handle both "never fires" and "fires with empty array" cases.

## Challenges and Lessons Learned

### Implementation Challenges

1. **TOC Load Detection**: There's no built-in "error" callback for `tocChanged` in `react-reader`. We need to use a timeout pattern to detect when TOC loading has stalled or failed.

2. **Loading vs Empty State Distinguishing between "still loading" and "loaded but empty" requires explicit state tracking since `tocChanged([])` looks the same as "not loaded yet".

3. **Chapter Name Display**: ReaderHeader shows `currentChapter` in the header. When TOC is unavailable, this needs to fall back to progress percentage without breaking the layout.

### Patterns Worth Remembering

1. **Timeout Pattern for Async Callbacks**: When using third-party library callbacks that may not fire (like `tocChanged`), always pair with a timeout effect to set a fallback state.

2. **Graceful Degradation**: When a feature (TOC) is unavailable, ensure the UI still functions. The reader should work without TOC — chapter navigation just becomes less convenient.

3. **Loading State Visibility**: Users should see feedback when asynchronous operations are in progress. A loading spinner in the TOC panel communicates that navigation data is being fetched.
