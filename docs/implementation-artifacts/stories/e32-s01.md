# Story 32.01: Add List Virtualization with @tanstack/react-virtual

## Story

As a learner with a growing course library,
I want large lists to render efficiently using virtualization,
so that the UI stays responsive even with hundreds of items.

## Acceptance Criteria

- AC1: Given the Courses page with 500+ items, when the page renders, then only visible items + overscan buffer are in the DOM
- AC2: Given the Notes page with 1000+ items, when scrolling through notes, then TipTap-heavy cards are virtualized to prevent memory issues
- AC3: Given the Authors page with 100+ authors, when the grid renders, then only visible author cards are in the DOM
- AC4: Given any virtualized list, when filters/search reduce items, then scroll position resets and the item count updates correctly
- AC5: Given variable-height cards (tags, progress bars differ per card), when scrolling, then the virtualizer handles dynamic row heights correctly

## Implementation Notes

- Using @tanstack/react-virtual for lightweight virtualization
- Variable-height support via `measureElement` callback
- Overscan of 5 items for smooth scrolling experience
- Grid virtualization achieved by virtualizing rows, each containing multiple columns
- Scroll position resets on filter/search changes via virtualizer.scrollToOffset(0)

## Testing Notes

- Manual testing with dev tools to verify DOM element count stays bounded
- Filter/search should reset scroll position to top
