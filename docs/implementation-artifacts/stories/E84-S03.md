# Story 84.3: Reading Themes and Font Controls

Status: done

## Story

As a learner,
I want to customize the reading appearance with themes, font size, and font family,
so that I can read comfortably in different lighting conditions and according to my preferences.

## Acceptance Criteria

1. **Given** the reader header menu is open **When** the user selects "Settings" **Then** a reader settings panel appears (Sheet from bottom on mobile, Popover on desktop) **And** the panel contains: theme selector (Light/Sepia/Dark as radio pill cards), font size slider (80%-200% in 10% steps with A-/A+ buttons), font family select (System, Serif, Sans, Mono), and line height select (1.2/1.5/1.8/2.0)

2. **Given** the user selects the Light theme **When** the theme is applied **Then** the reader background becomes `#FAF5EE`, text becomes `#1a1a1a`, and UI chrome uses Knowlune default tokens **And** contrast ratio is 14.5:1 (passes AAA)

3. **Given** the user selects the Sepia theme **When** the theme is applied **Then** the reader background becomes `#F4ECD8`, text becomes `#5B4636` **And** contrast ratio is 6.8:1 (passes AA)

4. **Given** the user selects the Dark theme **When** the theme is applied **Then** the reader background becomes `#1a1a1a`, text becomes `#d4d4d4` **And** contrast ratio is 12.1:1 (passes AAA)

5. **Given** the user adjusts any reading setting **When** the change is applied **Then** the setting takes effect immediately via `rendition.themes.override()` (font size), `rendition.themes.font()` (family) **And** all reader settings are persisted in `useReaderStore` and survive page reload via localStorage

## Dev Notes

### Architecture Constraints

- **epub.js themes API** (architecture decision 7): `rendition.themes.override()` injects CSS into the epub.js iframe. This is the correct API — do NOT attempt to modify iframe content DOM directly.
- **Reader chrome** (header/footer) must match the selected reading theme. The theme-aware background styles were stubbed in E84-S01 — this story fills them in with actual theme switching.
- **Design tokens for Knowlune UI**: The settings panel itself (Sheet/Popover) uses standard Knowlune design tokens (`bg-popover`, `text-foreground`). Only the reader content area and reader chrome use the reader-specific hex values.
- **Font rendering**: Add `-webkit-font-smoothing: antialiased` to epub.js content via theme override for crisp text.

### Key Dependencies (from E84-S01)

- EpubRenderer with rendition ref via `getRendition` callback
- useReaderStore with theme/font state fields
- ReaderHeader dropdown menu with "Settings" item
- ReaderHeader/ReaderFooter with theme-aware background placeholders

### shadcn Components Used

- `Sheet` (bottom variant for mobile)
- `Popover` (for desktop)
- `Slider` (font size)
- `Select` (font family, line height)
- `Button` (A-/A+ increment)
- `RadioGroup` or custom radio pills (theme selector)

### Accessibility

- Settings panel: focus trap when open
- Theme selector: `role="radiogroup"` with `aria-label="Reading theme"`
- Font size slider: `aria-label="Font size"` with `aria-valuemin="80"` `aria-valuemax="200"`
- All controls keyboard navigable
- `prefers-reduced-motion`: skip theme transition animations

### WCAG Compliance Verification

All three themes must pass WCAG AA minimum contrast (4.5:1 for body text):
- Light: 14.5:1 (AAA)
- Sepia: 6.8:1 (AA)
- Dark: 12.1:1 (AAA)
