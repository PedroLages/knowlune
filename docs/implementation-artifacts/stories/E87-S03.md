# Story 87.3: Speed Control and Sleep Timer

Status: ready-for-dev

## Story

As a learner,
I want to adjust playback speed without pitch distortion and set a sleep timer,
so that I can listen faster during focused sessions and fall asleep listening without the audiobook running all night.

## Acceptance Criteria

**Given** the audiobook player is active
**When** the user taps the speed button
**Then** a Popover appears with 9 speed options: 0.5x, 0.75x, 1.0x, 1.25x, 1.5x, 1.75x, 2.0x, 2.5x, 3.0x
**And** the active speed shows a checkmark and `text-brand font-medium`
**And** selecting a speed sets `audio.playbackRate` with `audio.preservesPitch = true`
**And** the speed preference persists in `useAudioPlayerStore`

**Given** the audiobook player is active
**When** the user taps the sleep button
**Then** a Popover appears with options: 15 minutes, 30 minutes, 45 minutes, 60 minutes, End of chapter, Off
**And** when a timer is active, the sleep button shows remaining time as a badge
**And** when the timer expires, audio fades out over 5 seconds and pauses
**And** "End of chapter" pauses when the current chapter's audio file ends
**And** on next app open after sleep timer end, a toast shows "Sleep timer ended"

### Audio API Details

```typescript
// Speed control
audioElement.playbackRate = 1.5
audioElement.preservesPitch = true // Prevents chipmunk effect

// Fade-out (simplified)
function fadeOut(audio: HTMLAudioElement, durationMs: number) {
  const startVolume = audio.volume
  const startTime = performance.now()
  function tick() {
    const elapsed = performance.now() - startTime
    const progress = Math.min(elapsed / durationMs, 1)
    audio.volume = startVolume * (1 - progress)
    if (progress < 1) requestAnimationFrame(tick)
    else { audio.pause(); audio.volume = startVolume }
  }
  requestAnimationFrame(tick)
}
```

### Browser Compatibility (NFR22)

- `preservesPitch` is supported in Chrome 86+, Firefox 101+, Safari 15.4+
- For older browsers, speed change will alter pitch (acceptable degradation)
