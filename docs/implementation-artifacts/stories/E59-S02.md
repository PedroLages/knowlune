# Story 59.02: FSRS Algorithm Wrapper

## Story

As a learner,
I want my flashcard scheduling powered by the FSRS algorithm,
so that I get more accurate review intervals based on proven memory science.

## Acceptance Criteria

**Given** the SM-2 implementation in `src/lib/spacedRepetition.ts`
**When** this story is implemented
**Then** `spacedRepetition.ts` is rewritten as a thin wrapper around `ts-fsrs`
**And** `calculateNextReview(card, rating, now)` returns FSRS scheduling fields (stability, difficulty, reps, lapses, state, elapsed_days, scheduled_days, due, last_review)
**And** `predictRetention({ last_review, stability }, now)` uses FSRS power-law forgetting curve instead of SM-2 exponential curve
**And** `isDue(card, now)` uses the `due` field (not `nextReviewAt`)
**And** the FSRS instance is configured with `request_retention: 0.9`, `maximum_interval: 365`, `enable_fuzz: true`, `enable_short_term: true`
**And** only `spacedRepetition.ts` imports from `ts-fsrs` (single gateway pattern enforced)
**And** rating string values ('again', 'hard', 'good', 'easy') are converted to ts-fsrs `Rating` enum only inside the wrapper
**And** ISO string dates are converted to `Date` objects only when passing to ts-fsrs API

**Given** a card with `last_review: undefined` (new card, never reviewed)
**When** `calculateNextReview` is called
**Then** it handles the undefined date without producing `Invalid Date`
**And** uses `createEmptyCard()` defaults from ts-fsrs

**Given** `enable_fuzz` is true in production
**When** unit tests run
**Then** a test-specific FSRS instance with `enable_fuzz: false` is used for deterministic assertions

## Implementation Notes

- Used `fsrs.next(card, now, grade)` instead of `fsrs.repeat()` for single-grade scheduling (more efficient)
- `forgetting_curve()` from ts-fsrs returns 0-1 probability; wrapper multiplies by 100 for percentage
- `createEmptyCard(now)` handles brand-new cards — sets due=now, stability/difficulty=0, state=New(0)
- Production FSRS: `enable_fuzz: true` adds natural variation to intervals; test FSRS: `enable_fuzz: false` for deterministic assertions
- `last_review` is always returned as ISO string (falls back to `now.toISOString()` if ts-fsrs returns undefined)
- Old `SpacedRepetitionState` interface removed; new `FSRSCardState` interface matches FSRS field structure
- Old `calculateNewEaseFactor`, `QUALITY_MAP`, `FIRST_REVIEW_INTERVALS`, `MIN_EASE_FACTOR` all removed (SM-2 artifacts)
- Removed `date-fns` import (`addDays`) — no longer needed since ts-fsrs handles date arithmetic internally

## Testing Notes

## Challenges and Lessons Learned

- ts-fsrs `fsrs.next()` is more efficient than `fsrs.repeat()` when you only need one grade's result (avoids computing all 4 grades)
- The `forgetting_curve()` function is exported directly from ts-fsrs — no need to reimplement the power-law formula
- FSRS new cards schedule very short initial intervals (1-10 minutes for again/hard/good) when `enable_short_term: true` — this is by design for learning steps
