# Story 88.3: Remote EPUB Streaming

Status: done

## Story

As a power user,
I want to read EPUBs streamed from my Calibre-Web server without downloading them,
so that I can read from my 500+ book library without consuming local storage.

## Acceptance Criteria

**Given** the user opens a book with `source.type: 'remote'`
**When** the BookReader page loads
**Then** the EPUB is fetched from the remote URL via `BookContentService.getEpubContent()` and passed to epub.js as an ArrayBuffer
**And** if the server requires authentication, the stored auth credentials are included in the request
**And** a loading indicator shows during fetch

**Given** the user opens a remote book and the server is unreachable
**When** the fetch fails
**Then** an error message displays: "Could not reach your library server. Check your connection and try again."
**And** if a cached version exists from a previous successful load, the user is offered "Read cached version"

**Given** the user reads a remote book
**When** highlights, reading position, and progress are saved
**Then** they are stored locally in Dexie (same as local books) and survive even if the server becomes unreachable
**And** the reading experience is identical to a locally-stored EPUB once loaded

## Dev Notes

### Architecture Patterns

- **Content source abstraction**: The architecture defines `ContentSource` as a union type (`local | remote | fileHandle`). `BookContentService` is the runtime resolution layer.
- **Cache API**: Use the browser Cache API (not IndexedDB) for caching remote EPUBs. Cache API is designed for request/response pairs and has good browser support.
- **epub.js**: `ReactReader` accepts either a URL string or an ArrayBuffer for the `url` prop. Pass ArrayBuffer for both local (from OPFS) and remote (from fetch) sources.

### Cache Strategy

```typescript
const CACHE_NAME = 'knowlune-epub-cache'
const MAX_CACHED_BOOKS = 10

async function cacheEpub(bookId: string, buffer: ArrayBuffer) {
  const cache = await caches.open(CACHE_NAME)
  const response = new Response(buffer)
  await cache.put(`/epub/${bookId}`, response)
  // LRU eviction if > MAX_CACHED_BOOKS
}

async function getCachedEpub(bookId: string): Promise<ArrayBuffer | null> {
  const cache = await caches.open(CACHE_NAME)
  const response = await cache.match(`/epub/${bookId}`)
  return response ? response.arrayBuffer() : null
}
```

### CORS Reminder

Remote EPUB fetch requires CORS headers from the server. This is the same CORS issue as E88-S01. If the server doesn't support CORS, the fetch will fail. Error message should mention this possibility.

## Challenges and Lessons Learned

- **Cache API for binary data**: The browser Cache API proved to be a natural fit for caching remote EPUBs. It stores request/response pairs natively and is origin-scoped, avoiding the complexity of IndexedDB for large binary blobs. LRU eviction uses a custom `X-Cached-At` response header since the Cache API doesn't expose insertion timestamps.

- **Structured error handling with RemoteEpubError**: Creating a typed error subclass with a `code` field (`network`, `auth`, `not-found`, `server`, `timeout`) enabled the UI layer to switch on error types without parsing message strings. The `hasCachedVersion` boolean on the error object lets the UI conditionally show the "Read cached version" button without a separate cache check.

- **Design token migration**: The existing BookReader had two instances of hardcoded `bg-[#FAF5EE]` which were migrated to `bg-background`. This was caught opportunistically during the story implementation and aligns with the design token enforcement strategy.

- **AbortController timeout pattern**: Using `AbortController` with `setTimeout` for fetch timeout is cleaner than `Promise.race()` because it actually cancels the underlying request rather than just ignoring the result. The `clearTimeout` call in both success and error paths prevents timer leaks.

- **Non-ASCII credentials risk**: `btoa()` throws on non-ASCII characters. This was identified in code review and needs a fix using `TextEncoder` + manual base64 for RFC 7617 compliance.
