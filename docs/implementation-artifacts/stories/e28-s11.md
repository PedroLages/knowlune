# Story 23.11: Transcript Fallback — Tier 2 (yt-dlp) & Tier 3 (Whisper)

## Story

As a learner with a home server,
I want transcripts for videos that don't have YouTube captions,
so that I can search and study all my YouTube course content, not just the ~90% with auto-generated captions.

## Acceptance Criteria

1. When Tier 1 transcript fetch fails with `no-captions-available` AND user has yt-dlp server URL configured → pipeline falls back to Tier 2: `POST /api/youtube/ytdlp/subtitles`
2. Vite middleware proxies request to configured server URL with SSRF validation via `isAllowedProxyUrl()`
3. yt-dlp VTT subtitles parsed using existing `parseVTT()` function, stored with `source: 'ytdlp'`
4. When Tier 1 and Tier 2 both fail AND user has Whisper endpoint configured → falls back to Tier 3: `POST /api/youtube/whisper/transcribe`
5. Whisper request proxied to user's faster-whisper Docker container, async with progress indicator
6. Whisper transcription completes within 60s per video (FR118), stored with `source: 'whisper'`
7. Transcript panel updates reactively without page refresh
8. When all three tiers fail → video marked `status: 'unavailable'` with reason string
9. yt-dlp metadata enrichment: `POST /api/youtube/ytdlp/metadata` returns chapter markers, cleaned descriptions
10. SSRF validation rejects invalid target URLs with 403
11. Unconfigured tiers terminate gracefully (no error thrown)

## Technical Notes

- Tier 2 and Tier 3 are Premium features — require user-provided infrastructure
- Vite middleware follows established `ollamaDevProxy()` pattern
- SSRF protection shared between Ollama and YouTube endpoints via `src/lib/ssrfProtection.ts`
- `parseVTT()` already exists in `src/lib/captions.ts`
- Key files: `src/lib/youtubeTranscriptPipeline.ts`, `src/lib/ssrfProtection.ts`, `vite.config.ts`

## Implementation Plan

1. Update `src/lib/youtubeTranscriptPipeline.ts` — add Tier 2 and Tier 3 fallback chain
2. Add Vite middleware in `vite-plugin-youtube-transcript.ts` for yt-dlp and Whisper proxy endpoints
3. Add unit tests for fallback chain logic, VTT parsing via proxy, SSRF enforcement

## Lessons Learned

- Existing infrastructure (parseVTT, SSRF protection, Settings UI) eliminated most boilerplate
- Tiered fallback pattern keeps each tier isolated and testable
