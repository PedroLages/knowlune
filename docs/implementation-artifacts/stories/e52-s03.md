# Story 52.03: Quiz Quality & Feedback

## Story

As a learner,
I want generated quiz questions to be high-quality and relevant to the transcript, with the ability to provide feedback and regenerate,
so that I can trust the quizzes test real comprehension and improve over time.

## Acceptance Criteria

**Given** the LLM generates a set of quiz questions for a chunk
**When** the QC pipeline runs
**Then** duplicate questions are detected via embedding cosine similarity (>0.85 threshold)
**And** duplicate questions are removed from the set

**Given** a generated question
**When** the QC pipeline validates answer uniqueness
**Then** it verifies `correctAnswer` is present in `options`
**And** all `options` are distinct strings

**Given** a generated question
**When** the QC pipeline checks transcript grounding
**Then** key terms from the question text must appear in the source transcript chunk
**And** questions that fail grounding check are flagged for retry

**Given** a chunk fails QC validation
**When** the system retries generation
**Then** it retries up to 2 times with the same chunk
**And** if all retries fail, the chunk is skipped and remaining valid questions still produce a quiz

**Given** a quiz is displayed to the learner
**When** the learner views an individual question
**Then** thumbs up and thumbs down feedback buttons are visible on each question
**And** clicking a feedback button stores the feedback locally in the quiz data

**Given** the learner wants fresh questions for a lesson
**When** they click "Regenerate"
**Then** a new quiz is generated (new `id`) while the previous quiz is preserved
**And** the learner can access both the old and new quiz

## Implementation Notes

**New files:**
- `src/ai/quizQualityControl.ts` — Deterministic QC pipeline (no LLM calls)

**Existing files to modify:**
- `src/ai/quizGenerationService.ts` — Integrate QC pipeline after generation stage
- Quiz question display component — Add thumbs up/down buttons
- `src/app/components/figma/GenerateQuizButton.tsx` — "Regenerate" label logic

**Key patterns:**
- QC pipeline is purely deterministic (no additional LLM calls)
- Duplicate detection uses existing `BruteForceVectorStore.cosineSimilarity()` or `generateEmbeddings`
- Transcript grounding: extract nouns/terms from question text, check presence in chunk
- Feedback stored as `{ questionId, feedback: 'up' | 'down', timestamp }` in quiz data

**Dependencies:**
- E52-S01 (Quiz Generation Pipeline)
- E52-S02 (Quiz Generation UI — for regeneration UX and feedback buttons)
