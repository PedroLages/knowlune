# Story 84.2: Page Navigation and Table of Contents

Status: ready-for-dev

## Story

As a learner,
I want to turn pages with clicks, swipes, or keyboard and jump to chapters via a table of contents,
so that I can navigate through a book naturally on any device.

## Acceptance Criteria

1. **Given** the reader is displaying EPUB content **When** the user clicks the right third of the page, swipes left, or presses the Right arrow key **Then** the reader navigates to the next page with a slide-left animation (200ms ease-out) **And** page navigation completes in <100ms (NFR3)

2. **Given** the reader is displaying EPUB content **When** the user clicks the left third of the page, swipes right, or presses the Left arrow key **Then** the reader navigates to the previous page with a slide-right animation (200ms ease-out)

3. **Given** the reader header menu is open **When** the user selects "Table of Contents" **Then** a Sheet slides in from the left showing the EPUB's TOC as a navigable list **And** the current chapter is highlighted with `text-brand font-medium` **And** tapping a chapter navigates to that chapter's start position **And** the TOC panel closes after navigation

4. **Given** the reader supports keyboard navigation **When** the user presses Space **Then** the reader advances to the next page **And** all navigation triggers the `locationChanged` callback for position tracking

## Dev Notes

### Architecture Constraints

- **react-reader handles most navigation internally** — `rendition.prev()` and `rendition.next()` are the APIs for page turns. The `locationChanged` prop on `ReactReader` fires automatically on every navigation. Do NOT reimplement epub.js pagination logic.
- **epub.js TOC** is available via `book.navigation.toc` after the book loads. Access it through the rendition ref stored in E84-S01's `getRendition` callback. The TOC is an array of `{ id, href, label, subitems }`.
- **Page turn animations**: epub.js handles the actual content rendering. The slide animation should be a CSS transition on the container, NOT on individual epub.js pages. Consider a brief directional slide or fade.
- **Sheet component**: Use shadcn `Sheet` with `side="left"` for TOC. This is consistent with how other sliding panels work in Knowlune.

### Key Dependencies (from E84-S01)

- EpubRenderer with `ReactReader` component and rendition ref
- ReaderHeader with dropdown menu (TOC menu item)
- ReaderFooter with progress display
- useReaderStore with `currentCfi`, `readingProgress`, `headerVisible`
- Page interaction zones (center zone toggle from E84-S01)

### Performance Requirements

- Page navigation must complete in <100ms (NFR3)
- Do NOT add heavy animation libraries — CSS transitions only
- epub.js default manager handles pagination efficiently

### Accessibility (UX-DR22, NFR12)

- All navigation via keyboard (Left/Right/Space)
- TOC panel must trap focus when open, release on close
- `aria-label` on navigation zones: "Previous page", "Next page", "Toggle reader controls"
- TOC list items: `role="listitem"` with chapter title as accessible name
- Current chapter in TOC: `aria-current="true"`
- Page turn: announce page change via `aria-live="polite"` region (e.g., "Page 128 of 372")
