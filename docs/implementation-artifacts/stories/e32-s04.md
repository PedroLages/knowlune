# Story 32.4: Implement Data Pruning for studySessions, aiUsageEvents, and Embeddings

## Story

As a power user,
I want old data to be automatically pruned,
So that IndexedDB storage doesn't grow unbounded over months of use.

## Acceptance Criteria

**Given** the app has study sessions older than 90 days
**When** the pruning job runs (on app start, debounced)
**Then** sessions older than the configurable TTL (default: 90 days) are deleted
**And** a summary is logged: "Pruned X study sessions older than 90 days"

**Given** embeddings that are no longer referenced by any course or note
**When** the pruning job runs
**Then** orphaned embeddings are deleted

**Given** aiUsageEvents older than the configured TTL (default: 90 days)
**When** the pruning job runs
**Then** expired events are deleted

**Given** the user wants to adjust retention periods
**When** they visit Settings > Data Management
**Then** they can configure TTL for each data type (30, 60, 90, 180 days, or "Keep forever")

## Implementation Notes

- Pruning settings stored in localStorage alongside app-settings
- Uses Dexie `.where('startTime').below(cutoff).delete()` for studySessions
- Uses Dexie `.where('timestamp').below(cutoff).delete()` for aiUsageEvents
- Orphaned embeddings: compare embedding noteIds against notes table
- Background task via `requestIdleCallback` / `setTimeout` deferral in main.tsx
- Pruning runs after app init, never blocks first paint
