# Story E77B-S03: Drive File Streaming and OPFS Cache

## Story

As a user,
I want to stream Drive-hosted video lessons with an OPFS cache for offline playback,
so that I can watch imported Drive videos instantly without waiting for full downloads.

## Acceptance Criteria

1. **Given** a Drive-sourced video lesson **When** it is first played **Then** `resolveFileUrl()` streams the file from Google Drive via the streaming endpoint, creates a blob URL for playback, and writes the stream to the OPFS cache in the background.
2. **Given** a Drive-sourced video lesson that has been cached **When** it is played offline **Then** the cached blob URL from OPFS is returned immediately without fetching from Drive.
3. **Given** a Drive-sourced video lesson that is not cached **When** it is played offline **Then** `DriveFileOfflineError` is thrown and the UI shows a connectivity error.
4. **Given** a 401 response from the Drive API **When** fetching a file **Then** the token is refreshed automatically via `refreshDriveToken()` and the request is retried once.
5. **Given** a 404 response from the Drive API **When** fetching a file **Then** `DriveFileNotFoundError` is thrown.
6. **Given** `useDriveFileUrl` resolves a Drive file ref **When** the component mounts **Then** it returns `{ blobUrl, error, loading }` matching the existing `useVideoFromHandle` contract.
7. **Given** a Drive-sourced lesson in `LocalVideoContent` **When** the video is rendered **Then** it uses `useDriveFileUrl` for Drive sources and `useVideoFromHandle` for local sources, with a clean `isDriveSource` branching path.
8. **Given** a Drive file playback error **When** the retry button is clicked **Then** `handleRecoveryNeeded` increments the retry key to regenerate the blob URL.

## Dev Notes

- Adds `src/lib/driveFileAccessService.ts` — core service with `resolveFileUrl()`, `clearDriveCache()`, OPFS cache helpers
- Adds `src/hooks/useDriveFileUrl.ts` — React hook mirroring `useVideoFromHandle` contract
- Updates `src/app/components/course/LocalVideoContent.tsx` — branching between Drive and local sources
- Adds `src/lib/__tests__/driveFileAccessService.test.ts` — comprehensive test suite
- OPFS cache lives under `knowlune/drive-cache/` in the Origin Private File System
- Cache writes are fire-and-forget (background), cache hits block on OPFS read
- The `buildStreamUrl()` function from `googleDriveFileService.ts` constructs the Drive media download URL

## Challenges and Lessons Learned

- **OPFS tee strategy**: Using `response.body.tee()` to create two readable streams — one for blob URL creation (blocking) and one for OPFS caching (fire-and-forget). This avoids waiting for the cache write before showing the video.
- **Blob URL lifecycle**: The hook carefully avoids revoking blob URLs while they're active in state, using `activeUrlRef` and deferred cleanup. On dependency change, old URLs are revoked atomically inside setState only when the new URL is ready.
- **Strict Mode safety**: The unmount cleanup uses `setTimeout(0)` to avoid URL revocation during React's double-mount cycle. The second mount sets a new `activeUrlRef` before the first mount's timeout fires.
- **Error/retry integration**: The `retryKey` mechanism from LocalVideoContent's existing error recovery system is reused for Drive files — incrementing `retryKey` triggers re-resolution via the effect dependency.
