# Story 24.1: Refactor Import Function into Scan and Persist

## Story

As a developer,
I want to split `importCourseFromFolder` into separate scan and persist phases,
so that the upcoming import wizard can preview/edit course metadata before saving.

## Acceptance Criteria

- AC1: `scanCourseFolder()` scans a user-selected folder, extracts metadata, and returns a `ScannedCourse` without persisting anything to IndexedDB.
- AC2: `persistScannedCourse(scanned, overrides?)` accepts a `ScannedCourse` and optional overrides (name, category, tags), persists to IndexedDB, updates Zustand store, and triggers auto-analysis/tagging.
- AC3: `importCourseFromFolder()` remains as a backwards-compatible one-shot wrapper that calls scan then persist.
- AC4: New types (`ScannedCourse`, `ScannedVideo`, `ScannedPdf`) are exported for wizard consumption.
- AC5: Unit tests cover both phases independently and verify integration equivalence.

## Tasks / Subtasks

- [x] Task 1: Define ScannedVideo, ScannedPdf, ScannedCourse types (AC: 4)
- [x] Task 2: Extract scan logic into scanCourseFolder() (AC: 1)
- [x] Task 3: Create persistScannedCourse() with overrides support (AC: 2)
- [x] Task 4: Refactor importCourseFromFolder() as thin wrapper (AC: 3)

## Implementation Notes

- Split the monolithic `importCourseFromFolder` into two phases: `scanCourseFolder()` (read-only) and `persistScannedCourse()` (write).
- `ScannedCourse` carries `fileHandle` references so the wizard can display file info without re-scanning.
- `persistScannedCourse` accepts optional `overrides` parameter for wizard-modified metadata (name, category, tags).
- Batch metadata extraction (10 concurrent) preserved in scan phase for memory management.
- Error handling with `ImportError` class and specific error codes (NO_FILES, PERMISSION_DENIED, SCAN_ERROR, DUPLICATE).
- Toast notifications with retry action for permission errors.

## Testing Notes

- 18 unit tests covering: scan phase (directory picker, permissions, duplicates, metadata extraction, empty folders, batching), persist phase (full course, overrides, PDF-only, empty videos), integration (scan+persist equivalence to one-shot), and error handling.
- Design review skipped — no UI changes (pure refactoring of library code).
- E2E tests skipped — no user-facing behavior change; existing import flow unchanged via backwards-compatible wrapper.

## Challenges and Lessons Learned

- Splitting a monolithic function into phases required careful handling of shared state (the Zustand import store progress tracking). The scan phase owns progress updates during metadata extraction, while the persist phase is stateless regarding progress.
- The `overrides` parameter pattern provides a clean API for the wizard without requiring the caller to reconstruct the entire course object.
- Keeping `importCourseFromFolder` as a thin wrapper ensures zero breaking changes for existing callers while enabling the new wizard flow.
