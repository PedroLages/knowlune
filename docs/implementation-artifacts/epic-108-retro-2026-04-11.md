## Lessons Learned

### 1. Abort state machine: set terminal state at the point of decision, not in shared post-loop blocks

In async processing loops using `AbortController`, calling `setPhase('cancelled')` must happen at the `break` point ‚Äî atomically, before the post-loop block executes. Post-loop blocks run regardless of how the loop exited (abort vs. normal completion). Setting terminal state only in a shared post-loop block creates a race: the abort checkpoint sets `cancelled` via `break`, then the post-loop block overwrites it with `done`. Fix: `setPhase('cancelled'); break;` at the abort checkpoint. Pattern: every exit path from a stateful async loop should explicitly set its own terminal state. (E108-S01, `useBulkImport`)

### 2. useCallback with object deps causes unnecessary re-instantiation ‚Äî destructure first

Passing a non-memoized object literal to a `useCallback` or `useMemo` dep array causes the callback to re-instantiate on every render, because the object reference changes even if its values are stable. Fix: destructure the object into the specific primitive or stable ref values you actually need before the dep array. The dep array should contain only the values that, when changed, should genuinely invalidate the memoized function. (E108-S01, `useBulkImport`)

### 3. Onboarding overlay in E2E tests: addInitScript with all three localStorage keys

Playwright E2E tests navigating to app routes are blocked by the onboarding overlay unless all three localStorage flags are pre-set via `page.addInitScript()`: `hasCompletedOnboarding`, `hasSeenWelcome`, `onboardingDismissed`. This must be applied in `beforeEach` before any `page.goto()` call. Canonical implementation belongs in `tests/helpers/` ‚Äî not per-story. Every E2E spec that navigates to a library or reader route needs this. (E108-S03, `useKeyboardShortcuts` E2E)

### 4. The 500-line component ESLint limit is a design signal, not just a style rule

When `AudiobookRenderer.tsx` hit the 500-line ESLint limit during S04 implementation, the required hook extraction produced `useAudiobookSettings` and `useAudiobookShortcuts` as independently testable units. The component's responsibilities became clearer, the renderer became readable, and no architectural debt was deferred. When this rule fires: don't suppress it, extract to hooks. The rule works. (E108-S04)

### 5. Run `git status` before staging to catch stray files

S05 shipped with a stray debug file in the repo root. A `git status` review before `git add` would have caught it. Pre-commit habit: check untracked files explicitly before staging. If a file is untracked and unexpected, delete it or add it to `.gitignore` before the commit. (E108-S05)

### 6. Wire service function consumers before implementing the service

In S05, `checkPagesGoalMet` was implemented before its consumer (`DailyGoalRing`) was wired, leaving it briefly as dead code. For service functions with known downstream consumers, stub the consumer first (even with a no-op), then implement the service. This ensures the interface is validated from the start, prevents dead-code drift during implementation, and surfaces integration issues before the function is complete. (E108-S05, `checkPagesGoalMet`)
