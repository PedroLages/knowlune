# Epic 84 Retrospective — EPUB Reader and Navigation
Date: 2026-04-05

## What Went Well

1. **epub.js API clarity**: The rendition.themes.default({body:{...}}) pattern (established in S01) propagated cleanly to all subsequent stories. No API confusion after the initial S01 discovery.

2. **Zero new dependencies for TTS**: The entire Read-Aloud feature (S05) was built using only the native Web Speech API — no extra npm packages. The sentence-level chunking workaround for Chrome's 15-second pause bug was clean and maintainable.

3. **Store architecture held up**: useReaderStore with localStorage persistence covered themes, font settings, and UI state perfectly across all stories. No schema changes needed after S01.

4. **Design token discipline**: All reader chrome (header/footer backgrounds, brand colors) consistently used design tokens. ESLint enforcement caught no violations across 5 stories.

## What Could Be Improved

1. **epub.js rendition API types are incomplete**: epub.js TypeScript types don't fully cover currentLocation(), getContents(), and book.locations. Multiple stories required `as unknown as` casts. A local type-augmentation file would reduce this boilerplate.

2. **Component size warning on BookReader**: BookReader grew to 350+ lines across stories S01–S05. The component-size ESLint rule required suppression. Future work should extract the position-save logic and TTS wiring into custom hooks.

3. **ContentPosition type mismatch in story doc**: The S04 story draft described ContentPosition as `{ type: 'cfi', cfi: string, percent: number }` but the actual type is `{ type: 'cfi', value: string }`. Story author should verify types against src/data/types.ts before writing.

## Key Lessons

- **Lesson 1**: epub.js `rendition.themes.override(name, value)` takes individual CSS property + string value. Use `rendition.themes.default({body:{...}})` for bulk theme application. Update architecture docs to clarify.
- **Lesson 2**: Chrome speechSynthesis pauses at ~15 seconds continuous speech. Always chunk TTS text into sentence-level utterances queued via `utterance.onend`.
- **Lesson 3**: epub iframe content is accessible via `rendition.getContents()[0].document` for DOM manipulation (highlighting). This is undocumented but stable across epub.js 0.3.x.
- **Lesson 4**: The `BookReader` route must be placed OUTSIDE the Layout wrapper in routes.tsx, or the sidebar will appear over the full-screen reader.

## NFR Validation

- **Performance**: epub.js is code-split (lazy-loaded) — no bundle impact on initial load. locations.generate() runs async, non-blocking. Position save debounced at 500ms.
- **Accessibility**: All interactive zones have aria-label. TTS control bar is role="toolbar" with keyboard-accessible controls. Progress announced via aria-live regions.
- **Persistence**: Reader settings survive reload (localStorage via useReaderStore). Reading position survives reload (Dexie via ContentPosition CFI).
- **WCAG**: All three themes verified: Light 14.5:1 (AAA), Sepia 6.8:1 (AA), Dark 12.1:1 (AAA).
