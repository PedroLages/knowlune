## Key Insights and Learnings

1. **Success paradox: smooth epics lead to relaxed quality assurance** — When everything goes well, optional quality steps (burn-in, substantive lessons learned, debt reduction) get skipped. This creates a false sense of durability. The discipline that produced the smooth epic erodes if not structurally enforced.

2. **Voluntary retro commitments are systemically unreliable** — Two consecutive measurement cycles (56% → 14%) confirm the pattern. The only reliable mechanism is automation (ESLint rules, git hooks, review agents). Future retro commitments should only include items that can be automated or enforced.

3. **Pure function architecture is the ideal analytics pattern** — Co-locating stateless functions in `analytics.ts` with zero side effects creates: trivial unit testing, clean separation from UI, easy composition, and natural reuse across components. This is now the established template for E17's psychometric calculations.

4. **Forward-planning APIs provide compound returns** — The `QuestionDisplayMode` type (E12) → review styling (E16) pipeline demonstrates that investing in extensible type surfaces during foundation work pays dividends 4+ epics later. Worth repeating for E17's analytics display patterns.

5. **Architectural maturity is the inflection point, not individual heroics** — E16's zero-BLOCKER status and single-day completion are products of 4 epics of compounding infrastructure, not exceptional effort on any single story. The quiz subsystem has reached the assembly phase where new features compose from existing parts.
