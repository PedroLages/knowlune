## What Went Well

### 1. `addInitScript` Pattern Transferred Immediately

E101's discovery that `page.route()` cannot intercept cross-origin fetches was applied from S01 onward. No time lost rediscovering this. The pattern also extended naturally to WebSocket mocking in S04 — the `addInitScript` approach for intercepting the `WebSocket` constructor followed the same logic.

### 2. LWW Conflict Resolution is Clean and Tested

`resolveConflict()` in S01 is an explicit, pure function with three test cases (ABS ahead, local ahead, equal timestamps). The decision logic is not buried in a `useEffect` — it's named, isolated, and verifiable. This is the right way to implement conflict resolution and should be the template for any future sync story.

### 3. Zero-Dependency Engine.IO Implementation

S04 achieved real-time Socket.IO communication without adding `socket.io-client` to the bundle. The protocol framing (~100 lines in `AudiobookshelfService.ts`) is readable, commented, and co-located with the rest of the ABS service layer. The approach of keeping it as pure service functions (not a class or singleton) is architecturally consistent with E101.

### 4. Collections Inline Expansion Consistency

Following E102-S02's series accordion pattern for collections (rather than introducing a separate navigation flow) kept the UX consistent. Users get the same interaction model for both organizational structures.

### 5. Series Pagination Implemented Correctly

`loadSeries()` implements the full multi-page loop: fetch total from first page, iterate remaining pages. The story spec explicitly called this out and the implementation follows through. Compare to collections where the spec gap led to a single-page implementation.

## What Didn't Go Well

### 1. S04 Quality Gates Skipped — "Optional Story" Framing

S04 was framed as optional ("implement only if REST polling proves insufficient"). Once the decision was made to ship it, four review gates were marked skipped: security, design, performance, and exploratory QA. The security skip in particular is significant: S04 introduces a new authenticated WebSocket transport channel, which is a mandatory security review scenario regardless of UI surface or story optionality.

### 2. Stale `isSocketConnected` — Repeated Stale Closure Pattern

`useAudiobookshelfSocket` returns `connectionRef.current?.isConnected ?? false`. Because `connectionRef` is a `useRef`, mutations to it don't trigger re-renders. The `forceUpdate.current += 1` mutation does nothing because it's never read by React's rendering. This is the same category of issue flagged in both E101-S02 and E101-S03 retrospective discussions, and an action item was explicitly created to add a stale-closure checklist item. The action item was not applied.

### 3. `seriesLoaded` / `collectionsLoaded` Not Keyed by Server

The `isLoaded` guards for series and collections are single booleans with no server awareness. Switching ABS servers returns the previous server's data without re-fetching. This pattern was not caught in story review because the current test environment uses a single server — the bug only manifests with multiple configured servers.

### 4. Collections Pagination Missing

`fetchCollections()` fetches only the first page. `loadSeries()` in the same file and the same commit implements multi-page correctly. The gap traces to the S03 story spec, which mentioned pagination in the response shape but did not explicitly require multi-page loading. The S02 spec said "implement same pagination pattern as `fetchLibraryItems()`" — S03 did not.

### 5. Socket Push Event Name Unverified

`pushProgressViaSocket()` emits `update_media_progress`. The story implementation notes say "verify against ABS v2.x source before implementing" — this was not done. ABS's socket architecture is primarily server-push; whether clients can push progress via a socket event (vs. the REST PATCH endpoint) is unconfirmed. The push path may be a silent no-op.

### 6. API Key in WebSocket URL

`connectSocket()` appends `?token={apiKey}` to the WebSocket URL. WebSocket URLs (including query params) appear in browser devtools Network tab and reverse-proxy access logs. The ABS Socket.IO client sends the token in the `auth` payload of the handshake packet instead — staying in the message frame, not the URL.

### 4. Add "isLoaded guard must be keyed per context" to engineering-patterns.md

Document: in-memory caches that use a boolean `isLoaded` flag are wrong when cached data differs by server, user, or other context key. The guard must store the key that was loaded and re-fetch when the active context changes. Example:

```typescript
// ❌ Wrong: single boolean ignores context
seriesLoaded: boolean

// ✅ Correct: keyed by context
seriesLoadedForServerId: string | null
```

**Owner:** Update `docs/engineering-patterns.md`

## Lessons Summary (Quick Reference)

| # | Lesson | Applies To |
|---|--------|-----------|
| L1 | `addInitScript` overrides `WebSocket` constructor for cross-origin E2E mocking — same principle as fetch override | All stories with real-time WebSocket connections |
| L2 | LWW conflict resolution belongs in a named, tested pure function — not buried in a `useEffect` | Any sync story with merge/conflict semantics |
| L3 | Engine.IO/Socket.IO wire protocol can be implemented without `socket.io-client` in ~100 lines | Any real-time feature under zero-dep constraint |
| L4 | `isLoaded` boolean guards must be keyed by context (serverId, userId, etc.) — not a global flag | All in-memory cache patterns |
| L5 | "Optional" applies to the build decision, not the quality bar — once shipped, all gates apply | All stories marked as optional/experimental |
| L6 | Reactive state belongs in Zustand state, not `useRef` — refs don't trigger re-renders | Any hook that needs to expose reactive connection state |
| L7 | Story specs must explicitly say "handle pagination" — response shape documenting `total`/`page` is not sufficient | Any service layer function calling a paginated endpoint |
| L8 | Verify external service event names/APIs before shipping — "verify before implementing" TODOs are blockers | All external integration stories |
