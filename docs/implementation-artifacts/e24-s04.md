# Story 24.4: AI Metadata Suggestions During Import

## Story

As a learner,
I want the import wizard to suggest tags and a description using AI when Ollama is available,
so that my courses are automatically organized without manual effort.

## Acceptance Criteria

- AC1: When Ollama is configured, the wizard shows an AI loading indicator during tag/description generation.
- AC2: AI-suggested tags are auto-applied to the tag input with a sparkle badge.
- AC3: AI-suggested description is auto-applied with an "AI Suggested" badge.
- AC4: User can override AI suggestions by editing tags or description manually.
- AC5: When Ollama is not configured, the wizard works identically to before (no AI indicators).
- AC6: Description field is passed through to `persistScannedCourse` on import.
- AC7: No E2E spec required (File System Access API cannot be tested in Playwright).

## Implementation Notes

- Extended `courseTagger.ts` with structured Ollama output for tags (JSON schema) and descriptions (plain text).
- Created `useAISuggestions` React hook with AbortController cleanup and cancelled-flag guard.
- AI suggestions only auto-apply when user hasn't manually entered tags/description yet.
- Sparkle icon badges distinguish AI-suggested values from user-entered ones.
- Graceful degradation: all AI functions return empty defaults on failure.

## Testing Notes

- 33 unit tests in ImportWizardDialog.test.tsx, all passing.
- Tests cover both AI-available and AI-unavailable paths.
- No E2E spec — File System Access API is not testable in Playwright.

## Challenges and Lessons Learned

- **Structured output parsing resilience**: Ollama models return structured output inconsistently. The `parseTagResponse` function implements a 4-strategy fallback chain (JSON parse, markdown fence extraction, regex array, raw split) to handle the variety. Future AI integrations should plan for multiple response formats from the start.
- **AbortController cleanup in hooks**: The `useAISuggestions` hook chains an external signal to an internal AbortController for proper cleanup. Testing revealed that checking `signal.aborted` before adding the abort listener is necessary for edge cases where the signal is already aborted when the hook runs.
- **Deduplicating async fetch patterns**: Code review caught that `generateCourseTags` and `generateCourseDescription` were structurally identical. Extracting a shared `callOllama` helper reduced duplication and simplified future additions.
