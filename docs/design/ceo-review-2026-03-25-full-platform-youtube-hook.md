## Vision

### 10x Check

The 10x version of Knowlune isn't just "a tool that organizes YouTube videos into courses." It's the **command center for self-directed learning** — the single place where everything you're learning, from every source, is tracked, searchable, and structured to maximize completion. The 10x version turns the guilt pile of unfinished courses into a dopamine-driven completion engine.

The YouTube Course Builder is the entry point. Transcripts + AI make it intelligent. Streaks + gamification make it addictive. The browser extension makes capture effortless. But the 10x version is when a user says: "I can't learn without this anymore."

### Delight Opportunities Surfaced

1. **YouTube chapter-aware structuring** — auto-create course sections matching the creator's intended chapters (ACCEPTED)
2. **Transcript search** — "What did this course say about X?" across all imported videos (ACCEPTED)
3. **AI flashcard generation** — automatically create spaced repetition cards from transcript key concepts (ACCEPTED)
4. **Playlist one-click import** — YouTube playlists → structured courses instantly (ACCEPTED)
5. **Browser extension** — "Save to Knowlune" button on every YouTube page (ACCEPTED)
6. **Faster-Whisper fallback** — transcribe captionless videos on local server (DEFERRED)

## Outside Voice Findings & Revisions

The independent outside voice raised 8 challenges. Three resulted in plan revisions:

### Revision 1: Browser Extension Cut
**Outside voice:** "A browser extension adds Chrome Web Store review, manifest v3, a separate codebase, Service Worker lifecycle — all for a button that a URL paste box handles."
**Decision:** DEFERRED. URL paste input is the v1 capture mechanism.

### Revision 2: Free/Premium Split Revised
**Outside voice:** "You're giving away the differentiator (YouTube course building) and charging for commodities (LLM wrappers users can get from ChatGPT)."
**Decision:** Moved transcript-based AI features to premium tier.
- **FREE:** YouTube import (URL paste), playlist import, YouTube chapters, basic progress tracking
- **PREMIUM ($12/mo):** Transcript extraction, transcript search, AI course structuring, AI summaries/Q&A, spaced review, auto-flashcards

### Revision 3: Sequencing Acknowledged
**Outside voice:** "27 epics + 0 users = building in a vacuum. Ship THIS WEEK."
**Decision:** Fix eng review items first (1-2 weeks), then continue original Phase 1 timeline. The urgency is noted — avoid scope creep that delays shipping further.

### Noted but Not Acted On
- **youtube-transcript single point of failure:** Valid concern. Mitigation: abstract behind interface so implementation can be swapped. The 90% coverage is acceptable for launch; Whisper fallback is deferred.
- **"Empty quadrant may be a demand vacuum":** Possible. Only way to know is to ship and validate. The plan now prioritizes getting to users faster.
- **Distribution strategy gap:** Valid. No concrete user acquisition plan beyond "YouTube hook + Product Hunt." This is a weakness. Suggest allocating 2 weeks post-launch specifically for distribution engineering (SEO, content marketing, community outreach).

## Architecture Decisions

### Browser Extension Communication
**Decision:** REST API + Service Worker
**Why:** Works even when the app tab is closed. Extension queues URLs in its own storage, Service Worker syncs to IndexedDB when available. Most robust pattern.

### Prompt Injection Mitigation
**Decision:** Sanitize YouTube descriptions + structured prompt guardrails (belt and suspenders)
**Why:** YouTube video descriptions can contain arbitrary text including prompt injection attempts. Sanitize input AND constrain LLM output format.

### YouTube Course Structuring Strategy
**Decision:** YouTube chapters as PRIMARY signal, AI-based topic clustering as FALLBACK
**Why:** Chapters are free, accurate, and respect creator intent. AI fallback for videos without chapters. Zero LLM cost for chaptered videos.

### Observability
**Decision:** Opt-in anonymous analytics to Supabase
**Why:** Solo dev with no users needs SOME feedback loop. Counts only (course_created_youtube, transcript_fetched, ai_structure_generated). No PII. Opt-in during onboarding.
