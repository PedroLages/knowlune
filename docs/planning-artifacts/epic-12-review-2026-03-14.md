# Epic 12 Review: Take Basic Quizzes

**Date:** 2026-03-14
**Stories Reviewed:** 6
**Sub-Agent Reviews:** 6 parallel adversarial reviews

---

## Epic-Level Verdict: **REJECT**

Two stories (12.2, 12.3) are individually REJECT-level, and 3 systemic issues affect every story in the epic. The schema version mismatch in Story 12.2 is a foundational blocker — implementing it as written risks data loss. Zero requirements traceability across all 6 stories makes it impossible to verify coverage against the 61 QFRs in the quiz UX spec.

---

## Systemic Findings

### Critical (affects multiple stories)

- **Zero FR/QFR traceability across ALL 6 stories.** No `FRs Covered` line at epic level. No `FRs Fulfilled` lines in any story. The quiz UX spec defines QFR1-QFR61, but not a single QFR is referenced anywhere in Epic 12. This makes coverage verification impossible and violates the project's documented traceability conventions. — *Affects: Stories 12.1-12.6*

- **Schema version fundamentally wrong.** Story 12.2 says "migrate from v2 to v3" but the actual codebase is at version 11. Architecture doc assigns v3 to Epic 19 (entitlements). The code snippet uses fabricated table names (`courses`, `sessions`) that don't match actual code (`importedCourses`, `studySessions`). Implementing as written would corrupt the migration chain. — *Affects: Story 12.2, and all downstream stories (12.3-12.6) that depend on quiz tables*

- **UX spec fields missing from type/store definitions.** `markedForReview` (required for question flagging and crash recovery) and timer accommodation type (`'standard' | '150%' | '200%' | 'untimed'`) are defined in the UX spec but absent from Story 12.1 (types), Story 12.3 (store), and Story 12.4 (UI). Adding these later forces breaking changes across the entire dependency chain. — *Affects: Stories 12.1, 12.3, 12.4*

### Cross-Reference Issues

- **`useProgressStore` does not exist.** Story 12.3 references `useProgressStore.getState().markLessonComplete()` — the actual store is `useContentProgressStore`. This is a build/runtime failure. — *Affects: Story 12.3, Story 12.6 (which calls submitQuiz)*

- **Type file location inconsistency.** Story 12.1 creates `src/types/quiz.ts` but all existing types live in `src/data/types.ts`. No justification or migration plan provided. — *Affects: Story 12.1, all consumers*

- **Architecture document is stale.** Table names, version numbers, and schema definitions in the Architecture doc don't match actual code. Multiple stories copied from the stale doc. — *Affects: Stories 12.1, 12.2*

- **Scoring function has runtime crash path.** Story 12.6's `calculatePartialCredit` has no return for the `multiple-select` case ("Implementation in Story 14.2"), producing `undefined` → `NaN` scores if any quiz contains multiple-select questions. — *Affects: Story 12.6*

## Per-Story Reviews

### Story 12.1: Create Quiz Type Definitions — REVISE

**Strengths:**
- Foundation-first sequencing is correct — types before schema before store before UI
- Zod + TypeScript dual approach eliminates drift between runtime and compile-time checks
- Polymorphic `correctAnswer: string | string[]` handles all question types pragmatically

**Key Issues:**
- File location (`src/types/quiz.ts`) contradicts existing convention (`src/data/types.ts`)
- `QuizProgress` missing `markedForReview: Set<string>` from UX spec
- Timer accommodation types absent
- `Question.media` sub-type undefined
- No scoring strategy representation in types
- Zod validation behavior (parse vs safeParse) unspecified
- Zero QFR traceability

### Story 12.2: Set Up Dexie Schema v3 Migration — REJECT

**Strengths:**
- Three-table design (quizzes, quizAttempts, quizProgress) is a reasonable decomposition
- Explicit no-backfill statement is correct

**Key Issues:**
- Version number wrong by 9 (v3 vs v12 needed)
- Every table name in code snippet is fabricated (courses vs importedCourses, etc.)
- 8 of 13 existing tables omitted — Dexie would delete them
- Version collision with Architecture doc's Epic 19 assignment
- No compound index for explicitly cited compound query
- `quizProgress` table conflicts with UX spec's localStorage directive
- No typed table registration on Dexie class

### Story 12.3: Create useQuizStore with Zustand — REJECT

**Strengths:**
- Follows established Zustand conventions (naming, selectors, persist pattern)
- Smart persistence partitioning (only currentProgress)
- Write amplification awareness (defer Dexie writes until submission)

**Key Issues:**
- References non-existent `useProgressStore` (actual: `useContentProgressStore`)
- Missing `markedForReview` and timer accommodation state
- Fisher-Yates shuffle order not persisted — crash recovery restores wrong question
- No timer tick/decrement mechanism specified
- No retry-before-rollback per Architecture convention
- `retakeQuiz` and `resumeQuiz` semantics undefined
- Rollback atomicity unclear for cross-store calls

### Story 12.4: Create Quiz Route and Basic QuizPlayer — REVISE

**Strengths:**
- Component decomposition follows project conventions
- Resume quiz edge case explicitly handled
- Testing requirements comprehensive (unit, E2E, accessibility)

**Key Issues:**
- Route uses `lessonId` but AC says "loads from quizId parameter" (contradiction)
- Missing UX spec elements: passing score, accessibility accommodations link, metadata badges
- `QuizPlayer` in title doesn't match any file in the spec
- No error state for invalid quiz URLs
- `QuestionDisplay` referenced but not defined or deferred

### Story 12.5: Display Multiple Choice Questions — REVISE

**Strengths:**
- Semantic HTML structure (`fieldset/legend` + RadioGroup) follows accessibility best practices
- Polymorphic QuestionDisplay pattern supports future question types cleanly

**Key Issues:**
- Only 2 of 5 UX visual states specified (missing correct, incorrect, disabled)
- Touch target 44px contradicts UX spec's 48px (h-12)
- Code snippet styling diverges from UX spec's label wrapper pattern
- `key={index}` anti-pattern (breaks with shuffled options)
- No `react-markdown`/`remark-gfm` dependency specified
- No option count validation for malformed data

### Story 12.6: Calculate and Display Quiz Score — REVISE

**Strengths:**
- Scoring algorithm is clean with zero-division handling
- Unanswered question confirmation flow well-specified
- Correct dependency chain identified

**Key Issues:**
- `multiple-select` case returns `undefined` → NaN scores (runtime crash)
- Fill-in-blank uses strict `===` but UX spec requires case-insensitive, whitespace-trimmed
- Missing animated SVG score indicator from UX spec
- Missing "Back to Lesson" link
- Encouraging messages vague vs UX spec's specific copy
- No error handling for `submitQuiz()` failure
- No loading state for results page
