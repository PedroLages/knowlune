# Epics 14–18 Batch Review: Quiz System Feature Set

**Date:** 2026-03-14
**Stories Reviewed:** 27 (across 5 epics)
**Sub-Agent Reviews:** 5 parallel adversarial reviews
**Scope:** Practice with Diverse Question Types → Timed Quizzes → Performance Review → Quiz Analytics → Accessibility & Integration

---

## Batch-Level Verdict: **REVISE**

All 5 epics require revision. Two stories are outright REJECTED (14.4, 17.4). The quiz system stories are structurally sound — well-scoped, logically ordered, with correct dependency chains — but every epic shares three systemic defects: pervasive hardcoded colors violating the design token enforcement system, zero QFR traceability lines, and code snippets that contradict project conventions (ESLint rules, Tailwind-first styling). These are document governance problems, not architectural ones.

---

## Systemic Findings

### CRITICAL — Affects All 5 Epics

**1. Hardcoded Colors Epidemic (~60+ violations across 27 stories)**

Every epic contains code snippets with hardcoded Tailwind colors (`text-gray-600`, `bg-green-100`, `text-red-600`, `#f5f5f5`, `#2563eb`, etc.) that the project's ESLint rule `design-tokens/no-hardcoded-colors` blocks at save-time. The UX specification provides correct design token mappings for every case. This is the single highest-volume defect.

|----------------|---------------|-------------|
| `text-gray-600` / `text-gray-700` | `text-muted-foreground` | ~15 |
| `text-green-600` / `text-green-700` | `text-success` | ~8 |
| `text-orange-600` / `text-orange-700` | `text-warning` | ~6 |
| `text-red-600` | `text-destructive` | ~3 |
| `text-amber-600` | `text-warning` | ~2 |
| `text-blue-600` / `text-blue-700` | `text-brand` | ~4 |

**Fix:** Global find-and-replace across all story code snippets using the UX spec token table (lines 366-419, 538-545, 615-623, 886-906).

**2. Zero QFR Traceability (All 27 stories)**

No story in Epics 14-18 contains a `**FRs Fulfilled:**` line mapping to specific QFR numbers. The PRD defines QFR9-QFR61 for the quiz system. The mapping is implicit but never stated, making requirements coverage unverifiable without manual cross-referencing.

**Fix:** Add `**FRs Fulfilled: QFR##**` line to every story.

**3. Missing `Question.topic` Field (Affects Epic 15.5, 17.3, 17.5)**

Story 15.5's topic-analysis feature (`analyzeTopicPerformance`) accesses `q.topic`, but the `Question` interface (Story 12.1) has no `topic` or `tags` field. This makes the performance insights feature non-functional — all questions fall into "General" topic.

**Fix:** Add `topic?: string` (or `tags?: string[]`) to the Question interface in Story 12.1 or a dedicated subtask.

### HIGH — Affects Multiple Epics

**4. Epic 12 Is a Hard Prerequisite (All 5 Epics)**

Every story depends on infrastructure from Epic 12: `quizzes` and `quizAttempts` Dexie tables (12.2), `Question`/`QuizAttempt` types (12.1), `useQuizStore` (12.3), `QuestionDisplay` (12.5), `scoring.ts` (12.6). None of this exists in the codebase (currently at Epic 9B, schema v12 with `aiUsageEvents`). Implementation cannot begin until Epic 12 ships.

**5. `useStreakStore` Does Not Exist (Epic 18.5)**

Story 18.5 depends on `useStreakStore` and a `streaks` Dexie table, both assumed to exist from Epic 1. Neither has been implemented. This blocks the study streak integration story.

**6. Code Snippets Use Anti-Patterns**
- Story 15.1: `onExpire` in useEffect dependency array creates stale closure bug (timer resets every render)
- Story 17.4: Variable `q = 1 - p` shadows outer `quiz.questions.map(q => ...)` — produces `questionId: undefined` for all results
- Story 18.1: `setTimeout(..., 0)` for focus management is brittle in React 18
- Story 18.5: `new Date().toISOString().split('T')[0]` is timezone-unsafe (UTC vs local)
